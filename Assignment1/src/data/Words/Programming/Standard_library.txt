library</h1>/n									<div id="bodyContent" class="mw-body-content">/n									<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>/n								<div id="contentSub"></div>/n												<div id="jump-to-nav" class="mw-jump">/n					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>/n				</div>/n				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p>A <b>standard library</b> in <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a> is the <a href="/wiki/Library_(computing)" title="Library (computing)">library</a> made available across implementations of a <a href="/wiki/Programming_language" title="Programming language">programming language</a>. These libraries are conventionally described in <a href="/wiki/Programming_language_specification" title="Programming language specification">programming language specifications</a>; however, contents of a language's associated library may also be determined (in part or whole) by more informal practices of a language's community.</p>/n<p>A language's standard library is often treated as part of the language by its <a href="/wiki/Programmer" title="Programmer">users</a>, although the designers may have treated it as a separate entity. Many language specifications define a core set that must be made available in all <a href="/wiki/Implementation#Computer_Science" title="Implementation">implementations</a>, in addition to other portions which may be optionally implemented. The line between a language and its libraries therefore differs from language to language. Indeed, some languages are designed so that the meanings of certain syntactic constructs cannot even be described without referring to the core library. For example, in <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, a <a href="/wiki/String_literal" title="String literal">string literal</a> is defined as an instance of the java.lang.String class; similarly, in <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a>, an anonymous function expression (a "block") constructs an instance of the library's BlockContext class. Conversely, <a href="/wiki/Scheme_(programming_language)" title="Scheme (programming language)">Scheme</a> contains multiple coherent subsets that suffice to construct the rest of the language as library macros, and so the language designers do not even bother to say which portions of the language must be implemented as language constructs, and which must be implemented as parts of a library.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#Contents"><span class="tocnumber">1</span> <span class="toctext">Contents</span></a></li>/n<li class="toclevel-1 tocsection-2"><a href="#Philosophies"><span class="tocnumber">2</span> <span class="toctext">Philosophies</span></a></li>/n<li class="toclevel-1 tocsection-3"><a href="#Examples_of_standard_libraries"><span class="tocnumber">3</span> <span class="toctext">Examples of standard libraries</span></a></li>/n<li class="toclevel-1 tocsection-4"><a href="#References"><span class="tocnumber">4</span> <span class="toctext">References</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="Contents">Contents</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Standard_library&amp;action=edit&amp;section=1" title="Edit section: Contents">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Standard libraries typically include definitions for commonly used algorithms, data structures, and mechanisms for input and output. Depending on the constructs made available by the host language, a standard library may include:</p>/n<ul>/n<li><a href="/wiki/Subroutine" title="Subroutine">Subroutines</a></li>/n<li><a href="/wiki/Macro_(computer_science)" title="Macro (computer science)">Macro</a> definitions</li>/n<li><a href="/wiki/Global_variable" title="Global variable">Global variables</a></li>/n<li><a href="/wiki/Class_(computer_science)" class="mw-redirect" title="Class (computer science)">Class</a> definitions</li>/n<li><a href="/wiki/Template_(C%2B%2B)" title="Template (C++)">Templates</a></li>/n</ul>/n<p>Most standard libraries include definitions for at least the following commonly used facilities:</p>/n<ul>/n<li><a href="/wiki/Algorithm" title="Algorithm">Algorithms</a> (such as <a href="/wiki/Sorting_algorithm" title="Sorting algorithm">sorting algorithms</a>)</li>/n<li><a href="/wiki/Data_structure" title="Data structure">Data structures</a> (such as <a href="/wiki/List_(computing)" class="mw-redirect" title="List (computing)">lists</a>, <a href="/wiki/Tree_(data_structure)" title="Tree (data structure)">trees</a>, and <a href="/wiki/Hash_table" title="Hash table">hash tables</a>)</li>/n<li>Interaction with the host platform, including input/output and operating system calls</li>/n</ul>/n<h2><span class="mw-headline" id="Philosophies">Philosophies</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Standard_library&amp;action=edit&amp;section=2" title="Edit section: Philosophies">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Philosophies of standard library design vary widely. For example, <a href="/wiki/Bjarne_Stroustrup" title="Bjarne Stroustrup">Bjarne Stroustrup</a>, designer of <a href="/wiki/C%2B%2B" title="C++">C++</a>, writes:</p>/n<blockquote class="templatequote">/n<p>What ought to be in the standard C++ library? One ideal is for a programmer to be able to find every interesting, significant, and reasonably general class, function, template, etc., in a library. However, the question here is not, "What ought to be in <i>some</i> library?" but "What ought to be in the <i>standard</i> library?" The answer "Everything!" is a reasonable first approximation to an answer to the former question but not the latter. A standard library is something every implementer must supply so that every programmer can rely on it.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup></p>/n</blockquote>/n<p>This suggests a relatively small standard library, containing only the constructs that "every programmer" might reasonably require when building a large collection of software. This is the philosophy that is used in the <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> and <a href="/wiki/C%2B%2B" title="C++">C++</a> standard libraries.</p>/n<p>By contrast, <a href="/wiki/Guido_van_Rossum" title="Guido van Rossum">Guido van Rossum</a>, designer of <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, has embraced a much more inclusive vision of the standard library; in the Python tutorial, he writes:</p>/n<blockquote class="templatequote">/n<p>Python has a "batteries included" philosophy. This is best seen through the sophisticated and robust capabilities of its larger packages.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup></p>/n</blockquote>/n<p>Van Rossum goes on to list libraries for processing <a href="/wiki/XML" title="XML">XML</a>, <a href="/wiki/XML-RPC" title="XML-RPC">XML-RPC</a>, email messages, and localization, facilities that the C++ standard library omits. This other philosophy is often found in <a href="/wiki/Scripting_language" title="Scripting language">scripting languages</a> (as in <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> or <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a>) or languages that use a <a href="/wiki/Virtual_machine" title="Virtual machine">virtual machine</a>, such as <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> or the <a href="/wiki/.NET_Framework" title=".NET Framework">.NET Framework</a> languages. In C++, such facilities are not part of the standard library, but instead other libraries, such as <a href="/wiki/Boost_(C%2B%2B_libraries)" title="Boost (C++ libraries)">Boost</a>.</p>/n<h2><span
