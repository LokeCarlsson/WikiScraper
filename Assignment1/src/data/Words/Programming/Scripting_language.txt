data-file-width="50" data-file-height="20" /></div>/n</td>/n<td class="mbox-text">/n<div class="mbox-text-span">It has been suggested that <i><a href="/wiki/Interpreted_language" title="Interpreted (<a href="/wiki/Talk:Scripting_language#Merge_with_.22Interpreted_language.22" title="Talk:Scripting language">Discuss</a>) <small><i>Proposed since July 2017.</i></small></div>/n</td>/n</tr>/n</table>/n<p>A <b>scripting</b> or <b>script language</b> is a <a href="/wiki/Programming_language" title="Programming language">programming language</a> that supports <b>scripts:</b> programs written for a special <a href="/wiki/Run-time_environment" class="mw-redirect" title="Run-time environment">run-time environment</a> that <a href="/wiki/Automate" class="mw-redirect" title="Automate">automate</a> the <a href="/wiki/Execution_(computing)" title="Execution (computing)">execution</a> of tasks that could alternatively be executed one-by-one by a human operator. Scripting languages are often interpreted (rather than <a href="/wiki/Compiler" title="Compiler">compiled</a>). Primitives are usually the elementary tasks or <a href="/wiki/Application_Programming_Interface" class="mw-redirect" title="Application Programming Interface">API</a> calls, and the language allows them to be combined into more complex programs. Environments that can be automated through scripting include <a href="/wiki/Software_application" class="mw-redirect" title="Software application">software applications</a>, <a href="/wiki/Web_page" title="Web page">web pages</a> within a <a href="/wiki/Web_browser" title="Web browser">web browser</a>, the <a href="/wiki/Shell_script" title="Shell script">shells</a> of <a href="/wiki/Operating_system" title="Operating system">operating systems</a> (OS), <a href="/wiki/Embedded_system" title="Embedded system">embedded systems</a>, as well as numerous games. A scripting language can be viewed as a <a href="/wiki/Domain-specific_language" title="Domain-specific language">domain-specific language</a> for a particular environment; in the case of scripting an application, this is also known as an <b>extension language</b>. Scripting languages are also sometimes referred to as <a href="/wiki/Very_high-level_programming_language" title="Very high-level programming language">very high-level programming languages</a>, as they operate at a high level of abstraction, or as <b>control languages</b>, particularly for job control languages on mainframes.</p>/n<p>The term "scripting language" is also used loosely to refer to <a href="/wiki/Dynamic_programming_language" title="Dynamic programming language">dynamic</a> <a href="/wiki/High-level_language" class="mw-redirect" title="High-level language">high-level</a> <a href="/wiki/General-purpose_programming_language" title="General-purpose programming language">general-purpose languages</a>, such as <a href="/wiki/Perl" title="Perl">Perl</a>,<sup id="cite_ref-sheppard00_1-0" class="reference"><a href="#cite_note-sheppard00-1">[1]</a></sup> <a href="/wiki/Tcl" title="Tcl">Tcl</a>, and <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>,<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> with the term "script" often used for small programs (up to a few thousand lines of code) in such languages, or in domain-specific languages such as the text-processing languages <a href="/wiki/Sed" title="Sed">sed</a> and <a href="/wiki/AWK" title="AWK">AWK</a>. Some of these languages were originally developed for use within a particular environment, and later developed into portable domain-specific or general-purpose languages. Conversely, many general-purpose languages have dialects that are used as scripting languages. This article discusses scripting languages in the narrow sense of languages for a specific environment.</p>/n<p>The spectrum of scripting languages ranges from very small and highly <a href="/wiki/Domain-specific_language" title="Domain-specific language">domain-specific languages</a> to <a href="/wiki/General-purpose_programming_language" title="General-purpose programming language">general-purpose programming languages</a> used for scripting. Standard examples of scripting languages for specific environments include: <a href="/wiki/Bash_(Unix_shell)" title="Bash (Unix shell)">Bash</a>, for the <a href="/wiki/Unix" title="Unix">Unix</a> or <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> <a href="/wiki/Operating_system" title="Operating system">operating systems</a>; <a href="/wiki/ECMAScript" title="ECMAScript">ECMAScript</a> (<a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>), for web browsers; and <a href="/wiki/Visual_Basic_for_Applications" title="Visual Basic for Applications">Visual Basic for Applications</a>, for <a href="/wiki/Microsoft_Office" title="Microsoft Office">Microsoft Office</a> applications. <a href="/wiki/Lua_(programming_language)" title="Lua (programming language)">Lua</a> is a language designed and widely used as an extension language. <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> is a general-purpose language that is also commonly used as an extension language, while ECMAScript is still primarily a scripting language for web browsers, but is also used as a general-purpose language. The <a href="/wiki/Emacs_Lisp" title="Emacs Lisp">Emacs Lisp</a> dialect of <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a> (for the <a href="/wiki/Emacs" title="Emacs">Emacs</a> editor) and the Visual Basic for Applications dialect of <a href="/wiki/Visual_Basic" title="Visual Basic">Visual Basic</a> are examples of scripting language dialects of general-purpose languages. Some game systems, notably the <a href="/wiki/Second_Life" title="Second Life">Second Life</a> <a href="/wiki/Virtual_world" title="Virtual world">virtual world</a> and the <a href="/wiki/Trainz" title="Trainz">Trainz</a> franchise of Railroad simulators have been extensively extended in functionality by scripting extensions. In other games like <a href="/wiki/Wesnoth" class="mw-redirect" title="Wesnoth">Wesnoth</a>, the variety of actual games played by players are scripts written by other users.</p>/n<p></p>/n<div class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Scripting_language&amp;action=edit&amp;section=1" title="Edit section: Characteristics">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Typically scripting languages are intended to be very fast to learn and write in, either as short source code files or interactively in a <a href="/wiki/Read%E2%80%93eval%E2%80%93print_loop" title="Read–eval–print loop">read–eval–print loop</a> (REPL, language shell).<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> This generally implies relatively simple <a href="/wiki/Syntax_(programming_languages)" title="Syntax (programming languages)">syntax</a> and <a href="/wiki/Semantics_(computer_science)" title="Semantics (computer science)">semantics</a>; typically a "script" (code written in the scripting language) is executed from start to finish, as a "script", with no explicit <a href="/wiki/Entry_point" title="Entry point">entry point</a>.</p>/n<p>For example, it is uncommon to characterise <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> as a scripting language because of its lengthy syntax and rules about which classes exist in which files, and it is not directly possible to execute Java interactively, because source files can only contain definitions that must be invoked externally by a <a href="/w/index.php?title=Host_application&amp;action=edit&amp;redlink=1" class="new" title="Host application (page does not exist)">host application</a> or <a href="/wiki/Application_launcher" class="mw-redirect" title="Application launcher">application launcher</a>.</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="kd">public</span> <span class="kd">class</span> <span class="nc">HelloWorld</span> <span class="o">{</span>/n  <span class="o">}</span>/n<span class="o">}</span>/n</pre></div>/n<p>This piece of code intended to print "Hello World" does nothing as <i>main()</i> is <i>not declared</i> in <b>HelloWorld</b> class.</p>/n<p>In contrast, <a href="/wiki/Python_(computer_language)" class="mw-redirect" title="Python (computer language)">Python</a> allows definition of some functions in a single file, or to avoid functions altogether and use <a href="/wiki/Imperative_programming" title="Imperative programming">imperative programming</a> style, or even use it interactively.</p>/n<p><code>print ("Hello World")</code></p>/n<p>This one line of python code prints "Hello World"; no <i>declarative</i> statement like <i>main()</i> is required here.</p>/n<p>A scripting language is usually <a href="/wiki/Interpreted_language" title="Interpreted language">interpreted</a> from source code or <a href="/wiki/Bytecode" title="Bytecode">bytecode</a>.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup> By contrast, the software environment the scripts are written for is typically written in a <a href="/wiki/Compiled_language" title="Compiled language">compiled language</a> and distributed in <a href="/wiki/Machine_code" title="Machine code">machine code</a> form.</p>/n<p>Scripting languages may be designed for use by end users of a program—<a href="/wiki/End-user_development" title="End-user development">end-user development</a>—or may be only for internal use by developers, so they can write portions of the program in the scripting language. Scripting languages typically use <a href="/wiki/Abstraction_(computer_science)" class="mw-redirect" title="Abstraction (computer science)">abstraction</a>, a form of <a href="/wiki/Information_hiding" title="Information hiding">information hiding</a>, to spare users the details of internal variable types, data storage, and <a href="/wiki/Memory_management" title="Memory management">memory management</a>.</p>/n<p>Scripts are often created or modified by the person executing them,<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup> but they are also often distributed, such as when large portions of games are written in a scripting language.</p>/n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Scripting_language&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Early <a href="/wiki/Mainframe_computer" title="Mainframe computer">mainframe computers</a> (in the 1950s) were non-interactive, instead using <a href="/wiki/Batch_processing" title="Batch processing">batch processing</a>. IBM's <a href="/wiki/Job_Control_Language" title="Job Control Language">Job Control Language</a> (JCL) is the archetype of languages used to control batch processing.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup></p>/n<p>The first interactive <a href="/wiki/Shell_(computing)" title="Shell (computing)">shells</a> were developed in the 1960s to enable remote operation of the first <a href="/wiki/Time-sharing" title="Time-sharing">time-sharing</a> systems, and these used <a href="/wiki/Shell_scripts" class="mw-redirect" title="Shell scripts">shell scripts</a>, which controlled running computer programs within a computer program, the shell. <a href="/wiki/Calvin_Mooers" title="Calvin Mooers">Calvin Mooers</a> in his <a href="/wiki/TRAC_(programming_language)" title="TRAC (programming language)">TRAC</a> language is generally credited with inventing <i>command substitution</i>, the ability to embed commands in scripts that when interpreted insert a character string into the script.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup> <a href="/wiki/Multics" title="Multics">Multics</a> calls these <i>active functions</i>.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup> <a href="/wiki/Louis_Pouzin" title="Louis Pouzin">Louis Pouzin</a> wrote an early processor for command scripts called RUNCOM for <a href="/wiki/Compatible_Time_Sharing_System" class="mw-redirect" title="Compatible Time Sharing System">CTSS</a> around 1964. <a href="/wiki/Stuart_Madnick" title="Stuart Madnick">Stuart Madnick</a> at MIT wrote a scripting language for IBM's <a href="/wiki/CP/CMS" title="CP/CMS">CP/CMS</a> in 1966. He originally called this processor COMMAND, later named <a href="/wiki/CMS_EXEC" title="CMS EXEC">EXEC</a>.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[9]</a></sup> Multics included an offshoot of CTSS RUNCOM, also called RUNCOM.<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">[10]</a></sup> <a href="/wiki/CMS_EXEC" title="CMS EXEC">EXEC</a> was eventually replaced by <a href="/wiki/EXEC_2" title="EXEC 2">EXEC 2</a> and <a href="/wiki/REXX" class="mw-redirect" title="REXX">REXX</a>.</p>/n<p>Languages such as <a href="/wiki/Tcl" title="Tcl">Tcl</a> and <a href="/wiki/Lua_(programming_language)" title="Lua (programming language)">Lua</a> were specifically designed as general-purpose scripting languages that could be embedded in any application. Other languages such as <a href="/wiki/Visual_Basic_for_Applications" title="Visual Basic for Applications">Visual Basic for Applications</a> (VBA) provided strong integration with the automation facilities of an underlying system. Embedding of such general-purpose scripting languages instead of developing a new language for each application also had obvious benefits, relieving the application developer of the need to code a language translator from scratch and allowing the user to apply skills learned elsewhere.</p>/n<p>Some software incorporates several different scripting languages. Modern <a href="/wiki/Web_browser" title="Web browser">web browsers</a> typically provide a language for writing extensions to the browser itself, and several standard embedded languages for controlling the browser, including <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> (a dialect of <a href="/wiki/ECMAScript" title="ECMAScript">ECMAScript</a>) or <a href="/wiki/XUL" title="XUL">XUL</a>.</p>/n<h2><span class="mw-headline" id="Types">Types</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a In this view, scripting is particularly <a href="/wiki/Glue_code" title="Glue code">glue code</a>, connecting <a href="/wiki/Software_component" class="mw-redirect" title="Software component">software components</a>, and a language specialized for this purpose is a <i>glue language</i>. Pipelines and shell scripting are archetypal examples of glue languages, and <a href="/wiki/Perl" title="Perl">Perl</a> was initially developed to fill this same role. <a href="/wiki/Web_development" title="Web development">Web development</a> can be considered a use of glue languages, interfacing between a <a href="/wiki/Database" title="Database">database</a> and <a href="/wiki/Web_server" title="Web server">web server</a>. But if a substantial amount of logic is written in script, it is better characterized as simply another software component, not "glue".</p>/n<p>Glue languages are especially useful for writing and maintaining:</p>/n<ul>/n<li>custom commands for a command shell;</li>/n<li>smaller programs than those that are better implemented in a compiled language;<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. or after running an application like a word processor, spreadsheet, data base, assembler, compiler, etc.;</li>/n<li>scripts that may change;</li>/n<li><a href="/wiki/Rapid_application_development" title="Rapid application development">rapid prototypes</a> of a solution eventually implemented in another, usually compiled, language.</li>/n</ul>/n<p>Glue language examples:</p>/n<table href="/wiki/Macro_(computer_science)" title="Macro (computer science)">Macro</a> languages exposed to operating system or application components can serve as glue languages. These include <a href="/wiki/Visual_Basic_for_Applications" title="Visual Basic for Applications">Visual Basic for Applications</a>, <a href="/wiki/WordBasic" class="mw-redirect" title="WordBasic">WordBasic</a>, <a href="/wiki/LotusScript" title="LotusScript">LotusScript</a>, <a rel="nofollow" class="external text" href="https://www.oberonplace.com/tutor/page1.htm">CorelScript</a>, Hummingbird Basic, QuickScript, <a rel="nofollow" class="external text" href="https://msdn.microsoft.com/en-us/library/ms994312.aspx">SaxBasic</a>, and <a href="/wiki/WinWrap_Basic" title="WinWrap Basic">WinWrap Basic</a>. Other tools like <a href="/wiki/AWK" title="AWK">AWK</a> can also be considered glue languages, as can any language implemented by an <a href="/wiki/Windows_Script_Host" title="Windows Script Host">Windows Script Host</a> engine (VBScript, JScript and VBA by default in Windows and third-party engines including implementations of Rexx, Perl, Tcl, Python, XSLT, Ruby, Modern Pascal, Delphi, &amp; C). A majority of applications can access and use operating system components via the <a href="/wiki/Object_model" title="Object model">object models</a> or its own functions.</p>/n<p>Other devices like programmable calculators may also have glue languages; the operating systems of PDAs such as Windows CE may have available native or third-party macro tools that glue applications together, in addition to implementations of common glue languages—including Windows NT, MS-DOS and some Unix shells, Rexx, Modern Pascal, PHP, and Perl. Depending upon the OS version, WSH and the default script engines (VBScript and JScript) are available.</p>/n<p>Programmable calculators can be programmed in glue languages in three ways. For example, the <a href="/wiki/Texas_Instruments" title="Texas Instruments">Texas Instruments</a> <a href="/wiki/TI-92" class="mw-redirect" title="TI-92">TI-92</a>, by factory default can be programmed with a command script language. Inclusion of the scripting and glue language <a href="/wiki/Lua_(programming_language)" title="Lua (programming language)">Lua</a> in the <a href="/w/index.php?title=TI-NSpire&amp;action=edit&amp;redlink=1" class="new" title="TI-NSpire (page does not exist)">TI-NSpire</a> series of calculators could be seen as a successor to this. The primary on-board high-level programming languages of most graphing calculators (most often Basic variants, sometimes Lisp derivatives, and more uncommonly, C derivatives) in many cases can glue together calculator functions—such as graphs, lists, matrices, etc. Third-party implementations of more comprehensive Basic version that may be closer to variants listed as glue languages in this article are available—and attempts to implement Perl, Rexx, or various operating system shells on the TI and HP graphing calculators are also mentioned. PC-based C <a href="/wiki/Cross-compilers" class="mw-redirect" title="Cross-compilers">cross-compilers</a> for some of the TI and HP machines used in conjunction with tools that convert between C and Perl, Rexx, AWK, as well as shell scripts to Perl, Modern Pascal, VBScript to and from Perl make it possible to write a program in a glue language for eventual implementation (as a compiled program) on the calculator.</p>/n<h3><span class="mw-headline" id="Job_control_languages_and_shells">Job control languages and shells</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Scripting_language&amp;action=edit&amp;section=5" title="Edit section: Job control languages and shells">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Shell_script" title="Shell script">Shell script</a></div>/n<p>A major class of scripting languages has grown out of the automation of <a href="/wiki/Job_control_(computing)" title="Job control (computing)">job control</a>, which relates to starting and controlling the behavior of system programs. (In this sense, one might think of shells as being descendants of IBM's JCL, or <a href="/wiki/Job_Control_Language" title="Job Control Language">Job Control Language</a>, which was used for exactly this purpose.) Many of these languages' interpreters double as <a href="/wiki/Command-line_interpreter" class="mw-redirect" title="Command-line interpreter">command-line interpreters</a> such as the <a href="/wiki/Unix_shell" title="Unix shell">Unix shell</a> or the MS-DOS <code><a href="/wiki/COMMAND.COM" title="COMMAND.COM">COMMAND.COM</a></code>. Others, such as <a href="/wiki/AppleScript" title="AppleScript">AppleScript</a> offer the use of English-like commands to build scripts.</p>/n<h3><span class="mw-headline" id="GUI_scripting">GUI scripting</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Scripting_language&amp;action=edit&amp;section=6" title="Edit section: GUI scripting">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>With the advent of graphical user interfaces, a specialized kind of scripting language emerged for controlling a computer. These languages interact with the same graphic windows, menus, buttons, and so on that a human user would. They do this by simulating the actions of a user. These languages are typically used to automate user actions. Such languages are also called "<a href="/wiki/Macro_(computer_science)#Keyboard_and_mouse_macros" title="Macro (computer science)">macros</a>" when control is through simulated key presses or mouse clicks, as well as tapping or pressing on a touch-activated screen.</p>/n<p>These languages could in principle be used to control any GUI application; but, in practice their use is limited because their use needs support from the application and from the <a href="/wiki/Operating_system" title="Operating system">operating system</a>. There are a few exceptions to this limitation. Some GUI scripting languages are based on recognizing graphical objects from their display screen <a href="/wiki/Pixel" title="Pixel">pixels</a>. These GUI scripting languages do not depend on support from the operating system or application.</p>/n<h3><span class="mw-headline" id="Application-specific_languages">Application-specific languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Scripting_language&amp;action=edit&amp;section=7" title="Edit section: Application-specific languages">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Application specific languages can be split in many different categories, i.e. standalone based app languages (executable) or internal application specific languages (postscript, xml, gscript as some of the widely distributed scripts, respectively implemented by Adobe, MS and Google) among others include an idiomatic scripting language tailored to the needs of the application user. Likewise, many <a href="/wiki/Computer_game" class="mw-redirect" title="Computer game">computer game</a> systems use a custom scripting language to express the programmed actions of <a href="/wiki/Non-player_character" title="Non-player character">non-player characters</a> and the game environment. Languages of this sort are designed for a single application; and, while they may superficially resemble a specific general-purpose language (e.g. <a href="/wiki/QuakeC" title="QuakeC">QuakeC</a>, modeled after C), they have custom features that distinguish them. <a href="/wiki/Emacs_Lisp" title="Emacs Lisp">Emacs Lisp</a>, while a fully formed and capable dialect of <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a>, contains many special features that make it most useful for extending the editing functions of Emacs. An application-specific scripting language can be viewed as a <a href="/wiki/Domain-specific_programming_language" class="mw-redirect" title="Domain-specific programming language">domain-specific programming language</a> specialized to a single application.</p>/n<h3><span id="Extension/embeddable_languages"></span><span class="mw-headline" id="Extension.2Fembeddable_languages">Extension/embeddable languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Scripting_language&amp;action=edit&amp;section=8" title="Edit section: Extension/embeddable languages">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>A number of languages have been designed for the purpose of replacing application-specific scripting languages by being embeddable in application programs. The application programmer (working in C or another systems language) includes "hooks" where the scripting language can control the application. These languages may be technically equivalent to an application-specific extension language but when an application embeds a "common" language, the user gets the advantage of being able to transfer skills from application to application. A more generic alternative is simply to provide a library (often a C library) that a general-purpose language can use to control the application, without modifying the language for the specific domain.</p>/n<p>JavaScript began as and primarily still is a language for scripting inside <a href="/wiki/Web_browser" title="Web browser">web browsers</a>; however, the standardization of the language as <a href="/wiki/ECMAScript" title="ECMAScript">ECMAScript</a> has made it popular as a general-purpose embeddable language. In particular, the <a href="/wiki/Mozilla" title="Mozilla">Mozilla</a> implementation <a href="/wiki/SpiderMonkey_(JavaScript_engine)" class="mw-redirect" title="SpiderMonkey (JavaScript engine)">SpiderMonkey</a> is embedded in several environments such as the <a href="/wiki/Yahoo!_Widget_Engine" class="mw-redirect" title="Yahoo! Widget Engine">Yahoo! Widget Engine</a>. Other applications embedding ECMAScript implementations include the <a href="/wiki/Adobe_Systems" title="Adobe Systems">Adobe</a> products <a href="/wiki/Adobe_Flash" title="Adobe Flash">Adobe Flash</a> (<a href="/wiki/ActionScript" title="ActionScript">ActionScript</a>) and <a href="/wiki/Adobe_Acrobat" title="Adobe Acrobat">Adobe Acrobat</a> (for scripting <a href="/wiki/PDF" class="mw-redirect" title="PDF">PDF</a> files).</p>/n<p><a href="/wiki/Tcl" title="Tcl">Tcl</a> was created as an extension language but has come to be used more frequently as a general-purpose language in roles similar to <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Perl" title="Perl">Perl</a>, and <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a>. On the other hand, <a href="/wiki/Rexx" title="Rexx">Rexx</a> was originally created as a job control language, but is widely used as an extension language as well as a general-purpose language. Perl is a general-purpose language, but had the Oraperl (1990) dialect, consisting of a <a href="/wiki/Perl" title="Perl">Perl</a> 4 binary with <a href="/wiki/Oracle_Call_Interface" title="Oracle Call Interface">Oracle Call Interface</a> compiled in. This has however since been replaced by a library (Perl Module), <a rel="nofollow" class="external text" href="https://metacpan.org/module/DBD::Oracle">DBD::Oracle</a>.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">[11]</a></sup><sup id="cite_ref-12" class="reference"><a href="#cite_note-12">[12]</a></sup></p>/n<p>Other complex and task-oriented applications may incorporate and expose an embedded programming language to allow their users more control and give them more functionality than can be available through a user interface, no matter how sophisticated. For example, <a href="/wiki/Autodesk_Maya" title="Autodesk Maya">Autodesk Maya</a> 3D authoring tools embed the <a href="/wiki/Autodesk_Maya#Mel_scripting" title="Autodesk Maya">MEL</a> scripting language, or <a href="/wiki/Blender_(software)" title="Blender (software)">Blender</a> which uses <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> to fill this role.</p>/n<p>Some other types of applications that need faster feature addition or tweak-and-run cycles (e.g. <a href="/wiki/Game_Engine" class="mw-redirect" title="Game Engine">game engines</a>) also use an embedded language. During the development, this allows them to prototype features faster and tweak more freely, without the need for the user to have intimate knowledge of the inner workings of the application or to rebuild it after each tweak (which can take a significant amount of time). The scripting languages used for this purpose range from the more common and more famous <a href="/wiki/Lua_(programming_language)" title="Lua (programming language)">Lua</a> and <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> to lesser-known ones such as <a href="/wiki/AngelScript" title="AngelScript">AngelScript</a> and <a href="/wiki/Squirrel_(programming_language)" title="Squirrel (programming language)">Squirrel</a>.</p>/n<p><a href="/wiki/Ch_(computer_programming)" title="Ch (computer programming)">Ch</a> is another C compatible scripting option for the industry to embed into C/C++ application programs.</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span
