href="/wiki/English_alphabet#Letter_names" title="English alphabet">pronounced</a> "tickle" or <i>tee cee ell</i>, <span class="nowrap"><span class="IPA nopopups noexcerpt"><a href="/wiki/Help:IPA/English" title="Help:IPA/English">/<span style="border-bottom:1px dotted"><span title="/ˈ/: primary stress follows">ˈ</span><span title="'t' in 'tie'">t</span><span title="/iː/: 'ee' in 'fleece'">iː</span></span> <span style="border-bottom:1px dotted"><span title="'s' in 'sigh'">s</span><span title="/iː/: 'ee' in 'fleece'">iː</span></span> <span style="border-bottom:1px dotted"><span title="/ɛ/: 'e' in 'dress'">ɛ</span><span title="'l' in 'lie'">l</span></span>/</a></span></span>) is a <a href="/wiki/High-level_programming_language" title="High-level programming language">high-level</a>, <a href="/wiki/General-purpose_programming_language" title="General-purpose programming language">general-purpose</a>, <a href="/wiki/Interpreter_(computing)" title="Interpreter (computing)">interpreted</a>, <a href="/wiki/Dynamic_programming_language" title="Dynamic programming language">dynamic programming language</a>. It was designed with the goal of being very simple but powerful.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup> Tcl casts everything into the mold of a <a href="/wiki/Command_(computing)" title="Command (computing)">command</a>, even programming constructs like variable assignment and procedure definition.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup> Tcl supports multiple <a href="/wiki/Programming_paradigm" title="Programming paradigm">programming paradigms</a>, including <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented</a>, <a href="/wiki/Imperative_programming" title="Imperative programming">imperative</a> and <a href="/wiki/Functional_programming" title="Functional programming">functional programming</a> or <a href="/wiki/Procedural_programming" title="Procedural programming">procedural</a> styles.</p>/n<p>It is commonly used embedded into <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> applications,<sup id="cite_ref-inception_8-0" class="reference"><a href="#cite_note-inception-8">[8]</a></sup> for <a href="/wiki/Rapid_application_development" title="Rapid application development">rapid prototyping</a>, scripted applications, GUIs, and testing.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[9]</a></sup> Tcl interpreters are available for many <a href="/wiki/Operating_system" title="Operating system">operating systems</a>, allowing Tcl code to run on a wide variety of systems. Because Tcl is a very compact language, it is used on <a href="/wiki/Embedded_systems" class="mw-redirect" title="Embedded systems">embedded systems</a> platforms, both in its full form and in several other small-footprint versions.<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">[10]</a></sup></p>/n<p>The popular combination of Tcl with the <a href="/wiki/Tk_(software)" title="Tk (software)">Tk</a> extension is referred to as <b>Tcl/Tk</b>, and enables building a <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a> (GUI) natively in Tcl. Tcl/Tk is included in the standard <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> installation in the form of <a href="/wiki/Tkinter" title="Tkinter">Tkinter</a>.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#History"><span while working at the <a href="/wiki/University_of_California,_Berkeley" title="University of California, Berkeley">University of California, Berkeley</a>.<sup id="cite_ref-11" class="reference"><a id="Safe-Tcl">Safe-Tcl</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=3" title="Edit section: Safe-Tcl">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Safe-Tcl is a subset of Tcl that has restricted features so that Tcl scripts cannot harm their hosting machine or application.<sup id="cite_ref-27" class="reference"><a href="#cite_note-27">[27]</a></sup> File system access is limited and arbitrary system commands are prevented from execution. It uses a dual interpreter model with the untrusted interpreter running code in an untrusted script. It was designed by <a href="/wiki/Nathaniel_Borenstein" title="Nathaniel Borenstein">Nathaniel Borenstein</a> and <a href="/wiki/Marshall_Rose" title="Marshall Rose">Marshall Rose</a> to include active messages in e-mail. Safe-Tcl can be included in e-mail when the <i>application/safe-tcl</i> and <i>multipart/enabled-mail</i> are supported. The functionality of Safe-Tcl has since been incorporated as part of the standard Tcl/Tk releases.<sup id="cite_ref-28" class="reference"><a href="#cite_note-28">[28]</a></sup><sup id="cite_ref-29" class="reference"><a href="#cite_note-29">[29]</a></sup></p>/n<h2><span class="mw-headline" id="Syntax_and_fundamental_semantics">Syntax and fundamental semantics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=4" title="Edit section: Syntax and fundamental semantics">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The syntax and semantics are covered by the twelve rules of the dodecalogue<sup id="cite_ref-30" class="reference"><a href="#cite_note-30">[30]</a></sup> (alternative wording<sup id="cite_ref-31" class="reference"><a href="#cite_note-31">[31]</a></sup>).</p>/n<p>A Tcl script consists of several command invocations. A command invocation is a list of words separated by whitespace and terminated by a newline or semicolon.</p>/n<pre>/nword0 word1 word2 ... wordN/n</pre>/n<p>The first word is the name of a command, which is not built into the language, but which is in the library. The following words are arguments. So we have:</p>/n<pre>/ncommandName argument1 argument2 ... argumentN/n</pre>/n<p>An example, using the <i>puts</i> command to display a string on the host console, is:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="nb">puts</span> <span class="s2">"Hello, World!"</span>/n</pre></div>/n<p>This sends the string "Hello, World!" to the 'stdout' device, with an appended newline character.</p>/n<p>Variables and the results of other commands can be substituted inside strings too, such as in this example where we use <i>set</i> and <i>expr</i> (no assignment operator <i>=</i>) to store a calculation result in a variable, and <i>puts</i> (short for "put string") to print the result together with some explanatory text:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="c"># expr evaluates text string as an expression</span>/n<span class="k">set</span> sum <span class="k">[expr</span> <span class="mi">1</span><span class="o">+</span><span class="mi">2</span><span class="o">+</span><span class="mi">3</span><span class="o">+</span><span class="mi">4</span><span class="o">+</span><span class="mi">5</span><span class="k">]</span>/n<span class="nb">puts</span> <span class="s2">"The sum of the numbers 1..5 is $sum."</span>/n</pre></div>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="c"># with curly braces, variable substitution is delayed until call time (dynamic scoping)</span>/n<span class="k">set</span> x <span class="mi">1</span>/n<span class="k">set</span> sum <span class="k">[expr</span> <span class="k">{</span><span class="nv">$x</span> <span class="o">+</span> <span class="nv">2</span> <span class="o">+</span> <span class="mi">3</span> <span class="o">+</span> <span class="mi">4</span> <span class="o">+</span> <span class="mi">5</span><span class="k">}];</span> <span class="c"># $x is not substituted, and the expression is preserved</span>/n<span class="nb">puts</span> <span class="s2">"The sum of the numbers 1..5 is $sum."</span><span class="k">;</span> <span class="c"># 1 is substituted for $x, and the expression is evaluated</span>/n</pre></div>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="c"># without curly braces, variable substitution occurs at the definition site (lexical scoping)</span>/n<span class="k">set</span> x <span class="mi">2</span>/n<span class="k">set</span> op <span class="o">*</span>/n<span class="k">set</span> y <span class="mi">3</span>/n<span class="k">set</span> res <span class="k">[expr</span> <span class="nv">$x$op$y</span><span class="k">];</span> <span class="c"># $x, $op, and $y are substituted, and the expression is evaluated</span>/n<span class="nb">puts</span> <span class="s2">"2 * 3 is $res."</span><span class="k">;</span> <span class="c"># 6 is substituted for $res</span>/n</pre></div>/n<p>There is one basic construct (the command) and a set of simple substitution rules.</p>/n<p>Formally, words are either written as-is; with double-quotes around them (allowing whitespace characters to be embedded); or with curly-brace characters around them, which suppresses all substitutions inside (except for backslash-newline elimination). In bare and double-quoted words, three types of substitution occur (once, in a single left-to-right scan through the word):</p>/n<ul>/n<li><b>Command substitution</b> replaces the contents of balanced square brackets with the result of evaluating the script contained inside. For example, “<b>[expr 1+2+3]</b>” is replaced with the result of evaluating the contained expression (i.e. 6) since that's what the <code>expr</code> command does.</li>/n<li><b>Variable substitution</b> replaces a dollar-sign followed by the name of a variable with the contents of the variable. For example, “<b>$foo</b>” is replaced with the contents of the variable called “foo”. The variable name may be surrounded in curly braces so as to delimit what is and isn't the variable name in otherwise ambiguous cases.</li>/n<li><b>Backslash substitution</b> replaces a backslash followed by a letter with another character. For example, “<b>\n</b>” is replaced with a newline.</li>/n</ul>/n<p>From Tcl 8.5 onwards, any word may be prefixed by “<b>{*}</b>” to cause that word to be split apart into its constituent sub-words for the purposes of building the command invocation (similar to the “<b>,@</b>” sequence of <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a>'s quasiquote feature).</p>/n<p>As a consequence of these rules, the result of any command may be used as an argument to any other command. Also, there is no operator or command for string concatenation, as the language concatenates directly. Note that, unlike in <a href="/wiki/Bourne_shell" title="Bourne shell">Unix command shells</a>, Tcl does not reparse any string unless explicitly directed to do so, which makes interactive use more cumbersome, but scripted use more predictable (e.g. the presence of spaces in filenames does not cause difficulties).</p>/n<p>The single equality sign (=) for example is not used at all, and the double equality sign (==) is the test for equality, and even then only in expression contexts such as the <code>expr</code> command or the first argument to <code>if</code>. (Both of those commands are just part of the standard library; they have no particularly special place in the library and can be replaced, if so desired.)</p>/n<p>The majority of Tcl commands, especially in the standard library, are <a href="/wiki/Variadic_function" title="Variadic function">variadic</a>, and the <code>proc</code> (the constructor for scripted command procedures) allows one to define default values for unspecified arguments and a catch-all argument to allow the code to process arbitrary numbers of arguments.</p>/n<p>Tcl is not statically typed: each variable may contain integers, floats, strings, lists, command names, dictionaries, or any other value; values are reinterpreted (subject to syntactic constraints) as other types on demand. However, values are <a href="/wiki/Immutable_object" title="Immutable object">immutable</a> and operations that appear to change them actually just return a new value instead.</p>/n<h3><span class="mw-headline" id="Basic_commands">Basic commands</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=5" title="Edit section: Basic commands">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>The most important commands that refer to program execution and data operations are:</p>/n<ul>/n<li><code>set</code> writes a new value to a variable (creates a variable if did not exist). If used only with one argument, it returns the value of the given variable (it must exist in this case).</li>/n<li><code>proc</code> defines a new command, whose execution results in executing a given Tcl script, written as a set of commands. <code>return</code> can be used to immediately return control to the caller.</li>/n</ul>/n<p>The usual execution control commands are:</p>/n<ul>/n<li><code><a href="/wiki/If-then-else" class="mw-redirect" title="If-then-else">if</a></code> executes given script body (second argument), if the condition (first argument) is satisfied. It can be followed by additional arguments starting from <code>elseif</code> with the alternative condition and body, or <code>else</code> with the complementary block.</li>/n<li><code><a href="/wiki/While_loop#Tcl" title="While loop">while</a></code> repeats executing given script body, as long as the condition (first argument) remains satisfied</li>/n<li><code><a href="/wiki/Foreach#Tcl" class="mw-redirect" title="Foreach">foreach</a></code> executes given body where the control variable is assigned list elements one by one.</li>/n<li><code>for</code> shortcut for initializing the control variable, condition (as in <code>while</code>) and the additional "next iteration" statement (command executed after executing the body)</li>/n</ul>/n<p>Those above looping commands can be additionally controlled by the following commands:</p>/n<ul>/n<li><code>break</code> interrupts the body execution and returns from the looping command</li>/n<li><code>continue</code> interrupts the body execution, but the control is still given back to the looping command. For <code>while</code> it means to loop again, for <code>for</code> and <code>foreach</code>, pick up the next iteration.</li>/n<li><code>return</code> interrupts the execution of the current body no matter how deep inside a procedure, until reaching the procedure boundary, and returns given value to the caller.</li>/n</ul>/n<h3><span class="mw-headline" id="Advanced_commands">Advanced commands</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=6" title="Edit section: Advanced commands">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<ul>/n<li><code>expr</code> passes the argument to a separate expression interpreter and returns the evaluated value. Note that the same interpreter is used also for "conditional" expression for <code>if</code> and looping commands.</li>/n<li><code>list</code> creates a <a href="/wiki/List_(computer_science)" class="mw-redirect" title="List (computer science)">list</a> comprising all the arguments, or an empty string if no argument is specified. The <code>lindex</code> command may be used on the result to re-extract the original arguments.</li>/n<li><code>array</code> manipulates <a href="/wiki/Associative_array" title="Associative array">array</a> variables.</li>/n<li><code>dict</code> manipulates <a href="/wiki/Associative_array" title="Associative array">dictionary</a> (since 8.5), which are lists with an even number of elements where every two elements are interpreted as a key/value pair.</li>/n<li><code>regexp</code> matches a regular expression against a string.</li>/n<li><code><a href="/wiki/Tcl#Uplevel" title="Tcl">uplevel</a></code> is a <a href="/wiki/Command_(computing)" title="Command (computing)">command</a> that allows a command script to be executed in a <a href="/wiki/Scope_(computer_science)" title="Scope (computer science)">scope</a> other than the current innermost scope on the stack.</li>/n<li><code><a href="/wiki/Tcl#Upvar" title="Tcl">upvar</a></code> creates a link to variable in a different stack frame.</li>/n<li><code>namespace</code> lets you create, access, and destroy separate contexts for commands and variables.</li>/n<li><code>apply</code> - apply an anonymous function (since 8.6).</li>/n<li><code>coroutine</code>, <code>yield</code>, <code>yieldto</code> create and produce values from <a href="/wiki/Coroutine" title="Coroutine">coroutines</a> (since 8.6).</li>/n<li><code>try</code> lets you trap and process errors and exceptions.</li>/n<li><code>catch</code> lets you trap exceptional returns.</li>/n<li><code>zlib</code> provides access to the compression and check-summing facilities of the Zlib library (since 8.6).</li>/n</ul>/n<h3><span class="mw-headline" id="Uplevel">Uplevel</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=7" title="Edit section: Uplevel">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p><code>uplevel</code> allows a command script to be executed in a <a href="/wiki/Scope_(computer_science)" title="Scope (computer science)">scope</a> other than the current innermost scope on the stack. Because the command script may itself call procedures that use the <i>uplevel</i> command, this has the net effect of transforming the <a href="/wiki/Call_stack" title="Call stack">call stack</a> into a call tree.<sup id="cite_ref-32" class="reference"><a href="#cite_note-32">[32]</a></sup></p>/n<p>It was originally implemented to permit Tcl procedures to reimplement built-in commands (like <i>for</i>, <i>if</i> or <i>while</i>) and still have the ability to manipulate local <a href="/wiki/Variable_(computer_science)" title="Variable (computer science)">variables</a>. For example, the following Tcl script is a reimplementation of the <i>for</i> command (omitting <a href="/wiki/Exception_handling" title="Exception handling">exception handling</a>):</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="k">proc</span> for <span class="k">{</span><span class="nv">initCmd</span> testExpr advanceCmd bodyScript<span class="k">}</span> <span class="k">{</span>/n    <span class="k">uplevel</span> <span class="mi">1</span> <span class="nv">$initCmd</span>/n    <span class="k">set</span> testCmd <span class="k">[</span><span class="nb">list</span> expr <span class="nv">$testExpr</span><span class="k">]</span>/n    <span class="k">while</span> <span class="k">{[uplevel</span> <span class="mi">1</span> <span class="nv">$testCmd</span><span class="k">]}</span> <span class="k">{</span>/n        <span class="k">uplevel</span> <span class="mi">1</span> <span class="nv">$bodyScript</span>/n        <span class="k">uplevel</span> <span class="mi">1</span> <span class="nv">$advanceCmd</span>/n    <span class="k">}</span>/n<span class="k">}</span>/n</pre></div>/n<h3><span class="mw-headline" id="Upvar">Upvar</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=8" title="Edit section: Upvar">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p><code>upvar</code> arranges for one or more local variables in the current procedure to refer to variables in an enclosing procedure call or to global variables. The upvar command simplifies the implementation of call-by-name procedure calling and also makes it easier to build new control constructs as Tcl procedures.<sup id="cite_ref-33" class="reference"><a href="#cite_note-33">[33]</a></sup></p>/n<p>A decr command that works like the built-in <code>incr</code> command except it subtracts the value from the variable instead of adding it:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="k">proc</span> decr <span class="k">{</span><span class="nv">varName</span> <span class="k">{</span><span class="nv">decrement</span> <span class="mi">1</span><span class="k">}}</span> <span class="k">{</span>/n    <span class="k">upvar</span> <span class="mi">1</span> <span class="nv">$varName</span> var/n    <span class="nb">incr</span> var <span class="k">[expr</span> <span class="k">{</span><span class="nv">-$decrement</span><span class="k">}]</span>/n<span class="k">}</span>/n</pre></div>/n<h3><span class="mw-headline" id="Object-oriented">Object-oriented</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=9" title="Edit section: Object-oriented">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<div role="note" class="hatnote navigation-not-searchable">See also: <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">Object-oriented programming</a></div>/n<p>Tcl 8.6 added a built-in dynamic object system, TclOO, in 2012.<sup id="cite_ref-tip257_25-1" class="reference"><a href="#cite_note-tip257-25">[25]</a></sup> It includes features such as:</p>/n<ul>/n<li>Class-based object system. This is what most programmers expect from OO.</li>/n<li>Allows per-object customization and dynamic redefinition of classes.</li>/n<li>Meta-classes</li>/n<li>Filters</li>/n<li>Mixins</li>/n<li>A system for implementing methods in custom ways, so that package authors that want significantly different ways of doing a method implementation may do so fairly simply.</li>/n</ul>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="nv">oo</span><span class="o">::</span>class create fruit <span class="k">{</span>/n    <span class="nv">method</span> eat <span did not have <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object oriented</a> (OO) syntax until 2012,<sup id="cite_ref-tip257_25-2" class="reference"><a href="#cite_note-tip257-25">[25]</a></sup> so various extension packages emerged to enable object-oriented programming. They are wide spread in existing Tcl source code. Popular such extensions include:</p>/n<ul>/n<li><a href="/wiki/Incr_Tcl" title="Incr Tcl">incr Tcl</a></li>/n<li><a href="/wiki/XOTcl" title="XOTcl">XOTcl</a></li>/n<li><a href="/wiki/Itk" title="Itk">Itk</a></li>/n<li><a href="/wiki/Snit" title="Snit">Snit</a></li>/n<li><a rel="nofollow" class="external text" href="http://jfontain.free.fr/stooop.html">STOOOP</a></li>/n</ul>/n<p>TclOO was not only added to build a strong object oriented system, but also to enable extension packages to build object oriented abstractions using it as a foundation. After the release of TclOO <a href="/wiki/Incr_Tcl" title="Incr Tcl">incr Tcl</a> was updated to use TclOO as its foundation.<sup id="cite_ref-tcl86release_23-1" class="reference"><a href="#cite_note-tcl86release-23">[23]</a></sup></p>/n<h2><span class="mw-headline" id="Interfacing_with_other_languages">Interfacing with other languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=10" title="Edit section: Interfacing with other languages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Tcl interfaces natively with the <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> language.<sup id="cite_ref-tclcapi_34-0" class="reference"><a href="#cite_note-tclcapi-34">[34]</a></sup> This is because it was originally written to be a framework for providing a syntactic front-end to commands written in C, and all commands in the language (including things that might otherwise be <a href="/wiki/Keyword_(computer_programming)" class="mw-redirect" title="Keyword (computer programming)">keywords</a>, such as <code>if</code> or <code>while</code>) are implemented this way. Each command implementation <a href="/wiki/Subroutine" title="Subroutine">function</a> is passed an array of values that describe the (already substituted) arguments to the command, and is free to interpret those values as it sees fit.</p>/n<p>Digital <a href="/wiki/Logic_simulation" title="Logic simulation">logic simulators</a> often include a Tcl scripting interface for simulating <a href="/wiki/Verilog" title="Verilog">Verilog</a>, <a href="/wiki/VHDL" title="VHDL">VHDL</a> and SystemVerilog <a href="/wiki/Hardware_description_language" title="Hardware description language">hardware languages</a>.</p>/n<p>Tools exist (e.g. <a href="/wiki/SWIG" title="SWIG">SWIG</a>, <a rel="nofollow" class="external text" href="http://elf.org/ffidl">Ffidl</a>) to automatically generate the necessary code to connect arbitrary C functions and the Tcl runtime, and <a rel="nofollow" class="external text" href="http://equi4.com/starkit/critcl.html">Critcl</a> does the reverse, allowing embedding of arbitrary C code inside a Tcl script and compiling it at runtime into a <a href="/wiki/Dynamic-link_library" title="Dynamic-link library">DLL</a>.</p>/n<div role="note" class="hatnote navigation-not-searchable">See also: <a href="/wiki/Tcl/Java" title="Tcl/Java">Tcl/Java</a></div>/n<h2><span class="mw-headline" id="Extension_packages">Extension packages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Tcl&amp;action=edit&amp;section=11" title="Edit section: Extension packages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The Tcl language has always allowed for extension packages, which provide additional functionality, such as a GUI, terminal-based application automation, database access, and so on. Commonly used extensions include:</p>/n<dl>/n<dt>Tk</dt>/n<dd>The most popular Tcl extension is the <a href="/wiki/Tk_(software)" title="Tk (software)">Tk</a> toolkit, which provides a <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a> library for a variety of <a href="/wiki/Operating_system" title="Operating system">operating systems</a>. Each GUI consists of one or more <i>frames</i>. Each frame has a layout manager.</dd>/n<dt>Expect</dt>/n<dd>One of the other very popular Tcl extensions is <a href="/wiki/Expect" title="Expect">Expect</a> extension. The early close relationship of Expect with Tcl is largely responsible for the popularity of Tcl in prolific areas of use such as in Unix testing, where Expect was (and still is today) employed very successfully to automate telnet, ssh, and serial sessions to perform many repetitive tasks (i.e., scripting of formerly interactive-only applications). Tcl was the only way to run Expect, so Tcl became very popular in these areas of industry.</dd>/n<dt>Tile/Ttk</dt>/n<dd>Tile/Ttk<sup id="cite_ref-tktable1208_35-0" class="reference"><a href="#cite_note-tktable1208-35">[35]</a></sup> is a styles and theming widget collection that can replace most of the widgets in Tk with variants that are truly platform native through calls to an <a href="/wiki/Operating_system" title="Operating system">operating system</a>'s <a href="/wiki/API" class="mw-redirect" title="API">API</a>. Themes covered in this way are <a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a>, Windows Classic, <a href="/wiki/Qt_(toolkit)" class="mw-redirect" title="Qt (toolkit)">Qt</a> (that hooks into the <a href="/wiki/X11" class="mw-redirect" title="X11">X11</a> <a href="/wiki/KDE" title="KDE">KDE</a> environment libraries) and <a href="/wiki/Aqua_(user_interface)" title="Aqua (user interface)">Aqua</a> (Mac OS X). A theme can also be constructed without these calls using widget definitions supplemented with image <a href="/wiki/Pixmap" class="mw-redirect" title="Pixmap">pixmaps</a>. Themes created this way include Classic Tk, Step, Alt/Revitalized, Plastik and Keramik. Under Tcl 8.4, this package is known as Tile, while in Tcl 8.5 it has been folded into the core distribution of Tk (as Ttk).</dd>/n<dt>Tix</dt>/n<dd>Tix, the Tk Interface eXtension, is a set of user interface components that expand the capabilities of Tcl/Tk and Python applications. It is an open source software package maintained by volunteers in the Tix Project Group and released under a BSD-style license.<sup id="cite_ref-tixlic96_36-0" class="reference"><a href="#cite_note-tixlic96-36">[36]</a></sup></dd>/n<dt>Itcl/IncrTcl</dt>/n<dd><a href="/wiki/Itcl" class="mw-redirect" title="Itcl">Itcl</a> is an object system for Tcl, and is normally named as [incr Tcl] (that being the way to increment in Tcl, similar in fashion to the name C++).</dd>/n<dt>Tcllib</dt>/n<dd><a href="/wiki/Tcllib" title="Tcllib">Tcllib</a> is a set of scripted packages for Tcl that can be used with no compilation steps.</dd>/n<dt>Tklib</dt>/n<dd>Tklib is a collection of utility modules for Tk, and a companion to Tcllib.</dd>/n<dt>TclUDP</dt>/n<dd>The TclUDP<sup id="cite_ref-tcludp1109_37-0" class="reference"><a href="#cite_note-tcludp1109-37">[37]</a></sup> extension provides a simple library to support <a href="/wiki/User_Datagram_Protocol" title="User Datagram Protocol">User Datagram Protocol</a> (UDP) sockets in Tcl.</dd>/n<dt>Databases</dt>/n<dd>Tcl Database Connectivity (TDBC), part of Tcl 8.6, is a common database access interface for Tcl scripts. It currently supports drivers for accessing <a href="/wiki/MySQL" title="MySQL">MySQL</a>, <a href="/wiki/ODBC" class="mw-redirect" title="ODBC">ODBC</a>, <a href="/wiki/PostgreSQL" title="PostgreSQL">PostgreSQL</a> and <a href="/wiki/SQLite" title="SQLite">SQLite</a> databases. More are planned for the future. Access to databases is also supported through database-specific extensions, of which there are many available.<sup id="cite_ref-tdbc1202_38-0" class="reference"><a href="#cite_note-tdbc1202-38">[38]</a></sup></dd>/n</dl>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a
