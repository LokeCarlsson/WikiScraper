href="/wiki/C_(programming_language)" title="C (programming language)">C</a></td>/n</tr>/n</table>/n<p><b>ALGOL</b> (short for <b>Algorithmic Language</b>)<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> is a family of <a href="/wiki/Imperative_programming" title="Imperative programming">imperative</a> computer <a href="/wiki/Programming_language" title="Programming language">programming languages</a>, originally developed in the mid-1950s, which greatly influenced many other languages and was the standard method for <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> description used by the <a href="/wiki/Association_for_Computing_Machinery" title="Association for Computing Machinery">ACM</a> in textbooks and academic sources for more than thirty years.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup></p>/n<p>In the sense that the syntax of most modern languages is "Algol-like",<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> it was arguably the most influential of the four high-level programming languages with which it was roughly contemporary: <a href="/wiki/FORTRAN" class="mw-redirect" title="FORTRAN">FORTRAN</a>, <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a>, and <a href="/wiki/COBOL" title="COBOL">COBOL</a>.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup> It was designed to avoid some of the perceived problems with FORTRAN and eventually gave rise to many other programming languages, including <a href="/wiki/PL/I" title="PL/I">PL/I</a>, <a href="/wiki/Simula" title="Simula">Simula</a>, <a href="/wiki/BCPL" title="BCPL">BCPL</a>, <a href="/wiki/B_(programming_language)" title="B (programming language)">B</a>, <a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a>, and <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>.</p>/n<p>ALGOL introduced code blocks and the <code>begin</code>…<code>end</code> pairs for delimiting them. It was also the first language implementing <a href="/wiki/Nested_function" title="Nested function">nested function</a> definitions with <a href="/wiki/Lexical_scope" class="mw-redirect" title="Lexical scope">lexical scope</a>. Moreover, it was the first programming language which gave detailed attention to formal language definition and through the <i><a href="/wiki/Algol_60#History" class="mw-redirect" title="Algol 60">Algol 60 Report</a></i> introduced <a href="/wiki/Backus%E2%80%93Naur_form" title="Backus–Naur form">Backus–Naur form</a>, a principal <a href="/wiki/Formal_grammar" title="Formal grammar">formal grammar</a> notation for language design.</p>/n<p>There were three major specifications, named after the year they were first published:</p>/n<ul>/n<li><a href="/wiki/ALGOL_58" title="ALGOL 58">ALGOL 58</a> – originally proposed to be called <i>IAL</i>, for <i>International Algebraic Language</i>.</li>/n<li><a href="/wiki/ALGOL_60" title="ALGOL 60">ALGOL 60</a> – first implemented as <i>X1 ALGOL 60</i> in mid-1960. Revised 1963.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup><sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup></li>/n<li><a href="/wiki/ALGOL_68" title="ALGOL 68">ALGOL 68</a> – introduced new elements including flexible arrays, slices, parallelism, operator identification. Revised 1973.<sup id="cite_ref-a68_7-0" class="reference"><a href="#cite_note-a68-7">[7]</a></sup></li>/n</ul>/n<p><a href="/wiki/Niklaus_Wirth" title="Niklaus Wirth">Niklaus Wirth</a> based his own <a href="/wiki/ALGOL_W" title="ALGOL W">ALGOL W</a> on ALGOL 60 before developing <a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a>. ALGOL-W was based on the proposal for the next generation ALGOL, but the ALGOL 68 committee decided on a design that was more complex and advanced, rather than a cleaned, simplified ALGOL 60.</p>/n<p>ALGOL 68 is substantially different from ALGOL 60 and was not well received, so that in general "Algol" means ALGOL 60 and dialects thereof.</p>/n<p></p>/n<div id="toc" class="toc">/n<div links</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="Important_implementations">Important implementations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ALGOL&amp;action=edit&amp;section=1" title="Edit section: Important implementations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The <a href="/wiki/ALGOL_58" title="ALGOL 58">International Algebraic Language</a> (IAL) was extremely influential and generally considered the ancestor of most of the modern programming languages (the so-called Algol-like languages). Additionally, <b>ALGOL object code</b> was a simple, compact, and stack-based <a href="/wiki/Instruction_set" class="mw-redirect" title="Instruction set">instruction set architecture</a> commonly used in teaching <a href="/wiki/Compiler" title="Compiler">compiler</a> construction and other high order languages (of which Algol is generally considered the first).</p>/n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ALGOL&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>ALGOL was developed jointly by a committee of European and American computer scientists in a meeting in 1958 at <a href="/wiki/ETH_Zurich" title="ETH Zurich">ETH Zurich</a> (cf. <a href="/wiki/ALGOL_58" title="ALGOL 58">ALGOL 58</a>). It specified three different syntaxes: a reference syntax, a publication syntax, and an implementation syntax. The different syntaxes permitted it to use different keyword names and conventions for decimal points (commas vs periods) for different languages.</p>/n<p>ALGOL was used mostly by research computer scientists in the United States and in Europe. Its use in commercial applications was hindered by the absence of standard input/output facilities in its description and the lack of interest in the language by large computer vendors other than <a href="/wiki/Burroughs_Corporation" title="Burroughs Corporation">Burroughs Corporation</a>. ALGOL 60 did however become the standard for the publication of algorithms and had a profound effect on future language development.</p>/n<p><a href="/wiki/John_Backus" title="John Backus">John Backus</a> developed the <a href="/wiki/Backus%E2%80%93Naur_form" title="Backus–Naur form">Backus normal form</a> method of describing programming languages specifically for ALGOL 58. It was revised and expanded by <a href="/wiki/Peter_Naur" title="Peter Naur">Peter Naur</a> for ALGOL 60, and at <a href="/wiki/Donald_Knuth" title="Donald Knuth">Donald Knuth</a>'s suggestion renamed <a href="/wiki/Backus%E2%80%93Naur_form" title="Backus–Naur form">Backus–Naur form</a>.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup></p>/n<p>Peter Naur: "As editor of the ALGOL Bulletin I was drawn into the international discussions of the language and was selected to be member of the European language design group in November 1959. In this capacity I was the editor of the ALGOL 60 report, produced as the result of the ALGOL 60 meeting in Paris in January 1960."<sup id="cite_ref-naur_acm_9-0" class="reference"><a href="#cite_note-naur_acm-9">[9]</a></sup></p>/n<p>The following people attended the meeting in Paris (from 1 to 16 January):</p>/n<ul>/n<li><a href="/wiki/Friedrich_L._Bauer" title="Friedrich L. Bauer">Friedrich L. Bauer</a>, <a href="/wiki/Peter_Naur" title="Peter Naur">Peter Naur</a>, <a href="/wiki/Heinz_Rutishauser" title="Heinz Rutishauser">Heinz Rutishauser</a>, <a href="/wiki/Klaus_Samelson" title="Klaus Samelson">Klaus Samelson</a>, <a href="/w/index.php?title=Bernard_Vauquois&amp;action=edit&amp;redlink=1" class="new" title="Bernard Vauquois (page does not exist)">Bernard Vauquois</a>, <a href="/wiki/Adriaan_van_Wijngaarden" title="Adriaan van Wijngaarden">Adriaan van Wijngaarden</a>, and <a href="/wiki/Michael_Woodger" class="mw-redirect" title="Michael Woodger">Michael Woodger</a> (from Europe)</li>/n<li><a href="/wiki/John_Backus" title="John Backus">John W. Backus</a>, <a href="/w/index.php?title=Julien_Green_(computer_scientist)&amp;action=edit&amp;redlink=1" class="new" title="Julien Green (computer scientist) (page does not exist)">Julien Green</a>, <a href="/wiki/Charles_Katz" title="Charles Katz">Charles Katz</a>, <a href="/wiki/John_McCarthy_(computer_scientist)" title="John McCarthy (computer scientist)">John McCarthy</a>, <a href="/wiki/Alan_Perlis" title="Alan Perlis">Alan J. Perlis</a>, and <a href="/wiki/Joseph_Henry_Wegstein" title="Joseph Henry Wegstein">Joseph Henry Wegstein</a> (from the USA).</li>/n</ul>/n<p>Alan Perlis gave a vivid description of the meeting: "The meetings were exhausting, interminable, and exhilarating. One became aggravated when one's good ideas were discarded along with the bad ones of others. Nevertheless, diligence persisted during the entire period. The chemistry of the 13 was excellent."</p>/n<p>ALGOL 60 inspired many languages that followed it. <a href="/wiki/C._A._R._Hoare" class="mw-redirect" title="C. A. R. Hoare">C. A. R. Hoare</a> remarked: "Here is a language so far ahead of its time that it was not only an improvement on its predecessors but also on nearly all its successors."<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">[10]</a></sup> The <a href="/wiki/Scheme_(programming_language)" title="Scheme (programming language)">Scheme</a> programming language, a variant of <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a> that adopted the block structure and lexical scope of ALGOL, also adopted the wording "Revised Report on the Algorithmic Language Scheme" for its standards documents in homage to ALGOL.<sup id="cite_ref-r3rs_11-0" class="reference"><a href="#cite_note-r3rs-11">[11]</a></sup></p>/n<h3><span class="mw-headline" id="Algol_and_programming_language_research">Algol and programming language research</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ALGOL&amp;action=edit&amp;section=3" title="Edit section: Algol and programming language research">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>As <a href="/wiki/Peter_Landin" title="Peter Landin">Peter Landin</a> noted, the language Algol was the first language to combine seamlessly imperative effects with the (call-by-name) <a href="/wiki/Lambda_calculus" title="Lambda calculus">lambda calculus</a>. Perhaps the most elegant formulation of the language is due to <a href="/wiki/John_C._Reynolds" title="John C. Reynolds">John C. Reynolds</a>, and it best exhibits its syntactic and semantic purity. Reynolds's idealized Algol also made a convincing methodological argument regarding the suitability of local effects in the context of call-by-name languages, to be contrasted with the global effects used by call-by-value languages such as <a href="/wiki/ML_(programming_language)" title="ML (programming language)">ML</a>. The conceptual integrity of the language made it one of the main objects of semantic research, along with <a href="/wiki/Programming_language_for_Computable_Functions" class="mw-redirect" title="Programming language for Computable Functions">PCF</a> and ML.<sup id="cite_ref-12" class="reference"><a href="#cite_note-12">[12]</a></sup></p>/n<h3><span class="mw-headline" id="IAL_implementations_timeline">IAL implementations timeline</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ALGOL&amp;action=edit&amp;section=4" title="Edit section: IAL implementations timeline">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>To date there have been at least 70 augmentations, extensions, derivations and sublanguages of Algol 60.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13">[13]</a></sup></p>/n<table class="wikitable sortable">/n<tr>/n<th>Name</th>/n<th>Year</th>/n<th>Author</th>/n<th>Country</th>/n<th>Description</th>/n<th>Target CPU</th>/n</tr>/n<tr>/n<td>ZMMD-implementation</td>/n<td>1958</td>/n<td><a href="/wiki/Friedrich_L._Bauer" title="Friedrich L. The latter is still used for Unisys MCP system software.</p>/n<h2><span class="mw-headline" id="Properties">Properties</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ALGOL&amp;action=edit&amp;section=5" title="Edit section: Properties">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>ALGOL 60 as officially defined had no <a href="/wiki/I/O" class="mw-redirect" title="I/O">I/O</a> facilities; implementations defined their own in ways that were rarely compatible with each other. In contrast, ALGOL 68 offered an extensive library of <i>transput</i> (input/output) facilities.</p>/n<p>ALGOL 60 allowed for two <a href="/wiki/Evaluation_strategy" title="Evaluation strategy">evaluation strategies</a> for <a href="/wiki/Parameter_(computer_science)" class="mw-redirect" title="Parameter (computer science)">parameter</a> passing: the common <a href="/wiki/Call-by-value" class="mw-redirect" title="Call-by-value">call-by-value</a>, and <a href="/wiki/Call-by-name" class="mw-redirect" title="Call-by-name">call-by-name</a>. Call-by-name has certain effects in contrast to <a href="/wiki/Call-by-reference" class="mw-redirect" title="Call-by-reference">call-by-reference</a>. For example, without specifying the parameters as <i>value</i> or <i>reference</i>, it is impossible to develop a procedure that will swap the values of two parameters if the actual parameters that are passed in are an integer variable and an array that is indexed by that same integer variable.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19">[19]</a></sup> Think of passing a pointer to swap(i, A[i]) in to a function. Now that every time swap is referenced, it is reevaluated. Say i&#160;:= 1 and A[i]&#160;:= 2, so every time swap is referenced it'll return the other combination of the values ([1,2], [2,1], [1,2] and so on). A similar situation occurs with a random function passed as actual argument.</p>/n<p>Call-by-name is known by many compiler designers for the interesting "<a href="/wiki/Thunk_(delayed_computation)" class="mw-redirect" title="Thunk (delayed computation)">thunks</a>" that are used to implement it. <a href="/wiki/Donald_Knuth" title="Donald Knuth">Donald Knuth</a> devised the "<a href="/wiki/Man_or_boy_test" title="Man or boy test">man or boy test</a>" to separate compilers that correctly implemented "<a href="/wiki/Recursion_(computer_science)" title="Recursion (computer science)">recursion</a> and non-local references." This test contains an example of call-by-name.</p>/n<p>ALGOL 68 was defined using a two-level grammar formalism invented by <a href="/wiki/Adriaan_van_Wijngaarden" title="Adriaan van Wijngaarden">Adriaan van Wijngaarden</a> and which bears his name. <a href="/wiki/Van_Wijngaarden_grammar" title="Van Wijngaarden grammar">Van Wijngaarden grammars</a> use a <a href="/wiki/Context-free_grammar" title="Context-free grammar">context-free grammar</a> to generate an infinite set of productions that will recognize a particular ALGOL 68 program; notably, they are able to express the kind of requirements that in many other programming language standards are labelled "semantics" and have to be expressed in ambiguity-prone natural language prose, and then implemented in compilers as <i>ad hoc</i> code attached to the formal language parser.</p>/n<h2><span class="mw-headline" id="Examples_and_portability_issues">Examples and portability issues</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ALGOL&amp;action=edit&amp;section=6" title="Edit section: Examples and portability issues">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<h3><span class="mw-headline" id="Code_sample_comparisons">Code sample comparisons</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ALGOL&amp;action=edit&amp;section=7" title="Edit section: Code sample comparisons">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<h4><span class="mw-headline" id="ALGOL_60">ALGOL 60</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ALGOL&amp;action=edit&amp;section=8" title="Edit section: ALGOL 60">edit</a><span class="mw-editsection-bracket">]</span></span></h4>/n<p>(The way the bold text has to be written depends on the implementation, e.g. 'INTEGER' -- quotation marks included -- for <b>integer</b>. section: ALGOL 58 (IAL)">edit</a><span class="mw-editsection-bracket">]</span></span></h4>/n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/ALGOL_58" title="ALGOL 58">ALGOL 58</a></div>/n<p>ALGOL 58 had no I/O facilities.</p>/n<h4><span class="mw-headline" id="ALGOL_60_family">ALGOL 60 family</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ALGOL&amp;action=edit&amp;section=12" title="Edit section: ALGOL 60 family">edit</a><span class="mw-editsection-bracket">]</span></span></h4>/n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/ALGOL_60" title="ALGOL 60">ALGOL 60</a></div>/n<p>Since ALGOL 60 had no I/O facilities, there is no portable <a href="/wiki/Hello_world_program" class="mw-redirect" title="Hello world program">hello world program</a> in ALGOL.</p>/n<pre>/nBEGIN/n  FILE F(KIND=REMOTE);/n  EBCDIC ARRAY E[0:11];/n  REPLACE E BY "HELLO WORLD!";/n  WRITE(F, *, E);/nEND./n</pre>/n<p>A simpler program using an inline format:</p>/n<pre>/nBEGIN/n  FILE F(KIND=REMOTE);/n  WRITE(F, &lt;"HELLO WORLD!"&gt;);/nEND./n</pre>/n<p>An even simpler program using the Display statement:</p>/n<pre>/nBEGIN DISPLAY("HELLO WORLD!") END./n</pre>/n<p>An alternative example, using Elliott Algol I/O is as follows. Elliott Algol used different characters for "open-string-quote" and "close-string-quote":</p>/n<pre>/n <b>program</b> HiFolks;/n <b>begin</b>/n    <b>print</b> ‘Hello world’;/n <b>end</b>;/n</pre>/n<p>Here's a version for the Elliott 803 Algol (A104) The standard Elliott 803 used 5 hole paper tape and thus only had upper case. The code lacked any quote characters so £ (UK Pound Sign) was used for open quote and&#160;? (Question Mark) for close quote. Special sequences were placed in double quotes (e.g. ££L?? produced a new line on the teleprinter).</p>/n<pre>/n  HIFOLKS'/n  BEGIN/n     PRINT £HELLO WORLD£L??'/n  END'/n</pre>/n<p>The <a href="/wiki/ICT_1900_series" title="ICT 1900 series">ICT 1900 series</a> Algol I/O version allowed input from paper tape or punched card. Paper tape 'full' mode allowed lower case. Output was to a line printer. The open and close quote characters were represented using '(' and ')' and spaces by&#160;%.<sup id="cite_ref-21" class="reference"><a href="#cite_note-21">[21]</a></sup></p>/n<pre>/n  'BEGIN'/n     WRITE TEXT('('HELLO%WORLD')');/n  'END'/n</pre>/n<h4><span class="mw-headline" id="ALGOL_68_2">ALGOL 68</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ALGOL&amp;action=edit&amp;section=13" title="Edit section: ALGOL 68">edit</a><span class="mw-editsection-bracket">]</span></span></h4>/n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/ALGOL_68" title="ALGOL 68">ALGOL 68</a></div>/n<p><b>ALGOL 68</b> code was published with reserved words typically in lowercase, but bolded or underlined.</p>/n<pre>/n<b>begin</b>/n  printf(($gl$,"Hello, world!"))/n<b>end</b>/n</pre>/n<p>In the language of the "Algol 68 Report" the input/output facilities were collectively called the "<a href="/wiki/Input/output" title="Input/output">Transput</a>".</p>/n<h3><span class="mw-headline" id="Timeline_of_ALGOL_special_characters">Timeline of ALGOL special characters</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ALGOL&amp;action=edit&amp;section=14" title="Edit section: Timeline of ALGOL special characters">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<table role="presentation" class="metadata mbox-small noprint selfref" style="background-color:#f9f9f9;border:1px solid #aaa;color:#000;width: 22em;">/n<tr>/n<td style="width:1px"></td>/n<td class="mbox-text plainlist" style="vertical-align:middle; font-size:95%;"><b>This article contains Unicode 6.0 "<a rel="nofollow" class="external text" href="http://www.unicode.org/charts/PDF/U2300.pdf">Miscellaneous Technical</a>" characters.</b> Without proper <a href="/wiki/Unicode#External_links" title="Unicode">rendering support</a>, you may see <a href="/wiki/Specials_(Unicode_block)#Replacement_character" title="Specials (Unicode block)">question marks, boxes, or other symbols</a> instead of something like "₁₀" (<a rel="nofollow" class="external text" href="http://mailcom.com/unicode/DecimalExponent.ttf">Decimal Exponent Symbol U+23E8 TTF</a>).</td>/n</tr>/n</table>/n<p>The ALGOLs were conceived at a time when character sets were diverse and evolving rapidly; also, the ALGOLs were defined so that only <i>uppercase</i> letters were required.</p>/n<p>1960: <a href="/wiki/IFIP" class="mw-redirect" title="IFIP">IFIP</a> – The Algol 60 language and report included several mathematical symbols which are available on modern computers and operating systems, but, unfortunately, were not supported on most computing systems at the time. For instance: ×, ÷, ≤, ≥, ≠, ¬, ∨, ∧, ⊂, ≡, ␣ and ⏨.</p>/n<p>1961 September: ASCII&#160;– The <a href="/wiki/ASCII" title="ASCII">ASCII</a> character set, then in an early stage of development, had the <a href="/wiki/%5C" class="mw-redirect" title="\">\</a> (Back slash) character added to it in order to support ALGOL's <a href="/wiki/Boolean_data_type" title="Boolean data type">boolean</a> operators <a href="/wiki/%5C/" class="mw-redirect" title="\/">/\</a> and <a href="/wiki/%5C/" class="mw-redirect" title="\/">\/</a>.<sup id="cite_ref-22" class="reference"><a href="#cite_note-22">[22]</a></sup></p>/n<p>1962: <a href="/wiki/ALCOR" title="ALCOR">ALCOR</a> – This character set included the unusual "᛭" (iron/runic cross<sup id="cite_ref-23" class="reference"><a href="#cite_note-23">[23]</a></sup>) character and the "⏨" (Decimal Exponent Symbol<sup id="cite_ref-unicode1_24-0" class="reference"><a href="#cite_note-unicode1-24">[24]</a></sup>) for floating point notation.<sup id="cite_ref-25" class="reference"><a href="#cite_note-25">[25]</a></sup><sup id="cite_ref-26" class="reference"><a href="#cite_note-26">[26]</a></sup><sup id="cite_ref-27" class="reference"><a href="#cite_note-27">[27]</a></sup></p>/n<p>1964: <a href="/wiki/GOST" title="GOST">GOST</a> – The 1964 Soviet standard <a href="/wiki/GOST_10859" title="GOST 10859">GOST 10859</a> allowed the encoding of 4-bit, 5-bit, 6-bit and 7-bit characters in ALGOL.<sup id="cite_ref-28" class="reference"><a href="#cite_note-28">[28]</a></sup></p>/n<p>1968: The "Algol 68 Report" – used existing ALGOL characters, and further adopted →, ↓, ↑, □, ⌊, ⌈, ⎩, ⎧, ○, ⊥ and ¢ characters which can be found on the <a href="/wiki/IBM_2741" title="IBM 2741">IBM 2741</a> keyboard with "golf-ball" print heads inserted (such as the <a href="/wiki/APL_syntax_and_symbols" title="APL syntax and symbols">APL</a> golfball). These became available in the mid-1960s while ALGOL 68 was being drafted. The report was translated into Russian, German, French and Bulgarian, and allowed programming in languages with larger character sets, e.g. <a href="/wiki/Cyrillic" class="mw-redirect" title="Cyrillic">Cyrillic</a> alphabet of the Soviet <a href="/wiki/BESM" title="BESM">BESM-4</a>. All ALGOL's characters are also part of the <a href="/wiki/Unicode" title="Unicode">Unicode</a> standard and most of them are available in several popular <a href="/wiki/Font" title="Font">fonts</a>.</p>/n<p>2009 October: <a href="/wiki/Unicode" title="Unicode">Unicode</a> – The "⏨" (Decimal Exponent Symbol<sup id="cite_ref-unicode1_24-1" class="reference"><a href="#cite_note-unicode1-24">[24]</a></sup>) for floating point notation was added to Unicode 5.2 for backward compatibility with historic <a href="/wiki/Buran_(spacecraft)" title="Buran (spacecraft)">Buran (spacecraft)</a> ALGOL software.<sup id="cite_ref-29" class="reference"><a href="#cite_note-29">[29]</a></sup></p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=ALGOL&amp;action=edit&amp;section=15" title="Edit section: See also">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div>/n<table class="multicol" role="presentation" style="border-collapse: collapse;
