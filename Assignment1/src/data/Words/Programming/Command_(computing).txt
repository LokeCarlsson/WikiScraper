It is not to be confused with <a href="/wiki/System_call" title="System call">system call</a>.</div>/n<div role="note" class="hatnote navigation-not-searchable">For other uses, see <a href="/wiki/Command_(disambiguation)#Computing" class="mw-redirect mw-disambig" title="Command (disambiguation)">Command#Computing</a>.</div>/n<table class="plainlinks metadata ambox ambox-content ambox-Unreferenced" role="presentation">/n<tr>/n<td class="mbox-image">/n<div style="width:52px"><a href="/wiki/File:Question_book-new.svg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/50px-Question_book-new.svg.png" width="50" height="39" srcset="//upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/75px-Question_book-new.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/100px-Question_book-new.svg.png 2x" data-file-width="512" data-file-height="399" /></a></div>/n</td>/n<td class="mbox-text">/n<div class="mbox-text-span">This article <b>does not <a href="/wiki/Wikipedia:Citing_sources" title="Wikipedia:Citing sources">cite</a> any <a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability">sources</a></b>. <span class="hide-when-compact">Please help <a class="external text" href="//en.wikipedia.org/w/index.php?title=Command_(computing)&amp;action=edit">improve this article</a> by <a href="/wiki/Help:Introduction_to_referencing_with_Wiki_Markup/1" title="Help:Introduction to referencing with Wiki Markup/1">adding citations to reliable sources</a>. Unsourced material may be challenged and <a href="/wiki/Wikipedia:Verifiability#Burden_of_evidence" title="Wikipedia:Verifiability">removed</a>.</span> <small><i>(May 2008)</i></small> <small class="hide-when-compact"><i>(<a href="/wiki/Help:Maintenance_template_removal" title="Help:Maintenance template removal">Learn how and when to remove this template message</a>)</i></small></div>/n</td>/n</tr>/n</table>/n<p>In <a href="/wiki/Computing" title="Computing">computing</a>, a <b>command</b> is a directive to a computer program acting as an interpreter of some kind, in order to perform a specific task. Most commonly a command is either a directive to some kind of <a href="/wiki/Command-line_interface" title="Command-line interface">command-line interface</a>, such as a <a href="/wiki/Shell_(computing)" title="Shell (computing)">shell</a>, or an event in a <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a> triggered by the user selecting an option in a <a href="/wiki/Menu_(computing)" title="Menu (computing)">menu</a>.</p>/n<p>Specifically, the term <i>command</i> is used in <a href="/wiki/Imperative_programming" title="Imperative programming">imperative</a> <a href="/wiki/Computer_language" title="Computer language">computer languages</a>. These languages are called this, because <a href="/wiki/Statement_(programming)" class="mw-redirect" title="Statement (programming)">statements</a> in these languages are usually written in a manner similar to the <a href="/wiki/Imperative_mood" title="Imperative mood">imperative mood</a> used in many <a href="/wiki/Natural_language" title="Natural language">natural languages</a>. If one views a statement in an <a href="/wiki/Imperative_language" class="mw-redirect" title="Imperative language">imperative language</a> as being like a sentence in a natural language, then a command is generally like a verb in such a language.</p>/n<p>Many programs allow specially formatted <a href="/wiki/Command-line_argument" class="mw-redirect" title="Command-line argument">arguments</a>, known as flags or options, which modify the default behaviour of the command, while further <a href="/wiki/Parameter_(computer_science)" class="mw-redirect" title="Parameter (computer science)">arguments</a> describe what the command acts on. Comparing to a natural language: the flags are adverbs, whilst the other arguments are <a href="/wiki/Object_(grammar)" title="Object (grammar)">objects</a>.</p>/n<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command_(computing)&amp;action=edit&amp;section=1" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Here are some commands given to a <a href="/wiki/Command-line_interpreter" class="mw-redirect" title="Command-line interpreter">command-line interpreter</a> (<a href="/wiki/Unix_shell" title="Unix shell">Unix shell</a>).</p>/n<p>The following command changes the user's place in the <a href="/wiki/Directory_tree" class="mw-redirect" title="Directory tree">directory tree</a> from their current position to the directory <tt>/home/pete</tt>. The command is <tt><a href="/wiki/Cd_(command)" title="Cd (command)">cd</a></tt> and the argument is <tt>/home/pete</tt></p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="nb">cd</span> /home/pete/n</pre></div>/n<p>The following command prints the text <tt>Hello World</tt> out to the <a href="/wiki/Standard_output" class="mw-redirect" title="Standard output">standard output</a> stream, which, in this case, will just print the text out on the screen. The command is <tt><a href="/wiki/Echo_(command)" title="Echo (command)">echo</a></tt> and the argument is <tt>"Hello World"</tt>. The quotes are used to prevent <tt>Hello</tt> and <tt>World</tt> being treated as separate arguments:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="nb">echo</span> <span class="s2">"Hello World"</span>/n</pre></div>/n<p>The following commands are equivalent. They list files in the directory <tt>/bin</tt>. The command is <tt><a href="/wiki/Ls" title="Ls">ls</a></tt>, the argument is <tt><a href="/wiki/Filesystem_Hierarchy_Standard" title="Filesystem Hierarchy Standard">/bin</a></tt> and there are three flags: <tt>-l</tt>, <tt>-t</tt> and <tt>-r</tt>.</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n ls -l -t -r  /bin/n ls -ltr  /bin/n</pre></div>/n<p>The following command displays the contents of the files <tt>ch1.txt</tt> and <tt>ch2.txt</tt>. The command is <tt><a href="/wiki/Cat_(Unix)" title="Cat (Unix)">cat</a></tt>, and <tt>ch1.txt</tt> and <tt>ch2.txt</tt> are both arguments.</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n cat ch1.txt ch2.txt/n</pre></div>/n<p>Here are some commands given to a different command-line interpreter (the <a href="/wiki/DOS" title="DOS">DOS</a>, <a href="/wiki/OS/2" title="OS/2">OS/2</a> and <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> command prompt). Notice that the flags are identified differently but that the concepts are the same:</p>/n<p>The following command lists all the contents of the current directory. The command is <tt><a href="/wiki/Dir_(command)" title="Dir (command)">dir</a></tt>, and "A" is a flag. There is no argument.</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n dir /A/n</pre></div>/n<p>The following command displays the contents of the file readme.txt. The command is <tt><a href="/wiki/TYPE_(DOS_command)" title="TYPE (DOS command)">type</a></tt>. The argument is "readme.txt". "P" is a parameter.</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="nb">type</span> /P readme.txt/n</pre></div>/n<h2><span class="mw-headline" id="See_also">See
