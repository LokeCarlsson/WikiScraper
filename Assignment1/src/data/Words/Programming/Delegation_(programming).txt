id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p><b>Delegation</b>, in <a href="/wiki/Computing" title="Computing">computing</a> or <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a>, refers generally to one entity passing something to another entity,<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> and narrowly to various specific forms of relationships. These include:</p>/n<ul>/n<li><a href="/wiki/Delegation_(object-oriented_programming)" title="Delegation (object-oriented programming)">Delegation (object-oriented programming)</a>, evaluating a member of one object (the receiver) in the context of another, original object (the sender)./n<ul>/n<li><a href="/wiki/Delegation_pattern" title="Delegation pattern">Delegation pattern</a>, a design pattern implementing this feature.</li>/n<li><a href="/wiki/Forwarding_(object-oriented_programming)" title="Forwarding (object-oriented programming)">Forwarding (object-oriented programming)</a>, an often-confused technique where a sending object uses the corresponding member of another object, without the receiving object having any knowledge of the original, sending object.</li>/n<li><a href="/wiki/Object_aggregation" class="mw-redirect" title="Object aggregation">Object aggregation</a> or consultation, general term for one object using another.</li>/n</ul>/n</li>/n<li><a href="/wiki/Delegation_(computer_security)" title="Delegation (computer security)">Delegation (computer security)</a>, one user or process allowing another user or process to use their credentials or permissions.</li>/n<li><a href="/wiki/Delegate_(CLI)" title="Delegate (CLI)">Delegate (CLI)</a>, a form of type-safe function pointer used by the Common Language Infrastructure (CLI), specifying both a method to call and optionally an object to call the method on.</li>/n</ul>/n<h2><span class="mw-headline" id="References">References</span><span class="mw-editsection"><span
