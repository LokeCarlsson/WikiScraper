The use of <i><a href="https://en.wiktionary.org/wiki/esoteric" class="extiw" title="wiktionary:esoteric">esoteric</a></i> distinguishes these languages from programming languages that working developers use to write software. Usually, an esolang's creators do not intend the language to be used for mainstream programming, although some esoteric features, such as <a href="/wiki/Visuospatial" class="mw-redirect" title="Visuospatial">visuospatial</a> <a href="/wiki/Syntax_(programming_languages)" title="Syntax (programming languages)">syntax</a>,<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> have inspired practical applications in the arts. Such languages are often popular among <a href="/wiki/Hacker_(hobbyist)" class="mw-redirect" title="Hacker (hobbyist)">hackers</a> and hobbyists.</p>/n<p>Usability is rarely a goal for esoteric programming language designers—often it is quite the opposite. Their usual aim is to remove or replace conventional language features while still maintaining a language that is <a href="/wiki/Turing-complete" class="mw-redirect" title="Turing-complete">Turing-complete</a>, or even one for which the <a href="/wiki/Computability_theory_(computer_science)" class="mw-redirect" title="Computability theory (computer science)">computational class</a> is unknown.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Esoteric_programming_language&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The earliest, and still the canonical example of an esoteric language was <a href="/wiki/INTERCAL" title="INTERCAL">INTERCAL</a>,<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> designed in 1972 by <a href="/wiki/Don_Woods_(programmer)" title="Don Woods (programmer)">Don Woods</a> and James M. Lyon, with the stated intention of being unlike any other programming language the authors were familiar with.<sup id="cite_ref-Raymond1996_3-0" class="reference"><a href="#cite_note-Raymond1996-3">[3]</a></sup><sup id="cite_ref-woods-lyon-intercal_4-0" class="reference"><a href="#cite_note-woods-lyon-intercal-4">[4]</a></sup> It <a href="/wiki/Parody" title="Parody">parodied</a> elements of established programming languages of the day, such as <a href="/wiki/Fortran" title="Fortran">Fortran</a>, <a href="/wiki/COBOL" title="COBOL">COBOL</a>, and <a href="/wiki/Assembly_language" title="Assembly language">assembly language</a>.</p>/n<p>For many years INTERCAL was represented only by paper copies of the INTERCAL manual. The language's revival in 1990 as an implementation in C under <a href="/wiki/Unix" title="Unix">Unix</a> stimulated a wave of interest in the intentional design of esoteric computer languages.</p>/n<p>In 1993, <a href="/w/index.php?title=Wouter_van_Oortmerssen&amp;action=edit&amp;redlink=1" class="new" title="Wouter van Oortmerssen (page does not exist)">Wouter van Oortmerssen</a> created FALSE, a small <a href="/wiki/Stack-oriented_programming_language" title="Stack-oriented programming language">stack-oriented programming language</a>, with syntax designed to make the code inherently obfuscated, confusing, and unreadable. It also has a compiler of only 1024 bytes.<sup id="cite_ref-Wouter_5-0" class="reference"><a href="#cite_note-Wouter-5">[5]</a></sup> This inspired Urban Müller to create an even smaller language, the now-infamous <a href="/wiki/Brainfuck" title="Brainfuck">brainfuck</a>, which consists of only eight recognized characters. Along with Chris Pressey's <a href="/wiki/Befunge" title="Befunge">Befunge</a> (like FALSE, but with a two-dimensional instruction pointer), brainfuck is now one of the best-supported esoteric programming languages. These are canonical examples of minimal <a href="/wiki/Turing_tarpit" title="Turing tarpit">Turing tarpits</a> and needlessly obfuscated language features. Brainfuck is related to the <a href="/wiki/P%E2%80%B2%E2%80%B2" title="P′′">P′′</a> family of <a href="/wiki/Turing_machine" title="Turing machine">Turing machines</a>.</p>/n<h2><span class="mw-headline" id="Esoteric_programming_terms">Esoteric programming terms</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Esoteric_programming_language&amp;action=edit&amp;section=2" title="Edit section: Esoteric programming terms">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<h3><span class="mw-headline" id="Turing_tarpit">Turing tarpit</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Esoteric_programming_language&amp;action=edit&amp;section=3" title="Edit section: Turing tarpit">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Turing_tarpit" title="Turing tarpit">Turing tarpit</a></div>/n<p>A <a href="/wiki/Turing_tarpit" title="Turing tarpit">Turing tarpit</a> is a <a href="/wiki/Turing-complete" class="mw-redirect" title="Turing-complete">Turing-complete</a> programming language in which any computable function could theoretically be written, but in which it is impractically difficult to do so. Esoteric languages may be described as Turing tarpits, especially when they aim to minimize the number of language features.</p>/n<h3><span class="mw-headline" id="Stateful_encoding">Stateful encoding</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Esoteric_programming_language&amp;action=edit&amp;section=4" title="Edit section: Stateful encoding">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>A method of encoding programs, such that each substring of the encoding is an instruction to both:</p>/n<ol>/n<li>Locate the next instruction in a list, and</li>/n<li>Apply it to transform the current program state.</li>/n</ol>/n<p>A single instruction always includes two sequential phases: choosing an operation, and executing it. The list of operations may either be static—as in <a rel="nofollow" class="external text" href="http://esolangs.org/wiki/ReMorse">reMorse</a> or <a rel="nofollow" class="external text" href="http://esolangs.org/wiki/THRAT">THRAT</a>—or dynamic—as in reMorse4ever.</p>/n<p>Here is an example based on reMorse or THRAT:</p>/n<pre>/nSelect Next Operation in list/nPerform Operation/n</pre>/n<h3><span class="mw-headline" id="Language_paradigm">Language paradigm</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Esoteric_programming_language&amp;action=edit&amp;section=5" title="Edit section: Language paradigm">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>The <a href="/wiki/Programming_paradigm" title="Programming paradigm">paradigm of a language</a> can fall into a number of categories, and these categories are used to get a general understanding of the way that a specific language operates. These include imperative languages such as <a href="/wiki/Brainfuck" title="Brainfuck">brainfuck</a>, in which instructions describe how to change data; functional languages such as <a href="/wiki/Unlambda_programming_language" class="mw-redirect" title="Unlambda programming language">Unlambda</a>, in which data and code are more or less interchangeable and execution is the repeated application of functions to the results of other functions; and rewriting languages such as <a href="/wiki/Thue_(programming_language)" title="Thue (programming language)">Thue</a>, in which transformation functions are applied to an initial state.</p>/n<h3><span class="mw-headline" id="Funges">Funges</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Esoteric_programming_language&amp;action=edit&amp;section=6" title="Edit section: Funges">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>A <b>funge</b> is an esoteric programming language which models its programs as <a href="/wiki/Metric_space" title="Metric space">metric spaces</a> with <a href="/wiki/Coordinate_systems" class="mw-redirect" title="Coordinate systems">coordinate systems</a> (often, but not necessarily, <a href="/wiki/Cartesian_coordinate_system" title="Cartesian coordinate system">Cartesian</a>) and which execute instructions located at points in their program space by moving an instruction pointer (a <a href="/wiki/Position_vector" class="mw-redirect" title="Position vector">position vector</a> which indicates the currently executing instruction) through that space. Different instructions determine the direction in which the instruction pointer moves, and consequently, the sequence of instructions that is executed.</p>/n<p>The current official standard for the behaviour of these programming languages is the Funge-98 specification. This specification is a generalisation of the semantics of the <a href="/wiki/Befunge" title="Befunge">Befunge</a> programming language, which has a two-<a href="/wiki/Dimension" title="Dimension">dimensional</a> <a href="/wiki/Torus" title="Torus">toroidal</a> topology. Languages which adhere closely to this standard, such as Unefunge (one-dimensional) and Trefunge (three-dimensional), are sometimes called <i>funges</i>, while more "distant relatives" which differ in significant respects, such as Wierd, are referred to as <i>fungeoids</i>.</p>/n<h3><span class="mw-headline" id="One_instruction_set_computer">One instruction set computer</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Esoteric_programming_language&amp;action=edit&amp;section=7" title="Edit section: One instruction set computer">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>A <a href="/wiki/One_instruction_set_computer" title="One instruction set computer">one instruction set computer</a> is a machine which supports only one operation.</p>/n<h3><span class="mw-headline" id="Nondeterministic_language">Nondeterministic language</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Esoteric_programming_language&amp;action=edit&amp;section=8" title="Edit section: Nondeterministic language">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>For a deterministic language, if one is given the current state of a program, the next state can always be predicted. This is not true for a <a href="/wiki/Nondeterministic_programming" title="Nondeterministic programming">nondeterministic language</a>. Most languages are deterministic, but some languages, such as Befunge, supply a built-in randomization instruction. Moreover, languages such as <a rel="nofollow" class="external text" href="http://p-nand-q.com/programming/languages/java2k/">Java2k</a> have only randomized instructions. Thus, getting even trivial programs to have a reliable output is often a monumental, if not outright impossible, task.</p>/n<p>Nondeterministic languages can be used to explore large search spaces, such as grammars, where exhaustive search is impractical. Random text generators such as <a rel="nofollow" class="external text" href="http://dev.null.org/dadaengine/">the Dada Engine</a> and <a rel="nofollow" class="external text" href="http://sourceforge.net/projects/rmutt/">rmutt</a> are examples of this kind of nondeterminstic language.</p>/n<p>More esoterically, nondeterministic algorithms have been employed in the theoretical investigation of <a href="/wiki/Hypercomputation" title="Hypercomputation">hypercomputation</a>.</p>/n<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Esoteric_programming_language&amp;action=edit&amp;section=9" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Below are some characteristic examples of esoteric programming languages:</p>/n<h3><span class="mw-headline" id="AsciiDots">AsciiDots</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Esoteric_programming_language&amp;action=edit&amp;section=10" title="Edit section: AsciiDots">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p><a href="/w/index.php?title=AsciiDots&amp;action=edit&amp;redlink=1" class="new" title="AsciiDots (page does not exist)">AsciiDots</a> is an esoteric programming language based on ascii art.
