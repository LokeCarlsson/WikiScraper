id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p>A <b>system programming language</b> usually refers to a programming language used for <a href="/wiki/System_programming" title="System programming">system programming</a>; such languages are designed for writing <a href="/wiki/System_software" title="System software">system software</a>, which usually requires different development approaches when compared with <a href="/wiki/Application_software" title="Application software">application software</a>.</p>/n<p>System software is computer software designed to operate and control the computer hardware, and to provide a platform for running application software. System software includes software categories such as <a href="/wiki/Operating_system" title="Operating system">operating systems</a>, <a href="/wiki/Utility_software" class="mw-redirect" title="Utility software">utility software</a>, <a href="/wiki/Device_driver" title="Device driver">device drivers</a>, <a href="/wiki/Compiler" title="Compiler">compilers</a>, and <a href="/wiki/Linker_(computing)" title="Linker (computing)">linkers</a>.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 links</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=System_programming_language&amp;action=edit&amp;section=1" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>In contrast with application languages, system programming languages typically offer more-direct access to the physical hardware of the machine: an archetypical system programming language in this sense was <a href="/wiki/BCPL" title="BCPL">BCPL</a>. System programming languages often lack built-in <a href="/wiki/Input/output" title="Input/output">input/output</a> (I/O) facilities because a system-software project usually develops its own I/O mechanisms or builds on top of basic monitor I/O or screen management facilities. The distinction between languages used for system programming and application programming became blurred over time with the widespread popularity of <a href="/wiki/PL/I" title="PL/I">PL/I</a>, <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> and <a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a>.</p>/n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=System_programming_language&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The earliest system software was written in <a href="/wiki/Assembly_language" title="Assembly language">assembly language</a> primarily because there was no alternative, but also for reasons including efficiency of object code, compilation time, and ease of debugging. Application languages such as <a href="/wiki/FORTRAN" class="mw-redirect" title="FORTRAN">FORTRAN</a> were used for system programming, although they usually still required some routines to be written in assembly language.<sup id="cite_ref-Sammet_1-0" class="reference"><a href="#cite_note-Sammet-1">[1]</a></sup></p>/n<h3><span class="mw-headline" id="Mid-level_languages">Mid-level languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=System_programming_language&amp;action=edit&amp;section=3" title="Edit section: Mid-level languages">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Mid-level languages "have much of the syntax and facilities of a higher level language, but also provide direct access in the language (as well as providing assembly language) to machine features."<sup id="cite_ref-Sammet_1-1" class="reference"><a href="#cite_note-Sammet-1">[1]</a></sup> The earliest of these was <a href="/wiki/Executive_Systems_Problem_Oriented_Language" title="Executive Systems Problem Oriented Language">ESPOL</a> on Burroughs mainframes in about 1960, followed by Niklaus Wirth's <a href="/wiki/PL360" title="PL360">PL360</a> (initially written on a Burroughs system as a <a href="/wiki/Cross_compiler" title="Cross compiler">cross compiler</a>), which had the general syntax of ALGOL 60 but whose statements directly manipulated CPU registers and memory. Other languages in this category include <a href="/wiki/MOL-360" title="MOL-360">MOL-360</a> and <a href="/wiki/PL/S" class="mw-redirect" title="PL/S">PL/S</a>.</p>/n<p>As an example, a typical PL360 statement is <code>R9&#160;:= R8 and R7 shll 8 or R6</code>, signifying that registers 8 and 7 should be and'ed together, the result shifted left 8 bits, the result of that or'ed with the contents of register 6, and the result placed into register 9.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup></p>/n<h3><span class="mw-headline" id="Higher-level_languages">Higher-level languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=System_programming_language&amp;action=edit&amp;section=4" title="Edit section: Higher-level languages">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>While PL360 is at the semantic level of assembly language, another kind of system programming language operates at a higher semantic level, but has specific extensions designed to make the language suitable for system programming. An early example of this kind of language is LRLTRAN,<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> which extended Fortran with features for character and bit manipulation, pointers, and directly addressed jump tables.</p>/n<p>Subsequently, languages such as C were developed, where the combination of features was sufficient to write system software, and a compiler could be developed that generated efficient object programs on modest hardware. Such a language generally omits features that cannot be implemented efficiently, and adds a small number of machine-dependent features needed to access specific hardware capabilities; inline assembly code, such as C's <tt>asm</tt> statement, is often used for this purpose. Although many such languages were developed,<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup> C and C++ are the ones that have survived.</p>/n<p><i>System Programming Language</i> (SPL) is also the name of a specific language on the <a href="/wiki/HP_3000" title="HP 3000">HP 3000</a> computer series, used for its operating system <a href="/wiki/HP_Multi-Programming_Executive" title="HP Multi-Programming Executive">HP Multi-Programming Executive</a>, and other parts of its system software.</p>/n<h2><span class="mw-headline"
