id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p>In <a href="/wiki/Logic" title="Logic">logic</a>, the term <b>decidable</b> refers to the <a href="/wiki/Decision_problem" title="Decision problem">decision problem</a>, the question of the existence of an <a href="/wiki/Effective_method" title="Effective method">effective method</a> for determining membership in a set of formulas, or, more precisely, an algorithm that can and will return a boolean true or false value that is correct (instead of looping indefinitely, crashing, returning "don't know" or returning a wrong answer). <a href="/wiki/Formal_system" title="Formal system">Logical systems</a> such as <a href="/wiki/Propositional_calculus" title="Propositional calculus">propositional logic</a> are decidable if membership in their set of <a href="/wiki/Validity" title="Validity">logically valid</a> formulas (or theorems) can be effectively determined. A <a href="/wiki/Theory_(mathematical_logic)" title="Theory (mathematical logic)">theory</a> (set of sentences <a href="/wiki/Deductive_closure" title="Deductive closure">closed</a> under <a href="/wiki/Logical_consequence" title="Logical consequence">logical consequence</a>) in a fixed logical system is decidable if there is an effective method for determining whether arbitrary formulas are included in the theory. Many important problems are <a href="/wiki/Undecidable_problem" title="Undecidable problem">undecidable</a>, that is, it has been proven that no effective method for determining membership (returning a correct answer after finite, though possibly very long, time in all cases) can exist for them.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decidability_(logic)&amp;action=edit&amp;section=1" title="Edit section: Relationship to computability">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>As with the concept of a <a href="/wiki/Decidable_set" class="mw-redirect" title="Decidable set">decidable set</a>, the definition of a decidable theory or logical system can be given either in terms of <i><a href="/wiki/Effective_method" title="Effective method">effective methods</a></i> or in terms of <i><a href="/wiki/Computable_function" title="Computable function">computable functions</a></i>. These are generally considered equivalent per <a href="/wiki/Church%27s_thesis" class="mw-redirect" title="Church's thesis">Church's thesis</a>. Indeed, the proof that a logical system or theory is undecidable will use the formal definition of computability to show that an appropriate set is not a decidable set, and then invoke Church's thesis to show that the theory or logical system is not decidable by any effective method (Enderton 2001, pp.&#160;206<i>ff.</i>).</p>/n<h2><span class="mw-headline" id="Decidability_of_a_logical_system">Decidability of a logical system</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decidability_(logic)&amp;action=edit&amp;section=2" title="Edit section: Decidability of a logical system">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Each <a href="/wiki/Logical_system" class="mw-redirect" title="Logical system">logical system</a> comes with both a <a href="/wiki/Syntax_(logic)" title="Syntax (logic)">syntactic component</a>, which among other things determines the notion of <a href="/wiki/Formal_proof" title="Formal proof">provability</a>, and a <a href="/wiki/Formal_semantics_(logic)" class="mw-redirect" title="Formal semantics (logic)">semantic component</a>, which determines the notion of <a href="/wiki/Logical_validity" class="mw-redirect" title="Logical validity">logical validity</a>. The logically valid formulas of a system are sometimes called the <b>theorems</b> of the system, especially in the context of first-order logic where <a href="/wiki/G%C3%B6del%27s_completeness_theorem" title="Gödel's completeness theorem">Gödel's completeness theorem</a> establishes the equivalence of semantic and syntactic consequence. In other settings, such as <a href="/wiki/Linear_logic" title="Linear logic">linear logic</a>, the syntactic consequence (provability) relation may be used to define the theorems of a system.</p>/n<p>A logical system is decidable if there is an effective method for determining whether arbitrary formulas are theorems of the logical system. For example, <a href="/wiki/Propositional_logic" class="mw-redirect" title="Propositional logic">propositional logic</a> is decidable, because the <a href="/wiki/Truth_table" title="Truth table">truth-table</a> method can be used to determine whether an arbitrary <a href="/wiki/Propositional_formula" title="Propositional formula">propositional formula</a> is logically valid.</p>/n<p><a href="/wiki/First-order_logic" title="First-order logic">First-order logic</a> is not decidable in general; in particular, the set of logical validities in any <a href="/wiki/Signature_(logic)" title="Signature (logic)">signature</a> that includes equality and at least one other predicate with two or more arguments is not decidable.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> Logical systems extending first-order logic, such as <a href="/wiki/Second-order_logic" title="Second-order logic">second-order logic</a> and <a href="/wiki/Type_theory" title="Type theory">type theory</a>, are also undecidable.</p>/n<p>The validities of <a href="/wiki/Monadic_predicate_calculus" title="Monadic predicate calculus">monadic predicate calculus</a> with identity are decidable, however. This system is first-order logic restricted to signatures that have no function symbols and whose relation symbols other than equality never take more than one argument.</p>/n<p>Some logical systems are not adequately represented by the set of theorems alone. (For example, <a href="/wiki/Ternary_logic" class="mw-redirect" title="Ternary logic">Kleene's logic</a> has no theorems at all.) In such cases, alternative definitions of decidability of a logical system are often used, which ask for an effective method for determining something more general than just validity of formulas; for instance, validity of <a href="/wiki/Sequent" title="Sequent">sequents</a>, or the <a href="/wiki/Logical_consequence" title="Logical consequence">consequence relation</a> {(Г, <i>A</i>) | Г ⊧ <i>A</i>} of the logic.</p>/n<h2><span class="mw-headline" id="Decidability_of_a_theory">Decidability of a theory</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decidability_(logic)&amp;action=edit&amp;section=3" title="Edit section: Decidability of a theory">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>A <a href="/wiki/Theory_(mathematical_logic)" title="Theory (mathematical logic)">theory</a> is a set of formulas, which here is assumed to be closed under <a href="/wiki/Logical_consequence" title="Logical consequence">logical consequence</a>. The question of decidability for a theory is whether there is an effective procedure that, given an arbitrary formula in the signature of the theory, decides whether the formula is a member of the theory or not. This problem arises naturally when a theory is defined as the set of logical consequences of a fixed set of axioms. Examples of decidable first-order theories include the theory of <a href="/wiki/Real_closed_field" title="Real closed field">real closed fields</a>, and <a href="/wiki/Presburger_arithmetic" title="Presburger arithmetic">Presburger arithmetic</a>, while the theory of <a href="/wiki/Group_(mathematics)" title="Group (mathematics)">groups</a> and <a href="/wiki/Robinson_arithmetic" title="Robinson arithmetic">Robinson arithmetic</a> are examples of undecidable theories.</p>/n<p>There are several basic results about decidability of theories. Every inconsistent theory is decidable, as every formula in the signature of the theory will be a logical consequence of, and thus member of, the theory. Every <a href="/wiki/Complete_theory" title="Complete theory">complete</a> <a href="/wiki/Recursively_enumerable" class="mw-redirect" title="Recursively enumerable">recursively enumerable</a> first-order theory is decidable. An extension of a decidable theory may not be decidable. For example, there are undecidable theories in propositional logic, although the set of validities (the smallest theory) is decidable.</p>/n<p>A consistent theory that has the property that every consistent extension is undecidable is said to be <b>essentially undecidable</b>. In fact, every consistent extension will be essentially undecidable. The theory of fields is undecidable but not essentially undecidable. <a href="/wiki/Robinson_arithmetic" title="Robinson arithmetic">Robinson arithmetic</a> is known to be essentially undecidable, and thus every consistent theory that includes or interprets Robinson arithmetic is also (essentially) undecidable.</p>/n<h2><span class="mw-headline" id="Some_decidable_theories">Some decidable theories</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decidability_(logic)&amp;action=edit&amp;section=4" title="Edit section: Some decidable theories">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Some decidable theories include (Monk 1976, p.&#160;234):<sup id="cite_ref-Monk1976_2-0" class="reference"><a href="#cite_note-Monk1976-2">[2]</a></sup></p>/n<ul>/n<li>The set of first-order logical validities in the signature with only equality, established by <a href="/wiki/Leopold_L%C3%B6wenheim" title="Leopold Löwenheim">Leopold Löwenheim</a> in 1915.</li>/n<li>The set of first-order logical validities in a signature with equality and one unary function, established by Ehrenfeucht in 1959.</li>/n<li>The first-order theory of the natural numbers in the signature with equality and addition, also called <a href="/wiki/Presburger_arithmetic" title="Presburger arithmetic">Presburger arithmetic</a>. The completeness was established by <a href="/wiki/Moj%C5%BCesz_Presburger" title="Mojżesz Presburger">Mojżesz Presburger</a> in 1929.</li>/n<li>The first-order theory of the natural numbers in the signature with equality and multiplication, also called <a href="/wiki/Skolem_arithmetic" title="Skolem arithmetic">Skolem arithmetic</a>.</li>/n<li>The first-order theory of Boolean algebras, decidability include <a href="/wiki/Quantifier_elimination" title="Quantifier elimination">quantifier elimination</a>, <a href="/wiki/Model_completeness" class="mw-redirect" title="Model completeness">model completeness</a>, and <a href="/wiki/Vaught%27s_test" class="mw-redirect" title="Vaught's test">Vaught's test</a>.</p>/n<h2><span class="mw-headline" id="In_context_of_games">In context of games</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decidability_(logic)&amp;action=edit&amp;section=5" title="Edit section: In class="reference"><a href="#cite_note-4">[4]</a></sup></li>/n<li><a href="/wiki/Infinite_chess" title="Infinite chess">Infinite chess</a> (with limitations on rules and gamepieces) is decidable.<sup This is closely related to the concept of a <a href="/wiki/Many-one_reduction" title="Many-one reduction">many-one reduction</a> in computability theory.</p>/n<h2><span class="mw-headline" id="Semidecidability">Semidecidability</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decidability_(logic)&amp;action=edit&amp;section=7" title="Edit section: Semidecidability">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>A property of a theory or logical system weaker than decidability is <b>semidecidability</b>. A theory is semidecidable if there is an effective method which, given an arbitrary formula, will always tell correctly when the formula is in the theory, but may give either a negative answer or no answer at all when the formula is not in the theory. A logical system is semidecidable if there is an effective method for generating theorems (and only theorems) such that every theorem will eventually be generated. This is different from decidability because in a semidecidable system there may be no effective procedure for checking that a formula is <i>not</i> a theorem.</p>/n<p>Every decidable theory or logical system is semidecidable, but in general the converse is not true; a theory is decidable if and only if both it and its complement are semi-decidable. For example, the set of logical validities <i>V</i> of first-order logic is semi-decidable, but not decidable. In this case, it is because there is no effective method for determining for an arbitrary formula <i>A</i> whether <i>A</i> is not in <i>V</i>. Similarly, the set of logical consequences of any <a href="/wiki/Recursively_enumerable_set" title="Recursively enumerable set">recursively enumerable set</a> of first-order axioms is semidecidable. Many of the examples of undecidable first-order theories given above are of this form.</p>/n<h2><span class="mw-headline" id="Relationship_with_completeness">Relationship with completeness</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decidability_(logic)&amp;action=edit&amp;section=8" title="Edit section: Relationship with completeness">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Decidability should not be confused with <a href="/wiki/Complete_theory" title="Complete theory">completeness</a>. For example, the theory of <a href="/wiki/Algebraically_closed_field" title="Algebraically closed field">algebraically closed fields</a> is decidable but incomplete, whereas the set of all true first-order statements about nonnegative integers in the language with + and × is complete but undecidable. Unfortunately, as a terminological ambiguity, the term "undecidable statement" is sometimes used as a synonym for <a href="/wiki/Independence_(mathematical_logic)" title="Independence (mathematical logic)">independent statement</a>.</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Decidability_(logic)&amp;action=edit&amp;section=9" title="Edit section: See also">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div role="navigation" aria-label="Portals" class="noprint portal plainlist tright" style="margin:0.5em 0 0.5em 1em;border:solid #aaa 1px">/n<ul style="display:table;box-sizing:border-box;padding:0.1em;max-width:175px;background:#f9f9f9;font-size:85%;line-height:110%;font-style:italic;font-weight:bold">/n<li style="display:table-row"><span style="display:table-cell;padding:0.2em;vertical-align:middle;text-align:center"><a href="/wiki/File:Logic_portal.svg" class="image"><img alt="icon" src="//upload.wikimedia.org/wikipedia/commons/thumb/7/7c/Logic_portal.svg/32px-Logic_portal.svg.png" width="32" height="24" class="noviewer" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/7/7c/Logic_portal.svg/48px-Logic_portal.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/7/7c/Logic_portal.svg/64px-Logic_portal.svg.png 2x" data-file-width="287" data-file-height="218" /></a></span><span style="display:table-cell;padding:0.2em 0.2em 0.2em 0.3em;vertical-align:middle"><a href="/wiki/Portal:Logic" title="Portal:Logic">Logic portal</a></span></li>/n</ul>/n</div>/n<ul>/n<li><a href="/wiki/L%C3%A1szl%C3%B3_Kalm%C3%A1r" title="László Kalmár">László Kalmár</a> (1936)</li>/n<li><a href="/wiki/Alonzo_Church" title="Alonzo Church">Alonzo Church</a> (1956)</li>/n<li><a href="/wiki/W.V.O._Quine" class="mw-redirect" title="W.V.O.
