lang="en">Fifth-generation programming language</h1>/n									<div id="bodyContent" class="mw-body-content">/n									<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>/n								<div id="contentSub"><span class="mw-redirectedfrom">  (Redirected from <a href="/w/index.php?title=Fifth_generation_programming_language&amp;redirect=no" class="mw-redirect" title="Fifth generation programming language">Fifth generation programming language</a>)</span></div>/n												<div id="jump-to-nav" class="mw-jump">/n					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>/n				</div>/n				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p>A <b>fifth-generation programming language</b> (<b>5GL</b>) is a <a href="/wiki/Programming_language" title="Programming language">programming language</a> based on solving using constraints given to the program, rather than using an <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> written by a programmer.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (September 2010)">citation needed</span></a></i>]</sup> Most <a href="/wiki/Constraint_programming" title="Constraint programming">constraint-based</a> and <a href="/wiki/Logic_programming" title="Logic programming">logic programming</a> languages and some other <a href="/wiki/Declarative_language" class="mw-redirect" title="Declarative language">declarative languages</a> are fifth-generation languages.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#History"><span class="tocnumber">1</span> <span class="toctext">History</span></a></li>/n<li class="toclevel-1 tocsection-2"><a href="#Common_misconception"><span class="tocnumber">2</span> <span class="toctext">Common misconception</span></a></li>/n<li class="toclevel-1 tocsection-3"><a href="#See_also"><span class="tocnumber">3</span> <span class="toctext">See also</span></a></li>/n<li class="toclevel-1 tocsection-4"><a href="#References"><span class="tocnumber">4</span> <span class="toctext">References</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Fifth-generation_programming_language&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>While <a href="/wiki/Fourth-generation_programming_language" title="Fourth-generation programming language">fourth-generation programming languages</a> are designed to build specific programs, fifth-generation languages are designed to make the computer solve a given problem without the programmer. This way, the user only needs to worry about what problems need to be solved and what conditions need to be met, without worrying about how to implement a routine or algorithm to solve them. Fifth-generation languages are used mainly in <a href="/wiki/Artificial_intelligence" title="Artificial intelligence">artificial intelligence</a> research. <a href="/wiki/Prolog" title="Prolog">Prolog</a>, <a href="/wiki/OPS5" title="OPS5">OPS5</a> and <a href="/wiki/Mercury_(programming_language)" title="Mercury (programming language)">Mercury</a> are examples of fifth-generation languages.<sup id="cite_ref-bala_1-0" class="reference"><a href="#cite_note-bala-1">[1]</a></sup></p>/n<p>These types of languages were also built upon <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a>, many originating on the <a href="/wiki/Lisp_machine" title="Lisp machine">Lisp machine</a>, such as <a href="/wiki/ICAD_(software)" title="ICAD (software)">ICAD</a>. Then, there are many <a href="/wiki/Frame_language" title="Frame language">frame languages</a>, such as <a href="/wiki/KL-ONE" title="KL-ONE">KL-ONE</a>.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (September 2010)">citation needed</span></a></i>]</sup></p>/n<p>In the 1980s, fifth-generation languages were considered to be the way of the future, and some predicted that they would replace all other languages for system development, with the exception of low-level languages.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (July 2009)">citation needed</span></a></i>]</sup> Most notably, from 1982 to 1993, <a href="/wiki/Japan" title="Japan">Japan</a><sup id="cite_ref-Ref1_2-0" class="reference"><a href="#cite_note-Ref1-2">[2]</a></sup><sup id="cite_ref-Ref2_3-0" class="reference"><a href="#cite_note-Ref2-3">[3]</a></sup> put much research and money into their <a href="/wiki/Fifth_generation_computer" title="Fifth generation computer">fifth-generation computer systems project</a>, hoping to design a massive computer network of machines using these tools.</p>/n<p>However, as larger programs were built, the flaws of the approach became more apparent. It turns out that, given a set of constraints defining a particular problem, deriving an efficient algorithm to solve it is a very difficult problem in itself. This crucial step cannot yet be automated and still requires the insight of a human programmer.</p>/n<h2><span class="mw-headline" id="Common_misconception">Common misconception</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Fifth-generation_programming_language&amp;action=edit&amp;section=2" title="Edit section: Common misconception">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Vendors have been known on occasion to advertise their languages as 5GL.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup> Most of the time they actually sell <a href="/wiki/Fourth-generation_programming_language" title="Fourth-generation programming language">4GLs</a> with a higher level of automation and <a href="/wiki/Knowledge_base" title="Knowledge base">knowledge base</a>. Because the hype of the 1980s faded away and the projects were eventually all dropped, 5GL awareness has also dropped; this has opened doors to the vendors to re-use the term in marketing their new tools, without causing much controversy among the current generations of programmers.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (October 2013)">citation needed</span></a></i>]</sup></p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Fifth-generation_programming_language&amp;action=edit&amp;section=3" title="Edit section: See also">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<ul>/n<li><a href="/wiki/Constraint_programming" title="Constraint programming">Constraint programming</a></li>/n<li><a href="/wiki/List_of_programming_languages_for_artificial_intelligence" title="List of programming languages for artificial intelligence">List of programming languages for artificial intelligence</a></li>/n<li><a href="/wiki/Programming_paradigms" class="mw-redirect" title="Programming paradigms">Programming paradigms</a></li>/n</ul>/n<h2><span class="mw-headline" id="References">References</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Fifth-generation_programming_language&amp;action=edit&amp;section=4" title="Edit section: References">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div class="reflist" style="list-style-type: decimal;">/n<div class="mw-references-wrap">/n<ol class="references">/n<li id="cite_note-bala-1"><span class="mw-cite-backlink"><b><a href="#cite_ref-bala_1-0">^</a></b></span> <span class="reference-text">E.
