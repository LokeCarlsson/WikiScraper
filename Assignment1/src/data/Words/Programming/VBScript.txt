href="/wiki/Windows_PowerShell" class="mw-redirect" title="Windows PowerShell">Windows PowerShell</a></td>/n</tr>/n</table>/n<p><b>VBScript</b> (<i>"<a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> <a href="/wiki/Visual_Basic" title="Visual Basic">Visual Basic</a> Scripting Edition"</i>) is an <a href="/wiki/Active_Scripting" title="Active Scripting">Active Scripting</a> language developed by Microsoft that is modeled on Visual Basic. It allows <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> <a href="/wiki/System_administrator" title="System administrator">system administrators</a> to generate powerful tools for managing computers with <a href="/wiki/Error_handling" class="mw-redirect" title="Error handling">error handling</a>, <a href="/wiki/Subroutine" title="Subroutine">subroutines</a>, and other advanced programming constructs. It can give the user complete control over many aspects of their computing environment.</p>/n<p>VBScript uses the <a href="/wiki/Component_Object_Model" title="Component Object Model">Component Object Model</a> to access elements of the environment within which it is running; for example, the FileSystemObject (FSO) is used to <a href="/wiki/Create,_read,_update_and_delete" title="Create, read, update and delete">create, read, update and delete</a> <a href="/wiki/Computer_file" title="Computer file">files</a>. VBScript has been installed by default in every desktop release of <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> since <a href="/wiki/Windows_98" title="Windows 98">Windows 98</a>;<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> in <a href="/wiki/Windows_Server" title="Windows Server">Windows Server</a> since <a href="/wiki/Windows_NT_4.0#Option_Pack" title="Windows NT 4.0">Windows NT 4.0 Option Pack</a>;<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> and optionally with <a href="/wiki/Windows_CE" class="mw-redirect" title="Windows CE">Windows CE</a> (depending on the device it is installed on).</p>/n<p>A VBScript script must be executed within a <a href="/wiki/Hosting_environment" title="Hosting environment">host environment</a>, of which there are several provided with Microsoft Windows, including: <a href="/wiki/Windows_Script_Host" title="Windows Script Host">Windows Script Host</a> (WSH), <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> (IE), and <a href="/wiki/Internet_Information_Services" title="Internet Information Services">Internet Information Services</a> (IIS).<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> Additionally, the VBScript hosting environment is embeddable in other programs, through technologies such as the <a rel="nofollow" class="external text" href="https://msdn.microsoft.com/en-us/library/aa227633(v=vs.60).aspx">Microsoft Script Control</a> (<tt>msscript.ocx</tt>).</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#History"><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VBScript&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>VBScript began as part of the Microsoft Windows Script Technologies, launched in 1996. This technology (which also included <a href="/wiki/JScript" title="JScript">JScript</a>) was initially targeted at web developers. During a period of just over two years, VBScript advanced from version 1.0 to 2.0, and over that time it gained support from Windows <a href="/wiki/System_administrator" title="System administrator">system administrators</a> seeking an automation tool more powerful than the <a href="/wiki/Batch_file" title="Batch file">batch language</a> first developed in the early 1980s.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup></p>/n<p>On March 6, 1988, <a href="/wiki/Alan_Cooper" title="Alan Cooper">Alan Cooper</a> showed <a href="/wiki/Bill_Gates" title="Bill Gates">Bill Gates</a> his shell prototype that allowed widgets to be added dynamically. On March 20, 1991, <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> adopted "Quick Basic". This allowed users to create Windows apps quickly and easily with a <a href="/wiki/GUI" class="mw-redirect" title="GUI">GUI</a>. Finally, on August 1, 1996, <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> was made with features that included VBScript.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup></p>/n<p>In version 5.0, the functionality of VBScript was increased with new features including <a href="/wiki/Regular_expression" title="Regular expression">regular expressions</a>; <a href="/wiki/Class_(computer_science)" class="mw-redirect" title="Class (computer science)">classes</a>; the <i>With</i> statement;<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup> the <i>Eval</i>, <i>Execute</i>, and <i>ExecuteGlobal</i> functions to evaluate and execute script commands built during the execution of another script; a function-pointer system via GetRef,<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup> and <a href="/wiki/Distributed_Component_Object_Model" title="Distributed Component Object Model">Distributed COM</a> (DCOM) support.</p>/n<p>In version 5.5, <i>SubMatches</i><sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup> were added to the <i>regular expression</i> class in VBScript, to finally allow script authors to capture the text within the expression's groups. That capability had already been available in JScript.</p>/n<p>With the advent of the <a href="/wiki/.NET_framework" class="mw-redirect" title=".NET framework">.NET framework</a>, the scripting team took the decision to implement future support for VBScript within <a href="/wiki/ASP.NET" title="ASP.NET">ASP.NET</a> for web development,<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[9]</a></sup> and therefore no new versions of the VBScript engine would be developed. It would henceforth be supported by Microsoft's <i>Sustaining Engineering Team</i>, who are responsible for bug fixes and security enhancements. For Windows system administrators, Microsoft suggests migrating to <a href="/wiki/Windows_PowerShell" class="mw-redirect" title="Windows PowerShell">Windows PowerShell</a>. However, the scripting engine will continue to be shipped with future releases of Microsoft Windows and IIS.</p>/n<h2><span class="mw-headline" id="Environments">Environments</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VBScript&amp;action=edit&amp;section=2" title="Edit section: Environments">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>When employed for client-side web development in Microsoft <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a>, VBScript is similar in function to <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>. It is used to write executable functions that are embedded in or included from HTML pages and interact with the <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a> (DOM) of the page, to perform tasks not possible in HTML alone. However, other web browsers such as <a href="/wiki/Firefox" title="Firefox">Firefox</a> and <a href="/wiki/Opera_(web_browser)" title="Opera (web browser)">Opera</a> do not have built-in support for VBScript. This means that where client-side scripting and cross-browser compatibility are required, developers usually choose JavaScript over VBScript.</p>/n<p>VBScript is also used for server-side processing of web pages, most notably with Microsoft <a href="/wiki/Active_Server_Pages" title="Active Server Pages">Active Server Pages</a> (ASP). The ASP engine and type library, <tt>asp.dll</tt>, invokes <tt>vbscript.dll</tt> to run VBScript scripts. VBScript that is embedded in an ASP page is contained within &lt;% and&#160;%&gt; context switches. The following example of an ASP page with VBScript displays the current time in 24-hour format, but it is also used as the default script for error messages with the Windows operating systems.</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n <span class="nt">&lt;%</span> <span class="k">Option</span> <span class="n">Explicit</span>/n <span href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>. The simplest example is a script that makes use of the <a href="/wiki/Windows_Script_Host" title="Windows Script Host">Windows Script Host</a> (WSH) environment. Such a script is usually in a stand-alone file with the <a href="/wiki/File_extension" class="mw-redirect" title="File extension">file extension</a> <code>.vbs</code>. The script can be invoked in two ways: <tt>Wscript.exe</tt> is used to display output and receive input through a GUI, such as dialog and input boxes. <tt>Cscript.exe</tt> is used in a <a href="/wiki/Command_line_interface" class="mw-redirect" title="Command line interface">command line environment</a>.</p>/n<p>VBScript can be included in two other types of scripting files: <a href="/wiki/Windows_Script_File" title="Windows Script File">Windows Script Files</a>, and <a href="/wiki/HTML_Application" title="HTML Application">HTML Applications</a>.</p>/n<p>A Windows Script File (WSF) is styled after XML. A WSF file can include multiple VBS files. As a result, WSF files provide a means for code reuse: a library of classes or functions can be stored in one or more <tt>.vbs</tt> files, which can be included in <tt>.wsf</tt> files, so the functionality can be reused in a modular way. The files have extension <tt>.wsf</tt> and can be executed using <tt>wscript.exe</tt> or <tt>cscript.exe</tt>, just as a <tt>.vbs</tt> file can.</p>/n<p>An HTML Application (HTA) is styled after HTML. The HTML in the file is used to generate the user interface, and a scripting language such as VBScript is used for the program logic. The files have extension <tt>.hta</tt> and can be executed using <tt>mshta.exe</tt>.</p>/n<p>VBScript (and JScript) can also be used in a Windows Script Component, an ActiveX-enabled script class that can be invoked by other <a href="/wiki/Component_Object_Model" title="Component Object Model">COM</a>-enabled applications.<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">[10]</a></sup> These files have extension <tt>.wsc</tt>.</p>/n<p>VBScript is simple to create and can be coded using an easy to use text editor like Notepad. A simple VBScript saved by the ".vbs" extension can be run on a Windows system by double clicking it.</p>/n<h2><span class="mw-headline" id="Functionality">Functionality</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VBScript&amp;action=edit&amp;section=3" title="Edit section: Functionality">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<h3><span class="mw-headline" id="Language_features">Language features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VBScript&amp;action=edit&amp;section=4" title="Edit section: Language features">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>The language of VBScript is modeled on Visual Basic, and therefore can be reviewed using similar categories: procedures, control structures, constants, variables, user interaction, array handling, date/time functions, error handling, mathematical functions, objects, regular expressions, string manipulation, and so on.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">[11]</a></sup></p>/n<p>The following are some key points of introduction to the VBScript language.</p>/n<p>A "procedure" is the main construct in VBScript for separating code into smaller modules. VBScript distinguishes between a function, which can return a result in an assignment statement, and a subroutine, which cannot. Parameters are positional, and can be passed by value or by reference.</p>/n<p>Control structures include the usual iterative and conditional Do Loops, If-Then-Else statements, and Case statements, with some more complex variants, such as ElseIf and nested control structures.</p>/n<p>As a memory aid in coding, and certainly for readability, there are a large number of constants, such as True and False for logical values, vbOKCancel and vbYesNo for MsgBox codes, vbBlack and vbYellow for color values, vbCR for the carriage return character, and many others.</p>/n<p>Variables have "<a href="/wiki/Variant_type" title="Variant type">Variant</a>" type by default, but it is possible (and sometimes necessary) to force a particular type (integer, date, etc.) using conversion functions (CInt, CDate, etc.)</p>/n<p>User interaction is provided through the functions <code>MsgBox</code> and <code>InputBox</code> which provide a simple dialogue box format for messages and input. Both functions display prompting messages, with the former returning a standard response, and the latter returning one user-supplied text or numeric value. For more elaborate GUI interaction with controls, VBScript can be used in combination with HTML, for example, in an <a href="/wiki/HTML_Application" title="HTML Application">HTML Application</a>. Event-driven forms are not supported as in Visual Basic or Visual Basic for Applications.</p>/n<p>Names are not case-sensitive. However, it is considered a best practice of VBScript style to be consistent and to capitalize judiciously.</p>/n<h3><span class="mw-headline" id="VBScript_functionalities">VBScript functionalities</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VBScript&amp;action=edit&amp;section=5" title="Edit section: VBScript functionalities">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>When hosted by the <a href="/wiki/Windows_Script_Host" title="Windows Script Host">Windows Script Host</a>, VBScript provides numerous features which are common to scripting languages, but not available from <a href="/wiki/Visual_Basic" title="Visual Basic">Visual Basic 6.0</a>. These features include direct or easy access to:</p>/n<ul>/n<li>Named and unnamed command line arguments,</li>/n<li><a href="/wiki/Stdin" class="mw-redirect" title="Stdin">Stdin</a> and <a href="/wiki/Stdout" class="mw-redirect" title="Stdout">stdout</a>, which could be redirected,</li>/n<li>WSH.Echo which writes to the console and cannot be redirected,</li>/n<li>WSH.ExitCode which can be tested from DOS batch files, or by the process which invoked the script file,</li>/n<li>Network Printers,</li>/n<li>Network Shares,</li>/n<li>Special Folders, e.g. Desktop, Favorites, MyDocuments and so on,</li>/n<li>Network user information, such as group membership,</li>/n<li>Methods for runtime execution of text defined at runtime: Eval and Execute,</li>/n<li>Methods for executing scripts on remote machines,</li>/n<li>Windows Management Instrumentation (WMI)</li>/n<li>Functionality for embedding a VBScript engine in other applications, using a widely known language.</li>/n</ul>/n<p>CScript, the command line runner, provides options for:</p>/n<ul>/n<li>Interactive or Batch Mode,</li>/n<li>Invoking Debug Mode from the command line,</li>/n<li>Error reporting including the line number.</li>/n</ul>/n<h3><span class="mw-headline" id="Additional_functionality">Additional functionality</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VBScript&amp;action=edit&amp;section=6" title="Edit section: Additional functionality">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>File system management, file modification, and streaming text operations are implemented with the Scripting Runtime Library <code>scrrun.dll</code>. This provides objects such as FileSystemObject, File, and TextStream, which expose the Windows file system to the programmer.</p>/n<p>Binary file and memory I/O are provided by the "ADODB.Stream" class, which can also be used for string builders (to avoid excessive string concatenation, which can be costly), and to interconvert byte arrays and strings. Database access is made possible through <a href="/wiki/ActiveX_Data_Objects" title="ActiveX Data Objects">ActiveX Data Objects</a> (ADO), and the <a href="/wiki/IIS_Metabase" title="IIS Metabase">IIS Metabase</a> can be manipulated using the GetObject() function with sufficient permissions (useful for creating and destroying sites and virtual directories). XML files and schemas can be manipulated with the <a href="/wiki/MSXML" title="MSXML">Microsoft XML Library</a> <a href="/wiki/Application_Programming_Interface" class="mw-redirect" title="Application Programming Interface">Application Programming Interfaces</a> (<tt>msxml6.dll</tt>, <tt>msxml3.dll</tt>), which also can be used to retrieve content from the World Wide Web via the XMLHTTP and ServerXMLHTTP objects (class strings "MSXML2.XMLHTTP.6.0" and "MSXML2.ServerXMLHTTP.6.0", respectively).</p>/n<p>Functionality can also be added through ActiveX technologies. Security concerns have led to many ActiveX controls being blacklisted in the Internet Explorer process by Microsoft, which deploys the <a href="/wiki/Killbit" title="Killbit">killbit</a> via monthly Windows security updates to disable vulnerable Microsoft and third party code.<sup id="cite_ref-12" class="reference"><a href="#cite_note-12">[12]</a></sup><sup id="cite_ref-13" class="reference"><a href="#cite_note-13">[13]</a></sup></p>/n<h2><span class="mw-headline" id="Development_tools">Development tools</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VBScript&amp;action=edit&amp;section=7" title="Edit section: Development tools">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Microsoft does not routinely make available an IDE (<a href="/wiki/Integrated_Development_Environment" class="mw-redirect" title="Integrated Development Environment">Integrated Development Environment</a>) for VBScript, although the <a href="/wiki/Microsoft_Script_Editor" title="Microsoft Script Editor">Microsoft Script Editor</a> has been bundled with certain versions of Microsoft Office.</p>/n<p>For debugging purposes the Microsoft Script Debugger can still be used in current Windows versions, even though the tool has not been updated in years. It allows the user to set break points in the VBScript code but the user interface is more than clumsy.</p>/n<p>There are VBScript debuggers available from third-party sources,<sup id="cite_ref-14" class="reference"><a href="#cite_note-14">[14]</a></sup><sup id="cite_ref-15" class="reference"><a href="#cite_note-15">[15]</a></sup> and many <a href="/wiki/List_of_text_editors" title="List of text editors">text editors</a> offer <a href="/wiki/Syntax_highlighting" title="Syntax highlighting">syntax highlighting</a> for the language.</p>/n<p>During execution, when an error occurs, the script host issues a message stating the type of error and the number of the offending line.</p>/n<h2><span class="mw-headline" id="Uses">Uses</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=VBScript&amp;action=edit&amp;section=8" title="Edit section: Uses">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Although VBScript is a general-purpose scripting language, several particular areas of use are noteworthy. First, it is widely used among system administrators in the Microsoft environment.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16">[16]</a></sup> This situation may change with the promotion and increased use of <a href="/wiki/Windows_PowerShell" class="mw-redirect" title="Windows PowerShell">Windows PowerShell</a>. Second, VBScript is the scripting language for <a href="/wiki/HP_QuickTest_Professional" title="HP QuickTest Professional">Quick Test Professional</a>, a test automation tool.<sup id="cite_ref-17" class="reference"><a href="#cite_note-17">[17]</a></sup> A third area to note is the adoption of VBScript as the internal scripting language for some embedded applications, such as industrial operator interfaces and human machine interfaces. The hierarchical DBMS <a href="/wiki/InterSystems" title="InterSystems">InterSystems</a> <a href="/wiki/Cach%C3%A9_(software)" class="mw-redirect" title="Caché (software)">Caché</a> (which has its roots in the language <a href="/wiki/MUMPS" title="MUMPS">MUMPS</a>) also supports an implementation of VBScript, Cache BASIC, for programming stored code.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18">[18]</a></sup><sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (September 2008)">citation needed</span></a></i>]</sup></p>/n<p>VBScript omits several useful features of the full Visual Basic, such as strong typing, extended error trapping and the ability to pass a variable number of parameters to a subroutine. However, its use is relatively widespread because it is easy to learn and because those who implement code in the language need not pay royalties to Microsoft as long as the VBScript trade mark is acknowledged.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (September 2014)">citation needed</span></a></i>]</sup> When an organization licenses Visual Basic for Applications (VBA) from Microsoft, as companies such as Autodesk, StatSoft, Great Plains Accounting and Visio (subsequently acquired by Microsoft) have done, it is allowed to redistribute the full VBA code-writing and debugging environment with its product.</p>/n<p>VBScript is used in place of VBA as the macro language of Outlook 97.</p>/n<p>VBScript can be effectively used for automating day to day office tasks as well as monitoring in the Windows-based environment. It's a first choice tool for server administrators, MS Office users and other tech contributors. It can also be used in collaboration with ADODB <a href="/wiki/ActiveX_Data_Objects" title="ActiveX Data Objects">ActiveX Data Objects</a> (ADODB) for effective database connectivity.</p>/n<h2><span
