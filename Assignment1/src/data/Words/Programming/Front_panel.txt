"blinkenlights" front panel</div>/n</div>/n</div>/n<p>A <b>front panel</b> was used on early electronic computers to display and allow the alteration of the state of the machine's internal <a href="/wiki/CPU_register" class="mw-redirect" title="CPU register">registers</a> and <a href="/wiki/Computer_memory" title="Computer memory">memory</a>. The front panel usually consisted of arrays of <a href="/wiki/Lamp_(electrical_component)" class="mw-redirect" title="Lamp (electrical component)">indicator lamps</a>, <a href="/wiki/Toggle_switch" class="mw-redirect" title="Toggle switch">toggle switches</a>, and <a href="/wiki/Button_(control)" class="mw-redirect" title="Button (control)">push buttons</a> mounted on a sheet metal face plate. In early machines, <a href="/wiki/Cathode_ray_tube" title="Cathode ray tube">CRTs</a> might also be present (as an oscilloscope, or, for example, to mirror the contents of <a href="/wiki/Williams_tube" title="Williams tube">Williams-Kilburn tube</a> memory). Prior to the development of CRT <a href="/wiki/System_console" title="System console">system consoles</a>, many computers such as the <a href="/wiki/IBM_1620" title="IBM 1620">IBM 1620</a> had console typewriters.</p>/n<p>Usually the contents of one or more <a href="/wiki/Hardware_register" title="Hardware register">hardware registers</a> would be represented by a row of lights, allowing the contents to be read directly when the machine was stopped. The switches allowed direct entry of data and address values into registers or memory.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#Details"><span class="tocnumber">1</span> <span class="toctext">Details</span></a></li>/n<li class="toclevel-1 tocsection-2"><a href="#Common_usage"><span class="tocnumber">2</span> <span class="toctext">Common usage</span></a>/n<ul>/n<li class="toclevel-2 tocsection-3"><a href="#Booting"><span class="tocnumber">2.1</span> <span class="toctext">Booting</span></a></li>/n<li class="toclevel-2 tocsection-4"><a href="#Debugging"><span class="tocnumber">2.2</span> <span class="toctext">Debugging</span></a></li>/n<li class="toclevel-2 tocsection-5"><a href="#Entertainment"><span class="tocnumber">2.3</span> <span class="toctext">Entertainment</span></a></li>/n</ul>/n</li>/n<li class="toclevel-1 tocsection-6"><a href="#Example"><span class="tocnumber">3</span> <span class="toctext">Example</span></a></li>/n<li class="toclevel-1 tocsection-7"><a href="#References"><span class="tocnumber">4</span> <span class="toctext">References</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="Details">Details</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Front_panel&amp;action=edit&amp;section=1" title="Edit section: Details">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>On some machines, certain lights and switches were reserved for use under program control. These were often referred to as <i>sense lights</i> and <i><a href="/wiki/Sense_switch" title="Sense switch">sense switches</a></i>. For example, the original <a href="/wiki/Fortran" title="Fortran">Fortran</a> compiler for the <a href="/wiki/IBM_704" title="IBM 704">IBM 704</a> contained specific statements for testing and manipulation of the 704's sense lights and switches. These switches were often used by the program to control optional behavior, for example information might be printed only if a particular sense switch was set.</p>/n<p>Operating systems made for computers with <i><a href="/wiki/Blinkenlights" title="Blinkenlights">blinkenlights</a></i>, for example, <a href="/wiki/RSTS/E" title="RSTS/E">RSTS/E</a> and <a href="/wiki/RSX-11" title="RSX-11">RSX-11</a>, would frequently have an <a href="/wiki/Idle_(CPU)" title="Idle (CPU)">idle</a> task blink the panel lights in some recognizable fashion. System programmers often became very familiar with these light patterns and could tell from them how busy the system was and, sometimes, exactly what it was doing at the moment. The <a href="/wiki/Master_Control_Program" class="mw-redirect" title="Master Control Program">Master Control Program</a> for the <a href="/wiki/Burroughs_Corporation" title="Burroughs Corporation">Burroughs Corporation</a> <a href="/wiki/Burroughs_large_systems" title="Burroughs large systems">B6700 mainframe</a> would display a large block-letter "B" when the system was idle.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup></p>/n<p>Switches and lights required little additional logic circuitry and usually no software support, important when logic hardware components were costly and software often limited.</p>/n<p>This baroque style of front panels began to die out in 1964 when <a href="/wiki/Seymour_Cray" title="Seymour Cray">Seymour Cray</a> designed his <a href="/wiki/CDC_6000_series" title="CDC 6000 series">CDC 6600 supercomputer</a> with a very simple and elegant <a href="/wiki/CDC_6000_series#Display_console" title="CDC 6000 series">display console</a> containing only 2 <a href="/wiki/Cathode_ray_tube" title="Cathode ray tube">CRT</a> displays and a <a href="/wiki/Keyboard_(computing)" class="mw-redirect" title="Keyboard (computing)">keyboard</a>, replacing all the hundreds of switches, buttons, and blinking lights. The 6600 had support from ten supporting "peripheral processors" whose duties included reading the keyboard and driving the graphics displays.</p>/n<p>Early <a href="/wiki/Microcomputers" class="mw-redirect" title="Microcomputers">microcomputers</a> such as the 1975 <a href="/wiki/Altair_8800" title="Altair 8800">Altair 8800</a> also relied on front panels, but since the introduction of the <a href="/wiki/Apple_II" title="Apple II">Apple II</a> in June 1977, the vast majority of micros came with keyboards and connections for TV screens or other monitors.</p>/n<h2><span class="mw-headline" id="Common_usage">Common usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Front_panel&amp;action=edit&amp;section=2" title="Edit section: Common usage">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>An <a href="/wiki/Computer_operator" title="Computer operator">operator</a> would use the front panel to <a href="/wiki/Booting" title="Booting">bootstrap</a> the computer, to <a href="/wiki/Debug" class="mw-redirect" title="Debug">debug</a> running programs, and to find <a href="/wiki/Computer_hardware" title="Computer hardware">hardware</a> faults.</p>/n<h3><span class="mw-headline" id="Booting">Booting</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Front_panel&amp;action=edit&amp;section=3" title="Edit section: Booting">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Typically, the operator would have a written procedure containing a short series of bootstrap instructions that would be hand-entered using the toggle switches. First, the operator would set the "address" switch and enter the address in binary using the switches. For easier entry and readout, on some computers (such as the DEC <a href="/wiki/PDP-8" title="PDP-8">PDP-8</a> or MITS <a href="/wiki/Altair_8800" title="Altair 8800">Altair 8800</a>) binary digits were grouped into threes on the front panel, with each group of lights or switches representing a single <a href="/wiki/Octal" title="Octal">octal</a> digit between 0 and 7. Decimal computers like the <a href="/wiki/IBM_1620" title="IBM 1620">IBM 1620</a> used <a href="/wiki/Binary-coded_decimal" title="Binary-coded decimal">binary-coded decimal</a> for memory addresses.</p>/n<p>Next the operator would set the "value" switch, and then enter the value intended for that address. After entering several of these instructions (most computers had a "deposit next" button, which would deposit subsequent values in subsequent addresses, relieving the operator of needing to toggle in addresses), the operator would then set the starting address of the bootstrap program and press the "run" switch to begin the execution of the program. The bootstrap program usually read a somewhat longer program from punched paper-tape, punched cards, magnetic tape or disk which in turn would load the operating system from disk.</p>/n<p>Some machines accelerated the bootstrap process by allowing the operator to set the switches to contain one or two <a href="/wiki/Machine_code" title="Machine code">machine language</a> instructions and then directly executing those instructions. Other machines allowed I/O devices to be explicitly commanded from the front panel (for example, "Read-In Preset" on the <a href="/wiki/PDP-10" title="PDP-10">PDP-10</a> or the accessing of <a href="/wiki/Memory-mapped_I/O" title="Memory-mapped I/O">memory-mapped I/O</a> devices on a <a href="/wiki/PDP-11" title="PDP-11">PDP-11</a>). Some machines also contained various bootstrap programs in <a href="/wiki/Read-only_memory" title="Read-only memory">ROM</a> and all that was required to <i>boot</i> the system was to start it executing at the address of the correct ROM program.</p>/n<h3><span class="mw-headline" id="Debugging">Debugging</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Front_panel&amp;action=edit&amp;section=4" title="Edit section: Debugging">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Front panels were often used to debug programs when operating support was limited and most programs were run in stand-alone mode. Typically a front-panel switch could cause the computer to <i>single-step</i>, that is run a single instruction and stop until the programmer pressed a button to execute the next instruction. An <i>address stop</i> could be set to stop a running program when it attempted to execute an instruction or access data at a specified address. The contents of registers and memory would be displayed in the front-panel lights. The programmer could read and alter register contents, change program instructions or data in memory or force a branch to another section of code.</p>/n<p>When multiprogramming became the norm it was no longer acceptable to tie up an entire machine for debugging, except for special situations. Programs called <i><a href="/wiki/Debugger" title="Debugger">debuggers</a></i> were written which provided the programmer with the equivalent of the front-panel functions without requiring the entire machine.</p>/n<h3><span class="mw-headline" id="Entertainment">Entertainment</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Front_panel&amp;action=edit&amp;section=5" title="Edit section: Entertainment">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>For fun, bored programmers would create programs to display animated light shows. Front panels in the late 1960s and early 1970s were quite brightly colored. When bootstrap <a href="/wiki/Read-only_memory" title="Read-only memory">ROMs</a> enabled computers to start themselves without operator intervention in the late 1970s and early 1980s, most computers were built without a front switch panel. High-powered calculators, such as the <a href="/wiki/HP_9830" class="mw-redirect" title="HP 9830">HP 9830</a> based on <a href="/wiki/Read-only_memory" title="Read-only memory">ROM</a>, were among the first computers to do away with front panels, and operators.</p>/n<p>Huge banks of "blinkenlights" and "blowenfuzen" were featured on TV and movies as the popular image of the "computer" during the 1950s to 1970. (A <a href="/wiki/Burroughs_Corporation" title="Burroughs Corporation">Burroughs</a> B205 was used as a Hollywood prop for many of these shows.)</p>/n<h2><span class="mw-headline" id="Example">Example</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Front_panel&amp;action=edit&amp;section=6" title="Edit section: Example">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div class="thumb tright">/n<div class="thumbinner" style="width:222px;"><a href="/wiki/File:Digital_pdp8-e2.jpg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/8/8c/Digital_pdp8-e2.jpg/220px-Digital_pdp8-e2.jpg" width="220" height="103" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/8/8c/Digital_pdp8-e2.jpg/330px-Digital_pdp8-e2.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/8/8c/Digital_pdp8-e2.jpg/440px-Digital_pdp8-e2.jpg 2x" data-file-width="4636" data-file-height="2173" /></a>/n<div class="thumbcaption">/n<div class="magnify"><a href="/wiki/File:Digital_pdp8-e2.jpg" class="internal" title="Enlarge"></a></div>/nPDP-8/E front panel</div>/n</div>/n</div>/n<p>The following procedure would bootstrap a <a href="/wiki/PDP-8" title="PDP-8">PDP-8</a> system from an <a href="/wiki/RK05" title="RK05">RK05</a> moving-head <a href="/wiki/Magnetic_storage" title="Magnetic storage">magnetic disk</a>:</p>/n<ol>/n<li>Ensure that the machine is halted by lowering and raising the <b>Halt</b> switch; the front panel "RUN" light should then be off.</li>/n<li>Set the 12 data switches to 0030 (Octal address 30), depress the <b>Load Address</b> ("ADDR LOAD") switch. The address lights will change to "0030".</li>/n<li>Set the switches to 6743, raise the <b>Deposit</b> switch. The data lights will show this instruction.</li>/n<li>Set the switches to 5031, raise the <b>Deposit</b> switch. The data lights will show this instruction.</li>/n<li>Set the switches to 0030 (Octal address 30), depress the <b>Load Address</b> switch. The address lights will change back to "0030".</li>/n<li>Depress the <b>Clear</b> switch.</li>/n<li>Depress the <b>Continue</b> switch. The "RUN" light will illuminate and the operating system on the disk will be bootstrapped.</li>/n</ol>/n<p>This process works by depositing a simple, two-instruction program in memory and executing it. The first instruction commands the disk controller to begin reading the disk from the current disk address into the current memory address. The second instruction is a <a href="/wiki/JMP_(x86_instruction)" title="JMP (x86 instruction)">JMP instruction</a> that jumps to itself endlessly. When "Clear" is pressed, the disk controller's current disk address is set to sector 0 and its current memory address is set to memory location 0000. When the read is commanded, the program stored in disk sector 0 overlays the bootstrap program and, once the JMP instruction is overlaid, the disk program takes control of the machine.</p>/n<h2><span class="mw-headline" id="References">References</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Front_panel&amp;action=edit&amp;section=7" title="Edit section: References">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div class="reflist" style="list-style-type: decimal;">/n<div class="mw-references-wrap">/n<ol class="references">/n<li id="cite_note-1"><span class="mw-cite-backlink"><b><a href="#cite_ref-1">^</a></b></span> <span class="reference-text"><cite class="citation web"><a rel="nofollow" class="external text" href="http://www.retrocomputingtasmania.com/home/projects/burroughs-b6700-mainframe#TOC-B6700-Display-Panel">"Burroughs B6700"</a>. <i>retroComputingTasmania</i><span class="reference-accessdate">.
