class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_educational_programming_languages&amp;action=edit&amp;section=1" title="Edit section: Learning paths">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Many educational programming languages position themselves inside a learning path, that is, a sequence of languages each designed to build on the others moving a student from easy to understand and entertaining environments to full professional environments. Some of the better known are presented below.</p>/n<h3><span class="mw-headline" id="Assembly_language">Assembly language</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_educational_programming_languages&amp;action=edit&amp;section=2" title="Edit section: Assembly language">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Originally, <a href="/wiki/Machine_code" title="Machine code">machine code</a> was the first and only way to program computers. <a href="/wiki/Assembly_language" title="Assembly language">Assembly language</a> was the next type of language used, and thus is one of the oldest families of computer languages in use today. Many dialects and implementations are available, usually some for each computer processor architecture. It is very basic and termed a <a href="/wiki/Low_level_programming_language" class="mw-redirect" title="Low level programming language">low level programming language</a>. It is one of the more difficult languages to work with being <a href="/wiki/Type_system" title="Type system">untyped</a> and rigid, but this is how computers work at low level. Several simplified dialects exist for education.</p>/n<p>Low level languages must be written for a specific <a href="/wiki/Central_processing_unit" title="Central processing unit">processor</a> <a href="/wiki/Computer_architecture" title="Computer architecture">architecture</a> and cannot be written or taught in isolation without referencing the processor for which it was written. Unlike higher level languages, using an educational assembly language needs a representation of a processor, whether virtualized or physical. Assembly is the most helpful language to use for learning about fundamental computer processor operation.</p>/n<ul>/n<li><i><a href="/wiki/Little_Man_Computer" class="mw-redirect" title="Little Man Computer">Little Man Computer</a></i> (LMC) is an instructional model of a simple <a href="/wiki/Von_Neumann_architecture" title="Von Neumann architecture">von Neumann architecture</a> computer with all basic features of modern computers. It can be programmed in machine code (usually decimal) or assembly. It is based on the concept of having a little man locked in a small room. At one end of the room are 100 mailboxes as memory; each can hold a three digit instruction or data. At the other end of the room are two mailboxes labeled INBOX and OUTBOX which receive and emit data. In the middle of the room is a work area with a simple two function (add and subtract) calculator called the Accumulator and a resettable counter called the Program Counter. The counter is similar to what a doorperson uses to count how many people have entered a facility; it can count up 1, or can be reset to 0. As specified by the <a href="/wiki/Von_Neumann_architecture" title="Von Neumann architecture">von Neumann architecture</a>, memory holds both instructions and data. The user loads data into the mailboxes and then signals the little man to begin executing.</li>/n<li><i><a href="/wiki/Next_Byte_Codes" title="Next Byte Codes">Next Byte Codes</a></i> (NBC) is a simple language with assembly language syntax that is used to program <a href="/wiki/Lego_Mindstorms_NXT" title="Lego Mindstorms NXT">Lego Mindstorms NXT</a> programmable bricks. The command line compiler emits NXT compatible machine code, and supports Windows, Mac OS X and Linux.</li>/n<li><i>Little Computer 3</i> (<a href="/wiki/LC-3" title="LC-3">LC-3</a>), is an assembly language with a simplified instruction set, but can be used to write moderately complex assembly programs and is a theoretically viable target for C compilers. It is simpler than x86 assembly but has many features similar to those in more complex languages. These features make it useful for teaching basic programming and computer architecture to beginning college computer science and computer engineering students, which is its most common use.</li>/n<li><i><a href="/wiki/DLX" title="DLX">DLX</a></i> is a <a href="/wiki/Reduced_instruction_set_computer" title="Reduced instruction set computer">reduced instruction set computer</a> (RISC) processor architecture by the main designers of the <a href="/wiki/MIPS_architecture" title="MIPS architecture">MIPS</a> and the <a href="/wiki/Berkeley_RISC" title="Berkeley RISC">Berkeley RISC</a> designs, two benchmark examples of RISC design. DLX is essentially a cleaned up, simplified MIPS, with a simple 32-bit load/store architecture. It is widely used in <a href="/wiki/College" title="College">college</a>-level computer architecture courses.</li>/n<li><i><a href="/wiki/MIX" title="MIX">MIX</a></i> and <i><a href="/wiki/MMIX" title="MMIX">MMIX</a></i> are hypothetical computers used in <a href="/wiki/Donald_Knuth" title="Donald Knuth">Donald Knuth</a>'s monograph, <i><a href="/wiki/The_Art_of_Computer_Programming" title="The Art of Computer Programming">The Art of Computer Programming</a></i> (<i>TAOCP</i>). Paraphrasing Knuth: <i>The MIX systems are computers intended to illustrate machine-level aspects of programming, so its machine language is simple, elegant, easy to learn. It also includes all the complexities needed for high performance in practice, so in principle it can be built and perhaps be competitive with some of the fast general-purpose computers.</i> MIX is hybrid programmable in <a href="/wiki/Binary_numeral_system" class="mw-redirect" title="Binary numeral system">binary</a> and <a href="/wiki/Decimal" title="Decimal">decimal</a> numbers; most programs written for it will work using either form. Software implementations for MIX and MMIX have been developed by Knuth and made freely available. Several versions of both emulators exist. MIX is a 1960s-style computer. It is superseded by MMIX, a newer modern computer architecture, a <a href="/wiki/64-bit" class="mw-redirect" title="64-bit">64-bit</a> <a href="/wiki/Reduced_instruction_set_computer" title="Reduced instruction set computer">RISC</a> <a href="/wiki/Instruction_set" class="mw-redirect" title="Instruction set">instruction set</a> architecture (ISA). For MMIX, Knuth collaborated with the architects of the <a href="/wiki/MIPS_architecture" title="MIPS architecture">MIPS</a> and <a href="/wiki/DEC_Alpha" title="DEC Alpha">Alpha</a> ISAs.</li>/n</ul>/n<h3><span class="mw-headline" id="BASIC">BASIC</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=List_of_educational_programming_languages&amp;action=edit&amp;section=3" title="Edit section: BASIC">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p><i><a href="/wiki/BASIC" title="BASIC">BASIC</a></i> which stands for Beginners All-purpose Symbolic Instruction Code, is a language invented in 1964 to provide computer access to non-science students. It became popular on minicomputers during the 1960s, and became a standard computing language for <a href="/wiki/Microcomputer" title="Microcomputer">microcomputers</a> during the late 1970s and early 1980s. The goals of BASIC were focused on the needs of learning to program easily: be easy for beginners to use, be interactive, provide clear and friendly error messages, respond quickly, do not require an understanding of computer hardware or operating systems. What made BASIC particularly useful for education was the small size of programs. Useful programs to illustrate a concept could be written in a dozen lines. At the same time BASIC did not require mathematical or computer science sophistication.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> BASIC continues to this day to be frequently self-taught with excellent tutorials and implementations. See <a href="/wiki/List_of_BASIC_dialects_by_platform" class="mw-redirect" title="List of BASIC dialects by platform">List of BASIC dialects by platform</a> for a complete list. BASIC offers a learning path from learning oriented BASICs such as <a href="/wiki/Microsoft_Small_Basic" title="Microsoft Small Basic">Microsoft Small Basic</a>, <a href="/wiki/BASIC-256" class="mw-redirect" title="BASIC-256">BASIC-256</a> and <a href="/wiki/SiMPLE" title="SiMPLE">SiMPLE</a>, to more full featured BASICs like <a href="/wiki/Visual_Basic_.NET" title="Visual Basic .NET">Visual Basic .NET</a> and <a href="/wiki/Gambas" title="Gambas">Gambas</a>.</p>/n<ul>/n<li><i><a href="/wiki/Visual_Basic_.NET" title="Visual Basic .NET">Visual Basic .NET</a></i> with a freely available <a rel="nofollow" class="external text" href="http://www.microsoft.com/Express/VB/">Visual Basic Express Edition</a> including an <a rel="nofollow" class="external text" href="http://msdn.microsoft.com/en-us/beginner/bb308754.aspx">K-12 learning center</a> and <a rel="nofollow" class="external text" href="http://msdn.microsoft.com/en-us/beginner/default.aspx">Beginner adult learning center</a></li>/n<li><i><a href="/wiki/Microsoft_Small_Basic" title="Microsoft Small Basic">Microsoft Small Basic</a></i> is a restricted version of Visual Basic designed as a first language, "aimed at bringing 'fun' back to programming". The language is explicitly quite small with only 15 intuitive keywords. By including object specific libraries for things of general interest to children, children can create entertaining, interactive programs, on the net or on the desktop. Lisp was originally created as a practical mathematical notation for computer programs, based on <a href="/wiki/Lambda_calculus" title="Lambda calculus">lambda calculus</a>, which makes it particularly well suited for teaching theories of computing. As one of the earliest languages, Lisp pioneered many ideas in <a href="/wiki/Computer_science" title="Computer science">computer science</a>, including <a href="/wiki/Tree_(data_structure)" title="Tree (data structure)">tree data structures</a>, <a href="/wiki/Garbage_collection_(computer_science)" title="Garbage collection (computer science)">automatic storage management</a>, <a href="/wiki/Type_system" title="Type system">dynamic typing</a>, <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented programming</a>, and the <a href="/wiki/Self-hosting" title="Self-hosting">self-hosting</a> <a href="/wiki/Compiler" title="Compiler">compiler</a> all of which are useful for learning computer science.</p>/n<p>The name <i>LISP</i> derives from "LISt Processing language". <a href="/wiki/Linked_list" title="Linked list">Linked lists</a> are one of the languages' major <a href="/wiki/Data_structure" title="Data structure">data structures</a>, and Lisp <a href="/wiki/Source_code" title="Source code">source code</a> is made of lists. Thus, Lisp programs can manipulate source code as a data structure, giving rise to the <a href="/wiki/Macro_(computer_science)" title="Macro (computer science)">macro</a> systems that allow programmers to create new syntax or even new <a href="/wiki/Domain-specific_language" title="Domain-specific language">domain-specific languages</a> embedded in Lisp. So Lisps are useful for learning language design, and creating custom languages.</p>/n<p>A reasonable learning path would be Logo followed by any educational variant such as Scheme or newLISP, followed by a professional variant such as Common LISP.</p>/n<ul>/n<li><i><a href="/wiki/Logo_(programming_language)" title="Logo (programming language)">Logo</a></i> is a language that was specifically designed to introduce children to programming. The first part of learning Logo deals with "turtle graphics" (derived from <a href="/wiki/Turtle_(robot)" title="Turtle (robot)">turtle robots</a> used as early as 1969 with proto-Logo. In modern implementations, an abstract drawing device, called the <i>turtle</i>, is used to make programming for children very attractive by concentrating on doing <a href="/wiki/Turtle_graphics" title="Turtle graphics">turtle graphics</a>. <a href="/wiki/Seymour_Papert" title="Seymour Papert">Seymour Papert</a>, one of the creators of Logo, was a major thinker in <a href="/wiki/Constructionism_(learning_theory)" title="Constructionism (learning theory)">constructionism</a>, a variety of <a href="/wiki/Constructivism_(learning_theory)" class="mw-redirect" title="Constructivism (learning theory)">constructivist learning theory</a>. Papert argued that activities like writing would naturally be learned by much younger children providing that they adopted a computing culture.<sup id="cite_ref-RedC_5-0" class="reference"><a href="#cite_note-RedC-5">[5]</a></sup> Logo was thus designed not only to teach programming, and computing concepts, but to enhance a child's entire well being in a culture increasingly dominated by technology, "<i>more important than having an early start on intellectual building, is being saved from a long period of dependency during which one learns to think of learning as something that has to be dished out by a more powerful other...Such children would not define themselves or allow society to define them as intellectually helpless.</i>"<sup id="cite_ref-RedC_5-1" class="reference"><a href="#cite_note-RedC-5">[5]</a></sup> It has been used with children as young as 3 and has a track record of 30 years of success in education. Since Logo is actually a streamlined version of <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a> with more advanced students it can be used to introduce the basic concepts of computer science and even artificial intelligence. <a rel="nofollow" class="external text" href="http://www.cs.berkeley.edu/~bh/">Brian Harvey</a> wrote a series <i>Computer Science Logo Style</i><sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup> for self study of computer science based on Logo. Logo is widely available on virtually every platform, in both free and commercial versions.</li>/n</ul>/n<ul>/n<li><i><a href="/wiki/Scheme_(programming_language)" title="Scheme (programming language)">Scheme</a></i> was originally designed in 1975 to serve a tutorial purpose.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup> LISPs of the day used non-recursive control structures to implement <a href="/wiki/Lambda_calculus" title="Lambda calculus">lambda calculus</a>, primarily since LISPs were still being implemented for efficiency reasons in hardware. <a href="/wiki/Guy_L._Steele,_Jr." class="mw-redirect" title="Guy L. Steele, Jr.">Guy L. Steele, Jr.</a> and <a href="/wiki/Gerald_Jay_Sussman" title="Gerald Jay Sussman">Gerald Jay Sussman</a> constructed Scheme as a fast interpreted language atop an underlying LISP with cheap procedure calls.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup> Pedagogically this allowed for teaching programming in terms of <a href="/wiki/Domain-specific_language" title="Domain-specific language">domain-specific languages</a> and <a href="/wiki/Meta-circular_evaluator" title="Meta-circular evaluator">meta-circular evaluators</a>. The publication of <a href="/wiki/Structure_and_Interpretation_of_Computer_Programs" title="Structure and Interpretation of Computer Programs">Structure and Interpretation of Computer Programs</a> in 1984 incorporated this educational philosophy into a canonical textbook, which changed the predominance of Pascal as the university programming language.</li>/n</ul>/n<table class="cquote" role="presentation" style="margin:auto; border-collapse: collapse; border: none; background-color: transparent; width: auto;">/n<tr>/n<td style="width: 20px; vertical-align: top; border: none; color: #B2B7F2; font-size: 40px; font-family: 'Times New Roman', Times, serif; font-weight: bold; line-height: .6em; text-align: left; padding: 10px 10px;">“</td>/n<td style="vertical-align: top; border: none; padding: 4px 10px;">The publication of Abelson and Sussman's Structure and Interpretation of Computer Programs (sicp) (Abelson et al., 1985) revolutionized the landscape of the introductory computing curriculum in the 1980s. Most importantly, the book liberated the introductory course from the tyranny of syntax. Instead of arranging a course around the syntax of a currently fashionable language, sicp focused the first course on the study of important ideas in computing: functional abstraction, data abstraction, streams, data-directed programming, implementation of message-passing objects, interpreters, compilers, and <a href="/wiki/Register_machine" title="Register machine">register machines</a>./n<p>Over a short period, many universities in the US and around the world switched their ﬁrst course to sicp and Scheme. The book became a major bestseller for MIT Press. Along with sicp, the Scheme language became widely used. It was no longer the subject of a few individual courses at Indiana University, MIT, and Yale, but the language of choice in introductory courses all over the world.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[9]</a></sup></p>/n</td>/n<td style="width: 20px; vertical-align: bottom; border: none; color: #B2B7F2; font-size: 40px; font-family: 'Times New Roman', Times, serif; font-weight: bold; line-height: .6em; text-align: right; padding: 10px 10px;">”</td>/n</tr>/n</table>/n<dl>/n<dd>Since then the Scheme community has introduced several pedagogic programming environments for more modern approaches. The Kodu language is designed specifically for game development and provides specialized primitives derived from gaming scenarios. An education specific compiler / IDE, called <a href="/wiki/Helium_(Haskell)" title="Helium (Haskell)">Helium</a> has been created. Another advantage of Haskell is in teaching inductive methods. Because of the advantages of Haskell's syntax inductive proofs become as easy or easier as they are on paper, unlike the LISP/Scheme family which introduces additional syntax.<sup id="cite_ref-26" class="reference"><a href="#cite_note-26">[26]</a></sup><sup id="cite_ref-27" class="reference"><a href="#cite_note-27">[27]</a></sup></li>/n</ul>/n<ul>/n<li><i><a href="/wiki/Flowgorithm_(programming_language)" class="mw-redirect" title="Flowgorithm (programming language)">Flowgorithm</a></i> is a graphical <a href="/wiki/Authoring_tool" class="mw-redirect" title="Authoring tool">authoring tool</a> for writing and executing programs via <a href="/wiki/Flowchart" title="Flowchart">flowcharts</a>. The approach is designed to emphasize the algorithm rather than the syntax of a given language. It is built on a semantic framework based in category theory, with a syntax similar to that of Pascal or <a href="/wiki/Modula-2" title="Modula-2">Modula-2</a>.
