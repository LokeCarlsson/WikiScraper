implementation</h1>/n									<div id="bodyContent" class="mw-body-content">/n									<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>/n								<div id="contentSub"><span class="mw-redirectedfrom">  (Redirected from <a href="/w/index.php?title=Reference_implementation_(computing)&amp;redirect=no" class="mw-redirect" title="Reference implementation (computing)">Reference implementation (computing)</a>)</span></div>/n												<div id="jump-to-nav" class="mw-jump">/n					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>/n				</div>/n				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p>In the <a href="/wiki/Software_development_process" title="Software development process">software development process</a>, a <b>reference implementation</b> (or, less frequently, <i>sample implementation</i> or <i>model implementation</i>) is the standard from which all other implementations and corresponding customizations are derived. An improvement to a reference implementation reflects an unchanging <a href="/wiki/Specification_(technical_standard)" title="Specification (technical standard)">specification</a>. Conversely, a failed attempt at an implementation may prove that the specification is not suitable and needs improvement itself. <a href="/wiki/Software_testing" title="Software testing">Testing</a> the implementation-vs.-specification relationship further enhances the <a href="/wiki/Product_family_engineering" title="Product family engineering">production</a>'s inter-process efficiencies:</p>/n<blockquote class="templatequote">/n<p>A reference implementation is, in general, an implementation of a specification to be used as a definitive interpretation for that specification (This definition is a bit grandiloquent but works). During the development of the ... conformance test suite, at least one relatively trusted implementation of each interface is necessary to (1) discover errors or ambiguities in the specification, and (2) validate the correct functioning of the test suite.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup></p>/n</blockquote>/n<blockquote class="templatequote">/n<p>Characteristics of a Reference Implementation:</p>/n<ol>/n<li>Developed concurrently with the specification and test suite;</li>/n<li>Verifies that specification is implementable;</li>/n<li>Enables the test suite to be tested;</li>/n<li>Serves as a Gold Standard against which other implementations can be measured;</li>/n<li>Helps to clarify the intent of the specification in situations where <a href="/wiki/Conformance_testing" title="Conformance testing">conformance tests</a> are inadequate<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup></li>/n</ol>/n</blockquote>/n<p>Reference implementations of algorithms, for instance <a href="/wiki/Cryptographic_algorithm" class="mw-redirect" title="Cryptographic algorithm">cryptographic algorithms</a>, are often the result or the input of <a href="/wiki/Standardization" title="Standardization">standardization</a> processes. In this function they are often dedicated to the <a href="/wiki/Public_domain" title="Public domain">public domain</a> with their <a href="/wiki/Source_code" title="Source code">source code</a> as <a href="/wiki/Public_domain_software" class="mw-redirect" title="Public domain software">public domain software</a>. Examples are the first <a href="/wiki/CERN_HTTPd" class="mw-redirect" title="CERN HTTPd">CERN's httpd</a>,<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> <a href="/wiki/Serpent_(cipher)" title="Serpent (cipher)">Serpent</a> cipher, <a href="/wiki/Base64" title="Base64">base64</a> variants,<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup> and <a href="/wiki/SHA-3" title="SHA-3">SHA-3</a>.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup> The <a href="/wiki/Openwall_Project" title="Openwall Project">Openwall Project</a> maintains a list of several algorithms with their reference <a href="/wiki/Source_code" title="Source code">source code</a> in the public domain.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup></p>/n<p>A reference implementation may or may not be production quality. For example, the <a href="/wiki/Fraunhofer_Society" title="Fraunhofer Society">Fraunhofer</a> reference implementation of the <a href="/wiki/MP3" title="MP3">MP3</a> standard usually does not compare favorably to other common implementations, such as <a href="/wiki/LAME" title="LAME">LAME</a>, in listening tests that determine sound quality.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2012)">citation needed</span></a></i>]</sup> In contrast, <a href="/wiki/CPython" title="CPython">CPython</a>, the reference implementation of the <a href="/wiki/Python_programming_language" class="mw-redirect" title="Python programming language">Python programming language</a><sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup>, is also the implementation most widely used in production.</p>/n<h2><span class="mw-headline" id="References">References</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Reference_implementation&amp;action=edit&amp;section=1" title="Edit section: References">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div class="reflist columns references-column-width" style="-moz-column-width: 30em; -webkit-column-width: 30em; column-width: 30em; list-style-type: decimal;">/n<ol class="references">/n<li id="cite_note-1"><span class="mw-cite-backlink"><b><a href="#cite_ref-1">^</a></b></span> <span class="reference-text">Dalci, Eric; Fong, Elizabeth; Goldfine, Alan (2003). <i>Requirements for GSC-IS Reference Implementations</i>.
