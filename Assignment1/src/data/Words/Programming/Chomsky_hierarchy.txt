id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p>In the <a href="/wiki/Formal_language" title="Formal language">formal languages</a> of <a href="/wiki/Computer_science" title="Computer science">computer science</a> and <a href="/wiki/Linguistics" title="Linguistics">linguistics</a>, the <b>Chomsky hierarchy</b> (occasionally referred to as <b>Chomsky–Schützenberger hierarchy</b>) is a <a href="/wiki/Containment_hierarchy" class="mw-redirect" title="Containment hierarchy">containment hierarchy</a> of classes of <a href="/wiki/Formal_grammar" title="Formal grammar">formal grammars</a>. This hierarchy of grammars was described by <a href="/wiki/Noam_Chomsky" title="Noam Chomsky">Noam Chomsky</a> in&#160;1956.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> It is also named after <a href="/wiki/Marcel-Paul_Sch%C3%BCtzenberger" title="Marcel-Paul Schützenberger">Marcel-Paul Schützenberger</a>, who played a crucial role in the development of the theory of <a href="/wiki/Formal_language" title="Formal language">formal languages</a>.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 grammars</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Chomsky_hierarchy&amp;action=edit&amp;section=1" title="Edit section: Formal grammars">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Formal_grammar" title="Formal grammar">Formal grammar</a></div>/n<p>A formal grammar of this type consists of a finite set of <i><a href="/wiki/Formal_grammar#Introductory_example" title="Formal grammar">production rules</a></i> (<i>left-hand side</i> → <i>right-hand side</i>), where each side consists of a finite sequence of the following symbols:</p>/n<ul>/n<li>a finite set of <i><a href="/wiki/Nonterminal_symbol" class="mw-redirect" title="Nonterminal symbol">nonterminal symbols</a></i> (indicating that some production rule can yet be applied)</li>/n<li>a finite set of <i><a href="/wiki/Terminal_symbol" class="mw-redirect" title="Terminal symbol">terminal symbols</a></i> (indicating that no production rule can be applied)</li>/n<li>a <i>start symbol</i> (a distinguished nonterminal symbol)</li>/n</ul>/n<p>A <a href="/wiki/Formal_grammar" title="Formal grammar">formal grammar</a> provides an <a href="/wiki/Axiom_schema" title="Axiom schema">axiom schema</a> for (or <i>generates</i>) a <i>formal language</i>, which is a (usually infinite) set of <a href="/wiki/String_(computer_science)" title="String (computer science)">finite-length sequences of symbols</a> that may be constructed by applying <a href="/wiki/Production_(computer_science)" title="Production (computer science)">production rules</a> to another sequence of symbols (which initially contains just the start symbol). A rule may be applied by replacing an occurrence of the symbols on its left-hand side with those that appear on its right-hand side. A sequence of rule applications is called a <i>derivation</i>. Such a grammar defines the formal language: all words consisting solely of terminal symbols which can be reached by a derivation from the start symbol.</p>/n<p>Nonterminals are often represented by uppercase letters, terminals by lowercase letters, and the start symbol by <span class="texhtml mvar" style="font-style:italic;">S</span>. For example, the grammar with terminals <span class="texhtml">{<i>a, b</i>}</span>, nonterminals <span class="texhtml">{<i>S, A, B</i>}</span>, production rules</p>/n<dl>/n<dd><span class="texhtml mvar" style="font-style:italic;">S</span> → <span class="texhtml mvar" style="font-style:italic;">AB</span></dd>/n<dd><span class="texhtml mvar" style="font-style:italic;">S</span> → ε (where ε is the empty string)</dd>/n<dd><span class="texhtml mvar" style="font-style:italic;">A</span> → <span class="texhtml mvar" style="font-style:italic;">aS</span></dd>/n<dd><span class="texhtml mvar" style="font-style:italic;">B</span> → <span class="texhtml mvar" style="font-style:italic;">b</span></dd>/n</dl>/n<p>and start symbol <span class="texhtml mvar" style="font-style:italic;">S</span>, defines the language of all words of the form <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math defines the same language: Terminals <span class="texhtml">{<i>a, b</i>}</span>, Nonterminals <span class="texhtml">{<i>S</i>}</span>, Start symbol <span class="texhtml mvar" style="font-style:italic;">S</span>, Production rules</p>/n<dl>/n<dd><span class="texhtml mvar" style="font-style:italic;">S</span> → <span class="texhtml mvar" These are defined by rules of the form <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math xmlns="http://www.w3.org/1998/Math/MathML" >/n  <semantics>/n    <mrow class="MJX-TeXAtom-ORD">/n      <mstyle displaystyle="true" scriptlevel="0">/n        <mi>A</mi>/n        <mo stretchy="false">&#x2192;<!-- → --></mo>/n        <mi>&#x03B3;<!-- γ --></mi>/n      </mstyle>/n    </mrow>/n    <annotation encoding="application/x-tex">{\displaystyle A\rightarrow \gamma }</annotation>/n  </semantics>/n</math></span><img src="https://wikimedia.org/api/rest_v1/media/math/render/svg/91998fd76701871c7df169d4e9114b5970c61877" class="mwe-math-fallback-image-inline" aria-hidden="true" style="vertical-align: -0.838ex; width:6.651ex; height:2.843ex;" alt="A \rightarrow \gamma" /></span> with <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math xmlns="http://www.w3.org/1998/Math/MathML" >/n  <semantics>/n    <mrow class="MJX-TeXAtom-ORD">/n      <mstyle displaystyle="true" scriptlevel="0">/n        <mi>A</mi>/n      </mstyle>/n    </mrow>/n    <annotation encoding="application/x-tex">{\displaystyle A}</annotation>/n  </semantics>/n</math></span><img src="https://wikimedia.org/api/rest_v1/media/math/render/svg/7daff47fa58cdfd29dc333def748ff5fa4c923e3" class="mwe-math-fallback-image-inline" aria-hidden="true" style="vertical-align: -0.338ex; Context-free languages—or rather its subset of <a href="/wiki/Deterministic_context-free_language" title="Deterministic context-free language">deterministic context-free language</a>—are the theoretical basis for the phrase structure of most <a href="/wiki/Programming_language" title="Programming language">programming languages</a>, though their syntax also includes context-sensitive name resolution due to declarations and <a href="/wiki/Scope_(computer_science)" title="Scope (computer science)">scope</a>. Such a grammar restricts its rules to a single nonterminal on the left-hand side and a right-hand side consisting of a single terminal, possibly followed by a single nonterminal (right regular). Alternatively, the right-hand side of the grammar can consist of a single terminal, possibly preceded by a single nonterminal (left regular). These generate the same languages. However, if left-regular rules and right-regular rules are combined, the language need no longer be regular. The rule <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math xmlns="http://www.w3.org/1998/Math/MathML" >/n  <semantics>/n    <mrow class="MJX-TeXAtom-ORD">/n      <mstyle displaystyle="true" scriptlevel="0">/n        <mi>S</mi>/n        <mo stretchy="false">&#x2192;<!-- → --></mo>/n        <mi>&#x03F5;<!-- ϵ --></mi>/n      </mstyle>/n    </mrow>/n    <annotation encoding="application/x-tex">{\displaystyle S\rightarrow \epsilon }</annotation>/n  </semantics>/n</math></span><img src="https://wikimedia.org/api/rest_v1/media/math/render/svg/b118a2c5fb5cbe4c11e07687a2b702019d508431" class="mwe-math-fallback-image-inline" aria-hidden="true" style="vertical-align: -0.338ex; width:6.089ex; height:2.176ex;" alt="S \rightarrow \epsilon" /></span> is also allowed here if <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math xmlns="http://www.w3.org/1998/Math/MathML" >/n  <semantics>/n    <mrow class="MJX-TeXAtom-ORD">/n      <mstyle displaystyle="true" scriptlevel="0">/n        <mi>S</mi>/n      </mstyle>/n    </mrow>/n    <annotation encoding="application/x-tex">{\displaystyle S}</annotation>/n  </semantics>/n</math></span><img src="https://wikimedia.org/api/rest_v1/media/math/render/svg/4611d85173cd3b508e67077d4a1252c9c05abca2" class="mwe-math-fallback-image-inline" aria-hidden="true" style="vertical-align: -0.338ex; width:1.51ex; height:2.176ex;" alt="S" /></span> does not appear on the right side of any rule. These languages are exactly all languages that can be decided by a <a href="/wiki/Finite_state_automaton" class="mw-redirect" title="Finite state automaton">finite state automaton</a>. Additionally, this family of formal languages can be obtained by <a href="/wiki/Regular_expression" title="Regular expression">regular expressions</a>.
