class="mw-redirect" title="Windows PowerShell">Windows PowerShell</a> 1.0, running on <a href="/wiki/Windows_Vista" title="Windows Vista">Windows Vista</a></div>/n</div>/n</div>/n<p>A <b>command-line interface</b> or <b>command language interpreter</b> (<b>CLI</b>), also known as <b>command-line user interface</b>, <b>console user interface</b><sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> and <b>character user interface</b> (<b>CUI</b>), is a means of interacting with a <a href="/wiki/Computer_program" title="Computer program">computer program</a> where the <a href="/wiki/User_(computing)" title="User (computing)">user</a> (or <a href="/wiki/Client_(computing)" title="Client (computing)">client</a>) issues <a href="/wiki/Command_(computing)" title="Command (computing)">commands</a> to the program in the form of successive lines of text (command lines). A program which handles the interface is called a <b>command language interpreter</b> or shell.</p>/n<p>The CLI was the primary means of interaction with most computer systems on <a href="/wiki/Computer_terminal" title="Computer terminal">computer terminals</a> in the mid-1960s, and continued to be used throughout the 1970s and 1980s on <a href="/wiki/OpenVMS" title="OpenVMS">OpenVMS</a>, <a href="/wiki/Unix" title="Unix">Unix</a> systems and personal computer systems including <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a>, <a href="/wiki/CP/M" title="CP/M">CP/M</a> and <a href="/wiki/Apple_DOS" title="Apple DOS">Apple DOS</a>. The interface is usually implemented with a command line <a href="/wiki/Shell_(computing)" title="Shell (computing)">shell</a>, which is a program that accepts commands as text input and converts commands into appropriate operating system functions.</p>/n<p>Command-line interfaces to computer operating systems are less widely used by casual computer users, who favor <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interfaces</a> or menu-driven interaction.</p>/n<p>Alternatives to the command line include, but are not limited to <a href="/wiki/Text_user_interface" class="mw-redirect" title="Text user interface">text user interface</a> <a href="/wiki/Menu_(computing)" title="Menu (computing)">menus</a> (see <a href="/wiki/IBM_AIX_SMIT" class="mw-redirect" title="IBM AIX SMIT">IBM AIX SMIT</a> for example), <a href="/wiki/Keyboard_shortcut" title="Keyboard shortcut">keyboard shortcuts</a>, and various other <a href="/wiki/Desktop_metaphor" title="Desktop metaphor">desktop metaphors</a> centered on the <a href="/wiki/Pointer_(graphical_user_interfaces)" class="mw-redirect" title="Pointer (graphical user interfaces)">pointer</a> (usually controlled with a <a href="/wiki/Mouse_(computing)" class="mw-redirect" title="Mouse (computing)">mouse</a>). Examples of this include the Windows versions 1, 2, 3, 3.1, and 3.11 (an OS shell that runs in DOS), <a href="/wiki/DOSSHELL" class="mw-redirect" title="DOSSHELL">DosShell</a>, and Mouse Systems PowerPanel.</p>/n<p>Command-line interfaces are often preferred by more advanced computer users, as they often provide a more concise and powerful means to control a program or operating system.</p>/n<p>Programs with command-line interfaces are generally easier to automate via <a href="/wiki/Scripting_language" title="Scripting language">scripting</a>.</p>/n<p>Command line interfaces for software other than operating systems include a number of programming languages such as Tcl/Tk, PHP and others, as well as utilities such as the compression utilities WinZip and UltimateZip, and some FTP and ssh/telnet clients.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a command line requires fewer system resources to implement. Since options to commands are given in a few characters in each command line, an experienced user finds the options easier to access. Automation of repetitive tasks is simplified - most operating systems using a command line interface support some mechanism for storing frequently used sequences in a disk file, for re-use; this may extend to a <a href="/wiki/Scripting_language" title="Scripting language">scripting language</a> that can take parameters and variable options. A command-line history can be kept, allowing review or repetition of commands.</p>/n<p>A command-line system may require paper or on-line manuals for the user's reference, although often a "help" option provides a concise review of the options of a command. The command line environment may not provide the graphical enhancements such as different fonts or extended edit windows found in a GUI. It may be difficult for a new user to become familiar with all the commands and options available, compared with the drop-down menus of a graphical user interface, without repeated reference to manuals.</p>/n<h2><span class="mw-headline" id="Operating_system_command-line_interfaces">Operating system command-line interfaces</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=2" title="Edit section: Operating system command-line interfaces">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Operating system (OS) command line interfaces are usually distinct programs supplied with the operating system. A program that implements such a text interface is often called a command-line interpreter, command processor or <a href="/wiki/Shell_(computing)" title="Shell (computing)">shell</a>.</p>/n<p>Examples of command-line interpreters include <a href="/wiki/Digital_Equipment_Corporation" title="Digital Equipment Corporation">DEC's</a> <a href="/wiki/DIGITAL_Command_Language" title="DIGITAL Command Language">DIGITAL Command Language</a> (DCL) in <a href="/wiki/OpenVMS" title="OpenVMS">OpenVMS</a> and <a href="/wiki/RSX-11" title="RSX-11">RSX-11</a>, the various <a href="/wiki/Unix_shell" title="Unix shell">Unix shells</a> (<a href="/wiki/Bourne_shell" title="Bourne shell">sh</a>, <a href="/wiki/KornShell" title="KornShell">ksh</a>, <a href="/wiki/C_shell" title="C shell">csh</a>, <a href="/wiki/Tcsh" title="Tcsh">tcsh</a>, <a href="/wiki/Bash_(Unix_shell)" title="Bash (Unix shell)">bash</a>, etc.), the historical <a href="/wiki/CP/M" title="CP/M">CP/M</a> CCP, and <a href="/wiki/DOS" title="DOS">MS-DOS/IBM-DOS/DR-DOS</a>'s <a href="/wiki/COMMAND.COM" title="COMMAND.COM">COMMAND.COM</a>, as well as the <a href="/wiki/OS/2" title="OS/2">OS/2</a> and the Windows <a href="/wiki/CMD.EXE" class="mw-redirect" title="CMD.EXE">CMD.EXE</a> programs, the latter groups being based heavily on DEC's RSX-11 and <a href="/wiki/RSTS/E" title="RSTS/E">RSTS</a> CLIs. Under most operating systems, it is possible to replace the default shell program with alternatives; examples include <a href="/wiki/4DOS" title="4DOS">4DOS</a> for DOS, <a href="/wiki/4OS2" title="4OS2">4OS2</a> for OS/2, and <a href="/wiki/Take_Command_Console" title="Take Command Console">4NT</a> or <a href="/wiki/Take_Command_(command_line_interpreter)" title="Take Command (command line interpreter)">Take Command</a> for Windows.</p>/n<p>Although the term 'shell' is often used to describe a command-line interpreter, strictly speaking a 'shell' can be any program that constitutes the user-interface, including fully graphically oriented ones. For example, the default Windows GUI is a shell program named EXPLORER.EXE, as defined in the SHELL=EXPLORER.EXE line in the WIN.INI configuration file. These programs are shells, but not CLIs.</p>/n<h2><span class="mw-headline" id="Application_command-line_interfaces">Application command-line interfaces</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=3" title="Edit section: Application command-line interfaces">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Application programs (as opposed to operating systems) may also have command line interfaces.</p>/n<p>An application program may support none, any, or all of these three major types of command line interface mechanisms:</p>/n<ol>/n<li><b>Parameters</b>: Most operating systems support a means to pass additional information to a program when it is launched. When a program is launched from an OS command line shell, additional text provided along with the program name is passed to the launched program.</li>/n<li><b>Interactive command line sessions</b>: After launch, a program may provide an operator with an independent means to enter commands in the form of text.</li>/n<li><b>OS inter-process communication</b>: Most operating systems support means of <a href="/wiki/Inter-process_communication" title="Inter-process communication">inter-process communication</a> (for example; <a href="/wiki/Standard_streams" title="Standard streams">standard streams</a> or <a href="/wiki/Named_pipe" title="Named pipe">named pipes</a>). Command lines from client processes may be redirected to a CLI program by one of these methods.</li>/n</ol>/n<p>Some applications support only a CLI, presenting a CLI prompt to the user and acting upon command lines as they are entered. Other programs support both a CLI and a GUI. In some cases, a GUI is simply a <a href="/wiki/Wrapper_function" title="Wrapper function">wrapper</a> around a separate CLI <a href="/wiki/Executable_file" class="mw-redirect" title="Executable file">executable file</a>. In other cases, a program may provide a CLI as an optional alternative to its GUI. CLIs and GUIs often support different functionality. For example, all features of <a href="/wiki/MATLAB" title="MATLAB">MATLAB</a>, a <a href="/wiki/Numerical_analysis" title="Numerical analysis">numerical analysis</a> computer program, are available via the CLI, whereas the MATLAB GUI exposes only a subset of features.</p>/n<p>The early Sierra games, like the first three <a href="/wiki/King%27s_Quest" title="King's Quest">King's Quest</a> games (1984–1986), used commands from an internal command line to move the character around in the graphic window.</p>/n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=4" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The command-line interface evolved from a form of dialog once conducted by humans over <a href="/wiki/Teleprinter" title="Teleprinter">teleprinter</a> (TTY) machines, in which human operators remotely exchanged information, usually one line of text at a time. Early computer systems often used teleprinter machines as the means of interaction with a human operator. The computer became one end of the human-to-human teleprinter model. So instead of a human communicating with another human over a teleprinter, a human communicated with a computer.</p>/n<p>The mechanical teleprinter was replaced by a <a href="/wiki/Dumb_terminal#Early_VDUs" class="mw-redirect" title="Dumb terminal">"glass tty"</a>, a keyboard and screen emulating the teleprinter. <a href="/wiki/Computer_terminal#.22Intelligent.22_terminals" title="Computer terminal">"Smart" terminals</a> permitted additional functions, such as cursor movement over the entire screen, or local editing of data on the terminal for transmission to the computer. As the <a href="/wiki/Microcomputer_revolution" title="Microcomputer revolution">microcomputer revolution</a> replaced the traditional&#160;– minicomputer + terminals&#160;– <a href="/wiki/Time_sharing" class="mw-redirect" title="Time sharing">time sharing</a> architecture, hardware terminals were replaced by <a href="/wiki/Terminal_emulator" title="Terminal emulator">terminal emulators</a>&#160;— PC software that interpreted terminal signals sent through the PC's <a href="/wiki/Serial_port" title="Serial port">serial ports</a>. These were typically used to interface an organization's new PC's with their existing mini- or mainframe computers, or to connect PC to PC. Some of these PCs were running <a href="/wiki/Bulletin_Board_System" class="mw-redirect" title="Bulletin Board System">Bulletin Board System</a> software.</p>/n<p>Early operating system CLIs were implemented as part of <a href="/wiki/Resident_monitor" title="Resident monitor">resident monitor</a> programs, and could not easily be replaced. The first implementation of the shell as a replaceable component was part of the <a href="/wiki/Multics" title="Multics">Multics</a> <a href="/wiki/Time-sharing" title="Time-sharing">time-sharing</a> <a href="/wiki/Operating_system" title="Operating system">operating system</a>.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup>In 1964, <a href="/wiki/MIT_Computation_Center" title="MIT Computation Center">MIT Computation Center</a> staff member <a href="/wiki/Louis_Pouzin" title="Louis Pouzin">Louis Pouzin</a> developed the <a href="/wiki/RUNCOM" class="mw-redirect" title="RUNCOM">RUNCOM</a> tool for executing command scripts while allowing argument substitution.<sup id="cite_ref-multicians_3-0" class="reference"><a href="#cite_note-multicians-3">[3]</a></sup> Pouzin coined the term "<a href="/wiki/Shell_(computing)" title="Shell (computing)">shell</a>" to describe the technique of using commands like a programming language, and wrote a paper about how to implement the idea in the <a href="/wiki/Multics" title="Multics">Multics</a> operating system.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup> Pouzin returned to his native France in 1965, and the first Multics shell was developed by <a href="/wiki/Glenda_Schroeder" title="Glenda Schroeder">Glenda Schroeder</a>.<sup id="cite_ref-multicians_3-1" class="reference"><a href="#cite_note-multicians-3">[3]</a></sup></p>/n<p>The first <a href="/wiki/Unix_shell" title="Unix shell">Unix shell</a>, the <a href="/wiki/Thompson_shell" title="Thompson shell">V6 shell</a>, was developed by <a href="/wiki/Ken_Thompson" title="Ken Thompson">Ken Thompson</a> in 1971 at <a href="/wiki/Bell_Labs" title="Bell Labs">Bell Labs</a> and was modeled after Schroeder's Multics shell.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup><sup id="cite_ref-ibm_6-0" class="reference"><a href="#cite_note-ibm-6">[6]</a></sup> The <a href="/wiki/Bourne_shell" title="Bourne shell">Bourne shell</a> was introduced in 1977 as a replacement for the V6 shell. Although it is used as an interactive command interpreter, it was also intended as a scripting language and contains most of the features that are commonly considered to produce structured programs. The Bourne shell led to the development of the <a href="/wiki/Korn_shell" class="mw-redirect" title="Korn shell">Korn shell</a> (ksh), <a href="/wiki/Almquist_shell" title="Almquist shell">Almquist shell</a> (ash), and the popular <a href="/wiki/Bourne-again_shell" class="mw-redirect" title="Bourne-again shell">Bourne-again shell</a> (or Bash).<sup id="cite_ref-ibm_6-1" class="reference"><a href="#cite_note-ibm-6">[6]</a></sup></p>/n<p>Early microcomputers themselves were based on a command-line interface such as <a href="/wiki/CP/M" title="CP/M">CP/M</a>, <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a> or <a href="/wiki/AppleSoft_BASIC" class="mw-redirect" title="AppleSoft BASIC">AppleSoft BASIC</a>. Throughout the 1980s and 1990s—especially after the introduction of the <a href="/wiki/Apple_Macintosh" class="mw-redirect" title="Apple Macintosh">Apple Macintosh</a> and <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>—command line interfaces were replaced in popular usage by the <a href="/wiki/Graphical_User_Interface" class="mw-redirect" title="Graphical User Interface">Graphical User Interface</a>. The command line remains in use, however, by <a href="/wiki/System_Administrator" class="mw-redirect" title="System Administrator">system administrators</a> and other advanced users for system administration, <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a>, and <a href="/wiki/Batch_processing" title="Batch processing">batch processing</a>.</p>/n<p>In November 2006, <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> released version 1.0 of <a href="/wiki/Windows_PowerShell" class="mw-redirect" title="Windows PowerShell">Windows PowerShell</a> (formerly codenamed <i>Monad</i>), which combined features of traditional Unix shells with their proprietary object-oriented <a href="/wiki/.NET_Framework" title=".NET Framework">.NET Framework</a>. <a href="/wiki/MinGW" title="MinGW">MinGW</a> and <a href="/wiki/Cygwin" title="Cygwin">Cygwin</a> are <a href="/wiki/Open_source" class="mw-redirect" title="Open source">open-source</a> packages for Windows that offer a Unix-like CLI. Microsoft provides <a href="/wiki/MKS_Inc." title="MKS Inc.">MKS Inc.</a>'s <a href="/wiki/Korn_shell" class="mw-redirect" title="Korn shell">ksh</a> implementation <i>MKS Korn shell</i> for Windows through their <a href="/wiki/Windows_Services_for_UNIX" title="Windows Services for UNIX">Services for UNIX</a> add-on.</p>/n<p>Since 2001, the <a href="/wiki/Macintosh" title="Macintosh">Macintosh</a> operating system is based on a variation of Unix called <a href="/wiki/Darwin_(operating_system)" title="Darwin (operating system)">Darwin</a>. On these computers, users can access a Unix-like command-line interface called <a href="/wiki/Terminal_(Mac_OS_X)" class="mw-redirect" title="Terminal (Mac OS X)">Terminal</a> found in the Applications Utilities folder. This terminal uses <a href="/wiki/Bash_(Unix_shell)" title="Bash (Unix shell)">bash</a> by default.</p>/n<div class="thumb tright">/n<div class="thumbinner" style="width:302px;"><a GUI</div>/n</div>/n</div>/n<h2><span class="mw-headline" id="Usage">Usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a This is typically the case with <a href="/wiki/Operating_system_shell" class="mw-redirect" title="Operating system shell">operating system command shells</a>. CLIs are also used by systems with insufficient resources to support a graphical user interface. Some computer language systems (such as <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Forth_(programming_language)" title="Forth (programming language)">Forth</a>, <a href="/wiki/LISP" class="mw-redirect" title="LISP">LISP</a>, <a href="/wiki/Rexx" title="Rexx">Rexx</a>, and many dialects of <a href="/wiki/BASIC" title="BASIC">BASIC</a>) provide an interactive command-line mode to allow for rapid evaluation of code.</p>/n<p>CLIs are often used by programmers and system administrators, in engineering and scientific environments, and by technically advanced personal computer users. CLIs are also popular among people with visual disability, since the commands and responses can be displayed using <a href="/wiki/Refreshable_Braille_display" class="mw-redirect" title="Refreshable Braille display">Refreshable Braille displays</a>.</p>/n<h2><span class="mw-headline" id="Anatomy_of_a_shell_CLI">Anatomy of a shell CLI</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=6" title="Edit section: Anatomy of a shell CLI">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<table class="plainlinks metadata ambox ambox-content ambox-Refimprove" role="presentation">/n<tr>/n<td class="mbox-image">/n<div style="width:52px"><a href="/wiki/File:Question_book-new.svg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/50px-Question_book-new.svg.png" width="50" height="39" srcset="//upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/75px-Question_book-new.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/100px-Question_book-new.svg.png 2x" data-file-width="512" data-file-height="399" /></a></div>/n</td>/n<td class="mbox-text">/n<div class="mbox-text-span">This section <b>needs additional citations for <a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability">verification</a></b>. <span class="hide-when-compact">Please help <a class="external text" href="//en.wikipedia.org/w/index.php?title=Command-line_interface&amp;action=edit">improve this article</a> by <a href="/wiki/Help:Introduction_to_referencing_with_Wiki_Markup/1" title="Help:Introduction to referencing with Wiki Markup/1">adding citations to reliable sources</a>. Unsourced material may be challenged and removed.</span> <small><i>(July 2015)</i></small> <small class="hide-when-compact"><i>(<a href="/wiki/Help:Maintenance_template_removal" title="Help:Maintenance template removal">Learn how and when to remove this template message</a>)</i></small></div>/n</td>/n</tr>/n</table>/n<p>The general pattern of an OS command line interface<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup><sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup> is:</p>/n<p><a href="#Command_prompt">Prompt</a> command param1 param2 param3 ... paramN</p>/n<ul>/n<li><a href="#Command_prompt">Prompt</a> - generated by the program to provide context for the client.</li>/n<li>Command&#160;— provided by the client. Commands are usually one of three classes:/n<ol>/n<li>Internal&#160;— recognized and processed by the command line interpreter itself and not dependent upon any external executable file.</li>/n<li>Included&#160;— A separate executable file generally considered part of the operating environment and always included with the OS.</li>/n<li>External&#160;— External executable files not part of the basic OS, but added by other parties for specific purposes and applications.</li>/n</ol>/n</li>/n<li>param1 ...paramN&#160;— Optional parameters provided by the client. The format and meaning of the parameters depends upon the command issued. In the case of Included or External commands, the values of the parameters are delivered to the program (specified by the Command) as it is launched by the OS. Parameters may be either <a href="/wiki/Argument_(computer_science)#Parameters_and_arguments" class="mw-redirect" title="Argument (computer science)">Arguments</a> or <a href="/wiki/Command-line_option" class="mw-redirect" title="Command-line option">Options</a>.</li>/n</ul>/n<p>In this example, the delimiters between command line elements are <a href="/wiki/Whitespace_character" title="Whitespace character">whitespace characters</a> and the end-of-line delimiter is the <a href="/wiki/Newline" title="Newline">newline</a> delimiter. This is a widely used (but not universal) convention for command-line interfaces.</p>/n<p>A CLI can generally be considered as consisting of <a href="/wiki/Syntax_(programming_languages)" title="Syntax (programming languages)">syntax</a> and <a href="/wiki/Semantics" title="Semantics">semantics</a>. The <i>syntax</i> is the grammar that all commands must follow. In the case of <a href="/wiki/Operating_system" title="Operating system">operating systems</a> (OS), <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a> and <a href="/wiki/Unix" title="Unix">Unix</a> each define their own set of rules that all commands must follow. In the case of <a href="/wiki/Embedded_systems" class="mw-redirect" title="Embedded systems">embedded systems</a>, each vendor, such as <a href="/wiki/Nortel" title="Nortel">Nortel</a>, <a href="/wiki/Juniper_Networks" title="Juniper Networks">Juniper Networks</a> or <a href="/wiki/Cisco_Systems" title="Cisco Systems">Cisco Systems</a>, defines their own proprietary set of rules that all commands within their CLI conform to. These rules also dictate how a user navigates through the system of commands. The <i>semantics</i> define what sort of operations are possible, on what sort of data these operations can be performed, and how the grammar represents these operations and data—the symbolic meaning in the syntax.</p>/n<p>Two different CLIs may agree on either syntax or semantics, but it is only when they agree on both that they can be considered sufficiently similar to allow users to use both CLIs without needing to learn anything, as well as to enable re-use of scripts.</p>/n<p>A simple CLI will display a prompt, accept a "command line" typed by the user terminated by the <a href="/wiki/Enter_key" title="Enter key">Enter key</a>, then execute the specified command and provide textual display of results or error messages. Advanced CLIs will validate, interpret and parameter-expand the command line before executing the specified command, and optionally capture or redirect its output.</p>/n<p>Unlike a button or menu item in a GUI, a command line is typically self-documenting, stating exactly what the user wants done. In addition, command lines usually include many <a href="/wiki/Default_(computer_science)" title="Default (computer science)">defaults</a> that can be changed to customize the results. Useful command lines can be saved by assigning a <a href="/wiki/Character_string" class="mw-redirect" title="Character string">character string</a> or <a href="/wiki/Alias_(command)" title="Alias (command)">alias</a> to represent the full command, or several commands can be grouped to perform a more complex sequence&#160;– for instance, compile the program, install it, and run it&#160;— creating a single entity, called a command procedure or script which itself can be treated as a command. These advantages mean that a user must figure out a complex command or series of commands only once, because they can be saved, to be used again.</p>/n<p>The commands given to a CLI shell are often in one of the following forms:</p>/n<ul>/n<li><i>doSomething how toFiles</i></li>/n<li><i>doSomething how sourceFile destinationFile</i></li>/n<li><i>doSomething how &lt; inputFile &gt; outputFile</i></li>/n<li><i>doSomething how | doSomething how | doSomething how &gt; outputFile</i></li>/n</ul>/n<p>where <i>doSomething</i> is, in effect, a <a href="/wiki/Verb" title="Verb">verb</a>, <i>how</i> an <a href="/wiki/Adverb" title="Adverb">adverb</a> (for example, should the command be executed "verbosely" or "quietly") and <i>toFiles</i> an object or objects (typically one or more files) on which the command should act. The <code>&gt;</code> in the third example is a <a href="/wiki/Redirection_operator" class="mw-redirect" title="Redirection operator">redirection operator</a>, telling the command-line interpreter to send the output of the command not to its own standard output (the screen) but to the named file. This will overwrite the file. Using <code>&gt;&gt;</code> will redirect the output and append it to the file. Another redirection operator is the <a href="/wiki/Vertical_bar" title="Vertical bar">vertical bar</a> (<code>|</code>), which creates a <a href="/wiki/Pipeline_(Unix)" title="Pipeline (Unix)">pipeline</a> where the output of one command becomes the input to the next command.</p>/n<h3><span class="mw-headline" id="CLI_and_resource_protection">CLI and resource protection</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=7" title="Edit section: CLI and resource protection">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>One can modify the set of available commands by modifying which paths appear in the <a href="/wiki/PATH_(variable)" title="PATH (variable)">PATH</a> environment variable. Under Unix, commands also need be marked as <a href="/wiki/Executable" title="Executable">executable</a> files. The directories in the path variable are searched in the order they are given. By re-ordering the path, one can run e.g. \OS2\MDOS\E.EXE instead of \OS2\E.EXE, when the default is the opposite. Renaming of the executables also works: people often rename their favourite editor to EDIT, for example.</p>/n<p>The command line allows one to restrict available commands, such as access to advanced internal commands. The Windows <a href="/wiki/CMD.EXE" class="mw-redirect" title="CMD.EXE">CMD.EXE</a> does this. Often, shareware programs will limit the range of commands, including printing a command 'your administrator has disabled running batch files' from the prompt.</p>/n<p>Some CLIs, such as those in <a href="/wiki/Network_router" class="mw-redirect" title="Network router">network routers</a>, have a hierarchy of <a href="/wiki/Mode_(computer_interface)" title="Mode (computer interface)">modes</a>, with a different set of commands supported in each mode. The set of commands are grouped by association with security, system, interface, etc. In these systems the user might traverse through a series of sub-modes. For example, if the CLI had two modes called <i>interface</i> and <i>system</i>, the user might use the command <i>interface</i> to enter the interface mode. At this point, commands from the system mode may not be accessible and the user exits the interface mode and enters the system mode.</p>/n<h3><span class="mw-headline" id="Command_prompt">Command prompt</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=8" title="Edit section: Command prompt">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<div role="note" class="hatnote navigation-not-searchable">"Command prompt" redirects here. For the Windows component named Command Prompt, see <a href="/wiki/Cmd.exe" title="Cmd.exe">cmd.exe</a>.</div>/n<p>A command prompt (or just <i>prompt</i>) is a sequence of (one or more) characters used in a command-line interface to indicate readiness to accept commands. It literally <a href="https://en.wiktionary.org/wiki/prompt" class="extiw" title="wiktionary:prompt">prompts</a> the user to take action. A prompt usually ends with one of the characters <code>$</code>, <code>%</code>, <code>#</code>, <code>:</code>, <code>&gt;</code> and often includes other information, such as the path of the current <a href="/wiki/Working_directory" title="Working directory">working directory</a> and the <a href="/wiki/Hostname" title="Hostname">hostname</a>.</p>/n<p>On many <a href="/wiki/Unix" title="Unix">Unix</a> and <a href="/wiki/Unix-like" title="Unix-like">derivative systems</a>, the prompt commonly ends in <code>$</code> or <code>%</code> if the user is a normal user, but in <code>#</code> if the user is a <a href="/wiki/Superuser" title="Superuser">superuser</a> ("root" in Unix terminology).</p>/n<p>End-users can often modify prompts. Depending on the environment, they may include colors, special characters, and other elements (like variables and functions for the current time, user, shell number or working directory) in order, for instance, to make the prompt more informative or visually pleasing, to distinguish sessions on various machines, or to indicate the current level of nesting of commands. On some systems, special tokens in the definition of the prompt can be used to cause external programs to be called by the command-line interpreter while displaying the prompt.</p>/n<p>In DOS's COMMAND.COM and in Windows NT's <a href="/wiki/Cmd.exe" title="Cmd.exe">cmd.exe</a> users can modify the prompt by issuing a <code>prompt</code> command or by directly changing the value of the corresponding <code>%PROMPT%</code> <a href="/wiki/Environment_variable" title="Environment variable">environment variable</a>. The default of most modern systems, the <code>C:\&gt;</code> style is obtained, for instance, with <code>prompt $P$G</code>. The default of older DOS systems, <code>C&gt;</code> is obtained by just <code>prompt</code>, although on some systems this produces the newer <code>C:\&gt;</code> style, unless used on floppy drives A: or B:; on those systems <code>prompt $N$G</code> can be used to override the automatic default and explicitly switch to the older style.</p>/n<p>Many Unix systems feature the <code>$PS1</code> variable (Prompt String 1),<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[9]</a></sup> although other variables also may affect the prompt (depending on the <a href="/wiki/Unix_shell" title="Unix shell">shell</a> used). In the <a href="/wiki/Bash_(Unix_shell)" title="Bash (Unix shell)">bash</a> shell, a prompt of the form:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="o">[</span>time<span class="o">]</span> user@host: work_dir $/n</pre></div>/n<p>could be set by issuing the command</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="nb">export</span> <span class="nv">PS1</span><span class="o">=</span><span class="s1">'[\t] \u@\H: \W $'</span>/n</pre></div>/n<p>In <a href="/wiki/Zsh" class="mw-redirect" title="Zsh">zsh</a> the <code>$RPROMPT</code> variable controls an optional "prompt" on the right-hand side of the display. It is not a real prompt in that the location of text entry does not change. It is used to display information on the same line as the prompt, but right-justified.</p>/n<p>In <a href="/wiki/RISC_OS" title="RISC OS">RISC OS</a> the command prompt is a <code>*</code> symbol, and thus (OS)CLI commands are often referred to as "star commands".<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">[10]</a></sup> One can also access the same commands from other command lines (such as the <a href="/wiki/BBC_BASIC" title="BBC BASIC">BBC BASIC</a> command line), by preceding the command with a <code>*</code>.</p>/n<p><span id="Arguments_section"></span></p>/n<h3><span class="mw-headline" id="Arguments">Arguments</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=9" title="Edit section: Arguments">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<div class="thumb tright">/n<div class="thumbinner" style="width:222px;"><a href="/wiki/File:COMMAND_LINE.svg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/8/8a/COMMAND_LINE.svg/220px-COMMAND_LINE.svg.png" width="220" height="59" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/8/8a/COMMAND_LINE.svg/330px-COMMAND_LINE.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/8/8a/COMMAND_LINE.svg/440px-COMMAND_LINE.svg.png 2x" data-file-width="518" data-file-height="138" /></a>/n<div class="thumbcaption">/n<div class="magnify"><a href="/wiki/File:COMMAND_LINE.svg" class="internal" title="Enlarge"></a></div>/nAn MS DOS command line, illustrating parsing into command and arguments</div>/n</div>/n</div>/n<p>A <b>command-line argument</b> or <b><a href="/wiki/Parameter#Computing" title="Parameter">parameter</a></b> is an item of information provided to a program when it is started. A program can have many command-line arguments that identify sources or destinations of information, or that alter the operation of the program.</p>/n<p>When a command processor is active a program is typically invoked by typing its name followed by command-line arguments (if any). For example, in <a href="/wiki/Unix" title="Unix">Unix</a> and <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> environments, an example of a command-line argument is:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/nrm file.s/n</pre></div>/n<p>"file.s" is a command-line argument which tells the program <a href="/wiki/Rm_(Unix)" title="Rm (Unix)">rm</a> to remove the file "file.s".</p>/n<p>Some programming languages, such as <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, <a href="/wiki/C%2B%2B" title="C++">C++</a> and <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, allow a program to interpret the command-line arguments by handling them as string parameters in the <a href="/wiki/Main_function_(programming)" class="mw-redirect" title="Main function (programming)">main function</a>. Other languages, such as <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, expose these arguments as global variables.</p>/n<p>In <a href="/wiki/Unix-like_operating_system" class="mw-redirect" title="Unix-like operating system">Unix-like operating systems</a>, a single hyphen-minus by itself is usually a special value specifying that a program should handle data coming from the <a href="/wiki/Standard_input" class="mw-redirect" title="Standard input">standard input</a> or send data to the <a href="/wiki/Standard_output" class="mw-redirect" title="Standard output">standard output</a>.</p>/n<h4><span class="mw-headline" id="Command-line_option"><span id="SwitChar"></span>Command-line option</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=10" title="Edit section: Command-line option">edit</a><span class="mw-editsection-bracket">]</span></span></h4>/n<p>A <b>command-line option</b> or simply <b>option</b> (also known as a <b>flag</b> or <b>switch</b>) modifies the operation of a command; the effect is determined by the command's program. Options follow the command name on the command line, separated by spaces. A space before the first option is not always required, for example <code>Dir/?</code> and <code style="white-space:nowrap">DIR /?</code> have the same effect in DOS (list the DIR command's options) whereas <code>dir --help</code> (in many versions of Unix) <i>does</i> require the option to be preceded by at least one space (and is case-sensitive).</p>/n<p>The format of options varies widely between operating systems. In most cases the syntax is by convention rather than an operating system requirement; the entire command line is simply a string passed to a program, which can process it in any way the programmer wants, so long as the interpreter can tell where the command name ends and its arguments and options begin.</p>/n<p>A few representative samples of command-line options, all relating to listing files in a directory, to illustrate some conventions:</p>/n<table class="wikitable">/n<tr>/n<th>Operating system</th>/n<th>Command</th>/n<th>Valid alternative</th>/n<th>Notes</th>/n</tr>/n<tr>/n<td><a of the files.<br />/n<small><i>Note the Directory command name is not case sensitive, and can be abbreviated to as few letters as required to remain unique.</i></small></td>/n</tr>/n<tr>/n<td>DOS</td>/n<td><code>dir/Q/O:S d*</code></td>/n<td><code>diR /q d* /o:s</code></td>/n<td>display ownership of files whose names begin with "D", sorted by size, smallest first.<br />/n<small><i>Note spaces around argument d* are required.</i></small></td>/n</tr>/n<tr>/n<td><a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> systems</td>/n<td><code>ls -lS D*</code></td>/n<td><code>ls -S -l D*</code></td>/n<td>display in long format files and directories beginning with "D" (but not "d"), sorted by size (largest first).<br 04-26-80/b</code></td>/n<td><code>List /S/E 4-26-80/B</code></td>/n<td>list every attribute for files created before 26 April 1980.<br />/n<small><i>Note the /B at the end of the date argument is a <b>local switch</b>, that modifies the meaning of that argument, while /S and /E are <b>global switches</b>, i.e. apply to the whole command.</i></small></td>/n</tr>/n</table>/n<h5><span class="mw-headline" id="Abbreviating_commands">Abbreviating commands</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=11" title="Edit section: Abbreviating commands">edit</a><span class="mw-editsection-bracket">]</span></span></h5>/n<p>In <a href="/wiki/Multics" title="Multics">Multics</a>, command-line options and subsystem keywords may be abbreviated. This idea appears to derive from the <a href="/wiki/PL/I_programming_language" class="mw-redirect" title="PL/I programming language">PL/I programming language</a>, with its shortened keywords (e.g., STRG for STRINGRANGE and DCL for DECLARE). For example, in the Multics "forum" subsystem, the <i>-long_subject</i> parameter can be abbreviated <i>-lgsj</i>. It is also common for Multics commands to be abbreviated, typically corresponding to the initial letters of the words that are strung together with underscores to form command names, such as the use of <i>did</i> for <i>delete_iacl_dir</i>.</p>/n<p>In some other systems abbreviations are automatic, such as permitting enough of the first characters of a command name to uniquely identify it (such as <code>SU</code> as an abbreviation for <code>SUPERUSER</code>) while others may have some specific abbreviations pre-programmed (e.g. MD for MKDIR in COMMAND.COM) or user-defined via batch scripts and aliases (e.g. <code>alias md mkdir</code> in <a href="/wiki/Tcsh" title="Tcsh">tcsh</a>). See also: <a href="/wiki/Tab_completion" class="mw-redirect" title="Tab completion">Tab completion</a>.</p>/n<h5><span id="Option_conventions_in_DOS,_Windows,_OS/2"></span><span class="mw-headline" id="Option_conventions_in_DOS.2C_Windows.2C_OS.2F2">Option conventions in DOS, Windows, OS/2</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=12" title="Edit section: Option conventions in DOS, Windows, OS/2">edit</a><span class="mw-editsection-bracket">]</span></span></h5>/n<p>On DOS, OS/2 and Windows, different programs called from their COMMAND.COM or CMD.EXE (or internal their commands) may use different syntax within the same operating system. For example:</p>/n<ul>/n<li>Options may be indicated by either of the "switch characters": <code>-</code>, <code>/</code>, or either may be allowed. See below.</li>/n<li>They may or may not be <a href="/wiki/Case-sensitive" class="mw-redirect" title="Case-sensitive">case-sensitive</a>.</li>/n<li>Sometimes options and their arguments are run together, sometimes separated by whitespace, and sometimes by a character, typically <code>:</code> or <code>=</code>; thus <code>Prog -fFilename</code>, <code>Prog -f Filename</code>, <code>Prog -f:Filename</code>, <code>Prog -f=Filename</code>.</li>/n<li>Some programs allow single-character options to be combined; others do not. The switch <code>-fA</code> may mean the same as <code>-f -A</code>, or it may be incorrect, or it may even be a valid but different parameter.</li>/n</ul>/n<p>In <a href="/wiki/DOS" title="DOS">DOS</a>, <a href="/wiki/OS/2" title="OS/2">OS/2</a> and <a href="/wiki/Windows" class="mw-redirect" title="Windows">Windows</a>, the forward slash (<code>/</code>) is most prevalent, although the hyphen-minus is also sometimes used. In many versions of DOS (MS-DOS/PC&#160;DOS 2.xx and higher, all versions of <a href="/wiki/DR-DOS" title="DR-DOS">DR-DOS</a> since 5.0, as well as <a href="/wiki/PTS-DOS" title="PTS-DOS">PTS-DOS</a>, <a href="/wiki/Embedded_DOS" class="mw-redirect" title="Embedded DOS">Embedded DOS</a>, <a href="/wiki/FreeDOS" title="FreeDOS">FreeDOS</a> and <a href="/w/index.php?title=RxDOS&amp;action=edit&amp;redlink=1" class="new" title="RxDOS (page does not exist)">RxDOS</a>) the <b>switch character</b> (sometimes abbreviated <b>switchar</b> or <b>switchchar</b>) to be used is defined by a value returned from a <a href="/wiki/System_call" title="System call">system call</a> (<a href="/wiki/INT_21h" class="mw-redirect" title="INT 21h">INT&#160;21h</a>/AH=<b>37h</b>). The default character returned by this API is <code>/</code>, but can be changed to a hyphen-minus on the above-mentioned systems, except for Datalight ROM-DOS and MS-DOS/PC&#160;DOS 5.0 and higher, which always return <code>/</code> from this call (unless one of many available <a href="/wiki/Terminate_%26_Stay_Resident" class="mw-redirect" title="Terminate &amp; Stay Resident">TSRs</a> to reenable the SwitChar feature is loaded). In some of these systems (MS-DOS/PC&#160;DOS 2.xx, DOS Plus 2.1, DR-DOS 7.02 and higher, PTS-DOS, Embedded DOS, FreeDOS and RxDOS), the setting can also be pre-configured by a <a href="/wiki/SWITCHAR_(CONFIG.SYS_directive)" class="mw-redirect" title="SWITCHAR (CONFIG.SYS directive)">SWITCHAR</a> directive in <a href="/wiki/CONFIG.SYS" title="CONFIG.SYS">CONFIG.SYS</a>. Embedded DOS provides a SWITCH command for the same purpose, whereas <a href="/wiki/4DOS" title="4DOS">4DOS</a> allows the setting to be changed via <code>SETDOS /W:n</code>. Under DR-DOS, if the setting has been changed from <code>/</code>, the first directory separator <code>\</code> in the display of the <a href="/wiki/PROMPT_(DOS_command)" class="mw-redirect" title="PROMPT (DOS command)">PROMPT</a> parameter <code>$G</code> will change to a forward slash <code>/</code> (which is also a valid directory separator in DOS, FlexOS, 4680 OS, 4690 OS, OS/2 and Windows) thereby serving as a visual clue to indicate the change. Some versions of DR-DOS <a href="/wiki/COMMAND.COM" title="COMMAND.COM">COMMAND.COM</a> also support a PROMPT token <code>$/</code> to display the current setting. COMMAND.COM since DR-DOS 7.02 and <a href="/wiki/4DOS" title="4DOS">4DOS</a> also provide a <a href="/wiki/Pseudo-environment_variable" class="mw-redirect" title="Pseudo-environment variable">pseudo-environment variable</a> named <code><a href="/wiki/%25/%25" class="mw-redirect" title="%/%">%/%</a></code> to allow portable batchjobs to be written. Several external DR-DOS commands additionally support an <a href="/wiki/Environment_variable" title="Environment variable">environment variable</a> <code><a href="/wiki/%25SWITCHAR%25" class="mw-redirect" title="%SWITCHAR%">%SWITCHAR%</a></code> to override the system setting.</p>/n<p>However, many programs are hardwired to use <code>/</code> only, rather than retrieving the switch setting before parsing command line arguments. A very small number, mainly ports from Unix-like systems, are programmed to accept "-" even if the switch character is not set to it (for example netstat and ping, supplied with <a href="/wiki/Windows" class="mw-redirect" title="Windows">Windows</a>, will accept the /? option to list available options, and yet the list will specify the "-" convention).</p>/n<h5><span class="mw-headline" id="Option_conventions_in_Unix-like_systems">Option conventions in Unix-like systems</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=13" title="Edit section: Option conventions in Unix-like systems">edit</a><span class="mw-editsection-bracket">]</span></span></h5>/n<table class="plainlinks metadata ambox ambox-content ambox-Unreferenced" role="presentation">/n<tr>/n<td class="mbox-image">/n<div style="width:52px"><a href="/wiki/File:Question_book-new.svg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/50px-Question_book-new.svg.png" width="50" height="39" srcset="//upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/75px-Question_book-new.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/100px-Question_book-new.svg.png 2x" data-file-width="512" data-file-height="399" /></a></div>/n</td>/n<td class="mbox-text">/n<div class="mbox-text-span">This section <b>does not <a href="/wiki/Wikipedia:Citing_sources" title="Wikipedia:Citing sources">cite</a> any <a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability">sources</a></b>. <span class="hide-when-compact">Please help <a class="external text" href="//en.wikipedia.org/w/index.php?title=Command-line_interface&amp;action=edit">improve this section</a> by <a href="/wiki/Help:Introduction_to_referencing_with_Wiki_Markup/1" title="Help:Introduction to referencing with Wiki Markup/1">adding citations to reliable sources</a>. Double hyphen-minuses are also sometimes used to prefix "long options" where more descriptive option names are used. This is a common feature of <a href="/wiki/GNU" title="GNU">GNU</a> software. The <i><a href="/wiki/Getopt" title="Getopt">getopt</a></i> function and program, and the <i><a href="/wiki/Getopts" title="Getopts">getopts</a></i> command are usually used for parsing command-line options.</p>/n<p>Unix command names, arguments and options are case-sensitive (except in a few examples, mainly where popular commands from other operating systems have been ported to Unix).</p>/n<h5><span class="mw-headline" id="Options_conventions_in_other_systems">Options conventions in other systems</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=14" title="Edit section: Options conventions in other systems">edit</a><span class="mw-editsection-bracket">]</span></span></h5>/n<p><a href="/wiki/FlexOS" title="FlexOS">FlexOS</a>, <a href="/wiki/IBM_4680_OS" class="mw-redirect" title="IBM 4680 OS">4680 OS</a> and <a href="/wiki/IBM_4690_OS" class="mw-redirect" title="IBM 4690 OS">4690 OS</a> use <code>-</code>.</p>/n<p><a href="/wiki/CP/M" title="CP/M">CP/M</a> typically used <code>[</code>.</p>/n<p><a href="/wiki/Conversational_Monitor_System" title="Conversational Monitor System">Conversational Monitor System</a> (CMS) uses a single <a href="/wiki/Bracket#Parentheses_.28_.29" title="Bracket">left parenthesis</a> to separate options at the end of the command from the other arguments. For example, in the following command the options indicate that the target file should be replaced if it exists, and the date and time of the source file should be retained on the copy: <code>COPY source file a target file b (REPLACE OLDDATE</code></p>/n<p><a href="/wiki/Data_General" title="Data General">Data General</a>'s CLI under their <a href="/wiki/Data_General_RDOS" title="Data General the switch character, is case-insensitive, and allows "local switches" on some arguments to control the way they are interpreted, such as <code>MAC/U LIB/S A B C $LPT/L</code> has the global option (April 2015)">citation needed</span></a></i>]</sup> To overcome this limitation, many CLI programs display a brief summary of its valid parameters, typically when invoked with no arguments or one of <code>?</code>, <code>-?</code>, <code>-h</code>, <code>-H</code>, <code>/?</code>, <code>/h</code>, <code>/H</code>, <code>-help</code>, or <code>--help</code>.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (April 2015)">citation needed</span></a></i>]</sup></p>/n<p>However, entering a program name without parameters in the hope that it will display usage help can be hazardous, as some programs and scripts execute without further notice.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (April 2015)">citation needed</span></a></i>]</sup></p>/n<p>Although desirable at least for the help parameter, programs may not support all option lead-in characters exemplified above. Under DOS, where the default <a href="/wiki/Command-line_interface#Option_conventions_in_DOS.2C_Windows.2C_OS.2F2" title="Command-line interface">command line option character</a> can be changed from <code>/</code> to <code>-</code>, programs may query the <a href="/wiki/SwitChar" class="mw-redirect" title="SwitChar">SwitChar</a> API in order to determine the current setting. So, if a program is not hard-wired to support them all, a user may need to know the current setting even to be able to reliably request help. If the SwitChar has been changed to <code>-</code> and therefore the <code>/</code> character is accepted as alternative path delimiter also at the DOS command line, programs may misinterpret options like <code>/h</code> or <code>/H</code> as paths rather than help parameters. However, if given as first or only parameter, most DOS programs will, by convention, accept it as request for help regardless of the current SwitChar setting.</p>/n<p>In some cases, different levels of help can be selected for a program. Some programs supporting this allow to give a verbosity level as an optional argument to the help parameter (as in <code>/H:1</code>, <code>/H:2</code>, etc.) or they give just a short help on help parameters with question mark and a longer help screen for the other help options.</p>/n<p>Depending on the program, additional or more specific help on accepted parameters is sometimes available by either providing the parameter in question as an argument to the help parameter or vice versa (as in <code>/H:W</code> or in <code>/W:?</code> (assuming <code>/W</code> would be another parameter supported by the program)).</p>/n<p>In a similar fashion to the help parameter, but much less common, some programs provide additional information about themselves (like mode, status, version, author, license or contact information) when invoked with an "about" parameter like <code>-!</code>, <code>/!</code>, <code>-about</code>, or <code>--about</code>.</p>/n<p>Since the <code>?</code> and <code>!</code> characters typically also serve other purposes at the command line, they may not be available in all scenarios, therefore, they should not be the only options to access the corresponding help information.</p>/n<p>If more detailed help is necessary than provided by a program's built-in internal help, many systems support a dedicated external "<code><a href="/wiki/HELP_(command)" class="mw-redirect" title="HELP (command)">HELP</a> <i>command</i></code>" command (or similar), which accepts a command name as calling parameter and will invoke an external help system.</p>/n<p>In the DR-DOS family, typing <code>/?</code> or <code>/H</code> at the <a href="/wiki/COMMAND.COM" title="COMMAND.COM">COMMAND.COM</a> prompt instead of a command itself will display a dynamically generated list of available internal commands; <a href="/wiki/4DOS" title="4DOS">4DOS</a> and <a href="/wiki/NDOS" class="mw-redirect" title="NDOS">NDOS</a> support the same feature by typing <code>?</code> at the prompt (which is also accepted by newer versions of DR-DOS COMMAND.COM); internal commands can be individually disabled or reenabled via <code>SETDOS /I</code>. In addition to this, some newer versions of DR-DOS COMMAND.COM also accept a <code>?%</code> command to display a list of available built-in <a href="/wiki/DR-DOS_system_information_variable" class="mw-redirect" title="DR-DOS system information variable">pseudo-environment variables</a>. Besides their purpose as quick help reference this can be used in batchjobs to query the facilities of the underlying command line processor.</p>/n<h3><span class="mw-headline" id="Command_description_syntax">Command description syntax</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=16" title="Edit section: Command description syntax">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Built-in usage help and <a href="/wiki/Man_page" title="Man page">man pages</a> commonly employ a small syntax to describe the valid command form:<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">[11]</a></sup><sup id="cite_ref-12" class="reference"><a href="#cite_note-12">[12]</a></sup></p>/n<ul>/n<li>angle brackets for <i>required</i> parameters: <code>ping &lt;hostname&gt;</code></li>/n<li>square brackets for <i>optional</i> parameters: <code>mkdir [-p] &lt;dirname&gt;</code></li>/n<li>ellipses for <i>repeated</i> items: <code>cp &lt;source1&gt; [source2...] &lt;dest&gt;</code></li>/n<li>vertical bars for <i>choice</i> of items: <code>netstat {-t|-u}</code></li>/n</ul>/n<p>Notice that these characters have different meanings than when used directly in the shell. Angle brackets may be omitted when confusing the parameter name with a literal string is not likely.</p>/n<h3><span class="mw-headline" id="The_space_character">The space character</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=17" title="Edit section: The space character">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>In many areas of computing, but particularly in the command line, the <a href="/wiki/Space_character" class="mw-redirect" title="Space character">space character</a> can cause problems as it has two distinct and incompatible functions: as part of a command or parameter, or as a parameter or name <a href="/wiki/Delimiter" title="Delimiter">separator</a>. Ambiguity can be prevented either by prohibiting embedded spaces in file and directory names in the first place (for example, by substituting them with <a href="/wiki/Underscores" class="mw-redirect" title="Underscores">underscores</a> <code>_</code>), or by enclosing a name with embedded spaces between quote characters or using an <a href="/wiki/Escape_character" title="Escape character">escape character</a> before the space, usually a <a href="/wiki/Backslash" title="Backslash">backslash</a> (<code>\</code>). For example</p>/n<dl>/n<dd><code>Long path/Long program name Parameter one Parameter two</code> ...</dd>/n</dl>/n<p>is ambiguous (is "program name" part of the program name, or two parameters?); however</p>/n<dl>/n<dd><code>Long_path/Long_program_name Parameter_one Parameter_two</code> ...,</dd>/n<dd><code>LongPath/LongProgramName ParameterOne ParameterTwo</code> ...,</dd>/n<dd><code>"Long path/Long program name" "Parameter one" "Parameter two"</code> ...</dd>/n</dl>/n<p>and</p>/n<dl>/n<dd><code>Long\ path/Long\ program\ name Parameter\ one Parameter\ two</code> ...</dd>/n</dl>/n<p>are not ambiguous. <a href="/wiki/Unix" title="Unix">Unix</a>-based operating systems minimize the use of embedded spaces to minimize the need for quotes. In <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>, one often has to use quotes because embedded spaces (such as in directory names) are common.</p>/n<h2><span class="mw-headline" id="Command-line_interpreter">Command-line interpreter</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=18" title="Edit section: Command-line interpreter">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div role="note" class="hatnote navigation-not-searchable">See also: <a href="/wiki/List_of_command-line_interpreters" title="List of command-line interpreters">List of command-line interpreters</a></div>/n<blockquote class="templatequote">/n<p>Although most users think of the shell as an interactive command interpreter, it is really a programming language in which each statement runs a command. Because it must satisfy both the interactive and programming aspects of command execution, it is a strange language, shaped as much by history as by design.</p>/n<div class="templatequotecite">— <cite><a href="/wiki/Brian_Kernighan" title="Brian Kernighan">Brian Kernighan</a> &amp; <a href="/wiki/Rob_Pike" title="Rob Pike">Rob Pike</a><sup id="cite_ref-13" class="reference"><a href="#cite_note-13">[13]</a></sup></cite></div>/n</blockquote>/n<p>The terms <b>command-line interpreter</b>, <b>command line <a href="/wiki/Shell_(computing)" title="Shell (computing)">shell</a></b>, <b>command language interpreter</b>, or identical abbreviation&#160;<b>CLI</b>, are applied to <a href="/wiki/Computer_program" title="Computer program">computer programs</a> designed to <a href="/wiki/Interpreter_(computing)" title="Interpreter (computing)">interpret</a> a sequence of lines of text which may be entered by a user, read from a <a href="/wiki/Computer_file" title="Computer file">file</a> or another kind of <a href="/wiki/Data_stream" title="Data stream">data stream</a>. The context of interpretation is usually one of a given <a href="/wiki/Operating_system" title="Operating system">operating system</a> or <a href="/wiki/Programming_language" title="Programming language">programming language</a>.</p>/n<p>Command-line interpreters allow users to issue various commands in a very efficient (and often terse) way. This requires the user to know the names of the commands and their parameters, and the syntax of the <a href="/wiki/Programming_language" title="Programming language">language</a> that is interpreted.</p>/n<p>The Unix <a href="/wiki/Shebang_(Unix)" title="Shebang (Unix)"><code>#!</code></a> mechanism and OS/2 EXTPROC command facilitate the passing of batch files to external processors. One can use these mechanisms to write specific command processors for dedicated uses, and process external data files which reside in batch files.</p>/n<p>Many graphical interfaces, such as the OS/2 <a href="/wiki/Presentation_Manager" title="Presentation Manager">Presentation Manager</a> and early versions of Microsoft Windows use command-lines to call helper programs to open documents and programs. The commands are stored in the graphical shell<sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag may need clarification or removal of jargon. (May 2012)">clarification needed</span></a></i>]</sup> or in files like the registry or the <a href="/wiki/OS/2" title="OS/2">OS/2</a> <code>os2user.ini</code> file.</p>/n<h3><span class="mw-headline" id="Early_history">Early history</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=19" title="Edit section: Early history">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>The earliest computers did not support interactive input/output devices, often relying on <a href="/wiki/Sense_switch" title="Sense switch">sense switches</a> and lights to communicate with the <a href="/wiki/Computer_operator" title="Computer operator">computer operator</a>. This was adequate for <a href="/wiki/Batch_processing" title="Batch processing">batch</a> systems that ran one program at a time, often with the programmer acting as operator. This also had the advantage of low overhead, since lights and switches could be tested and set with one machine instruction. Later a single <a href="/wiki/System_console" title="System console">system console</a> was added to allow the operator to communicate with the system.</p>/n<p>From the 1960s onwards, user interaction with computers was primarily by means of command-line interfaces, initially on machines like the <a href="/wiki/Teletype_Model_33" title="Teletype Model 33">Teletype Model 33</a> ASR, but then on early <a href="/wiki/Cathode_ray_tube" title="Cathode ray tube">CRT</a>-based <a href="/wiki/Computer_terminals" class="mw-redirect" title="Computer terminals">computer terminals</a> such as the <a href="/wiki/VT52" title="VT52">VT52</a>.</p>/n<p>All of these devices were purely text based, with no ability to display graphic or pictures.<sup id="cite_ref-14" class="reference"><a href="#cite_note-14">[14]</a></sup> For business <a href="/wiki/Application_software" title="Application software">application programs</a>, text-based <a href="/wiki/Menu_(computing)" title="Menu (computing)">menus</a> were used, but for more general interaction the command line was the interface.</p>/n<p>Around 1964 <a href="/wiki/Louis_Pouzin" title="Louis Pouzin">Louis Pouzin</a> introduced the concept and the name <i>shell</i> in <a href="/wiki/Multics" title="Multics">Multics</a>, building on earlier, simpler facilities in the <a href="/wiki/Compatible_Time-Sharing_System" title="Compatible Time-Sharing System">Compatible Time-Sharing System</a> (CTSS).<sup id="cite_ref-15" class="reference"><a href="#cite_note-15">[15]</a></sup><sup class="noprint Inline-Template noprint Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:NOTRS" class="mw-redirect" title="Wikipedia:NOTRS"><span title="This claim needs references to better sources. interpreters support <a href="/wiki/Script_(computer_programming)" class="mw-redirect" title="Script (computer programming)">scripting</a>, to various extents. (They are, after all, interpreters of an <a href="/wiki/Interpreted_programming_language" class="mw-redirect" title="Interpreted programming language">interpreted programming language</a>, albeit in many cases the language is unique to the particular command-line interpreter.) They will interpret scripts (variously termed <a href="/wiki/Shell_script" title="Shell script">shell scripts</a> or <a href="/wiki/Batch_file" title="Batch file">batch files</a>) written in the <a href="/wiki/Programming_language" title="Programming language">language</a> that they interpret. Some command-line interpreters also incorporate the interpreter engines of other languages, such as <a href="/wiki/REXX" class="mw-redirect" title="REXX">REXX</a>, in addition to their own, allowing the executing of scripts, in those languages, directly within the command-line interpreter itself.</p>/n<p>Conversely, <a href="/wiki/Scripting_programming_language" class="mw-redirect" title="Scripting programming language">scripting programming languages</a>, in particular those with an <a href="/wiki/Eval" title="Eval">eval</a> <a href="/wiki/Function_(programming)" class="mw-redirect" title="Function (programming)">function</a> (such as <a href="/wiki/REXX" class="mw-redirect" title="REXX">REXX</a>, <a href="/wiki/Perl" title="Perl">Perl</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a> or <a href="/wiki/Jython" title="Jython">Jython</a>), can be used to implement command-line interpreters and filters. For a few <a href="/wiki/Operating_system" title="Operating system">operating systems</a>, most notably <a href="/wiki/DOS" title="DOS">DOS</a>, such a command interpreter provides a more flexible command line interface than the one supplied. In other cases, such a command interpreter can present a highly customised user interface employing the user interface and input/output facilities of the language.</p>/n<h2><span class="mw-headline" id="Other_command-line_interfaces">Other command-line interfaces</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=22" title="Edit section: Other command-line interfaces">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The command line provides an interface between programs as well as the user. In this sense, a command line is an alternative to a <a href="/wiki/Dialog_box" title="Dialog box">dialog box</a>. Editors and data-bases present a command line, in which alternate command processors might run. On the other hand, one might have options on the command line which opens a dialog box. The latest version of 'Take Command' has this feature. DBase used a dialog box to construct command lines, which could be further edited before use.</p>/n<p>Programs like Basic, Diskpart, Edlin, and QBasic all provide command-line interfaces, some of which use the system shell. Basic is modeled on the default interface for 8-bit Intel computers. Calculators can be run as command-line or dialog interfaces.</p>/n<p><a href="/wiki/Emacs" title="Emacs">Emacs</a> provides a command line interface in the form of its minibuffer. Commands and arguments can be entered using Emacs standard text editing support, and output is displayed in another buffer.</p>/n<p>There are a number of pre-mouse games, like <i><a href="/wiki/Colossal_Cave_Adventure" title="Colossal Cave Adventure">Adventure</a></i> or <i><a href="/wiki/King%27s_Quest#Series" title="King's Quest">King's Quest 1-3</a></i>, which relied on the user typing commands at the bottom of the screen. One controls the character by typing commands like 'get ring' or 'look'. The program returns a text which describes how the character sees it, or makes the action happen. The <a href="/wiki/Text_adventure" class="mw-redirect" title="Text adventure">text adventure</a> <i><a href="/wiki/The_Hitchhiker%27s_Guide_to_the_Galaxy_(video_game)" title="The Hitchhiker's Guide to the Galaxy (video game)">The Hitchhiker's Guide to the Galaxy</a></i>, a piece of <a href="/wiki/Interactive_fiction" title="Interactive fiction">interactive fiction</a> based on <a href="/wiki/Douglas_Adams" title="Douglas Adams">Douglas Adam's</a> book of the same name, is a teletype-style command-line game.</p>/n<p>The most notable of these interfaces is the <a href="/wiki/Standard_streams" title="Standard streams">standard streams</a> interface, which allows the output of one command to be passed to the input of another. Text files can serve either purpose as well. This provides the interfaces of piping, filters and redirection. Under Unix, <a href="/wiki/Everything_is_a_file" title="Everything is a file">devices are files</a> too, so the normal type of file for the shell used for stdin,stdout and stderr is a <a href="/wiki/Teleprinter#Teleprinters_in_computing" title="Teleprinter">tty</a> device file.</p>/n<p>Another command-line interface allows a shell program to launch helper programs, either to launch documents or start a program. The command is processed internally by the shell, and then passed on to another program to launch the document. The graphical interface of Windows and OS/2 rely heavily on command-lines passed through to other programs&#160;– console or graphical, which then usually process the command line without presenting a user-console.</p>/n<p>Programs like the OS/2 <a href="/wiki/E_(editor)" class="mw-redirect" title="E (editor)">E</a> editor and some other IBM <a href="/wiki/Text_editor" title="Text editor">editors</a>, can process command-lines normally meant for the shell, the output being placed directly in the document window.</p>/n<p>A web browser's URL input field can be used as a command line. It can be used to "launch" <a href="/wiki/Web_app" class="mw-redirect" title="Web app">web apps</a>, <a href="/wiki/About:config" class="mw-redirect" title="About:config">access browser configuration</a>, as well as perform a search. <a href="/wiki/Google" title="Google">Google</a>, which has been called "the command line of the internet" will perform a domain-specific search when it detects search parameters in a known format.<sup id="cite_ref-26" class="reference"><a href="#cite_note-26">[26]</a></sup> This functionality is present whether the search is triggered from a browser field or one on Google's web site.</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Command-line_interface&amp;action=edit&amp;section=23" title="Edit section: See also">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<ul>/n<li><a href="/wiki/Comparison_of_command_shells" title="Comparison of command shells">Comparison of command shells</a></li>/n<li><a href="/wiki/List_of_command-line_interpreters" title="List of command-line interpreters">List of command-line interpreters</a></li>/n<li><a href="/wiki/Orthodox_file_manager" class="mw-redirect" title="Orthodox file manager">Orthodox file manager</a>, also known as command-based file manager<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Accuracy_dispute#Disputed_statement" title="Wikipedia:Accuracy dispute"><span title="The material near this tag is possibly inaccurate or nonfactual.
