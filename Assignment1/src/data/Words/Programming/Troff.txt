class="hatnote navigation-not-searchable">This article is about the Unix document processing system. For the command in BASIC, see <a href="/wiki/TROFF" class="mw-redirect" title="TROFF">TROFF</a>.</div>/n<div role="note" class="hatnote navigation-not-searchable">Not to be confused with <a href="/wiki/Trough_(disambiguation)" class="mw-redirect mw-disambig" title="Trough (disambiguation)">Trough (disambiguation)</a>.</div>/n<p><b>troff</b> <span class="nowrap"><span class="IPA nopopups noexcerpt"><a href="/wiki/Help:IPA/English" title="Help:IPA/English">/<span style="border-bottom:1px dotted"><span title="/ˈ/: primary stress follows">ˈ</span><span title="'t' in 'tie'">t</span><span title="/iː/: 'ee' in 'fleece'">iː</span><span title="/./: syllable break">.</span><span title="'r' in 'rye'">r</span><span title="/ɒ/: 'o' in 'body'">ɒ</span><span title="'f' in 'find'">f</span></span>/</a></span></span> is the major component of a <a href="/wiki/Document_processing_system" class="mw-redirect" title="Document processing system">document processing system</a> developed by <a href="/wiki/AT%26T_Corporation" title="AT&amp;T Corporation">AT&amp;T Corporation</a> for the <a href="/wiki/Unix" title="Unix">Unix</a> operating system.</p>/n<p><i>troff</i> features commands to designate fonts, spacing, paragraphs, margins, footnotes and more. Unlike many other text formatters, <i>troff</i> can position characters arbitrarily on a page, even overlapping them, and has a fully programmable input language. Separate preprocessors are used for more convenient production of tables, diagrams, and mathematics. Inputs to troff are plain text files that can be created by any text editor.</p>/n<p>Extensive <a href="/wiki/Troff_macros" class="mw-redirect" title="Troff macros">macro packages</a> have been created for various document styles. A typical distribution of <b>troff</b> includes the <b>me</b> macros for formatting research papers, <b>man</b> and <b>mdoc</b> macros for creating Unix <a href="/wiki/UNIX_manual" class="mw-redirect" title="UNIX manual">man pages</a>, <b>mv</b> macros for creating mountable <a href="/wiki/Overhead_projector" title="Overhead projector">transparencies</a>, and the <b>ms</b> and <b>mm</b> macros for letters, books, technical memoranda, and reports.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#External_links"><span class="tocnumber">6</span> <span class="toctext">External links</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Troff&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p><i>troff</i> can trace its origins back to a <a href="/wiki/Text_formatting" class="mw-redirect" title="Text formatting">text-formatting</a> <a href="/wiki/Computer_program" title="Computer program">program</a> called <a href="/wiki/RUNOFF" class="mw-redirect" title="RUNOFF">RUNOFF</a>, written by <a href="/wiki/Jerome_H._Saltzer" class="mw-redirect" title="Jerome H. Saltzer">Jerome H. Saltzer</a> for <a href="/wiki/Massachusetts_Institute_of_Technology" title="Massachusetts Institute of Technology">MIT</a>'s <a href="/wiki/Compatible_Time-Sharing_System" title="Compatible Time-Sharing System">CTSS</a> <a href="/wiki/Operating_system" title="Operating system">operating system</a> in the mid-1960s. (The name allegedly came from the phrase <i>I'll run off a document</i>.)</p>/n<p><a href="/wiki/Robert_Morris_(cryptographer)" title="Robert Morris (cryptographer)">Bob Morris</a> ported it to the <a href="/wiki/GE_635" class="mw-redirect" title="GE 635">GE 635</a> architecture and called the program <a href="/wiki/Roff_(computer_program)" title="Roff (computer program)">roff</a> (an abbreviation of <i>runoff</i>). It was rewritten as <a href="/w/index.php?title=Rf_(program)&amp;action=edit&amp;redlink=1" class="new" title="Rf (program) (page does not exist)">rf</a> for the <a href="/wiki/PDP-7" title="PDP-7">PDP-7</a>, and at the same time (1969), <a href="/wiki/Doug_McIlroy" class="mw-redirect" title="Doug McIlroy">Doug McIlroy</a> rewrote an extended and simplified version of <i>roff</i> in the <a href="/wiki/BCPL" title="BCPL">BCPL</a> <a href="/wiki/Programming_language" title="Programming language">programming language</a>.</p>/n<p>The first version of <a href="/wiki/Unix" title="Unix">Unix</a> was developed on a PDP-7 which was sitting around <a href="/wiki/Bell_Labs" title="Bell Labs">Bell Labs</a>. In 1971 the developers wanted to get a <a href="/wiki/PDP-11" title="PDP-11">PDP-11</a> for further work on the operating system. In order to justify the cost for this system, they proposed that they would implement a document-formatting system for the AT&amp;T patents division. This first formatting program was a reimplementation of McIllroy's <i>roff</i>, written by <a href="/wiki/Joe_Ossanna" title="Joe Ossanna">Joe F. Ossanna</a>.</p>/n<p>When they needed a more flexible language, a new version of <i>roff</i> called <a href="/wiki/Nroff" title="Nroff">nroff</a> (<i>newer "roff"</i>) was written, which provided the basis for all future versions. When they got a <a href="/wiki/CAT_(phototypesetter)" title="CAT (phototypesetter)">Graphic Systems CAT phototypesetter</a>, Ossanna modified <i>nroff</i> to support multiple fonts and <a href="/wiki/Proportional_spacing" class="mw-redirect" title="Proportional spacing">proportional spacing</a>. Dubbed <i>troff</i>, for <i>typesetter roff</i>, its sophisticated output amazed the typesetter manufacturer and confused <a href="/wiki/Peer_review" title="Peer review">peer reviewers</a>, who thought that manuscripts using <i>troff</i> had been published before.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup><sup id="cite_ref-reader_2-0" class="reference"><a href="#cite_note-reader-2">[2]</a></sup> As such, the name <i>troff</i> is pronounced <span class="nowrap"><span class="IPA nopopups noexcerpt"><a href="/wiki/Help:IPA/English" title="Help:IPA/English">/<span style="border-bottom:1px dotted"><span title="/ˈ/: primary stress follows">ˈ</span><span title="'t' in 'tie'">t</span><span title="/iː/: 'ee' in 'fleece'">iː</span><span title="/./: syllable break">.</span><span title="'r' in 'rye'">r</span><span title="/ɒ/: 'o' in 'body'">ɒ</span><span title="'f' in 'find'">f</span></span>/</a></span></span> rather than *<span class="nowrap"><span class="IPA nopopups noexcerpt"><a href="/wiki/Help:IPA/English" title="Help:IPA/English">/<span style="border-bottom:1px dotted"><span title="/ˈ/: primary stress follows">ˈ</span><span title="'t' in 'tie'">t</span><span title="'r' in 'rye'">r</span><span title="/ɒ/: 'o' in 'body'">ɒ</span><span title="'f' in 'find'">f</span></span>/</a></span></span>.</p>/n<p>With <i>troff</i> came <i>nroff</i> (they were actually almost the same program), which was for producing output for <a href="/wiki/Line_printer" title="Line printer">line printers</a> and character <a href="/wiki/Computer_terminal" title="Computer terminal">terminals</a>. It understood everything <i>troff</i> did, and ignored the commands which were not applicable, e.g., <a href="/wiki/Typeface" title="Typeface">font</a> changes.</p>/n<p>Ossanna's <i>troff</i> was written in <a href="/wiki/PDP-11" title="PDP-11">PDP-11</a> <a href="/wiki/Assembly_language" title="Assembly language">assembly language</a> and produced output specifically for the CAT <a href="/wiki/Phototypesetter" class="mw-redirect" title="Phototypesetter">phototypesetter</a>. He rewrote it in <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, although it was now 7000 lines of uncommented code and still dependent on the CAT. As the CAT became less common, and was no longer supported by the manufacturer, the need to make it support other devices became a priority. Ossanna died before this task was completed, so <a href="/wiki/Brian_Kernighan" title="Brian Kernighan">Brian Kernighan</a> took on the task of rewriting <i>troff</i>. The newly rewritten version produced a device-independent code which was very easy for post-processors to read and translate to the appropriate printer codes. Also, this new version of <i>troff</i> (often called <b>ditroff</b> for <i>device independent troff</i>) had several extensions, which included drawing functions.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> The program's documentation defines the output format of <i>ditroff</i>, which is used by many modern <i>troff</i> clones like GNU <a href="/wiki/Groff_(software)" title="Groff (software)">groff</a>.</p>/n<p>The <i>troff</i> collection of tools (including <a href="#Preprocessors">pre</a>- and post-processors) was eventually called <a href="/w/index.php?title=Documenter%27s_WorkBench&amp;action=edit&amp;redlink=1" class="new" title="Documenter's WorkBench (page does not exist)"><i>Documenter's WorkBench</i> (DWB)</a>, and was under continuous development in Bell Labs and later at the spin-off <a href="/wiki/Unix_System_Laboratories" title="Unix System Laboratories">Unix System Laboratories</a> (USL) through 1994. At that time, <a href="/wiki/SoftQuad" class="mw-redirect" title="SoftQuad">SoftQuad</a> took over the maintenance, although Brian Kernighan continued to improve <i>troff</i> on his own. Thus, there are at least the following variants of the original Bell Labs <i>troff</i> in use:</p>/n<ul>/n<li>the SoftQuad DWB, based on USL DWB 2.0 from 1994;</li>/n<li>the DWB 3.4 from <a href="/wiki/Lucent" title="Lucent">Lucent</a> Software Solutions (formerly USL);</li>/n<li>troff, <a href="/wiki/Plan_9_from_Bell_Labs" title="Plan 9 from Bell Labs">Plan 9</a> edition.</li>/n</ul>/n<p>While <i>troff</i> has been supplanted by other programs such as <a href="/wiki/Interleaf" title="Interleaf">Interleaf</a>, <a href="/wiki/FrameMaker" class="mw-redirect" title="FrameMaker">FrameMaker</a>, and <a href="/wiki/LaTeX" title="LaTeX">LaTeX</a>, it is still being used quite extensively. It remains the default formatter for the <a href="/wiki/UNIX_manual" class="mw-redirect" title="UNIX manual">UNIX documentation</a>.</p>/n<p>The software was reimplemented as <a href="/wiki/Groff_(software)" title="Groff (software)">groff</a> for the <a href="/wiki/GNU%27s_Not_Unix" class="mw-redirect" title="GNU's Not Unix">GNU</a> system beginning in 1990. In addition, due to the <a href="/wiki/Open_source" class="mw-redirect" title="Open source">open sourcing</a> of <a href="/wiki/Ancient_UNIX" title="Ancient UNIX">Ancient UNIX</a> systems, as well as modern successors such as the ditroff based open-sourced versions found on <a href="/wiki/OpenSolaris" title="OpenSolaris">OpenSolaris</a> and <a href="/wiki/Plan_9_from_Bell_Labs" title="Plan 9 from Bell Labs">Plan 9 from Bell Labs</a>, there are several versions of AT&amp;T troff (CAT and ditroff-based<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup>) available under various open-source licenses.</p>/n<h2><span class="mw-headline" id="Preprocessors">Preprocessors</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Troff&amp;action=edit&amp;section=2" title="Edit section: Preprocessors">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>As <i>troff</i> evolved, since there are several things which cannot be done easily in <i>troff</i>, several <a href="/wiki/Preprocessor" title="Preprocessor">preprocessors</a> were developed. These programs transform certain parts of a document into <i>troff</i> input, fitting naturally into the use of "pipelines" in Unix — sending the output of one program as the input to another (see <a href="/wiki/Pipes_and_filters" class="mw-redirect" title="Pipes and filters">pipes and filters</a>). Typically, each preprocessor translates only sections of the input file that are specially marked, passing the rest of the file through unchanged. The embedded preprocessing instructions are written in a simple application-specific programming language, which provides a high degree of power and flexibility.</p>/n<ul>/n<li><a href="/wiki/Eqn" title="Eqn">eqn</a> preprocessor allows mathematical formulae to be specified in simple and intuitive manner.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup></li>/n<li><a href="/wiki/Tbl" title="Tbl">tbl</a> is a preprocessor for formatting tables.</li>/n<li><a href="/wiki/Refer_(software)" title="Refer (software)">refer</a> (and the similar program <i>bib</i>) processes citations in a document according to a <a href="/wiki/Bibliographic" class="mw-redirect" title="Bibliographic">bibliographic</a> database.</li>/n</ul>/n<p>Three preprocessors provide <i>troff</i> with drawing capabilities by defining a <a href="/wiki/Domain-specific_language" title="Domain-specific language">domain-specific language</a> for describing the picture.</p>/n<ul>/n<li><i><a href="/wiki/Pic_(software)" class="mw-redirect" title="Pic (software)">pic</a></i> is a <a href="/wiki/Procedural_programming_language" class="mw-redirect" title="Procedural programming language">procedural programming language</a> providing various drawing functions like <i>circle</i> and <i>box</i>.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup></li>/n<li><i>ideal</i> allows the drawing of pictures <a href="/wiki/Declarative_programming" title="Declarative programming">declaratively</a>, deriving the picture by solving a system of <a href="/wiki/Simultaneous_equations" title="Simultaneous equations">simultaneous equations</a> based on vectors and transformations described by its input.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup></li>/n<li><i>grn</i> describes the pictures through graphical elements drawn at absolute coordinates, based on the <i><a href="/w/index.php?title=Gremlin_(file_format)&amp;action=edit&amp;redlink=1" class="new" title="Gremlin (file format) (page does not exist)">gremlin file format</a></i> defined by an early graphics workstation.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup></li>/n</ul>/n<p>Yet more preprocessors allow the drawing of more complex pictures by generating output for <i>pic</i>.</p>/n<ul>/n<li><i>grap</i> draws <a href="/wiki/Chart" title="Chart">charts</a>, like <a href="/wiki/Scatter_plots" class="mw-redirect" title="Scatter plots">scatter plots</a> and <a href="/wiki/Histograms" class="mw-redirect" title="Histograms">histograms</a>.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[9]</a></sup></li>/n<li><i>chem</i> draws <a href="/wiki/Structural_formula" title="Structural formula">chemical structure diagrams</a>.<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">[10]</a></sup></li>/n<li><i>dformat</i> draws <a href="/wiki/Row_(database)" title="Row (database)">record-based</a> <a href="/wiki/Data_structures" class="mw-redirect" title="Data structures">data structures</a>.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">[11]</a></sup></li>/n</ul>/n<h2><span class="mw-headline" id="Reimplementations">Reimplementations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Troff&amp;action=edit&amp;section=3" title="Edit section: Reimplementations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<ul>/n<li><a href="/wiki/Groff_(software)" title="Groff (software)">groff</a> is <a href="/wiki/GNU_Project" title="GNU Project">GNU Project</a>'s <a href="/wiki/Free_software" title="Free software">free</a> replacement for troff and nroff.</li>/n<li><a rel="nofollow" class="external text" href="http://www-rn.informatik.uni-bremen.de/software/unroff/">unroff</a> is an extensible replacement of troff written in <a href="/wiki/Scheme_(programming_language)" title="Scheme (programming language)">Scheme</a></li>/n<li><a rel="nofollow" class="external text" href="http://heirloom.sf.net/doctools.html">Heirloom troff</a> is based on troff from <a href="/wiki/OpenSolaris" title="OpenSolaris">OpenSolaris</a>. It includes support for <a href="/wiki/OpenType" title="OpenType">OpenType</a> fonts, improved support for <a href="/wiki/Type_1_font" class="mw-redirect" title="Type 1 font">Type 1</a> fonts, support for <a href="/wiki/Unicode" title="Unicode">Unicode</a>, a new paragraph formatting algorithm, and a <a href="/wiki/Groff_(software)" title="Groff (software)">groff</a> compatibility
