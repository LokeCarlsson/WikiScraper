A structured document with Content, sections and subsections for explanations of sentences forms a NLP document, which is actually a <a href="/wiki/Computer_program" title="Computer program">computer program</a>. Natural languages and <a href="/wiki/Natural_language_user_interface" title="Natural language user interface">natural language user interfaces</a> include <a href="/wiki/Inform#Inform_7" title="Inform">Inform7</a>, a natural programming language for making interactive fiction, <a href="/wiki/Ring_(programming_language)" title="Ring (programming language)">Ring</a>, a general purpose language, <a href="/wiki/Shakespeare_(programming_language)" class="mw-redirect" title="Shakespeare (programming language)">Shakespeare</a>, an <a href="/wiki/Esoteric_programming_language" title="Esoteric programming language">esoteric</a> natural programming language in the style of the plays of <a href="/wiki/William_Shakespeare" title="William Shakespeare">William Shakespeare</a>, and <a href="/wiki/Wolfram_Alpha" title="Wolfram Alpha">Wolfram Alpha</a>, a computational knowledge engine, using natural language input.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Natural_language_programming&amp;action=edit&amp;section=1" title="Edit section: Interpretation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The smallest unit of statement in NLP is a sentence. Each sentence is stated in terms of concepts from the underlying ontology, attributes in that ontology and named objects in <a href="/wiki/Capital_letter" class="mw-redirect" title="Capital letter">capital letters</a>. In an NLP text every sentence unambiguously <a href="/wiki/Compiler" title="Compiler">compiles</a> into a <a href="/wiki/Procedure_call" class="mw-redirect" title="Procedure call">procedure call</a> in the underlying <a href="/wiki/High-level_programming_language" title="High-level programming language">high-level programming language</a> such as <a href="/wiki/MATLAB" title="MATLAB">MATLAB</a>, <a href="/wiki/GNU_Octave" title="GNU Octave">Octave</a>, <a href="/wiki/SciLab" class="mw-redirect" title="SciLab">SciLab</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, etc.</p>/n<p>Symbolic languages such as <a href="/wiki/Mathematica" class="mw-redirect" title="Mathematica">Mathematica</a> are capable of <a href="/wiki/Interpreted_language" title="Interpreted language">interpreted</a> processing of queries by sentences. This can allow interactive requests such as that implemented in Wolfram Alpha.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup><sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> The difference between these and NLP is that the latter builds up a single program or a library of routines that are programmed through natural language sentences using an ontology that defines the available data structures in a high level programming language.</p>/n<p>An example text from an English language NLP program (in <a href="/w/index.php?title=SEnglish&amp;action=edit&amp;redlink=1" class="new" title="SEnglish (page does not exist)">sEnglish</a>) is as follows:</p>/n<blockquote>/n<p>If U_ is 'smc01-control', then do the following. Define surface weights Alpha as "[0.5, 0.5]". Initialise matrix Phi as a 'unit matrix'. Define J as the 'inertia matrix' of Spc01. Compute matrix J2 as the inverse of J . Compute position velocity error Ve and angular velocity error Oe from dynamical state X, guidance reference Xnow . Define the joint sliding surface G2 from the position velocity error Ve and angular velocity error Oe using the surface weights Alpha. Compute the smoothed sign function SG2 from the joint sliding surface G2 with sign threshold 0.01. Compute special dynamical force F from dynamical state X and surface weights Alpha. Compute control torque T and control force U from matrix J2, surface weights Alpha, special dynamical force F, smoothed sign function SG2. Finish conditional actions.</p>/n</blockquote>/n<p>that defines a feedback control scheme using a <a href="/wiki/Sliding_mode_control" title="Sliding mode control">sliding mode control</a> method.</p>/n<h2><span class="mw-headline" id="Software_paradigm">Software paradigm</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Natural_language_programming&amp;action=edit&amp;section=2" title="Edit section: Software paradigm">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Natural language programming is a top down method of writing software. Its stages are as follows:</p>/n<ul>/n<li>Definition of an <a href="/wiki/Ontology_(information_science)" title="Ontology (information science)">ontology</a> - taxonomy - of concepts needed to describe tasks in the topic addressed. Each concept and all their attributes are defined in natural language words. This ontology will define the data structures the NLP can use in sentences.</li>/n<li>Definition of one or more top level sentences in terms of concepts from the ontology. These sentences are later used to invoke the most important activities in the topic.</li>/n<li>Defining of each of the top level sentences in terms of a sequence of sentences.</li>/n<li>Defining each of the lower level sentences in terms of other sentences or by a simple sentence of the form <b>Execute code "...".</b> where ... stands for a code in terms of the associated high level programming language.</li>/n<li>Repeating the previous step until you have no sentences left undefined. During this process each of sentences can be classified to belong to a section of the document to be produced in HTML or Latex format to form the final NLP program.</li>/n<li>Testing the meaning of each sentence by executing its code using testing objects.</li>/n<li>Providing a library of procedure calls (in the underlying high level language) which are needed in the code definitions of some low-level-sentence meanings.</li>/n<li>Providing a title, author data and compiling the sentences into an <a href="/wiki/HTML" title="HTML">HTML</a> or <a href="/wiki/LaTex" class="mw-redirect" title="LaTex">LaTex</a> file.</li>/n<li>Publishing the NLP program as a webpage on the Internet or as a <a href="/wiki/PDF" class="mw-redirect" title="PDF">PDF</a> file compiled from the LaTex document.</li>/n</ul>/n<h2><span class="mw-headline" id="Publication_value_of_NLP_programs_and_documents">Publication value of NLP programs and documents</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Natural_language_programming&amp;action=edit&amp;section=3" title="Edit section: Publication value of NLP programs and documents">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>An NLP program is a precise <a href="/wiki/Formal_system" title="Formal system">formal</a> description of some procedure that its author created. It is human readable and it can also be read by a suitable software agent. For example, a web page in an NLP format can be read by a software <a href="/wiki/Personal_assistant" title="Personal assistant">personal assistant</a> agent to a person and she or he can ask the agent to execute some sentences, i.e. carry out some task or answer a question. There is a <i>reader agent</i> available for English interpretation of HTML based NLP documents that a person can run on her <a href="/wiki/Personal_computer" title="Personal computer">personal computer</a> .</p>/n<h2><span class="mw-headline" id="Contribution_of_NLP_programs_to_machine_knowledge">Contribution of NLP programs to machine knowledge</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Natural_language_programming&amp;action=edit&amp;section=4" title="Edit section: Contribution of NLP programs to machine knowledge">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>An ontology class in a natural language program that is not a <a href="/wiki/Concept" title="Concept">concept</a> in the sense as humans use concepts. Concepts in an NLP are examples (samples) of generic human concepts. Each sentence in an NLP program is either (1) stating a relationship in a world model or (2) carries out an action in the environment or (3) carries out a computational procedure or (4) invokes an answering mechanism in response to a question.</p>/n<p>A set of NLP sentences, with associated ontology defined, can also be used as a <a href="/wiki/Pseudo_code" class="mw-redirect" title="Pseudo code">pseudo code</a> that does not provide the details in any underlying high level programming language. In such an application the sentences used become high level abstractions (conceptualisations) of computing procedures that are computer language and machine independent.</p>/n<h2><span
