href="/wiki/Acronym" title="Acronym">acronym</a> for <b>B</b>eginner's <b>A</b>ll-purpose <b>S</b>ymbolic <b>I</b>nstruction <b>C</b>ode)<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> is a family of <a href="/wiki/General-purpose_programming_language" title="General-purpose programming language">general-purpose</a>, <a href="/wiki/High-level_programming_language" title="High-level programming language">high-level programming languages</a> whose design philosophy emphasizes ease of use. In 1964, <a href="/wiki/John_G._Kemeny" title="John G. Kemeny">John G. Kemeny</a> and <a href="/wiki/Thomas_E._Kurtz" title="Thomas E. Kurtz">Thomas E. Kurtz</a> designed the original BASIC language at <a href="/wiki/Dartmouth_College" title="Dartmouth College">Dartmouth College</a> in <a href="/wiki/New_Hampshire" title="New Hampshire">New Hampshire</a>, <a href="/wiki/United_States" title="United States">United States</a>. They wanted to enable students in fields other than science and mathematics to use computers. At the time, nearly all use of computers required writing custom software, which was something only <a href="/wiki/Scientist" title="Scientist">scientists</a> and <a href="/wiki/Mathematician" title="Mathematician">mathematicians</a> tended to learn.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (August 2017)">citation needed</span></a></i>]</sup></p>/n<p>Versions of BASIC became widespread on <a href="/wiki/Microcomputer" title="Microcomputer">microcomputers</a> in the mid-1970s and 1980s. <a href="/wiki/Microcomputer" title="Microcomputer">Microcomputers</a> usually shipped with BASIC, often in the machine's <a href="/wiki/Firmware" title="Firmware">firmware</a>. Having an easy-to-learn language on these early <a href="/wiki/Personal_computer" title="Personal computer">personal computers</a> allowed <a href="/wiki/Small_business" title="Small business">small business</a> owners, professionals, <a href="/wiki/Hobbyist" class="mw-redirect" title="Hobbyist">hobbyists</a>, and <a href="/wiki/Consultant" title="Consultant">consultants</a> to develop <a href="/wiki/Custom_software" title="Custom software">custom software</a> on computers they could afford.<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:No_original_research" title="Wikipedia:No original research"><span title="The material near this tag possibly contains original research. (August 2017)">original research?</span></a></i>]</sup> In the 2010s, BASIC was popular in many <a href="/wiki/Dialect_(computing)" title="Dialect (computing)">computing dialects</a> and in new languages influenced by BASIC, such as <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>'s <a href="/wiki/Visual_Basic" title="Visual Basic">Visual Basic</a>.</p>/n<p></p>/n<div id="toc" <span class="toctext">External links</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=BASIC&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Before the mid-1960s, the only computers were huge <a href="/wiki/Mainframe_computer" title="Mainframe computer">mainframe computers</a>. Users submitted jobs (calculations or other requests) on <a href="/wiki/Punched_card" title="Punched card">punched cards</a> or similar media to specialist <a href="/wiki/Computer_operator" title="Computer operator">computer operators</a>. The computer stored these, then used a <a href="/wiki/Batch_processing" title="Batch processing">batch processing</a> system to run this queue of jobs one after another, allowing very high levels of utilization of these expensive machines. As the performance of computing hardware rose through the 1960s, <a href="/wiki/Multi-processing" class="mw-redirect" title="Multi-processing">multi-processing</a> was developed. This allowed a mix of batch jobs to be run together, but the real revolution was the development of <a href="/wiki/Time-sharing" title="Time-sharing">time-sharing</a>. <a href="/wiki/Time-sharing" title="Time-sharing">Time-sharing</a> allowed multiple remote interactive users to share use of the computer, interacting with the computer from <a href="/wiki/Computer_terminal" title="Computer terminal">computer terminals</a> with keyboards and <a href="/wiki/Teletype_printer" class="mw-redirect" title="Teletype printer">teletype printers</a>, and later <a href="/wiki/Computer_monitor" title="Computer monitor">display screens</a>, in much the same way as desktop computers or <a href="/wiki/Personal_computer" title="Personal computer">personal computers</a> would be used later.</p>/n<h3><span class="mw-headline" id="Origin">Origin</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=BASIC&amp;action=edit&amp;section=2" title="Edit section: Origin">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>The original BASIC language was released on May 1, 1964 by <a href="/wiki/John_G._Kemeny" title="John G. Kemeny">John G. Kemeny</a> and <a href="/wiki/Thomas_E._Kurtz" title="Thomas E. Kurtz">Thomas E. Kurtz</a><sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> and implemented under their direction by a team of <a href="/wiki/Dartmouth_College" title="Dartmouth College">Dartmouth College</a> students.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup><sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup><sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability"><span title="The material near this tag needs to be fact-checked with the cited source(s). (June 2017)">verification needed</span></a></i>]</sup> The <a href="/wiki/Acronym" title="Acronym">acronym</a> <i>BASIC</i> comes from the name of an unpublished paper by Thomas Kurtz.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup> BASIC was designed to allow students to write mainframe computer programs for the <a href="/wiki/Dartmouth_Time-Sharing_System" class="mw-redirect" title="Dartmouth Time-Sharing System">Dartmouth Time-Sharing System</a>. It was intended specifically for less technical users who did not have or want the mathematical background previously expected. Being able to use a computer to support teaching and research was quite novel at the time.</p>/n<p>The language was based on <a href="/wiki/FORTRAN_II" class="mw-redirect" title="FORTRAN II">FORTRAN II</a>, with some influences from <a href="/wiki/ALGOL_60" title="ALGOL 60">ALGOL 60</a> and with additions to make it suitable for timesharing. Initially, BASIC concentrated on supporting straightforward mathematical work, with <a href="/wiki/Matrix_(math)" class="mw-redirect" title="Matrix (math)">matrix</a> arithmetic support from its initial implementation as a batch language, and <a href="/wiki/Character_string" class="mw-redirect" title="Character string">character string</a> functionality being added by 1965. Wanting use of the language to become widespread, its designers made the compiler available free of charge. (In the 1960s, software became a chargeable commodity; until then, it was provided without charge as a service with the very expensive computers, usually available only to lease.) They also made it available to high schools in the <a href="/wiki/Hanover,_New_Hampshire" title="Hanover, New Hampshire">Hanover, New Hampshire</a> area and put considerable effort into promoting the language. In the following years, as other dialects of BASIC appeared, Kemeny and Kurtz's original BASIC dialect became known as <i><a href="/wiki/Dartmouth_BASIC" title="Dartmouth BASIC">Dartmouth BASIC</a></i></p>/n<h3><span class="mw-headline" id="Spread_on_minicomputers">Spread on minicomputers</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=BASIC&amp;action=edit&amp;section=3" title="Edit section: Spread on minicomputers">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<div class="thumb tright">/n<div class="thumbinner" style="width:222px;"><a href="/wiki/File:Pn-pravez-class-5.jpg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/1/18/Pn-pravez-class-5.jpg/220px-Pn-pravez-class-5.jpg" width="220" height="144" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/1/18/Pn-pravez-class-5.jpg/330px-Pn-pravez-class-5.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/1/18/Pn-pravez-class-5.jpg/440px-Pn-pravez-class-5.jpg 2x" data-file-width="1600" data-file-height="1045" /></a>/n<div class="thumbcaption">/n<div class="magnify"><a href="/wiki/File:Pn-pravez-class-5.jpg" class="internal" title="Enlarge"></a></div>/n"Train Basic every day!"&#160;— reads a poster (bottom center) in a Russian school. (ca. 1985–1986)</div>/n</div>/n</div>/n<p>Knowledge of the relatively simple BASIC became widespread for a computer language, and it was implemented by a number of manufacturers, becoming fairly popular on newer <a href="/wiki/Minicomputer" title="Minicomputer">minicomputers</a> such as the <a href="/wiki/Digital_Equipment_Corporation" title="Digital Equipment Corporation">DEC</a> <a href="/wiki/Programmed_Data_Processor" title="Programmed Data Processor">PDP</a> series, where <a href="/wiki/BASIC-PLUS" title="BASIC-PLUS">BASIC-PLUS</a> was an extended dialect for use on the <a href="/wiki/RSTS/E" title="RSTS/E">RSTS/E</a> time-sharing operating system. The BASIC language was available for the <a href="/wiki/Data_General_Nova" title="Data General Nova">Data General Nova</a>, and also central to the <a href="/wiki/HP_Time-Shared_BASIC" title="HP Time-Shared BASIC">HP Time-Shared BASIC</a> system in the late 1960s and early 1970s, where the language was implemented as an <a href="/wiki/Interpreter_(computing)" title="Interpreter (computing)">interpreter</a>. A version was a core part of the <a href="/wiki/Pick_operating_system" title="Pick operating system">Pick operating system</a> from 1973 onward, where a compiler renders it into <a href="/wiki/Bytecode" title="Bytecode">bytecode</a>, able to be interpreted by a virtual machine.</p>/n<p>During this period a number of simple <a href="/wiki/Computer_game" class="mw-redirect" title="Computer game">computer games</a> were written in BASIC, most notably Mike Mayfield's <i><a href="/wiki/Star_Trek_(text_game)" title="Star Trek (text game)">Star Trek</a></i>. A number of these were collected by DEC employee <a href="/wiki/David_H._Ahl" title="David H. Ahl">David H. Ahl</a> and published in a newsletter he compiled. He later collected a number of these into book form, <i>101 BASIC Computer Games</i>, published in 1973.<sup id="cite_ref-basicgames_6-0" class="reference"><a href="#cite_note-basicgames-6">[6]</a></sup> During the same period, Ahl was involved in the creation of a small computer for education use, an early <a href="/wiki/Personal_computer" title="Personal computer">personal computer</a>. When management refused to support the concept, Ahl left DEC in 1974 to found the seminal computer magazine, <i><a href="/wiki/Creative_Computing" class="mw-redirect" title="Creative Computing">Creative Computing</a></i>. The book remained popular, and was re-published on several occasions.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup></p>/n<h3><span class="mw-headline" id="Explosive_growth:_the_home_computer_era">Explosive growth: the home computer era</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=BASIC&amp;action=edit&amp;section=4" title="Edit section: Explosive growth: the home computer era">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<div class="thumb tright">/n<div class="thumbinner" style="width:222px;"><a href="/wiki/File:CommodoreBasic.JPG" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/1/14/CommodoreBasic.JPG/220px-CommodoreBasic.JPG" width="220" height="154" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/1/14/CommodoreBasic.JPG/330px-CommodoreBasic.JPG 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/1/14/CommodoreBasic.JPG/440px-CommodoreBasic.JPG 2x" data-file-width="756" data-file-height="530" /></a>/n<div class="thumbcaption">/n<div class="magnify"><a href="/wiki/File:CommodoreBasic.JPG" class="internal" title="Enlarge"></a></div>/n<a href="/wiki/Commodore_BASIC" title="Commodore BASIC">Commodore BASIC</a> v2.0 on the <a href="/wiki/Commodore_64" title="Commodore 64">Commodore 64</a></div>/n</div>/n</div>/n<div class="thumb tright">/n<div class="thumbinner" style="width:222px;"><a href="/wiki/File:Msxbasic.png" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/7/71/Msxbasic.png/220px-Msxbasic.png" width="220" height="165" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/7/71/Msxbasic.png 1.5x" data-file-width="256" data-file-height="192" /></a>/n<div class="thumbcaption">/n<div class="magnify"><a href="/wiki/File:Msxbasic.png" class="internal" title="Enlarge"></a></div>/n<a href="/wiki/MSX_BASIC" title="MSX BASIC">MSX BASIC</a> version 3.0</div>/n</div>/n</div>/n<p>The introduction of the first <a href="/wiki/Microcomputer" title="Microcomputer">microcomputers</a> in the mid-1970s was the start of explosive growth for BASIC. It had the advantage that it was fairly well known to the young designers and computer hobbyists who took an interest in microcomputers. Despite <a href="/wiki/Edsger_W._Dijkstra" title="Edsger W. Dijkstra">Dijkstra</a>'s famous judgement in 1975, "It is practically impossible to teach good programming to students that have had a prior exposure to BASIC: as potential programmers they are mentally mutilated beyond hope of regeneration",<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup> BASIC was one of the few languages that was both high-level enough to be usable by those without training and small enough to fit into the microcomputers of the day, making it the de facto standard programming language on early microcomputers.</p>/n<p>One of the first BASICs to appear was <a href="/wiki/Tiny_BASIC" title="Tiny BASIC">Tiny BASIC</a>, a simple BASIC variant designed by <a href="/wiki/Dennis_Allison" title="Dennis Allison">Dennis Allison</a> at the urging of <a href="/wiki/Bob_Albrecht" title="Bob Albrecht">Bob Albrecht</a> of the <a href="/wiki/Homebrew_Computer_Club" title="Homebrew Computer Club">Homebrew Computer Club</a>. He had seen BASIC on <a href="/wiki/Minicomputer" title="Minicomputer">minicomputers</a> and felt it would be the perfect match for new machines like the <a href="/wiki/Altair_8800" title="Altair 8800">MITS Altair 8800</a>. How to design and implement a stripped-down version of an <a href="/wiki/Interpreter_(computer_software)" class="mw-redirect" title="Interpreter (computer software)">interpreter</a> for the BASIC language was covered in articles by Allison in the first three quarterly issues of the <i><a href="/wiki/People%27s_Computer_Company" title="People's Computer Company">People's Computer Company</a></i> newsletter published in 1975 and implementations with source code published in <a href="/wiki/Dr._Dobb%27s_Journal_of_Tiny_BASIC_Calisthenics_%26_Orthodontia:_Running_Light_Without_Overbyte" class="mw-redirect" title="Dr. Dobb's Journal of Tiny BASIC Calisthenics &amp; Orthodontia: Running Light Without Overbyte">Dr. Dobb's Journal of Tiny BASIC Calisthenics &amp; Orthodontia: Running Light Without Overbyte</a>. Versions were written by <a href="/wiki/Li-Chen_Wang" title="Li-Chen Wang">Li-Chen Wang</a> and Tom Pittman.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[9]</a></sup> In 1975 MITS released <a href="/wiki/Altair_BASIC" title="Altair BASIC">Altair BASIC</a>, developed by <a href="/wiki/Bill_Gates" title="Bill Gates">Bill Gates</a> and <a href="/wiki/Paul_Allen" title="Paul Allen">Paul Allen</a> as the company Micro-Soft,<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">[10]</a></sup> which eventually grew into corporate giant <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>. The first Altair version was co-written by Gates, Allen, and <a href="/wiki/Monte_Davidoff" title="Monte Davidoff">Monte Davidoff</a>.</p>/n<p>Almost universally, <a href="/wiki/Home_computer" title="Home computer">home computers</a> of the 1980s had a <a href="/wiki/Read-only_memory" title="Read-only memory">ROM</a>-resident BASIC interpreter, which the machines <a href="/wiki/Booting" title="Booting">booted</a> directly into.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">[notes 1]</a></sup> When the <a href="/wiki/Apple_II" title="Apple II">Apple II</a>, <a href="/wiki/PET_2001" class="mw-redirect" title="PET 2001">PET 2001</a>, and <a href="/wiki/TRS-80" title="TRS-80">TRS-80</a> were all released in 1977, all three had BASIC as their primary programming language and operating environment. Upon boot, a BASIC interpreter in <a href="/wiki/Immediate_mode" class="mw-redirect" title="Immediate mode">immediate mode</a> was presented, not the <a href="/wiki/Command-line_interface" title="Command-line interface">command-line interface</a> used on systems running <a href="/wiki/CP/M" title="CP/M">CP/M</a> or <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a>. <a href="/wiki/Commodore_Business_Machines" class="mw-redirect" title="Commodore Business Machines">Commodore Business Machines</a> included a version of Microsoft BASIC. The Apple II and TRS-80 each had two versions of BASIC, a smaller introductory version introduced with the initial releases of the machines and a more advanced version developed as interest in the platforms increased. As new companies entered the field, additional versions were added that subtly changed the BASIC family. The <a href="/wiki/Atari_8-bit_family" title="Atari 8-bit family">Atari 8-bit family</a> had its own <a href="/wiki/Atari_BASIC" title="Atari BASIC">Atari BASIC</a> that was modified in order to fit on an 8&#160;kB ROM cartridge. The <a href="/wiki/BBC" title="BBC">BBC</a> published <a href="/wiki/BBC_BASIC" title="BBC BASIC">BBC BASIC</a>, developed by <a href="/wiki/Acorn_Computers_Ltd" class="mw-redirect" title="Acorn Computers Ltd">Acorn Computers Ltd</a>, incorporating many extra <a href="/wiki/Structured_programming" title="Structured programming">structured programming</a> keywords and advanced floating-point operation features.</p>/n<p>As the popularity of BASIC grew in this period, computer magazines published complete source code in BASIC for video games, utilities, and other programs. Given BASIC's straightforward nature, it was a simple matter to <a href="/wiki/Type-in_program" title="Type-in program">type in the code</a> from the magazine and execute the program. Different magazines were published featuring programs for specific computers, though some BASIC programs were considered universal and could be used in machines running any variant of BASIC (sometimes with minor adaptations). Many books of type-in programs were also available, and in particular, Ahl published versions of the original 101 BASIC games converted into the Microsoft dialect and published it from <i>Creative Computing</i> as <i><a href="/wiki/BASIC_Computer_Games" title="BASIC Computer Games">BASIC Computer Games</a></i>. This book, and its sequels, provided hundreds of ready-to-go programs that could be easily converted to practically any BASIC-running platform.<sup id="cite_ref-basicgames_6-1" class="reference"><a href="#cite_note-basicgames-6">[6]</a></sup><sup id="cite_ref-12" class="reference"><a href="#cite_note-12">[11]</a></sup><sup id="cite_ref-13" class="reference"><a href="#cite_note-13">[12]</a></sup> The book reached the stores in 1978, just as the <a href="/wiki/Home_computer" title="Home computer">home computer</a> market was starting off, and it became the first million-selling computer book. Later packages, such as <a href="/w/index.php?title=Learn_to_Program_BASIC&amp;action=edit&amp;redlink=1" class="new" title="Learn to Program BASIC (page does not exist)">Learn to Program BASIC</a> would also have gaming as an introductory focus. On the business-focused <a href="/wiki/CP/M" title="CP/M">CP/M</a> computers which soon became widespread in small business environments, <a href="/wiki/Microsoft_BASIC" title="Microsoft BASIC">Microsoft BASIC</a> (<a href="/wiki/MBASIC" title="MBASIC">MBASIC</a>) was one of the leading applications.<sup id="cite_ref-oldcomputer_14-0" class="reference"><a href="#cite_note-oldcomputer-14">[13]</a></sup></p>/n<h3><span class="mw-headline" id="IBM_PC_and_compatibles">IBM PC and compatibles</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=BASIC&amp;action=edit&amp;section=5" title="Edit section: IBM PC and compatibles">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<div class="thumb tright">/n<div class="thumbinner" style="width:222px;"><a href="/wiki/File:IBM_Cassette_BASIC.png" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/d/d1/IBM_Cassette_BASIC.png/220px-IBM_Cassette_BASIC.png" width="220" height="120" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/d/d1/IBM_Cassette_BASIC.png/330px-IBM_Cassette_BASIC.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/d/d1/IBM_Cassette_BASIC.png/440px-IBM_Cassette_BASIC.png 2x" data-file-width="640" data-file-height="350" /></a>/n<div class="thumbcaption">/n<div class="magnify"><a href="/wiki/File:IBM_Cassette_BASIC.png" class="internal" title="Enlarge"></a></div>/n<a href="/wiki/IBM_Cassette_BASIC" class="mw-redirect" title="IBM Cassette BASIC">IBM Cassette BASIC</a> 1.10</div>/n</div>/n</div>/n<p>When IBM was designing the <a href="/wiki/IBM_PC" class="mw-redirect" title="IBM PC">IBM PC</a> they followed the paradigm of existing home computers in wanting to have a built-in BASIC. They sourced this from Microsoft – <a href="/wiki/IBM_Cassette_BASIC" class="mw-redirect" title="IBM Cassette BASIC">IBM Cassette BASIC</a> – but Microsoft also produced several other versions of BASIC for <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a>/<a href="/wiki/PC_DOS" class="mw-redirect" title="PC DOS">PC DOS</a> including <a href="/wiki/IBM_Disk_BASIC" class="mw-redirect" title="IBM Disk BASIC">IBM Disk BASIC</a> (BASIC D), <a href="/wiki/IBM_BASICA" class="mw-redirect" title="IBM BASICA">IBM BASICA</a> (BASIC A), <a href="/wiki/Microsoft_GW-BASIC_interpreter" class="mw-redirect" title="Microsoft GW-BASIC interpreter">GW-BASIC</a> (a BASICA-compatible version that did not need IBM's ROM) and <a href="/wiki/QBasic" title="QBasic">QBasic</a>, all typically bundled with the machine. In addition they produced the Microsoft BASIC Compiler aimed at professional programmers. Turbo <a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a>-publisher <a href="/wiki/Borland" title="Borland">Borland</a> published <a href="/wiki/Turbo_Basic" class="mw-redirect" title="Turbo Basic">Turbo Basic</a> 1.0 in 1985 (successor versions are still being marketed by the original author under the name <a href="/wiki/PowerBASIC" title="PowerBASIC">PowerBASIC</a>). Microsoft wrote the windowed <a href="/wiki/AmigaBASIC" title="AmigaBASIC">AmigaBASIC</a> that was supplied with version 1.1 of the pre-emptive multitasking GUI Amiga computers (late 1985 / early 1986), although the product unusually did not bear any Microsoft marks. These languages introduced many extensions to the original home-computer BASIC, such as improved <a href="/wiki/String_manipulation" class="mw-redirect" title="String manipulation">string manipulation</a> and graphics support, access to the <a href="/wiki/File_system" title="File system">file system</a> and additional <a href="/wiki/Data_type" title="Data type">data types</a>. More important were the facilities for <a href="/wiki/Structured_programming" title="Structured programming">structured programming</a>, including additional <a href="/wiki/Control_structures" class="mw-redirect" title="Control structures">control structures</a> and proper <a href="/wiki/Subroutine" title="Subroutine">subroutines</a> supporting <a href="/wiki/Local_variable" title="Local variable">local variables</a>. However, by the latter half of the 1980s, users were increasingly using pre-made applications written by others, rather than learning programming themselves, while professional programmers now had a wide range of more advanced languages available on small computers. <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> and later <a href="/wiki/C%2B%2B" title="C++">C++</a> became the languages of choice for professional <a href="/wiki/Shrink_wrap_contract" title="Shrink wrap contract">"shrink wrap"</a> application development.<sup id="cite_ref-15" class="reference"><a href="#cite_note-15">[14]</a></sup><sup id="cite_ref-16" class="reference"><a href="#cite_note-16">[15]</a></sup></p>/n<h3><span class="mw-headline" id="Visual_Basic">Visual Basic</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=BASIC&amp;action=edit&amp;section=6" title="Edit section: Visual Basic">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>In 1991 Microsoft introduced <a href="/wiki/Visual_Basic" title="Visual Basic">Visual Basic</a>, an evolutionary development of QuickBasic. It included constructs from that language such as block-structured control statements, parameterized subroutines, and optional <a href="/wiki/Type_system#STATIC" title="Type system">static typing</a>, as well as <a href="/wiki/Object_oriented_language" class="mw-redirect" title="Object oriented language">object-oriented</a> constructs from other languages such as "With" and "For Each". The language retained some compatibility with its predecessors, such as the Dim keyword for declarations, "Gosub"/Return statements, and optional line numbers which could be used to locate errors. An important driver for the development of Visual Basic was as the new <a href="/wiki/Macro_language" class="mw-redirect" title="Macro language">macro language</a> for <a href="/wiki/Microsoft_Excel" title="Microsoft Excel">Microsoft Excel</a>, a <a href="/wiki/Spreadsheet" title="Spreadsheet">spreadsheet</a> program. To the surprise of many at Microsoft who still initially marketed it as a language for hobbyists, the language came into widespread use for small custom business applications shortly after the release of VB version 3.0, which is widely considered the first relatively stable version. While many advanced programmers still scoffed at its use, VB met the needs of <a href="/wiki/Small_business" title="Small business">small businesses</a> efficiently wherever ease of development was more of a concern than processing speed.<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:No_original_research" title="Wikipedia:No original research"><span title="The material near this tag possibly contains original research. (August 2017)">original research?</span></a></i>]</sup></p>/n<p>By that time, computers running Windows 3.1 had become fast enough that many business-related processes could be completed "in the blink of an eye" even using a "slow" language, as long as large amounts of data were not involved. Many small business owners found they could create their own small, yet useful applications in a few evenings to meet their own specialized needs. Eventually, during the lengthy lifetime of VB3, knowledge of Visual Basic had become a marketable job skill. Microsoft also produced <a href="/wiki/VBScript" title="VBScript">VBScript</a> in 1996 and <a href="/wiki/Visual_Basic_.NET" title="Visual Basic .NET">Visual Basic .NET</a> in 2001. The latter has essentially the same power as <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a> and <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> but with syntax that reflects the original Basic language.</p>/n<div class="thumb tright">/n<div class="thumbinner" style="width:222px;"><a href="/wiki/File:TriBasicExample.png" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/7/7a/TriBasicExample.png/220px-TriBasicExample.png" width="220" height="176" Basic, <a href="/wiki/OpenOffice.org_Basic" class="mw-redirect" title="OpenOffice.org Basic">OpenOffice.org Basic</a> and <a href="/wiki/Gambas" title="Gambas">Gambas</a></div>/n</div>/n</div>/n<h3><span class="mw-headline" id="Post-1990_versions_and_dialects">Post-1990 versions and dialects</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=BASIC&amp;action=edit&amp;section=7" title="Edit section: Post-1990 versions and dialects">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Many other BASIC dialects have also sprung up since 1990, including the <a href="/wiki/Open_source" class="mw-redirect" title="Open source">open source</a> <a href="/wiki/QB64" title="QB64">QB64</a> and <a href="/wiki/FreeBASIC" title="FreeBASIC">FreeBASIC</a>, inspired by QBasic, and the Visual Basic-styled <a href="/wiki/RapidQ" title="RapidQ">RapidQ</a>, <a href="/wiki/Basic_For_Qt" title="Basic For Qt">Basic For Qt</a> and <a href="/wiki/Gambas" title="Gambas">Gambas</a>. Modern commercial incarnations include <a href="/wiki/PureBasic" title="PureBasic">PureBasic</a>, <a href="/wiki/PowerBASIC" title="PowerBASIC">PowerBASIC</a>, <a href="/wiki/Xojo" title="Xojo">Xojo</a>, <a href="/wiki/Monkey_X" title="Monkey X">Monkey X</a> and <a href="/wiki/True_BASIC" title="True BASIC">True BASIC</a> (the direct successor to Dartmouth BASIC from a company controlled by Kurtz). Several web-based simple BASIC interpreters also now exist, including <a rel="nofollow" class="external text" href="http://www.quitebasic.com/">Quite BASIC</a> and Microsoft's <a href="/wiki/Microsoft_Small_Basic" title="Microsoft Small Basic">Small Basic</a> (educational software). Versions of BASIC have been showing up for use on <a href="/wiki/Smartphone" title="Smartphone">smartphones</a> and tablets. Apple <a href="/wiki/App_Store" class="mw-redirect" title="App Store">App Store</a> contains such implementations of BASIC programming language as <a href="/w/index.php?title=Smart_BASIC&amp;action=edit&amp;redlink=1" class="new" title="Smart BASIC (page does not exist)">smart BASIC</a>, <a href="/w/index.php?title=Basic!&amp;action=edit&amp;redlink=1" class="new" title="Basic! Android devices feature such implementations of BASIC as <a href="/w/index.php?title=RFO_BASIC&amp;action=edit&amp;redlink=1" class="new" title="RFO BASIC (page does not exist)">RFO BASIC</a> and <a href="/w/index.php?title=Mintoris_Basic&amp;action=edit&amp;redlink=1" class="new" title="Mintoris Basic (page does not exist)">Mintoris Basic</a>. More powerful versions had floating-point arithmetic, and variables could be labelled with names six or more characters long. There were some problems and restrictions in early implementations; for example, Applesoft allowed variable names to be several characters long, but only the first two were significant, thus it was possible to inadvertently write a program with variables "LOSS" and "LOAN", which would be treated as being the same; assigning a value to "LOAN" would silently overwrite the value intended as "LOSS". Keywords could not be used in variables in many early BASICs; "SCORE" would be interpreted as "SC" OR "E", where OR was a keyword. <a href="/wiki/String_(computer_science)" title="String (computer science)">String</a> variables are usually distinguished in many microcomputer dialects by having $ suffixed to their name, and values are often identified as strings by being delimited by "double quotation marks". Arrays in BASIC could contain integers, floating point or string variables.</p>/n<p>Some dialects of BASIC supported <a href="/wiki/Matrix_(mathematics)" title="Matrix (mathematics)">matrices and matrix operations</a>, useful for the solution of sets of simultaneous linear algebraic equations. These dialects would directly support matrix operations such as assignment, addition, multiplication (of compatible matrix types), and evaluation of a determinant. Many microcomputer BASICs did not support this data type; matrix operations were still possible, but had to be programmed explicitly on array elements.</p>/n<h3><span class="mw-headline"
