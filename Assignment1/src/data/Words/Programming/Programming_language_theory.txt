data-file-width="375" data-file-height="375" /></a>/n<div class="thumbcaption">/n<div class="magnify"><a href="/wiki/File:Lambda_lc.svg" class="internal" title="Enlarge"></a></div>/nThe lowercase <a href="/wiki/Greek_alphabet" title="Greek alphabet">Greek</a> letter λ (<a href="/wiki/Lambda" title="Lambda">lambda</a>) is an unofficial symbol of the field of programming language theory. This usage derives from the <a href="/wiki/Lambda_calculus" title="Lambda calculus">lambda calculus</a>, a <a href="/wiki/Model_of_computation" title="Model of computation">model of computation</a> introduced by <a href="/wiki/Alonzo_Church" title="Alonzo Church">Alonzo Church</a> in the 1930s and widely used by programming language researchers. It graces the cover of the classic text <i><a href="/wiki/Structure_and_Interpretation_of_Computer_Programs" title="Structure and Interpretation of Computer Programs">Structure and Interpretation of Computer Programs</a></i>, and the title of the so-called <a href="/wiki/Lambda_Papers" class="mw-redirect" title="Lambda Papers">Lambda Papers</a>, written by <a href="/wiki/Gerald_Jay_Sussman" title="Gerald Jay Sussman">Gerald Jay Sussman</a> and <a href="/wiki/Guy_Steele" class="mw-redirect" title="Guy Steele">Guy Steele</a>, the developers of the <a href="/wiki/Scheme_(programming_language)" title="Scheme (programming language)">Scheme</a> <a href="/wiki/Programming_language" title="Programming language">programming language</a>.</div>/n</div>/n</div>/n<p><b>Programming language theory</b> (<b>PLT</b>) is a branch of <a href="/wiki/Computer_science" title="Computer science">computer science</a> that deals with the design, implementation, analysis, characterization, and classification of <a href="/wiki/Programming_language" title="Programming language">programming languages</a> and their individual <a href="/wiki/Programming_language#Elements" title="Programming language">features</a>. It falls within the discipline of <a href="/wiki/Computer_science" title="Computer science">computer science</a>, both depending on and affecting <a href="/wiki/Mathematics" title="Mathematics">mathematics</a>, <a href="/wiki/Software_engineering" title="Software engineering">software engineering</a> and <a href="/wiki/Linguistics" title="Linguistics">linguistics</a>. It is a well-recognized branch of computer science, and an active research area, with results published in numerous <a href="/wiki/Academic_journal" title="Academic journal">journals</a> dedicated to PLT, as well as in general computer science and engineering publications.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Programming_language_theory&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>In some ways, the history of programming language theory predates even the development of programming languages themselves. The <a href="/wiki/Lambda_calculus" title="Lambda calculus">lambda calculus</a>, developed by <a href="/wiki/Alonzo_Church" title="Alonzo Church">Alonzo Church</a> and <a href="/wiki/Stephen_Cole_Kleene" title="Stephen Cole Kleene">Stephen Cole Kleene</a> in the 1930s, is considered by some to be the world's first programming language, even though it was intended to <a href="/wiki/Model_of_computation" title="Model of computation"><i>model</i></a> computation rather than being a means for programmers to <a href="/wiki/Computer_programming" title="Computer programming"><i>describe</i></a> algorithms to a computer system. Many modern <a href="/wiki/Functional_programming_language" class="mw-redirect" title="Functional programming language">functional programming languages</a> have been described as providing a "thin veneer" over the lambda calculus,<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> and many are easily described in terms of it.</p>/n<p>The first programming language to be invented was <a href="/wiki/Plankalk%C3%BCl" title="Plankalkül">Plankalkül</a>, which was designed by <a href="/wiki/Konrad_Zuse" title="Konrad Zuse">Konrad Zuse</a> in the 1940s, but not publicly known until 1972 (and not implemented until 1998). The first widely known and successful <a href="/wiki/High-level_programming_language" title="High-level programming language">high-level programming language</a> was <a href="/wiki/Fortran" title="Fortran">Fortran</a>, developed from 1954 to 1957 by a team of <a href="/wiki/IBM" title="IBM">IBM</a> researchers led by <a href="/wiki/John_Backus" title="John Backus">John Backus</a>. The success of FORTRAN led to the formation of a committee of scientists to develop a "universal" computer language; the result of their effort was <a href="/wiki/ALGOL_58" title="ALGOL 58">ALGOL 58</a>. Separately, <a href="/wiki/John_McCarthy_(computer_scientist)" title="John McCarthy (computer scientist)">John McCarthy</a> of <a href="/wiki/MIT" class="mw-redirect" title="MIT">MIT</a> developed the <a href="/wiki/Lisp_programming_language" class="mw-redirect" title="Lisp programming language">Lisp programming language</a> (based on the lambda calculus), the first language with origins in academia to be successful. With the success of these initial efforts, programming languages became an active topic of research in the 1960s and beyond.</p>/n<p>Some other key events in the history of programming language theory since then:</p>/n<h3><span class="mw-headline" id="1950s">1950s</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Programming_language_theory&amp;action=edit&amp;section=2" title="Edit section: 1950s">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<ul>/n<li><a href="/wiki/Noam_Chomsky" title="Noam Chomsky">Noam Chomsky</a> developed the <a href="/wiki/Chomsky_hierarchy" title="Chomsky hierarchy">Chomsky hierarchy</a> in the field of <a href="/wiki/Linguistics" title="Linguistics">linguistics</a>; a discovery which has directly impacted programming language theory and other branches of computer science.</li>/n</ul>/n<h3><span class="mw-headline" id="1960s">1960s</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Programming_language_theory&amp;action=edit&amp;section=3" title="Edit section: 1960s">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<ul>/n<li>The <a href="/wiki/Simula" title="Simula">Simula</a> language was developed by <a href="/wiki/Ole-Johan_Dahl" title="Ole-Johan Dahl">Ole-Johan Dahl</a> and <a href="/wiki/Kristen_Nygaard" title="Kristen Nygaard">Kristen Nygaard</a>; it is widely considered to be the first example of an <a href="/wiki/Object-oriented_programming_language" class="mw-redirect" title="Object-oriented programming language">object-oriented programming language</a>; Simula also introduced the concept of <a href="/wiki/Coroutine" title="Coroutine">coroutines</a>.</li>/n<li>In 1964, <a href="/wiki/Peter_Landin" title="Peter Landin">Peter Landin</a> is the first to realize <a href="/wiki/Alonzo_Church" title="Alonzo Church">Church</a>'s <a href="/wiki/Lambda_calculus" title="Lambda calculus">lambda calculus</a> can be used to model programming languages. He introduces the <a href="/wiki/SECD_machine" title="SECD machine">SECD machine</a> which "interprets" lambda expressions.</li>/n<li>In 1965, Landin introduces the <a href="/wiki/J_operator" title="J operator">J operator</a>, essentially a form of <a href="/wiki/Continuation" title="Continuation">continuation</a>.</li>/n<li>In 1966, Landin introduces <a href="/wiki/ISWIM" title="ISWIM">ISWIM</a>, an abstract computer <a href="/wiki/Programming_language" title="Programming language">programming language</a> in his article <i>The Next 700 Programming Languages</i>. It is influential in the design of languages leading to the <a href="/wiki/Haskell_(programming_language)" title="Haskell (programming language)">Haskell</a> programming language.</li>/n<li>In 1966, <a href="/wiki/Corrado_B%C3%B6hm" title="Corrado Böhm">Corrado Böhm</a> introduced the programming language <a href="/w/index.php?title=CUCH&amp;action=edit&amp;redlink=1" class="new" title="CUCH (page does not exist)">CUCH</a> (Curry-Church).<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup></li>/n<li>In 1967, <a href="/wiki/Christopher_Strachey" title="Christopher Strachey">Christopher Strachey</a> publishes his influential set of lecture notes <i><a href="/wiki/Fundamental_Concepts_in_Programming_Languages" title="Fundamental Concepts in Programming Languages">Fundamental Concepts in Programming Languages</a></i>, introducing the terminology <i><a href="/wiki/Value_(computer_science)" title="Value (computer science)">R-values</a></i>, <i><a href="/wiki/Value_(computer_science)" title="Value (computer science)">L-values</a></i>, <i><a href="/wiki/Parametric_polymorphism" title="Parametric polymorphism">parametric polymorphism</a></i>, and <i><a href="/wiki/Ad_hoc_polymorphism" title="Ad hoc polymorphism">ad hoc polymorphism</a></i>.</li>/n<li>In 1969, <a href="/wiki/J._Roger_Hindley" title="J. Roger Hindley">J. Roger Hindley</a> publishes <i>The Principal Type-Scheme of an Object in Combinatory Logic</i>, later generalized into the <a href="/wiki/Hindley%E2%80%93Milner" class="mw-redirect" title="Hindley–Milner">Hindley–Milner</a> <a href="/wiki/Type_inference" title="Type inference">type inference</a> algorithm.</li>/n<li>In 1969, <a href="/wiki/Tony_Hoare" title="Tony Hoare">Tony Hoare</a> introduces the <a href="/wiki/Hoare_logic" title="Hoare logic">Hoare logic</a>, a form of <a href="/wiki/Axiomatic_semantics" title="Axiomatic semantics">axiomatic semantics</a>.</li>/n<li>In 1969, <a href="/wiki/William_Alvin_Howard" title="William Alvin Howard">William Alvin Howard</a> observed that a "high-level" <a href="/wiki/Proof_calculus" title="Proof calculus">proof system</a>, referred to as <a href="/wiki/Natural_deduction" title="Natural deduction">natural deduction</a>, can be directly interpreted in its <a href="/wiki/Intuitionistic" class="mw-redirect" title="Intuitionistic">intuitionistic</a> version as a typed variant of the <a href="/wiki/Model_of_computation" title="Model of computation">model of computation</a> known as <a href="/wiki/Lambda_calculus" title="Lambda calculus">lambda calculus</a>. This became known as the <a href="/wiki/Curry%E2%80%93Howard_correspondence" title="Curry–Howard correspondence">Curry–Howard correspondence</a>.</li>/n</ul>/n<h3><span class="mw-headline" id="1970s">1970s</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Programming_language_theory&amp;action=edit&amp;section=4" title="Edit section: 1970s">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<ul>/n<li>In 1970, <a href="/wiki/Dana_Scott" title="Dana Scott">Dana Scott</a> first publishes his work on <a href="/wiki/Denotational_semantics" title="Denotational semantics">denotational semantics</a>.</li>/n<li>In 1972, <a href="/wiki/Logic_programming" title="Logic programming">Logic programming</a> and <a href="/wiki/Prolog" title="Prolog">Prolog</a> were developed thus allowing computer programs to be expressed as mathematical logic.</li>/n<li>In 1974, <a href="/wiki/John_C._Reynolds" title="John C. Reynolds">John C. Reynolds</a> discovers <a href="/wiki/System_F" title="System F">System F</a>. It had already been discovered in 1971 by the mathematical logician <a href="/wiki/Jean-Yves_Girard" title="Jean-Yves Girard">Jean-Yves Girard</a>.</li>/n<li>From 1975, Sussman and Steele develop the <a href="/wiki/Scheme_(programming_language)" title="Scheme (programming language)">Scheme programming language</a>, a Lisp dialect incorporating <a href="/wiki/Lexical_scoping" class="mw-redirect" title="Lexical scoping">lexical scoping</a>, a unified namespace, and elements from the <a href="/wiki/Actor_model" title="Actor model">Actor model</a> including first-class <a href="/wiki/Continuation" title="Continuation">continuations</a>.</li>/n<li>Backus, at the 1977 <a href="/wiki/ACM_Turing_Award" class="mw-redirect" title="ACM Turing Award">ACM Turing Award</a> lecture, assailed the current state of industrial languages and proposed a new class of programming languages now known as <a href="/wiki/Function-level_programming" title="Function-level Three common approaches to describe the semantics or "meaning" of a computer program are <a href="/wiki/Denotational_semantics" title="Denotational semantics">denotational semantics</a>, <a href="/wiki/Operational_semantics" title="Operational semantics">operational semantics</a> and <a href="/wiki/Axiomatic_semantics" title="Axiomatic semantics">axiomatic semantics</a>.</p>/n<h3><span class="mw-headline" id="Type_theory">Type theory</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Programming_language_theory&amp;action=edit&amp;section=9" title="Edit section: Type theory">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Type_theory" title="Type theory">Type theory</a></div>/n<p>Type theory is the study of <a href="/wiki/Type_system" title="Type system">type systems</a>;
