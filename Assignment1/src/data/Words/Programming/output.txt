style="text-align:right;font-size:115%">/n<div class="plainlinks hlist navbar mini">/n<ul>/n<li class="nv-view"><a href="/wiki/Template:OS" title="Template:OS"><abbr title="View this text" href="//en.wikipedia.org/w/index.php?title=Template:OS&amp;action=edit"><abbr title="Edit this template">e</abbr></a></li>/n</ul>/n</div>/n</td>/n</tr>/n</table>/n<p>In <a href="/wiki/Computing" title="Computing">computing</a>, <b>input/output</b> or <b>I/O</b> (or, informally, <b>io</b> or <b>IO</b>) is the communication between an <a href="/wiki/Information_processing_system" class="mw-redirect" title="Information processing system">information processing system</a>, such as a <a href="/wiki/Computer" title="Computer">computer</a>, and the outside world, possibly a human or another information processing system. <a href="/wiki/Information" title="Information">Inputs</a> are the signals or data received by the system and outputs are the signals or <a href="/wiki/Data_(computing)" title="Data (computing)">data</a> sent from it. The term can also be used as part of an action; to "perform I/O" is to perform an <a href="/wiki/I/O_scheduling" title="I/O scheduling">input or output operation</a>.</p>/n<p><b><span id="IO_DEVICE"></span><span id="I.2FO_devices">I/O devices</span></b> are the pieces of <a href="/wiki/Hardware_(computing)" class="mw-redirect" title="Hardware (computing)">hardware</a> used by a human (or other system) to communicate with a computer. For instance, a <a href="/wiki/Computer_keyboard" title="Computer keyboard">keyboard</a> or <a href="/wiki/Computer_mouse" title="Computer mouse">computer mouse</a> is an input device for a computer, while <a href="/wiki/Computer_monitor" title="Computer monitor">monitors</a> and <a href="/wiki/Computer_printer" class="mw-redirect" title="Computer printer">printers</a> are output devices. Devices for communication between computers, such as <a href="/wiki/Modem" title="Modem">modems</a> and <a href="/wiki/Network_card" class="mw-redirect" title="Network card">network cards</a>, typically perform both input and output operations.</p>/n<p>The designation of a device as either input or output depends on perspective. Mouse and keyboards take physical movements that the human user outputs and convert them into input signals that a computer can understand; the output from these devices is the computer's input. Similarly, printers and monitors take signals that a computer outputs as input, and they convert these signals into a representation that human users can understand. From the human <a href="/wiki/User_(computing)" title="User (computing)">user</a>'s perspective, the process of reading or seeing these representations is receiving input; this type of interaction between computers and humans is studied in the field of <a href="/wiki/Human%E2%80%93computer_interaction" title="Human–computer interaction">human–computer interaction</a>.</p>/n<p>In computer architecture, the combination of the <a href="/wiki/Central_processing_unit" title="Central processing unit">CPU</a> and <a href="/wiki/Main_memory" class="mw-redirect" title="Main memory">main memory</a>, to which the CPU can read or write directly using individual <a href="/wiki/Instruction_(computer_science)" class="mw-redirect" title="Instruction (computer science)">instructions</a>, is considered the brain of a computer. Any transfer of information to or from the CPU/memory combo, for example by reading data from a <a href="/wiki/Disk_drive" class="mw-redirect" title="Disk drive">disk drive</a>, is considered I/O.<sup id="cite_ref-teco_1-0" class="reference"><a href="#cite_note-teco-1">[1]</a></sup> The CPU and its supporting circuitry may provide <a href="/wiki/Memory-mapped_I/O" title="Memory-mapped I/O">memory-mapped I/O</a> that is used in low-level <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a>, such as in the implementation of <a href="/wiki/Device_driver" title="Device driver">device drivers</a>, or may provide access to <a href="/wiki/Channel_I/O" title="Channel I/O">I/O channels</a>. An <a href="/wiki/External_memory_algorithm" class="mw-redirect" title="External memory algorithm">I/O algorithm</a> is one designed to exploit locality and perform efficiently when exchanging data with a secondary storage device, such as a disk drive.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#Interface"><span class="tocnumber">1</span> <span class="toctext">Interface</span></a>/n<ul>/n<li class="toclevel-2 tocsection-2"><a href="#Higher-level_implementation"><span class="tocnumber">1.1</span> <span class="toctext">Higher-level implementation</span></a></li>/n</ul>/n</li>/n<li class="toclevel-1 tocsection-3"><a href="#Channel_I.2FO"><span class="tocnumber">2</span> <span class="toctext">Channel I/O</span></a></li>/n<li class="toclevel-1 tocsection-4"><a href="#Port-mapped_I.2FO"><span class="tocnumber">3</span> <span class="toctext">Port-mapped I/O</span></a></li>/n<li class="toclevel-1 tocsection-5"><a href="#See_also"><span class="tocnumber">4</span> <span class="toctext">See also</span></a></li>/n<li class="toclevel-1 tocsection-6"><a href="#References"><span class="tocnumber">5</span> <span class="toctext">References</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="Interface">Interface</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Input/output&amp;action=edit&amp;section=1" title="Edit section: Interface">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>An I/O interface is required whenever the I/O device is driven by the processor. Typically a CPU communicates with devices via a <a href="/wiki/Bus_(computing)" title="Bus (computing)">bus</a>. The interface must have necessary logic to interpret the device address generated by the processor. <a href="/wiki/Handshaking" title="Handshaking">Handshaking</a> should be implemented by the interface using appropriate commands (like BUSY, READY, and WAIT), and the processor can communicate with an I/O device through the interface. If different data formats are being exchanged, the interface must be able to convert serial data to parallel form and vice versa. Because it would be a waste for a processor to be idle while it waits for data from an input device there must be provision for generating <a href="/wiki/Interrupt" title="Interrupt">interrupts</a><sup id="cite_ref-fcoa_2-0" class="reference"><a href="#cite_note-fcoa-2">[2]</a></sup> and the corresponding type numbers for further processing by the processor if required.<sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="meaning unclear (December 2016)">clarification needed</span></a></i>]</sup></p>/n<p>A computer that uses <a href="/wiki/Memory-mapped_I/O" title="Memory-mapped I/O">memory-mapped I/O</a> accesses hardware by reading and writing to specific memory locations, using the same assembly language instructions that computer would normally use to access memory. An alternative method is via instruction-based I/O which requires that a CPU have specialised instructions for I/O.<sup id="cite_ref-teco_1-1" class="reference"><a href="#cite_note-teco-1">[1]</a></sup> Both input and output devices have a <a href="/wiki/Data_processing" title="Data processing">data processing</a> rate that can vary greatly.<sup id="cite_ref-fcoa_2-1" class="reference"><a href="#cite_note-fcoa-2">[2]</a></sup> With some devices able to exchange data at very high speeds <a href="/wiki/Direct_memory_access" title="Direct memory access">direct access</a> to memory (DMA) without the continuous aid of a CPU is required.<sup id="cite_ref-fcoa_2-2" class="reference"><a href="#cite_note-fcoa-2">[2]</a></sup></p>/n<h3><span class="mw-headline" id="Higher-level_implementation">Higher-level implementation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Input/output&amp;action=edit&amp;section=2" title="Edit section: Higher-level implementation">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Higher-level <a href="/wiki/Operating_system" title="Operating system">operating system</a> and programming facilities employ separate, more abstract I/O concepts and <a href="/wiki/Primitive_(computer_science)" class="mw-redirect" title="Primitive (computer science)">primitives</a>. For example, most operating systems provide application programs with the concept of <a href="/wiki/Computer_file" title="Computer file">files</a>. The <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a> and <a href="/wiki/C%2B%2B" title="C++">C++</a> programming languages, and operating systems in the <a href="/wiki/Unix" title="Unix">Unix</a> family, traditionally abstract files and devices as <a href="/wiki/Stream_(computing)" title="Stream (computing)">streams</a><a href="/wiki/Stream_(computing)" title="Stream (computing)"></a>, which can be read or written, or sometimes both. The <a href="/wiki/C_standard_library" title="C standard library">C standard library</a> provides <a href="/wiki/C_file_input/output" title="C file input/output">functions for manipulating streams</a> for input and output.</p>/n<p>In the context of the <a href="/wiki/ALGOL_68" title="ALGOL 68">ALGOL 68</a> programming language, the <i>input</i> and <i>output</i> facilities are collectively referred to as <i>transput</i>. The <i>ALGOL 68</i> transput library recognizes the following standard files/devices: <code>stand in</code>, <code>stand out</code>, <code>stand errors</code> and <code>stand back</code>.</p>/n<p>An alternative to special primitive functions is the <a href="/wiki/I/O_monad" class="mw-redirect" title="I/O monad">I/O monad</a>, which permits programs to just describe I/O, and the actions are carried out outside the program. This is notable because the <span class="nowrap">I/O</span> functions would introduce <a href="/wiki/Side-effect_(computer_science)" class="mw-redirect" title="Side-effect (computer science)">side-effects</a> to any programming language, but this allows <a href="/wiki/Purely_functional_programming" title="Purely functional programming">purely functional programming</a> to be practical.</p>/n<h2><span id="Channel_I/O"></span><span class="mw-headline" id="Channel_I.2FO">Channel I/O</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Input/output&amp;action=edit&amp;section=3" title="Edit section: Channel I/O">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p><a href="/wiki/Channel_I/O" title="Channel I/O">Channel I/O</a> requires the use of instructions that are specifically designed to perform I/O operations. The I/O instructions address the channel or the channel and device; the channel asynchronously accesses all other required addressing and control information. This is similar to DMA, but more flexible.</p>/n<h2><span id="Port-mapped_I/O"></span><span class="mw-headline" id="Port-mapped_I.2FO">Port-mapped I/O</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Input/output&amp;action=edit&amp;section=4" title="Edit section: Port-mapped I/O">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p><a href="/wiki/Memory-mapped_I/O" title="Memory-mapped I/O">Port-mapped I/O</a> also requires the use of special I/O instructions. Typically one or more ports are assigned to the device, each with a special purpose. The port numbers are in a separate address space from that used by normal instructions.</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span
