lang="en">Fourth-generation programming language</h1>/n									<div id="bodyContent" class="mw-body-content">/n									<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>/n								<div id="contentSub"></div>/n												<div id="jump-to-nav" class="mw-jump">/n					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>/n				</div>/n				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p>A <b>fourth-generation programming language</b> (<b>4GL</b>) is any <a href="/wiki/Computer" title="Computer">computer</a> <a href="/wiki/Programming_language" title="Programming language">programming language</a> that belongs to a class of languages envisioned as an advancement upon <a href="/wiki/Third-generation_programming_language" title="Third-generation programming language">third-generation programming languages</a> (3GL). Each of the <a href="/wiki/Programming_language_generations" title="Programming language generations">programming language generations</a> aims to provide a higher level of <a href="/wiki/Abstraction_layer" title="Abstraction layer">abstraction</a> of the internal <a href="/wiki/Computer_hardware" title="Computer hardware">computer hardware</a> details, making the language more <a href="/wiki/Programmer" title="Programmer">programmer</a>-friendly, powerful and versatile. While the definition of 4GL has changed over time, it can be typified by operating more with large collections of information at once rather than focusing on just <a href="/wiki/Bit" title="Bit">bits</a> and <a href="/wiki/Byte" title="Byte">bytes</a>. Languages claimed to be 4GL may include support for <a href="/wiki/Database" title="Database">database</a> management, <a href="/wiki/Report_generator" title="Report generator">report generation</a>, <a href="/wiki/Mathematical_optimization" title="Mathematical optimization">mathematical optimization</a>, <a href="/wiki/Graphical_user_interface_builder" title="Graphical user interface builder">GUI development</a>, or <a href="/wiki/Web_development" title="Web development">web development</a>. Some researchers state that 4GLs are a subset of <a href="/wiki/Domain-specific_language" title="Domain-specific language">domain-specific languages</a>.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup><sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup></p>/n<p>The concept of 4GL was developed from the 1970s through the 1990s, overlapping most of the development of 3GL. While 3GLs like <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, <a href="/wiki/C%2B%2B" title="C++">C++</a>, <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a>, <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, and <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> remain popular for a wide variety of uses, 4GLs as originally defined found narrower uses.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (September 2015)">citation needed</span></a></i>]</sup> Some advanced 3GLs like <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a>, and <a href="/wiki/Perl" title="Perl">Perl</a> combine some 4GL abilities within a general-purpose 3GL environment. Also, <a href="/wiki/Library_(computing)" title="Library (computing)">libraries</a> with 4GL-like features have been developed as add-ons for most popular 3GLs. This has blurred the distinction of 4GL and 3GL.</p>/n<p>In the 1980s and 1990s, there were efforts to develop <a href="/wiki/Fifth-generation_programming_language" title="Fifth-generation programming language">fifth-generation programming languages</a> (5GL).</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#History"><span class="tocnumber">1</span> <span class="toctext">History</span></a></li>/n<li class="toclevel-1 tocsection-2"><a href="#Types"><span class="tocnumber">2</span> <span class="toctext">Types</span></a></li>/n<li class="toclevel-1 tocsection-3"><a href="#Examples"><span class="tocnumber">3</span> <span class="toctext">Examples</span></a></li>/n<li class="toclevel-1 tocsection-4"><a href="#See_also"><span class="tocnumber">4</span> <span class="toctext">See also</span></a></li>/n<li class="toclevel-1 tocsection-5"><a href="#References"><span class="tocnumber">5</span> <span class="toctext">References</span></a></li>/n<li class="toclevel-1 tocsection-6"><a href="#External_links"><span class="tocnumber">6</span> <span class="toctext">External links</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Fourth-generation_programming_language&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Though used earlier in papers and discussions, the term 4GL was first used formally by <a href="/wiki/James_Martin_(author)" title="James Martin (author)">James Martin</a> in his 1981 book <i>Applications Development Without Programmers</i><sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> to refer to non-procedural, high-level <a href="/wiki/Specification_language" title="Specification language">specification languages</a>. In some primitive way, early 4GLs were included in the <a href="/wiki/Sterling_Software" title="Sterling Software">Informatics</a> <a href="/wiki/MARK-IV_(Software)" class="mw-redirect" title="MARK-IV (Software)">MARK-IV</a> (1967) product and <a href="/wiki/Sperry_Corporation" title="Sperry Corporation">Sperry</a>'s <a href="/wiki/MAPPER_(Software)" class="mw-redirect" title="MAPPER (Software)">MAPPER</a> (1969 internal use, 1979 release).</p>/n<p>The motivations for the '4GL' inception and continued interest are several. The term can apply to a large set of software products. It can also apply to an approach that looks for greater <a href="/wiki/Semantic" class="mw-redirect" title="Semantic">semantic</a> properties and implementation power. Just as the 3GL offered greater power to the programmer, so too did the 4GL open up the development environment to a wider population.</p>/n<p>In a sense, the 4GL is an example of '<a href="/wiki/Black_box" title="Black box">black box</a>' processing, each generation (in the sense of the page) is further from the machine (see the <a href="/wiki/Computer_Science" class="mw-redirect" title="Computer Science">Computer Science</a> history in regard to data structure improvements and <a href="/wiki/Information_hiding" title="Information hiding">information hiding</a>). It is this latter nature that is directly associated with 4GL having errors that are harder, in many cases, to debug. In terms of applications, a 4GL could be business oriented or it could deal with some technical domain. Being further from the machine implies being closer to domain. Given the wide disparity of concepts and methods across domains, 4GL limitations lead to recognition of the need for the <a href="/wiki/Fifth-generation_programming_language" title="Fifth-generation programming language">5GL</a>.<sup class="noprint Inline-Template" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:No_original_research" title="Wikipedia:No original research"><span title="The material near this tag possibly contains original research. (May 2008)">original research?</span></a></i>]</sup></p>/n<p>The early input scheme for the 4GL supported entry of data within the <a href="/wiki/Punched_card#IBM_80_column_punched_card_format" title="Punched card">72-character limit of the punched card</a> (8 bytes used for sequencing) where a card's tag would identify the type or function. With judicious use of a few cards, the <a href="/wiki/Computer_programming_in_the_punch_card_era" class="mw-redirect" title="Computer programming in the punch card era">4GL deck</a> could offer a wide variety of processing and reporting capability whereas the equivalent functionality coded in a <a href="/wiki/Third-generation_programming_language" title="Third-generation programming language">3GL</a> could subsume, perhaps, a whole box or more of <a href="/wiki/Computer_programming_in_the_punch_card_era" class="mw-redirect" title="Computer programming in the punch card era">cards</a>.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup></p>/n<p>The 72-character metaphor <a href="/wiki/Characters_per_line" title="Characters per line">continued for a while</a> as hardware progressed to larger memory and terminal interfaces. Even with its limitations, this approach supported highly sophisticated applications.</p>/n<p>As interfaces improved and allowed longer statement lengths and grammar-driven input handling, greater power ensued. An example of this is described on the <a href="/wiki/Nomad_software" title="Nomad software">Nomad</a> page.</p>/n<dl>/n<dd>/n<dl>/n<dd>Another example of Nomad's power is illustrated by Nicholas Rawlings in his comments for the Computer History Museum about NCSS (see citation below). He reports that <a href="/wiki/James_Martin_(author)" title="James Martin (author)">James Martin</a> asked Rawlings for a Nomad solution to a standard problem Martin called the <i>Engineer's Problem</i>: "give 6% raises to engineers whose job ratings had an average of 7 or better." Martin provided a "dozen pages of COBOL, and then just a page or two of <a href="/wiki/MARK-IV_(Software)" class="mw-redirect" title="MARK-IV (Software)">Mark IV</a>, from <a href="/wiki/Sterling_Software" title="Sterling Software">Informatics</a>." Rawlings offered the following single statement, performing a set-at-a-time operation...</dd>/n</dl>/n</dd>/n</dl>/n<p>The development of the 4GL was influenced by several factors, with the hardware and operating system constraints having a large weight. When the 4GL was first introduced, a disparate mix of hardware and operating systems mandated custom application development support that was specific to the system in order to ensure sales. One example is the <a href="/wiki/MAPPER_(Software)" class="mw-redirect" title="MAPPER (Software)">MAPPER</a> system developed by <a href="/wiki/Sperry_Corporation" title="Sperry Corporation">Sperry</a>. Though it has roots back to the beginning, the system has proven successful in many applications and has been ported to modern platforms. The latest variant is embedded in the BIS<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup> offering of <a href="/wiki/Unisys" title="Unisys">Unisys</a>. <a href="/wiki/MARK-IV_(Software)" class="mw-redirect" title="MARK-IV (Software)">MARK-IV</a> is now known as VISION:BUILDER and is offered by <a href="/wiki/Computer_Associates" class="mw-redirect" title="Computer Associates">Computer Associates</a>.</p>/n<p><a href="/wiki/Atchison,_Topeka_and_Santa_Fe_Railway" title="Atchison, Topeka and Santa Fe Railway">Santa Fe</a> railroad used <a href="/wiki/MAPPER_(Software)" class="mw-redirect" title="MAPPER (Software)">MAPPER</a> to develop a system, in a project that was an early example of 4GL, <a href="/wiki/Software_prototyping" title="Software prototyping">rapid prototyping</a>, and <a href="/wiki/End-user_computing" title="End-user computing">programming by users</a>.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup> The idea was that it was easier to teach railroad experts to use <a href="/wiki/MAPPER_(Software)" class="mw-redirect" title="MAPPER (Software)">MAPPER</a> than to teach programmers the "intricacies of railroad operations".<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup></p>/n<p>One of the early (and portable) languages that had 4GL properties was <a href="/wiki/Ramis_software" class="mw-redirect" title="Ramis software">Ramis</a> developed by Gerald C. Cohen at Mathematica, a mathematical software company. Cohen left Mathematica and founded Information Builders to create a similar reporting-oriented 4GL, called <a href="/wiki/FOCUS" title="FOCUS">FOCUS</a>.</p>/n<p>Later 4GL types are tied to a database system and are far different from the earlier types in their use of techniques and resources that have resulted from the general improvement of computing with time.</p>/n<p>An interesting twist to the 4GL scene is realization that graphical interfaces and the <a href="/wiki/End-user_computing" title="End-user computing">related reasoning</a> done by the user form a 'language' that is poorly understood.</p>/n<h2><span class="mw-headline" id="Types">Types</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Fourth-generation_programming_language&amp;action=edit&amp;section=2" title="Edit section: Types">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>A number of different types of 4GLs exist:</p>/n<ul>/n<li>Table-driven (codeless) programming, usually running with a runtime framework and libraries. Instead of using code, the developer defines their logic by selecting an operation in a pre-defined list of memory or data table manipulation commands. In other words, instead of coding, the developer uses table-driven algorithm programming (see also <a href="/wiki/Control_table" title="Control table">control tables</a> that can be used for this purpose). A good example of this type of 4GL language is <a href="/wiki/PowerBuilder" title="PowerBuilder">PowerBuilder</a>. These types of tools can be used for business application development usually consisting in a package allowing for both business data manipulation and reporting, therefore they come with GUI screens and report editors. They usually offer integration with lower level DLLs generated from a typical 3GL for when the need arise for more hardware/OS specific operations.</li>/n<li><a href="/wiki/Report_generator" title="Report generator">Report-generator programming languages</a> take a description of the data format and the report to generate and from that they either generate the required report directly or they generate a program to generate the report. See also <a href="/wiki/IBM_RPG" title="IBM RPG">RPG</a></li>/n<li>Similarly, forms generators manage online interactions with the application system users or generate programs to do so.</li>/n<li>More ambitious 4GLs (sometimes termed <i>fourth generation environments</i>) attempt to automatically generate whole systems from the outputs of <a href="/wiki/Computer-Aided_Software_Engineering" class="mw-redirect" title="Computer-Aided Software Engineering">CASE</a> tools, specifications of screens and reports, and possibly also the specification of some additional processing logic.</li>/n<li><a href="/wiki/Data_management" title="Data management">Data management</a> 4GLs such as <a href="/wiki/SAS_System" class="mw-redirect" title="SAS System">SAS</a>, <a href="/wiki/SPSS" title="SPSS">SPSS</a> and <a href="/wiki/Stata" title="Stata">Stata</a> provide sophisticated coding <a href="/wiki/Command_(computing)" title="Command (computing)">commands</a> for data manipulation, file reshaping, case selection and data documentation in the preparation of data for <a href="/wiki/Statistical_analysis" class="mw-redirect" title="Statistical analysis">statistical analysis</a> and reporting.</li>/n</ul>/n<p>Some 4GLs have integrated tools which allow for the easy specification of all the required information:</p>/n<ul>/n<li>James Martin's version of <i>Information Engineering</i> <a href="/wiki/Systems_development_methodology" class="mw-redirect" title="Systems development methodology">systems development methodology</a> was automated to allow the input of the results of system analysis and design in the form of <a href="/wiki/Data_flow_diagram" title="Data flow diagram">data flow diagrams</a>, <a href="/wiki/Entity_relationship_diagram" class="mw-redirect" title="Entity relationship diagram">entity relationship diagrams</a>, <a href="/wiki/Entity_life_history_diagram" class="mw-redirect" title="Entity life history diagram">entity life history diagrams</a> etc. from which hundreds of thousands of lines of <a href="/wiki/COBOL" title="COBOL">COBOL</a> would be generated overnight.</li>/n<li>More recently <a href="/wiki/Oracle_Corporation" title="Oracle Corporation">Oracle Corporation</a>'s <a href="/wiki/Oracle_Designer" title="Oracle Designer">Oracle Designer</a> and <a href="/wiki/Oracle_Developer_Suite" title="Oracle Developer Suite">Oracle Developer Suite</a> 4GL products could be integrated to produce database definitions and the forms and reports programs.</li>/n</ul>/n<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a
