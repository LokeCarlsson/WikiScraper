--></div>/n						<div class="mw-indicators mw-body-content">/n</div>/n			<h1 id="firstHeading" class="firstHeading" lang="en">Atlas Autocode</h1>/n									<div id="bodyContent" class="mw-body-content">/n									<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>/n								<div id="contentSub"></div>/n												<div id="jump-to-nav" class="mw-jump">/n					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>/n				</div>/n				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p><b>Atlas Autocode</b> (AA)<sup id="cite_ref-AA_1-0" class="reference"><a href="#cite_note-AA-1">[1]</a></sup><sup id="cite_ref-AA_ED_2-0" class="reference"><a href="#cite_note-AA_ED-2">[2]</a></sup> was a <a href="/wiki/Programming_language" title="Programming language">programming language</a> developed around 1965 at <a href="/wiki/Manchester_University" class="mw-redirect" title="Manchester University">Manchester University</a>. A variant of the <a href="/wiki/ALGOL" title="ALGOL">ALGOL</a> <a href="/wiki/Programming_language" title="Programming language">programming language</a>, it was developed by <a href="/wiki/Tony_Brooker" title="Tony Brooker">Tony Brooker</a> and Derrick Morris for the <a href="/wiki/Atlas_Computer" class="mw-redirect" title="Atlas Computer">Atlas Computer</a>.</p>/n<p>("Autocode" was basically an early term for "programming language"; different autocodes could be totally different).</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#Features"><span class="tocnumber">1</span> <span class="toctext">Features</span></a></li>/n<li class="toclevel-1 tocsection-2"><a href="#Syntax"><span class="tocnumber">2</span> <span class="toctext">Syntax</span></a></li>/n<li class="toclevel-1 tocsection-3"><a href="#References"><span class="tocnumber">3</span> <span class="toctext">References</span></a></li>/n<li class="toclevel-1 tocsection-4"><a href="#External_links"><span class="tocnumber">4</span> <span class="toctext">External links</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="Features">Features</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Atlas_Autocode&amp;action=edit&amp;section=1" title="Edit section: Features">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>It removed some Algol features such as "<a href="/wiki/Call_by_name#Call_by_name" class="mw-redirect" title="Call by name">passing parameters by name</a>" (which in Algol 60 means passing the address of a short subroutine to recalculate the parameter each time it was mentioned).</p>/n<p>It featured explicitly typed variables, subroutines, and functions.</p>/n<p>The AA compiler generated range-checking for array accesses, and allowed an array to have dimensions that were determined at run-time (i.e. you could declare an array as <code><u>integer</u> <u>array</u> Thing (i:j)</code>, where <code>i</code> and <code>j</code> were calculated values).</p>/n<p><a href="/wiki/Machine_code" title="Machine code">Machine code</a> could be included within the high-level AA routines either to make an inner loop more efficient or to effect some operation which could not easily be done otherwise.<sup id="cite_ref-AA_1-1" class="reference"><a href="#cite_note-AA-1">[1]</a></sup></p>/n<p>Atlas Autocode included a <code><u>complex</u></code> data type<sup id="cite_ref-AA_1-2" class="reference"><a href="#cite_note-AA-1">[1]</a></sup> to represent <a href="/wiki/Complex_number" title="Complex number">complex numbers</a>, partly because of pressure from the <a href="/wiki/Electrical_engineering" title="Electrical engineering">electrical engineering</a> department, as complex numbers are used to represent the behavior of <a href="/wiki/Alternating_current" title="Alternating current">alternating current</a>. The square root of -1 was represented by <code><u>i</u></code>, which was treated as a fixed complex constant = <i>i</i>.</p>/n<p>The <code><u>complex</u></code> data type was dropped when Atlas Autocode later evolved into the <a href="/wiki/Edinburgh_IMP" title="Edinburgh IMP">Edinburgh IMP</a> programming language. (Imp was an extension of AA and was used to write the <a href="/wiki/Edinburgh_Multiple_Access_System" title="Edinburgh Multiple Access System">EMAS</a> <a href="/wiki/Operating_system" title="Operating system">operating system</a>.)</p>/n<p>Atlas Autocode's second-greatest claim to fame (after being the progenitor of Imp and EMAS) was that it had many of the features of the original "<a href="/wiki/Compiler-compiler" title="Compiler-compiler">Compiler Compiler</a>". A variant of the AA compiler included run-time support for a top-down <a href="/wiki/Recursive_descent_parser" title="Recursive descent parser">recursive descent parser</a>. The style of parser used in the Compiler Compiler was in use continuously at Edinburgh from the 60's until almost the turn of the millennium.</p>/n<p>Other <a href="/wiki/Autocode" title="Autocode">Autocodes</a> were developed for the <a href="/wiki/Titan_(computer)" title="Titan (computer)">Titan computer</a>, a prototype Atlas 2 at Cambridge, and the <a href="/wiki/Ferranti" title="Ferranti">Ferranti</a> Mercury.</p>/n<h2><span class="mw-headline" id="Syntax">Syntax</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Atlas_Autocode&amp;action=edit&amp;section=2" title="Edit section: Syntax">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Atlas Autocode's syntax was largely similar to Algol, though it was influenced by the output device which the author had available, a <a href="/wiki/Friden,_Inc." title="Friden, Inc.">Friden Flexowriter</a>. Consequently, it allowed symbols like "½" for ".5" and the superscript <sup>2</sup> for "to the power of 2". The Flexowriter supported overstriking and therefore AA did as well — up to three characters could be overstruck as a single symbol. For example, the character set had no "↑" symbol, so exponentiation was an overstrike of "|" and "*". The aforementioned underlining of keywords could also be done using overstriking. The language is described in detail in the <a rel="nofollow" class="external text" href="http://history.dcs.ed.ac.uk/archive/docs/atlasautocode.html">Atlas Autocode Reference Manual</a>.</p>/n<p>Other Flexowriter characters that were found a use in Atlas Autocode were: <code>α</code> in floating-point numbers, <i>e.g.</i>, <code>3.56α-7</code> for modern <code>3.56e-7</code>&#160;; <code>β</code> to mean "the second half of a 48-bit Atlas memory <a href="/wiki/Word_(data_type)" class="mw-redirect" title="Word (data type)">word</a>"; <code>π</code> for the mathematical constant <a href="/wiki/Pi" title="Pi">pi</a>.</p>/n<p>When AA was ported to the <a href="/wiki/English_Electric_KDF9" title="English Electric KDF9">English Electric KDF9</a> computer, the character set was changed to <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a> and that compiler has been recovered from an old paper tape by the <a rel="nofollow" class="external text" href="http://history.dcs.ed.ac.uk/">Edinburgh Computer History Project</a> and is available online, as is a high-quality scan of the original Edinburgh version of the Atlas Autocode manual.<sup id="cite_ref-AA_ED_2-1" class="reference"><a href="#cite_note-AA_ED-2">[2]</a></sup></p>/n<p>Keywords in AA were distinguishable from other text by being underlined, which was implemented via overstrike in the Flexowriter (compare to bold in Algol). There were also two <a href="/wiki/Stropping_(syntax)" title="Stropping (syntax)">stropping</a> regimes. First, there was an "uppercasedelimiters" mode where all uppercase letters (outside strings) were treated as underlined lowercase. Second, in some versions (but not in the original Atlas version), it was possible to strop keywords by placing a "<code>%</code>" sign in front of them, for example the keyword <code><u>endofprogramme</u></code> could be typed as <code>%end&#160;%of&#160;%programme</code> or <code>%endofprogramme</code>. This significantly reduced typing, due to only needing one character, rather than overstriking the whole keyword. As in Algol, there were no <a href="/wiki/Reserved_word" title="Reserved word">reserved words</a> in the language as keywords were identified by underlining (or stropping), not by recognising reserved character sequences. In the statement <code><u>if</u> token=if <u>then</u> <u>result</u> = token</code>, there is both a keyword <code><u>if</u></code> and a variable named <code>if</code>.</p>/n<p>As in Algol, AA allowed spaces in variable names, such as <code><u>integer</u> previous value</code>. Spaces were not significant and were removed before parsing in a trivial pre-lexing stage called "<a href="/wiki/Line_reconstruction" class="mw-redirect" title="Line reconstruction">line reconstruction</a>". What the compiler would see in the above example would be "<code><u>if</u>token=if<u>thenresult</u>=token</code>". Spaces were possible due partly to keywords being distinguished in other ways, and partly because the source was processed by a <a href="/wiki/Scannerless_parser" class="mw-redirect" title="Scannerless parser">scannerless parser</a>, without a separate lexing phase, which allowed the lexical syntax to be context-sensitive.</p>/n<p>The syntax for expressions let the multiplication operator be omitted, e.g. <code>3a</code> was treated as <code>3*a</code>, and <code>a(i+j)</code> was treated as <code>a*(i+j)</code> if <code>a</code> was not an array. In ambiguous usages, the longest possible name was taken (<a href="/wiki/Maximal_munch" title="Maximal munch">maximal munch</a>), for example <code>ab</code> was not treated as <code>a*b</code>, whether or not <code>a</code> and <code>b</code> had been declared.</p>/n<p>In the original Atlas Autocode for the Atlas computer, Atlas machine code instructions could be interpolated between the Atlas Autocode statements.</p>/n<h2><span class="mw-headline" id="References">References</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Atlas_Autocode&amp;action=edit&amp;section=3" title="Edit section: References">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div class="reflist" style="list-style-type: decimal;">/n<div class="mw-references-wrap">/n<ol class="references">/n<li id="cite_note-AA-1"><span class="mw-cite-backlink">^ <a href="#cite_ref-AA_1-0"><sup><i><b>a</b></i></sup></a> <a href="#cite_ref-AA_1-1"><sup><i><b>b</b></i></sup></a> <a href="#cite_ref-AA_1-2"><sup><i><b>c</b></i></sup></a></span> <span class="reference-text">R.A.
