class="mw-indicators mw-body-content">/n</div>/n			<h1 id="firstHeading" class="firstHeading" lang="en">UNCOL</h1>/n									<div id="bodyContent" class="mw-body-content">/n									<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>/n								<div id="contentSub"></div>/n												<div id="jump-to-nav" class="mw-jump">/n					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>/n				</div>/n				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p><b>UNCOL</b> (<b>Universal Computer Oriented Language</b>) was a proposed universal <a href="/wiki/Intermediate_language" class="mw-redirect" title="Intermediate language">intermediate language</a> for <a href="/wiki/Compiler" title="Compiler">compilers</a> introduced by <a href="/wiki/Melvin_Conway" title="Melvin Conway">Melvin E. Conway</a> in 1958. It was never fully specified or implemented; in many ways it was more a concept than a language.</p>/n<p>UNCOL was intended to make compilers economically available for each new <a href="/wiki/Instruction_set" class="mw-redirect" title="Instruction set">instruction set</a> <a href="/wiki/Computer_architecture" title="Computer architecture">architecture</a> and <a href="/wiki/Programming_language" title="Programming language">programming language</a>. Each machine architecture would require just one compiler back end, and each programming language would require one compiler front end. This was a very ambitious goal in 1961 because compiler technology was in its infancy, and little was standardized in computer hardware and software.</p>/n<p>The concept of such a universal intermediate language is old: the <a href="/wiki/SHARE_(computing)" title="SHARE (computing)">SHARE</a> report (1958) already says "[it has] been discussed by many independent persons as long ago as 1954." Macrakis (1993) summarizes its fate:</p>/n<blockquote class="templatequote">/n<p>UNCOL was an ambitious effort for the early 1960s. An attempt to solve the compiler-writing problem, it ultimately failed because language and compiler technology were not yet mature.</p>/n<p>In the 1970s, <a href="/wiki/Compiler-compiler" title="Compiler-compiler">compiler-compilers</a> ultimately contributed to solving the problem that UNCOL set itself: the economical production of compilers for new languages and new machines.</p>/n</blockquote>/n<p>UNCOL is sometimes used as a generic term for the idea of a universal intermediate language. The <a href="/wiki/Architecture_Neutral_Distribution_Format" title="Architecture Neutral Distribution Format">Architecture Neutral Distribution Format</a> is an example of an UNCOL in this sense.</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=UNCOL&amp;action=edit&amp;section=1" title="Edit section: See also">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<ul>/n<li><a href="/wiki/LLVM" title="LLVM">LLVM</a></li>/n<li><a href="/wiki/GIMPLE" class="mw-redirect" title="GIMPLE">GIMPLE</a></li>/n</ul>/n<h2><span class="mw-headline" id="References">References</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=UNCOL&amp;action=edit&amp;section=2" title="Edit section: References">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<ul>/n<li>Melvin E.
