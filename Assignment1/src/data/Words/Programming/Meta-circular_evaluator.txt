id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p>In <a href="/wiki/Computing" title="Computing">computing</a>, a <b>meta-circular evaluator</b> or <b>meta-circular interpreter</b> is an <a href="/wiki/Interpreter_(computing)" title="Interpreter (computing)">interpreter</a> which defines each feature of the interpreted language using a similar facility of the interpreter's host language. For example, interpreting a lambda application may be implemented using function application.<sup id="cite_ref-HOP_1-0" class="reference"><a href="#cite_note-HOP-1">[1]</a></sup> Meta-circular evaluation is most prominent in the context of <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a>.<sup id="cite_ref-HOP_1-1" class="reference"><a href="#cite_note-HOP-1">[1]</a></sup> A <b>self-interpreter</b> is a meta-circular interpreter where the interpreted language is nearly identical to the host language; the two terms are often used synonymously.<sup id="cite_ref-SICP_2-0" class="reference"><a href="#cite_note-SICP-2">[2]</a></sup></p>/n<p></p>/n<div id="toc" class="toc">/n<div class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta-circular_evaluator&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div role="note" class="hatnote navigation-not-searchable">See also: <a href="/wiki/History_of_compiler_construction" title="History of compiler construction">History of compiler construction</a></div>/n<p>The dissertation of <a href="/wiki/Corrado_B%C3%B6hm" title="Corrado Böhm">Corrado Böhm</a><sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> describes the design of a <a href="/wiki/Self-hosting" title="Self-hosting">self-hosting</a> compiler. <sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup> Due to the difficulty of compiling <a href="/wiki/Higher-order_function" title="Higher-order function">higher-order functions</a>, many languages were instead defined via interpreters, most prominently Lisp.<sup id="cite_ref-HOP_1-2" class="reference"><a href="#cite_note-HOP-1">[1]</a></sup><sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup> The term itself was coined by <a href="/wiki/John_C._Reynolds" title="John C. Reynolds">John C. Reynolds</a>,<sup id="cite_ref-HOP_1-3" class="reference"><a href="#cite_note-HOP-1">[1]</a></sup> and popularized through its use in the book <i><a href="/wiki/Structure_and_Interpretation_of_Computer_Programs" title="Structure and Interpretation of Computer Programs">Structure and Interpretation of Computer Programs</a></i>.<sup id="cite_ref-SICP_2-1" class="reference"><a href="#cite_note-SICP-2">[2]</a></sup><sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup></p>/n<h2><span class="mw-headline" id="Self-interpreters">Self-interpreters</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta-circular_evaluator&amp;action=edit&amp;section=2" title="Edit section: Self-interpreters">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>A self-interpreter is a meta-circular interpreter where the host language is also the language being interpreted.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup> A self-interpreter displays a <a href="/wiki/Utm_theorem" title="Utm theorem">universal function</a> for the language in question, and can be helpful in learning certain aspects of the language.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup> A self-interpreter will provide a circular, <a href="/wiki/Vacuous" class="mw-redirect" title="Vacuous">vacuous</a> definition of most language constructs and thus provides little insight into the interpreted language's semantics, for example <a href="/wiki/Evaluation_strategy" title="Evaluation strategy">evaluation strategy</a>. Addressing these issues produces the more general notion of a "definitional interpreter".<sup id="cite_ref-HOP_1-4" class="reference"><a href="#cite_note-HOP-1">[1]</a></sup></p>/n<h2><span class="mw-headline" id="Uses">Uses</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta-circular_evaluator&amp;action=edit&amp;section=3" title="Edit section: Uses">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>In combination with an existing language implementation, meta-circular interpreters provide a baseline system from which to extend a language, either upwards by adding more features or downwards by compiling away features rather than interpreting them.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[9]</a></sup> They are also useful for writing tools that are tightly integrated with the programming language, such as sophisticated debuggers.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (April 2017)">citation needed</span></a></i>]</sup> A language designed with a meta-circular implementation in mind is often more suited for building languages in general, even ones completely different from the host language.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (April 2017)">citation needed</span></a></i>]</sup></p>/n<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Meta-circular_evaluator&amp;action=edit&amp;section=4" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<table class="plainlinks metadata ambox ambox-content ambox-Refimprove" role="presentation">/n<tr>/n<td class="mbox-image">/n<div style="width:52px"><a href="/wiki/File:Question_book-new.svg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/50px-Question_book-new.svg.png" width="50" height="39" srcset="//upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/75px-Question_book-new.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/100px-Question_book-new.svg.png 2x" data-file-width="512" data-file-height="399" /></a></div>/n</td>/n<td class="mbox-text">/n<div class="mbox-text-span">This section <b>needs additional citations for <a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability">verification</a></b>. <span class="hide-when-compact">Please help <a class="external text" href="//en.wikipedia.org/w/index.php?title=Meta-circular_evaluator&amp;action=edit">improve this article</a> by <a href="/wiki/Help:Introduction_to_referencing_with_Wiki_Markup/1" title="Help:Introduction to referencing with Wiki Markup/1">adding citations to reliable sources</a>.
