Language Runtime</b> (<b>CLR</b>), the <a href="/wiki/Virtual_machine" title="Virtual machine">virtual machine</a> component of <a href="/wiki/Microsoft" title="Microsoft">Microsoft's</a> <a href="/wiki/.NET_framework" class="mw-redirect" title=".NET framework">.NET framework</a>, manages the execution of .NET programs. A process known as <a href="/wiki/Just-in-time_compilation" title="Just-in-time compilation">just-in-time compilation</a> converts compiled code into machine instructions which the computer's <a href="/wiki/CPU" class="mw-redirect" title="CPU">CPU</a> then executes.<sup id="cite_ref-msdn-clr_1-0" class="reference"><a href="#cite_note-msdn-clr-1">[1]</a></sup> The CLR provides additional services including <a href="/wiki/Memory_management" title="Memory management">memory management</a>, <a href="/wiki/Type_safety" title="Type safety">type safety</a>, <a href="/wiki/Exception_handling" title="Exception handling">exception handling</a>, <a href="/wiki/Garbage_collection_(computer_science)" title="Garbage collection (computer science)">garbage collection</a>, security and <a href="/wiki/Thread_management" class="mw-redirect" title="Thread management">thread management</a>. All programs written for the .NET framework, regardless of <a href="/wiki/Programming_language" title="Programming language">programming language</a>, are executed by the CLR. All versions of the .NET framework include CLR.</p>/n<p>CLR implements the <a href="/wiki/Virtual_Execution_System" title="Virtual Execution System">Virtual Execution System</a> (VES) as defined in the <a href="/wiki/Common_Language_Infrastructure" title="Common Language Infrastructure">Common Language Infrastructure</a> (CLI) standard, initially developed by Microsoft itself.
