Unsourced material may be challenged and removed.</span> <small><i>(August 2016)</i></small> <small class="hide-when-compact"><i>(<a href="/wiki/Help:Maintenance_template_removal" title="Help:Maintenance template removal">Learn how and when to remove this template message</a>)</i></small></div>/n</td>/n</tr>/n</table>/n<p><b>Pseudocode</b><sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> is an informal <a href="/wiki/High-level_programming_language" title="High-level programming language">high-level</a> description of the operating principle of a <a href="/wiki/Computer_program" title="Computer program">computer program</a> or other <a href="/wiki/Algorithm" title="Algorithm">algorithm</a>.</p>/n<p>It uses the structural conventions of a normal <a href="/wiki/Programming_language" title="Programming language">programming language</a>, but is intended for human reading rather than machine reading. Pseudocode typically omits details that are essential for machine understanding of the algorithm, such as <a href="/wiki/Variable_declaration" class="mw-redirect" title="Variable declaration">variable declarations</a>, system-specific code and some <a href="/wiki/Subroutine" title="Subroutine">subroutines</a>. The programming language is augmented with <a href="/wiki/Natural_language" title="Natural language">natural language</a> description details, where convenient, or with compact mathematical notation. The purpose of using pseudocode is that it is easier for people to understand than conventional programming language code, and that it is an efficient and environment-independent description of the key principles of an algorithm. It is commonly used in textbooks and <a href="/wiki/Scientific_publications" class="mw-redirect" title="Scientific publications">scientific publications</a> that are documenting various algorithms, and also in planning of computer program development, for sketching out the structure of the program before the actual coding takes place.</p>/n<p>No standard for pseudocode syntax exists, as a program in pseudocode is not an executable program. Pseudocode resembles, but should not be confused with, <a href="/wiki/Skeleton_(computer_programming)" title="Skeleton (computer programming)">skeleton programs</a> which can be <a href="/wiki/Compiler" title="Compiler">compiled</a> without errors. <a href="/wiki/Flowchart" title="Flowchart">Flowcharts</a>, <a href="/wiki/DRAKON" title="DRAKON">drakon-charts</a> and <a href="/wiki/Unified_Modeling_Language" title="Unified Modeling Language">Unified Modeling Language</a> (UML) charts can be thought of as a graphical alternative to pseudocode, but are more spacious on paper. Languages such as <span class="plainlinks"><a href="/wiki/HAGGIS" title="HAGGIS">HAGGIS</a></span> bridge the gap between pseudocode and code written on programming languages. Its main use is to introduce students to high level languages through use of this hybrid language.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Pseudocode&amp;action=edit&amp;section=1" title="Edit section: Application">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Textbooks and <a href="/wiki/Scientific_publication" class="mw-redirect" title="Scientific publication">scientific publications</a> related to <a href="/wiki/Computer_science" title="Computer science">computer science</a> and <a href="/wiki/Numerical_computation" class="mw-redirect" title="Numerical computation">numerical computation</a> often use pseudocode in description of algorithms, so that all programmers can understand them, even if they do not all know the same programming languages. In textbooks, there is usually an accompanying introduction explaining the particular conventions in use. The level of detail of the pseudo-code may in some cases approach that of formalized general-purpose languages.</p>/n<p>A <a href="/wiki/Programmer" title="Programmer">programmer</a> who needs to implement a specific algorithm, especially an unfamiliar one, will often start with a pseudocode description, and then "translate" that description into the target programming language and modify it to interact correctly with the rest of the program. Programmers may also start a project by sketching out the code in pseudocode on paper before writing it in its actual language, as a <a href="/wiki/Top-down_and_bottom-up_design" title="Top-down and bottom-up design">top-down</a> structuring approach, with a process of steps to be followed as a refinement.</p>/n<h2><span class="mw-headline" id="Syntax">Syntax</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Pseudocode&amp;action=edit&amp;section=2" title="Edit section: Syntax">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>As the name suggests, pseudocode generally does not actually obey the <a href="/wiki/Syntax" title="Syntax">syntax</a> rules of any particular language; there is no systematic standard form, although any particular writer will generally borrow style and syntax; for example, control structures from some conventional programming language. Popular syntax sources include <a href="/wiki/Fortran" title="Fortran">Fortran</a>, <a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a>, <a href="/wiki/BASIC" title="BASIC">BASIC</a>, <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, <a href="/wiki/C%2B%2B" title="C++">C++</a>, <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a>, and <a href="/wiki/ALGOL" title="ALGOL">ALGOL</a>. Variable declarations are typically omitted. Function calls and blocks of code, such as code contained within a loop, are often replaced by a one-line natural language sentence.</p>/n<p>Depending on the writer, pseudocode may therefore vary widely in style, from a near-exact imitation of a real programming language at one extreme, to a description approaching formatted prose at the other.</p>/n<p>This is an example of pseudocode (for the class="mw-editsection-bracket">]</span></span></h2>/n<h3><span class="mw-headline" id="Natural_language_grammar_in_programming_languages">Natural language grammar in programming languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Pseudocode&amp;action=edit&amp;section=7" title="Edit section: Natural language grammar in programming languages">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Various attempts to bring elements of natural language grammar into computer programming have produced programming languages such as <a href="/wiki/HyperTalk" title="HyperTalk">HyperTalk</a>, <a href="/wiki/Lingo_(programming_language)" title="Lingo (programming language)">Lingo</a>, <a href="/wiki/AppleScript" title="AppleScript">AppleScript</a>, <a href="/wiki/SQL" title="SQL">SQL</a>, <a href="/wiki/Inform" title="Inform">Inform</a> and to some extent <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>. In these languages, parentheses and other special characters are replaced by prepositions, resulting in quite talkative code. These languages are typically <a href="/wiki/Dynamic_typing" class="mw-redirect" title="Dynamic typing">dynamically typed</a>, meaning that variable declarations and other <a href="/wiki/Boilerplate_code" title="Boilerplate code">boilerplate code</a> can be omitted. Such languages may make it easier for a person without knowledge about the language to understand the code and perhaps also to learn the language. However, the similarity to natural language is usually more cosmetic than genuine. The syntax rules may be just as strict and formal as in conventional programming, and do not necessarily make development of the programs easier.</p>/n<h3><span class="mw-headline" id="Mathematical_programming_languages">Mathematical programming languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Pseudocode&amp;action=edit&amp;section=8" title="Edit section: Mathematical programming languages">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>An alternative to using mathematical pseudocode (involving set theory notation or matrix operations) for documentation of algorithms is to use a formal mathematical programming language that is a mix of non-ASCII mathematical notation and program control structures. Then the code can be parsed and interpreted by a machine.</p>/n<p>Several formal <a href="/wiki/Specification_language" title="Specification language">specification languages</a> include set theory notation using special characters. Examples are:</p>/n<ul>/n<li><a href="/wiki/Z_notation" title="Z notation">Z notation</a></li>/n<li><a href="/wiki/Vienna_Development_Method" title="Vienna Development Method">Vienna Development Method</a> Specification Language (VDM-SL).</li>/n</ul>/n<p>Some <a href="/wiki/Array_programming" title="Array programming">array programming</a> languages include vectorized expressions and matrix operations as non-ASCII formulas, mixed with conventional control structures. Examples are:</p>/n<ul>/n<li><a href="/wiki/APL_(programming_language)" title="APL (programming language)">A programming language</a> (APL), and its dialects <a href="/wiki/APLX" title="APLX">APLX</a> and <a href="/wiki/A%2B_(programming_language)" title="A+ (programming language)">A+</a>.</li>/n<li><a href="/wiki/MathCAD" class="mw-redirect" title="MathCAD">MathCAD</a>.</li>/n</ul>/n<h2><span
