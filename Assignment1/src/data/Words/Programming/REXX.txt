(<b>Restructured Extended Executor</b>) is an <a href="/wiki/Interpreted_language" title="Interpreted language">interpreted</a> <a href="/wiki/Programming_language" title="Programming language">programming language</a> developed at <a href="/wiki/IBM" title="IBM">IBM</a> by <a href="/wiki/Mike_Cowlishaw" title="Mike Cowlishaw">Mike Cowlishaw</a>. It is a structured, <a href="/wiki/High-level_programming_language" title="High-level programming language">high-level programming language</a> designed for ease of learning and reading. Proprietary and <a href="/wiki/Open_source" class="mw-redirect" title="Open source">open source</a> REXX <a href="/wiki/Interpreter_(computing)" title="Interpreter (computing)">interpreters</a> exist for a wide range of computing platforms; <a href="/wiki/Compiler" title="Compiler">compilers</a> exist for IBM <a href="/wiki/Mainframe_computer" title="Mainframe computer">mainframe computers</a>.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup></p>/n<p>Rexx is used as a <a href="/wiki/Scripting_language" title="Scripting language">scripting</a> and <a href="/wiki/Macro_(computer_science)" title="Macro (computer science)">macro</a> language, and is often used for processing data and text and generating reports; these similarities with <a href="/wiki/Perl" title="Perl">Perl</a> mean that Rexx works well in <a href="/wiki/Common_Gateway_Interface" title="Common Gateway Interface">Common Gateway Interface</a> (CGI) programming and it is indeed used for this purpose. Rexx is the primary scripting language in some operating systems, e.g. <a href="/wiki/OS/2" title="OS/2">OS/2</a>, <a href="/wiki/MVS" title="MVS">MVS</a>, <a href="/wiki/VM_(operating_system)" title="VM (operating system)">VM</a>, <a href="/wiki/AmigaOS" title="AmigaOS">AmigaOS</a>, and is also used as an internal macro language in some other software, such as <a href="/wiki/XEDIT#PC_and_Unix_adaptations" title="XEDIT">KEDIT</a>, <a href="/wiki/The_Hessling_Editor" title="The Hessling Editor">THE</a> and the <a href="/wiki/ZOC_(software)" title="ZOC (software)">ZOC</a> terminal emulator. Additionally, the Rexx language can be used for scripting and macros in any program that uses Windows Scripting Host ActiveX scripting engines languages (e.g. VBScript and JScript) if one of the Rexx engines is installed.</p>/n<p>Rexx is supplied with VM/SP on up, TSO/E Version 2 on up, OS/2 (1.3 and later, where it is officially named <i>Procedures Language/2</i>), AmigaOS Version 2 on up, PC DOS (7.0 or 2000), and Windows NT 4.0 (Resource Kit: Regina). REXX scripts for OS/2 share the filename extension .cmd with other scripting languages, and the first line of the script specifies the interpreter to be used. REXX macros for REXX-aware applications use extensions determined by the application. In the late 1980s Rexx became the common scripting language for <a href="/wiki/IBM_Systems_Application_Architecture" title="IBM Systems Application Architecture">IBM Systems Application Architecture</a>, where it was renamed "SAA Procedure Language REXX."</p>/n<p>A Rexx script or command is sometimes referred to as an <i>EXEC</i> in a nod to Rexx's role as a replacement for the older <a href="/wiki/CMS_EXEC" title="CMS EXEC">EXEC</a> command language on <a href="/wiki/CP/CMS" title="CP/CMS">CP/CMS</a> and VM/370 and <a href="/wiki/EXEC_2" title="EXEC 2">EXEC 2</a> command language on VM/SP.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#Features"><span class="tocnumber">1</span> limitations</li>/n<li>Simplified I/O facilities</li>/n<li>Unconventional operators</li>/n<li>Only partly supports Unix style command line parameters, except specific implementations</li>/n<li>Provides no basic terminal control as part of the language, except specific implementations</li>/n<li>Provides no generic way to include functions and subroutines from external libraries, except specific implementations</li>/n</ul>/n<p>Rexx has just twenty-three, largely self-evident, instructions (such as <code>call</code>, <code>parse</code>, and <code>select</code>) with minimal punctuation and formatting requirements. It is essentially an almost <a href="/wiki/Free-form_language" title="Free-form language">free-form language</a> with only one data-type, the character string; this philosophy means that all data are visible (symbolic) and debugging and tracing are simplified.</p>/n<p>Rexx's syntax looks similar to <a href="/wiki/PL/I" title="PL/I">PL/I</a>, but has fewer notations; this makes it harder to parse (by program) but easier to use, except for cases where PL/I habits may lead to surprises. One of the REXX design goals was the <a href="/wiki/Principle_of_least_astonishment" title="Principle of least astonishment">principle of least astonishment</a>.<sup id="cite_ref-POLA_3-0" class="reference"><a href="#cite_note-POLA-3">[3]</a></sup></p>/n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Rexx&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Rexx was designed and first implemented, in <a href="/wiki/Assembly_language" title="Assembly language">assembly language</a>, as an 'own-time' project between 20 March 1979 and mid-1982 by <a href="/wiki/Mike_Cowlishaw" title="Mike Cowlishaw">Mike Cowlishaw</a> of IBM, originally as a <a href="/wiki/Scripting_programming_language" class="mw-redirect" title="Scripting programming language">scripting programming language</a> to replace the languages <a href="/wiki/EXEC" class="mw-redirect" title="EXEC">EXEC</a> and <a href="/wiki/EXEC_2" title="EXEC 2">EXEC 2</a>.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup> It was designed to be a <a href="/wiki/Macro_(computer_science)" title="Macro (computer science)">macro</a> or scripting language for any system. As such, Rexx is considered a precursor to <a href="/wiki/Tcl" title="Tcl">Tcl</a> and <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>. Rexx was also intended by its creator to be a simplified and easier to learn version of the <a href="/wiki/PL/I" title="PL/I">PL/I</a> programming language. However, some differences from PL/I may trip up the unwary.</p>/n<p>It was first described in public at the SHARE 56 conference in Houston, Texas, in 1981,<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup> where customer reaction, championed by Ted Johnston of <a href="/wiki/SLAC_National_Accelerator_Laboratory" title="SLAC National Accelerator Laboratory">SLAC</a>, led to it being shipped as an IBM product in 1982.</p>/n<p>Over the years IBM included Rexx in almost all of its operating systems (<a href="/wiki/VM_(operating_system)" title="VM (operating system)">VM/CMS</a>, <a href="/wiki/MVS" title="MVS">MVS TSO/E</a>, <a href="/wiki/IBM_System_i" title="IBM System i">AS/400</a>, <a href="/wiki/VSE_(operating_system)" title="VSE (operating system)">VSE/ESA</a>, <a href="/wiki/IBM_AIX" title="IBM AIX">AIX</a>, <a href="/wiki/PC_DOS" class="mw-redirect" title="PC DOS">PC DOS</a>, and <a href="/wiki/OS/2" title="OS/2">OS/2</a>), and has made versions available for <a href="/wiki/Novell_NetWare" class="mw-redirect" title="Novell NetWare">Novell NetWare</a>, <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a>, <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, and <a href="/wiki/Linux" title="Linux">Linux</a>.</p>/n<p>The first non-IBM version was written for <a href="/wiki/PC_DOS" class="mw-redirect" title="PC DOS">PC DOS</a> by Charles Daney in 1984/5. The first compiler version appeared in 1987, written for CMS by Lundin and Woodruff.<sup id="cite_ref-TR_6-0" class="reference"><a href="#cite_note-TR-6">[6]</a></sup> Other versions have also been developed for <a href="/wiki/Atari" title="Atari">Atari</a>, <a href="/wiki/AmigaOS" title="AmigaOS">AmigaOS</a>, <a href="/wiki/Unix" title="Unix">Unix</a> (many variants), <a href="/wiki/Solaris_(operating_system)" title="Solaris (operating system)">Solaris</a>, <a href="/wiki/Digital_Equipment_Corporation" title="Digital Equipment Corporation">DEC</a>, <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a>, <a href="/wiki/Windows_CE" class="mw-redirect" title="Windows CE">Windows CE</a>, <a href="/wiki/Pocket_PC" title="Pocket PC">Pocket PC</a>, <a href="/wiki/DOS" title="DOS">DOS</a>, <a href="/wiki/Palm_OS" title="Palm OS">Palm OS</a>, <a href="/wiki/QNX" title="QNX">QNX</a>, <a href="/wiki/OS/2" title="OS/2">OS/2</a>, <a href="/wiki/Linux" title="Linux">Linux</a>, <a href="/wiki/BeOS" title="BeOS">BeOS</a>, <a href="/wiki/EPOC32" class="mw-redirect" title="EPOC32">EPOC32</a>/<a href="/wiki/Symbian" title="Symbian">Symbian</a>, <a href="/wiki/AtheOS" title="AtheOS">AtheOS</a>, <a href="/wiki/OpenVMS" title="OpenVMS">OpenVMS</a>, Apple <a href="/wiki/Macintosh" title="Macintosh">Macintosh</a>, and <a href="/wiki/Mac_OS_X" class="mw-redirect" title="Mac OS X">Mac OS X</a>.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup></p>/n<p>The <a href="/wiki/Amiga" title="Amiga">Amiga</a> version of Rexx, called <a href="/wiki/ARexx" title="ARexx">ARexx</a>, was included with AmigaOS 2 onwards and was popular for scripting as well as application control. Many Amiga applications have an "ARexx port" built into them which allows control of the application from Rexx. One single Rexx script could even switch between different Rexx ports in order to control several running applications.</p>/n<p>In 1990, Cathy Dager of SLAC organized the first independent Rexx symposium, which led to the forming of the REXX Language Association. Symposia are held annually.</p>/n<p>Several <a href="/wiki/Freeware" title="Freeware">freeware</a> versions of Rexx are available. In 1992, the two most widely used <a href="/wiki/Open-source" class="mw-redirect" title="Open-source">open-source</a> ports appeared: Ian Collier's REXX/imc for Unix and Anders Christensen's Regina (later adopted by Mark Hessling) for Windows and Unix. <a rel="nofollow" class="external text" href="http://ftp.gwdg.de/pub/languages/rexx/brexx/html/rx.html">BREXX</a> is well known for WinCE and Pocket PC platforms, and has been "back-ported" to VM/370 and <a href="/wiki/MVS" title="MVS">MVS</a>.</p>/n<p>OS/2 had a visual development system from Watcom <a href="/wiki/VX-REXX" title="VX-REXX">VX-REXX</a> another dialect was VisPro REXX from Hockware.</p>/n<p>Portable Rexx by Kilowatt and Personal Rexx by Quercus are two Rexx interpreters designed for MS-DOS and can of course be run under Windows as well using a command prompt. Since the mid-1990s, two newer variants of Rexx have appeared:</p>/n<ul>/n<li><a href="/wiki/NetRexx" title="NetRexx">NetRexx</a>: compiles to <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> <a href="/wiki/Byte-code" class="mw-redirect" title="Byte-code">byte-code</a> via Java source code; this has no reserved keywords at all, and uses the Java object model, and is therefore not generally upwards-compatible with 'classic' Rexx.</li>/n<li><a href="/wiki/Object_REXX" title="Object REXX">Object REXX</a>: an <a href="/wiki/Object-oriented" class="mw-redirect" title="Object-oriented">object-oriented</a> generally upwards-compatible version of Rexx.</li>/n</ul>/n<p>In 1996 <a href="/wiki/American_National_Standards_Institute" title="American National Standards Institute">American National Standards Institute</a> (ANSI) published a standard for Rexx: ANSI X3.274–1996 "Information Technology&#160;– Programming Language REXX".<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup> More than two dozen books on Rexx have been published since 1985.</p>/n<p>Rexx marked its 25th anniversary on 20 March 2004, which was celebrated at the REXX Language Association's 15th International REXX Symposium in Böblingen, Germany, in May 2004.</p>/n<p>On October 12, 2004, IBM announced their plan to release their <a href="/wiki/Object_REXX" title="Object REXX">Object REXX</a> implementation's sources under the <a href="/wiki/Common_Public_License" title="Common Public License">Common Public License</a>. Recent releases of Object REXX contain an ActiveX WSH scripting engine implementing this version of the Rexx language.</p>/n<p>On February 22, 2005, the first public release of Open Object Rexx (ooRexx) was announced. This product contains a WSH scripting engine which allows for programming of the Windows operating system and applications with Rexx in the same fashion in which <a href="/wiki/Visual_Basic" title="Visual Basic">Visual Basic</a> and <a href="/wiki/JScript" title="JScript">JScript</a> are implemented by the default WSH installation and <a href="/wiki/Perl" title="Perl">Perl</a>, <a href="/wiki/Tcl" title="Tcl">Tcl</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a> third-party scripting engines.</p>/n<p>As of January 2017<sup href="/wiki/TIOBE_index" title="TIOBE index">TIOBE index</a> as one of the fifty languages in its top 100 not belonging to the top 50.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[9]</a></sup></p>/n<h3><span class="mw-headline" id="Toolkits">Toolkits</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a used in Rexx programmes in the same fashion as <a href="/wiki/Tcl/Tk" class="mw-redirect" title="Tcl/Tk">Tcl/Tk</a> is widely available.</p>/n<p>A Rexx IDE, RxxxEd, has been developed for Windows.<sup and tutorials.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">[11]</a></sup><sup id="cite_ref-12" class="reference"><a href="#cite_note-12">[12]</a></sup></p>/n<h3><span varieties. NetRexx uses the keyword <code>LOOP</code> instead of <code>DO</code> for looping, while ooRexx treats <code>LOOP</code> and <code>DO</code> as equivalent when looping.</p>/n<h4><span class="mw-headline" id="Conditional_loops">Conditional loops</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Rexx&amp;action=edit&amp;section=7" title="Edit section: Conditional loops">edit</a><span class="mw-editsection-bracket">]</span></span></h4>/n<p>Rexx supports a variety of traditional structured-programming loops while testing a condition either before (<code>do while</code>) or after (<code>do until</code>) the list of instructions are executed:</p>/n<div style="background-color: #ffffff; !important" class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="w"> </span><span class="kr">do</span><span class="w"> </span><span class="kr">while</span><span class="w"> </span><span class="err">[</span>condition<span class="err">]</span>/n<span class="w"> </span><span class="err">[</span>instructions<span class="err">]</span>/n<span class="w"> </span><span class="kr">end</span>/n</pre></div>/n<div style="background-color: #ffffff; !important" class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="w"> </span><span class="kr">do</span><span class="w"> </span>index<span class="w"> </span><span class="o">=</span><span class="w"> </span>start<span class="w"> </span><span class="err">[</span><span class="kr">to</span><span class="w"> </span>limit<span class="err">]</span><span class="w"> </span><span class="err">[</span><span class="kr">by</span><span class="w"> </span>increment<span class="err">]</span><span class="w"> </span><span class="err">[</span><span class="kr">for</span><span class="w"> </span>count<span class="err">]</span>/n<span class="w"> </span><span class="err">[</span>instructions<span class="err">]</span>/n<span class="w"> </span><span class="kr">end</span>/n</pre></div>/n<p>The increment may be omitted and defaults to 1. The limit can also be omitted, which makes the loop continue forever.</p>/n<p>Rexx permits counted loops, where an expression is computed at the start of the loop and the instructions within the loop are executed that many times:</p>/n<div style="background-color: #ffffff; !important" class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="w"> </span><span class="kr">do</span><span class="w"> </span>expression/n<span class="w"> </span><span class="err">[</span>instructions<span class="err">]</span>/n<span class="w"> </span><span class="kr">end</span>/n</pre></div>/n<p>Rexx can even loop until the program is terminated:</p>/n<div style="background-color: #ffffff; !important" class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="w"> </span><span class="kr">do</span><span class="w"> </span>expression<span class="w"> </span><span class="err">[</span><span class="kr">until</span><span class="w"> </span>condition<span class="err">]</span>/n<span class="w"> </span><span class="err">[</span>instructions<span class="err">]</span>/n<span class="w"> </span><span class="kr">end</span>/n</pre></div>/n<h3><span class="mw-headline" id="Conditionals">Conditionals</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Rexx&amp;action=edit&amp;section=10" title="Edit section: Conditionals">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Testing conditions with <code>IF</code>:</p>/n<div style="background-color: #ffffff; !important" class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="w"> </span><span class="kr">if</span><span class="w"> </span><span class="err">[</span>condition<span class="err">]</span><span class="w"> </span><span class="kr">then</span>/n<span class="w"> </span><span class="kr">do</span>/n<span class="w"> </span><span class="err">[</span>instructions<span class="err">]</span>/n<span class="w"> </span><span class="kr">end</span>/n<span class="w"> </span><span class="kr">else</span>/n<span class="w"> </span><span class="kr">do</span>/n<span class="w"> </span><span class="err">[</span>instructions<span class="err">]</span>/n<span class="w"> </span><span class="kr">end</span>/n</pre></div>/n<p>The <code>ELSE</code> clause is optional.</p>/n<p>For single instructions, <code>DO</code> and <code>END</code> can also be omitted:</p>/n<div style="background-color: #ffffff; !important" class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="w"> </span><span class="kr">if</span><span class="w"> </span><span class="err">[</span>condition<span class="err">]</span><span class="w"> </span><span class="kr">then</span>/n<span class="w"> </span><span class="err">[</span>instruction<span class="err">]</span>/n<span class="w"> </span><span class="kr">else</span>/n<span class="w"> </span><span class="err">[</span>instruction<span class="err">]</span>/n</pre></div>/n<p>Indentation is optional, but it helps improve the readability.</p>/n<h3><span class="mw-headline" id="Testing_for_multiple_conditions">Testing for multiple conditions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Rexx&amp;action=edit&amp;section=11" title="Edit section: Testing for multiple conditions">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p><code>SELECT</code> is Rexx's <a href="/wiki/Switch_statement" title="Switch statement">CASE structure</a>, like many other constructs derived from <a href="/wiki/PL/I" title="PL/I">PL/I</a>. There are seven system conditions: ERROR, FAILURE, HALT, NOVALUE, NOTREADY, LOSTDIGITS and SYNTAX. Handling of each can be switched on and off in the source code as desired.</p>/n<p>The following program will run until terminated by the user:</p>/n<div style="background-color: #ffffff; !important" class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="w"> </span><span class="kr">signal</span><span class="w"> </span><span class="kr">on</span><span class="w"> </span>halt<span class="err">;</span>/n<span class="w"> </span><span class="kr">do</span><span class="w"> </span>a<span class="w"> </span><span class="o">=</span><span class="w"> </span><span class="m">1</span>/n<span class="w">	 </span><span class="kr">say</span><span class="w"> </span>a/n<span class="w">	 </span><span class="kr">do</span><span class="w"> </span><span class="m">100000</span><span class="w"> </span><span class="cm">/* a delay */</span>/n<span class="w">	 </span><span class="kr">end</span>/n<span class="w"> </span><span class="kr">end</span>/n<span class="w"> </span><span class="nl">halt</span><span class="o">:</span>/n<span class="w"> </span><span class="kr">say</span><span class="w"> </span><span class="s">"The program was stopped by the user"</span>/n<span class="w"> </span><span class="kr">exit</span>/n</pre></div>/n<p>A <code class="mw-highlight" dir="ltr"><span class="kr">signal</span> <span class="kr">on</span> novalue</code> statement intercepts uses of undefined variables, which would otherwise get their own (upper case) name as their value.
