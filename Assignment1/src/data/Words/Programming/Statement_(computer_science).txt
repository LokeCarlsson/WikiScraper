id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p>In <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a>, a <b>statement</b> is the smallest standalone element of an <a href="/wiki/Imperative_programming" title="Imperative programming">imperative</a> <a href="/wiki/Programming_language" title="Programming language">programming language</a> that expresses some action to be carried out. It is an instruction written in a high-level language that commands the computer to perform a specified action.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup> A program written in such a language is formed by a sequence of one or more statements. A statement may have internal components (e.g., <a href="/wiki/Expression_(computer_science)" title="Expression (computer science)">expressions</a>).</p>/n<p>Many languages (e.g. <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>) make a distinction between statements and definitions, with a statement only containing executable code and a definition instantiating an <a href="/wiki/Identifier_(computer_programming)" class="mw-redirect" title="Identifier (computer programming)">identifier</a>, while an expression evaluates to a value only. A distinction can also be made between simple and compound statements; the latter may contain statements as components.</p>/n<p></p>/n<div id="toc" class="toc">/n<div id="Syntax">Syntax</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Statement_(computer_science)&amp;action=edit&amp;section=4" title="Edit section: Syntax">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The appearance of statements shapes the look of programs. Programming languages are characterized by the type of statements they use (e.g. the <a href="/wiki/Curly_brace_language" class="mw-redirect" title="Curly brace language">curly brace language</a> family). Many statements are introduced by <a href="/wiki/Identifier_(computer_programming)" class="mw-redirect" title="Identifier (computer programming)">identifiers</a> like <i>if</i>, <i>while</i> or <i>repeat</i>. Often statement <a href="/wiki/Keyword_(computer_programming)" class="mw-redirect" title="Keyword (computer programming)">keywords</a> are reserved such that they cannot be used as names of <a href="/wiki/Variable_(programming)" class="mw-redirect" title="Variable (programming)">variables</a> or <a href="/wiki/Function_(computer_science)" class="mw-redirect" title="Function (computer science)">functions</a>. Imperative languages typically use special syntax for each statement, which looks quite different from function calls. Common methods to describe the syntax of statements are <a href="/wiki/Backus%E2%80%93Naur_form" title="Backus–Naur form">Backus–Naur form</a> and <a href="/wiki/Syntax_diagram" title="Syntax diagram">syntax diagrams</a>.</p>/n<h2><span class="mw-headline" id="Semantics">Semantics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Statement_(computer_science)&amp;action=edit&amp;section=5" title="Edit section: Semantics">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Semantically many statements differ from <a href="/wiki/Subroutine" title="Subroutine">subroutine</a> calls by their handling of <a href="/wiki/Parameter_(computer_science)" class="mw-redirect" title="Parameter (computer science)">parameters</a>. Usually an actual subroutine parameter is evaluated once before the subroutine is called. This contrasts to many statement parameters that can be evaluated several times (e.g. the condition of a <a href="/wiki/While_loop" title="While loop">while loop</a>) or not at all (e.g. the loop body of a while loop). Technically such statement parameters are <a href="/wiki/Call-by-name" class="mw-redirect" title="Call-by-name">call-by-name</a> parameters. Call-by-name parameters are evaluated when needed (see also <a href="/wiki/Lazy_evaluation" title="Lazy evaluation">lazy evaluation</a>). When call-by-name parameters are available a statement like behaviour can be implemented with subroutines (see <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a>). For languages without call-by-name parameters the semantic description of a <a href="/wiki/Control_flow#Loops" title="Control flow">loop</a> or <a href="/wiki/Conditional_(programming)" class="mw-redirect" title="Conditional (programming)">conditional</a> is usually beyond the capabilities of the language. Therefore, standard documents often refer to semantic descriptions in natural language.</p>/n<h2><span class="mw-headline" id="Expressions">Expressions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Statement_(computer_science)&amp;action=edit&amp;section=6" title="Edit section: Expressions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>In most languages, statements contrast with <a href="/wiki/Expression_(programming)" class="mw-redirect" title="Expression (programming)">expressions</a> in that statements do not return results and are executed solely for their <a href="/wiki/Side_effect_(programming)" class="mw-redirect" title="Side effect (programming)">side effects</a>, while expressions always return a result and often do not have side effects at all. Among imperative programming languages, <a href="/wiki/Algol_68" class="mw-redirect" title="Algol 68">Algol 68</a> is one of the few in which a statement can return a result. In languages that mix imperative and <a href="/wiki/Functional_programming" title="Functional programming">functional</a> styles, such as the <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a> family, the distinction between expressions and statements is not made: even expressions executed in sequential contexts solely for their side effects and whose return values are not used are considered 'expressions'. In <a href="/wiki/Purely_functional_programming" title="Purely functional programming">purely functional programming</a>, there are no statements; everything is an expression.</p>/n<p>This distinction is frequently observed in wording: a statement is <i>executed</i>, while an expression is <i>evaluated</i>. This is found in the <code>exec</code> and <code><a href="/wiki/Eval" title="Eval">eval</a></code> functions found in some languages: in Python both are found, with <code>exec</code> applied to statements and <code>eval</code> applied to expressions.</p>/n<h2><span class="mw-headline" id="Extensibility">Extensibility</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Statement_(computer_science)&amp;action=edit&amp;section=7" title="Edit section: Extensibility">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Most languages have a fixed set of statements defined by the language, but there have been experiments with <a href="/wiki/Extensible_languages" class="mw-redirect" title="Extensible languages">extensible languages</a> that allow the programmer to define new statements.</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Statement_(computer_science)&amp;action=edit&amp;section=8" title="Edit section: See also">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<ul>/n<li><a href="/wiki/Comparison_of_programming_languages_(syntax)#Statements" title="Comparison of programming languages (syntax)">Comparison of Programming Languages -
