or 'Before C Programming Language' (common humorous <a href="/wiki/Backronym" title="Backronym">backronym</a>)<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> ) is a <a href="/wiki/Procedural_programming" title="Procedural programming">procedural</a>, <a href="/wiki/Imperative_programming" title="Imperative programming">imperative</a>, and <a href="/wiki/Structured_programming" title="Structured programming">structured</a> <a href="/wiki/Computer" title="Computer">computer</a> <a href="/wiki/Programming_language" title="Programming language">programming language</a>. Originally intended for writing <a href="/wiki/Compiler" title="Compiler">compilers</a> for other languages, BCPL is no longer in common use. However, its influence is still felt because a stripped down and syntactically changed version of BCPL, called <a href="/wiki/B_(programming_language)" title="B (programming language)">B</a>, was the language on which the <a href="/wiki/C_(programming_language)" title="C (programming language)">C programming language</a> was based. BCPL introduced several features of modern programming languages, including <a href="/wiki/Brace_programming_language" class="mw-redirect" title="Brace programming language">curley braces notation</a> for code blocks; compilation via <a href="/wiki/Virtual_machine" title="Virtual machine">virtual machine</a> <a href="/wiki/Byte_code" class="mw-redirect" title="Byte code">byte code</a>; and the world's first '<a href="/wiki/Hello_world" class="mw-redirect" title="Hello world">hello world</a>' demonstrator program.</p>/n<p></p>/n<div id="toc" class="toc">/n<div Further, the original compiler, itself written in BCPL, was easily portable. BCPL was thus a popular choice for <a href="/wiki/Bootstrapping_(compilers)" title="Bootstrapping (compilers)">bootstrapping</a> a system.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2017)">citation needed</span></a></i>]</sup> A major reason for the compiler's portability lay in its structure. It was split into two parts: the front end parsed the source and generated <a href="/wiki/O-code_machine" class="mw-redirect" title="O-code machine">O-code</a> for a <a href="/wiki/Virtual_machine" title="Virtual machine">virtual machine</a>, and the back end took the O-code and translated it into the code for the target machine. Only 1/5 of the compiler's code needed to be rewritten to support a new machine, a task that usually took between 2 and 5 <a href="/wiki/Man-month" class="mw-redirect" title="Man-month">man-months</a>. This approach became common practice later, e.g., <a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a> or <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, but the original BCPL compiler was the first to define a virtual machine for this purpose.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2017)">citation needed</span></a></i>]</sup></p>/n<p>The language is unusual in having only one <a href="/wiki/Data_type" title="Data type">data type</a>: a <a href="/wiki/Word_(data_type)" class="mw-redirect" title="Word (data type)">word</a>, a fixed number of bits, usually chosen to align with the architecture's machine word and of adequate capacity to represent any valid storage address. For many machines of the time, this data type was a 16-bit word. This choice later proved to be a significant problem when BCPL was used on machines in which the smallest addressable item was not a word but a <a href="/wiki/Byte" title="Byte">byte</a>, or on machines with larger word sizes such as 32-bit or 64-bit.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2017)">citation needed</span></a></i>]</sup></p>/n<p>The interpretation of any value was determined by the operators used to process the values. (For example, + added two values together treating them as <a href="/wiki/Integer" title="Integer">integers</a>;&#160;! indirected through a value, effectively treating it as a pointer.) In order for this to work, the implementation provided no <a href="/wiki/Type_checking" class="mw-redirect" title="Type checking">type checking</a>. The <a href="/wiki/Hungarian_notation" title="Hungarian notation">Hungarian notation</a> was developed to help programmers avoid inadvertent type errors.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2017)">citation needed</span></a></i>]</sup></p>/n<p>The mismatch between BCPL's <a href="/wiki/Word_(data_type)" class="mw-redirect" title="Word (data type)">word orientation</a> and <a href="/wiki/Byte_orientation" class="mw-redirect" title="Byte orientation">byte-oriented</a> hardware was addressed in several ways. One was providing standard library routines for packing and unpacking words into byte strings. Later, two language features were added: the bit-field selection operator and the infix byte indirection operator (denoted by the '%' character).<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2017)">citation needed</span></a></i>]</sup></p>/n<p>BCPL handles bindings spanning separate <a href="/wiki/Compilation_unit" class="mw-redirect" title="Compilation unit">compilation units</a> in a unique way. There are no user-declarable global variables; instead there is a global vector, which is similar to "blank common" in <a href="/wiki/Fortran" title="Fortran">Fortran</a>. All data shared between different compilation units comprises scalars and pointers to vectors stored in a pre-arranged place in the global vector. Thus the header files (files included during compilation using the "GET" directive) become the primary means of synchronizing global data between compilation units, containing "GLOBAL" directives that present lists of symbolic names, each paired with a number that associates the name with the corresponding numerically addressed word in the global vector. As well as variables, the global vector also contains bindings for external procedures. This makes dynamic loading of compilation units very simple to achieve. Instead of relying on the link loader of the underlying implementation, effectively BCPL gives the programmer control of the linking process.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2017)">citation needed</span></a></i>]</sup></p>/n<p>The global vector also made it very simple to replace or augment standard library routines. A program could save the pointer from the global vector to the original routine and replace it with a pointer to an alternative version. The alternative might call the original as part of its processing. This could be used as a quick <i>ad hoc</i> debugging aid.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2017)">citation needed</span></a></i>]</sup></p>/n<p>BCPL was the first <a href="/wiki/Brace_programming_language" class="mw-redirect" title="Brace programming language">brace programming language</a>, and the braces survived the syntactical changes and have become a common means of denoting program source code statements. In practice, on limited keyboards of the day, source programs often used the sequences <code>$(</code> and <code>$)</code> in place of the symbols <code>{</code> and <code>}</code>. The single-line <code>//</code> <a href="/wiki/Comment_(computer_programming)" title="Comment (computer programming)">comments</a> of BCPL, which were not adopted by <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, reappeared in <a href="/wiki/C%2B%2B" title="C++">C++</a> and later in <a href="/wiki/C99" title="C99">C99</a>.</p>/n<p>The book <i>BCPL: The language and its compiler</i> describes the philosophy of BCPL as follows:<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (December 2016)">citation needed</span></a></i>]</sup></p>/n<blockquote class="templatequote">/n<p>"The philosophy of BCPL is not one of the tyrant who thinks he knows best and lays down the law on what is and what is not allowed; rather, BCPL acts more as a servant offering his services to the best of his ability without complaint, even when confronted with apparent nonsense. The programmer is always assumed to know what he is doing and is not hemmed in by petty restrictions."<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2017)">citation needed</span></a></i>]</sup></p>/n</blockquote>/n<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=BCPL&amp;action=edit&amp;section=2" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Print factorials:</p>/n<pre>/nGET "LIBHDR"/n/nLET START() = VALOF $(/n        FOR I = 1 TO 5 DO/n                WRITEF("%N! BCPL was a response to difficulties with its predecessor, Cambridge Programming Language, later renamed <a href="/wiki/Combined_Programming_Language" class="mw-redirect" title="Combined Programming Language">Combined Programming Language</a> (CPL), which was designed during the early 1960s. Richards created BCPL by "removing those features of the full language which make compilation difficult". The first compiler implementation, for the <a href="/wiki/IBM_7094" class="mw-redirect" title="IBM 7094">IBM 7094</a> under <a href="/wiki/Compatible_Time-Sharing_System" title="Compatible Time-Sharing System">Compatible Time-Sharing System</a> (CTSS), was written while Richards was visiting <a href="/wiki/Project_MAC" class="mw-redirect" title="Project MAC">Project MAC</a> at the <a href="/wiki/Massachusetts_Institute_of_Technology" title="Massachusetts Institute of Technology">Massachusetts Institute of Technology</a> (MIT) in the spring of 1967. The language was first described in a paper presented to the 1969 Spring Joint Computer Conference.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2017)">citation needed</span></a></i>]</sup></p>/n<p>BCPL has been rumored to have originally stood for "Bootstrap Cambridge Programming Language", but CPL was never created since development stopped at BCPL, and the acronym was later reinterpreted for the BCPL book.<sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag may need clarification or removal of jargon. (January 2017)">clarification needed</span></a></i>]</sup><sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2017)">citation needed</span></a></i>]</sup></p>/n<p>BCPL is the language in which the original <a href="/wiki/Hello_world_program" class="mw-redirect" title="Hello world program">hello world program</a> was written.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> The first <a href="/wiki/MUD" title="MUD">MUD</a> was also written in BCPL (<a href="/wiki/MUD1" title="MUD1">MUD1</a>).</p>/n<p>Several <a href="/wiki/Operating_system" title="Operating system">operating systems</a> were written partially or wholly in BCPL (for example, <a href="/wiki/TRIPOS" title="TRIPOS">TRIPOS</a> and the earliest versions of <a href="/wiki/AmigaDOS" title="AmigaDOS">AmigaDOS</a>). BCPL was also the initial language used in the seminal <a href="/wiki/Xerox_PARC" class="mw-redirect" title="Xerox PARC">Xerox PARC</a> <a href="/wiki/Alto_(computer)" class="mw-redirect" title="Alto (computer)">Alto</a> project, the first modern <a href="/wiki/Personal_computer" title="Personal computer">personal computer</a>; among other projects, the <a href="/wiki/Bravo_(software)" title="Bravo (software)">Bravo</a> <a href="/wiki/Word_processor" title="Word processor">document preparation system</a> was written in BCPL.</p>/n<p>An early compiler, bootstrapped in 1969, by starting with a paper tape of the <a href="/wiki/O-code" title="O-code">O-code</a> of Martin Richards's <a href="/wiki/Titan_(computer)" title="Titan (computer)">Atlas 2</a> compiler, targeted the <a href="/wiki/ICT_1900_series" title="ICT 1900 series">ICT 1900 series</a>. The two machines had different word-lengths (48 vs 24 bits), different character encodings, and different packed string representations—and the successful bootstrapping increased confidence in the practicality of the method.</p>/n<p>By late 1970, implementations existed for the <a href="/wiki/GE-600_series" title="GE-600 series">Honeywell 635 and Honeywell 645</a>, the <a href="/wiki/IBM_360" class="mw-redirect" title="IBM 360">IBM 360</a>, the <a href="/wiki/PDP-10" title="PDP-10">PDP-10</a>, the <a href="/wiki/TX-2" title="TX-2">TX-2</a>, the <a href="/wiki/CDC_6400" class="mw-redirect" title="CDC 6400">CDC 6400</a>, the <a href="/wiki/UNIVAC_1108" class="mw-redirect" title="UNIVAC 1108">UNIVAC 1108</a>, the <a href="/wiki/PDP-9" title="PDP-9">PDP-9</a>, the <a href="/wiki/KDF_9" class="mw-redirect" title="KDF 9">KDF 9</a> and the <a href="/wiki/Titan_(computer)" title="Titan (computer)">Atlas 2</a>. In 1974 a dialect of BCPL was implemented at <a href="/wiki/BBN_Technologies" title="BBN Technologies">BBN</a> without using the intermediate <a href="/wiki/O-code" title="O-code">O-code</a>. The initial implementation was a <a href="/wiki/Cross-compiler" class="mw-redirect" title="Cross-compiler">cross-compiler</a> hosted on BBN's <a href="/wiki/TOPS-20#TENEX" title="TOPS-20">Tenex</a> <a href="/wiki/PDP-10" title="PDP-10">PDP-10s</a>, and directly targeted the <a href="/wiki/PDP-11" title="PDP-11">PDP-11s</a> used in BBN's implementation of the second generation <a href="/wiki/Interface_Message_Processor" title="Interface Message Processor">IMPs</a> used in the <a href="/wiki/Arpanet" class="mw-redirect" title="Arpanet">Arpanet</a>.</p>/n<p>There was also a version produced for the <a href="/wiki/BBC_Micro" title="BBC Micro">BBC Micro</a> in the mid-1980s, by Richards Computer Products, a company started by John Richards, the brother of Dr. Martin Richards.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup> The <a href="/wiki/BBC_Domesday_Project" title="BBC Domesday Project">BBC Domesday Project</a> made use of the language. Versions of BCPL for the <a href="/wiki/Amstrad_CPC" title="Amstrad CPC">Amstrad CPC</a> and <a href="/wiki/Amstrad_PCW" title="Amstrad PCW">Amstrad PCW</a> computers were also released in 1986 by UK software house Arnor Ltd. MacBCPL was released for the Apple Macintosh in 1985 by Topexpress Ltd, of Kensington, England.</p>/n<p>Both the design and philosophy of BCPL strongly influenced <a href="/wiki/B_(programming_language)" title="B (programming language)">B</a>, which in turn influenced <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2017)">citation needed</span></a></i>]</sup> Programmers at the time debated whether an eventual successor to C would be called "D" (the next letter in the alphabet) or "P" (the next letter in the parent language name). As it happened, the actual name turned out to be "C++" (a pun on the C language increment operator).<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2017)">citation needed</span></a></i>]</sup></p>/n<p>In 1979, implementations of BCPL existed for at least 25 architectures; the language gradually fell out of favour as C became popular on non-Unix systems.</p>/n<p>Martin Richards maintains a modern Linux version of BCPL on his website, last updated in 2011. He continues to program in it, including for his research on musical <a href="/w/index.php?title=Automated_score_following&amp;action=edit&amp;redlink=1" class="new" title="Automated score following (page does not exist)">automated score following</a>.</p>/n<h2><span class="mw-headline" id="References">References</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=BCPL&amp;action=edit&amp;section=4" title="Edit section: References">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div class="reflist" style="list-style-type: decimal;">/n<div class="mw-references-wrap">/n<ol class="references">/n<li id="cite_note-hgophers-1"><span class="mw-cite-backlink"><b><a href="#cite_ref-hgophers_1-0">^</a></b></span> <span class="reference-text"><cite class="citation web">Pike, Rob (2014-04-24). <a rel="nofollow" class="external text" href="https://talks.golang.org/2014/hellogophers.slide#21">"Hello Gophers"</a><span class="reference-accessdate">.
