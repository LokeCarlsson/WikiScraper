English</h1>/n									<div id="bodyContent" class="mw-body-content">/n									<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>/n								<div id="contentSub"></div>/n												<div id="jump-to-nav" class="mw-jump">/n					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>/n				</div>/n				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p><b>Structured English</b> is the use of the <a href="/wiki/English_language" title="English language">English language</a> with the <a href="/wiki/Programming_language_syntax" class="mw-redirect" title="Programming language syntax">syntax</a> of <a href="/wiki/Structured_programming" title="Structured programming">structured programming</a> to communicate the design of a computer program to non-technical users by breaking it down into logical steps using straightforward English words. Structured English gives aims to get the benefits of both the programming logic and natural language: program logic helps to attain precision, whilst natural language helps with the familiarity of the spoken word.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup></p>/n<p>It is the basis of some programming languages such as SQL (Structured Query Language) "for use by people who have need for interaction with a large database but who are not trained programmers".<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup></p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#Elements"><span class="tocnumber">1</span> <span class="toctext">Elements</span></a></li>/n<li class="toclevel-1 tocsection-2"><a href="#Example_of_Structured_English"><span class="tocnumber">2</span> <span class="toctext">Example of Structured class="tocnumber">5</span> <span class="toctext">References</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="Elements">Elements</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Structured_English&amp;action=edit&amp;section=1" title="Edit section: Elements">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Structured English is a limited-form "<a href="/wiki/Pseudocode" title="Pseudocode">pseudocode</a>" and consists of the following elements:</p>/n<ol>/n<li>Operation statements written as English phrases executed from the top down</li>/n<li>Conditional blocks indicated by keywords such as IF, THEN, and ELSE</li>/n<li>Repetition blocks indicated by keywords such as DO, WHILE, and UNTIL</li>/n</ol>/n<p>The following guidelines are used when writing Structured English:<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup></p>/n<ol>/n<li>All logic should be expressed in operational, conditional, and repetition blocks</li>/n<li>Statements should be clear and unambiguous</li>/n<li>Logical blocks should be indented to show relationship and hierarchy</li>/n<li>Use one line per logical element, or indent the continuation line</li>/n<li>Keywords should be capitalized</li>/n<li>Group blocks of statements together, with a capitalized name that describes their function and end with an EXIT.</li>/n<li>Underline words or phrases defined in a data dictionary</li>/n<li>Mark comment lines with an asterisk</li>/n</ol>/n<h2><span class="mw-headline" id="Example_of_Structured_English">Example of Structured English</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Structured_English&amp;action=edit&amp;section=2" title="Edit section: Example of Structured English">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>APPROVE LOAN</p>/n<pre>/n IF customer has a Bank Account THEN/n   IF Customer has no dues from previous account THEN/n      Allow loan facility/n/n APPROVE LOAN/n IF customer has a Bank Account THEN/n   IF Customer has no dues from previous account THEN/n      Allow loan facility/n   ELSE/n      IF Management Approval is obtained THEN/n         Allow loan facility/n      ELSE/n         Reject/n      ENDIF/n   ENDIF/nELSE/n   Reject/nENDIF/nEXIT/n</pre>/n<h2><span class="mw-headline" id="Criticism">Criticism</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Structured_English&amp;action=edit&amp;section=3" title="Edit section: Criticism">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Though useful for planning programs, modules and routines, or describing algorithms it is less useful when numerous decisions need to be made.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup></p>/n<h3><span class="mw-headline" id="Other_specification_tools">Other specification tools</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Structured_English&amp;action=edit&amp;section=4" title="Edit section: Other specification tools">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>System processes at a lower level involve lot of computations and require more precision and clarity. This can be achieved with tools such as <a href="/wiki/Decision_tree" title="Decision tree">decision trees</a> or <a href="/wiki/Decision_table" title="Decision table">decision tables</a>.</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a
