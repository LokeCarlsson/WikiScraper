href="/wiki/Help:Maintenance_template_removal" title="Help:Maintenance template removal">Learn how and when to remove this template message</a>)</i></small></div>/n</td>/n</tr>/n</table>/n<p>In <a href="/wiki/Computability_theory" title="Computability theory">computability theory</a>, a system of data-manipulation rules (such as a computer's <a href="/wiki/Instruction_set" class="mw-redirect" title="Instruction set">instruction set</a>, a <a href="/wiki/Programming_language" title="Programming language">programming language</a>, or a <a href="/wiki/Cellular_automaton" title="Cellular automaton">cellular automaton</a>) is said to be <b>Turing complete</b> or <b>computationally universal</b> if it can be used to simulate any <a href="/wiki/Turing_machine" title="Turing machine">Turing machine</a>. The concept is named after English mathematician and computer scientist <a href="/wiki/Alan_Turing" title="Alan Turing">Alan Turing</a>. A classic example is <a href="/wiki/Lambda_calculus" title="Lambda calculus">lambda calculus</a>.</p>/n<p>A closely related concept is that of <b>Turing equivalence</b>&#160;–  two computers P and Q are called equivalent if P can simulate Q and Q can simulate P. The <a href="/wiki/Church%E2%80%93Turing_thesis" title="Church–Turing thesis">Church–Turing thesis</a> conjectures that any function whose values can be computed by an <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> can be computed by a Turing machine, and therefore that if any real-world computer can simulate a Turing machine, it is Turing equivalent to a Turing machine. A <a href="/wiki/Universal_Turing_machine" title="Universal Turing machine">universal Turing machine</a> can be used to simulate any Turing machine and by extension the computational aspects of any possible real-world computer.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[NB 1]</a></sup></p>/n<p>To show that something is Turing complete, it is enough to show that it can be used to simulate some Turing complete system. For example, an <a href="/wiki/Imperative_language" class="mw-redirect" title="Imperative language">imperative language</a> is Turing complete if it has <a href="/wiki/Conditional_branching" class="mw-redirect" title="Conditional branching">conditional branching</a> (<i>e.g.</i>, "if" and "goto" statements, or a "branch if zero" instruction; see <a href="/wiki/One_instruction_set_computer" title="One instruction set computer">one instruction set computer</a>) and the ability to change an arbitrary amount of <a href="/wiki/Computer_memory" title="Computer memory">memory</a> (<i>e.g.</i>, the ability to maintain an arbitrary number of variables). Since this is almost always the case, most if not all imperative languages are Turing complete if the limitations of finite memory are ignored.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toclevel-1 tocsection-15"><a href="#External_links"><span class="tocnumber">13</span> <span class="toctext">External links</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="Non-mathematical_usage">Non-mathematical usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_completeness&amp;action=edit&amp;section=1" title="Edit section: Non-mathematical usage">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>In <a href="/wiki/Colloquial" class="mw-redirect" title="Colloquial">colloquial</a> usage, the terms "Turing complete" or "Turing equivalent" are used to mean that any real-world general-purpose computer or computer language can approximately simulate the computational aspects of any other real-world general-purpose computer or computer language.</p>/n<p>Real computers constructed so far are essentially similar to a single-tape Turing machine; thus the associated mathematics can apply by abstracting their operation far enough. However, real computers have limited physical resources, so they are only <a href="/wiki/Linear_bounded_automaton" title="Linear bounded automaton">linear bounded automaton</a> complete. In contrast, a <a href="/wiki/Universal_computer" class="mw-redirect" title="Universal computer">universal computer</a> is defined as a device with a Turing complete instruction set, infinite memory, and infinite available time.</p>/n<h2><span class="mw-headline" id="Formal_definitions">Formal definitions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_completeness&amp;action=edit&amp;section=2" title="Edit section: Formal definitions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>In <a href="/wiki/Computability_theory_(computation)" class="mw-redirect" title="Computability theory (computation)">computability theory</a>, several closely related terms are used to describe the computational power of a computational system (such as an <a href="/wiki/Abstract_machine" title="Abstract machine">abstract machine</a> or <a href="/wiki/Programming_language" title="Programming language">programming language</a>):</p>/n<dl>/n<dt>Turing completeness</dt>/n<dd>A computational system that can compute every Turing-<a href="/wiki/Computable_function" title="Computable function">computable function</a> is called Turing-complete (or Turing-powerful). Alternatively, such a system is one that can simulate a <a href="/wiki/Universal_Turing_machine" title="Universal Turing machine">universal Turing machine</a>.</dd>/n</dl>/n<dl>/n<dt>Turing equivalence</dt>/n<dd>A Turing-complete system is called Turing equivalent if every function it can compute is also Turing computable; i.e., it computes precisely the same class of functions as do <a href="/wiki/Turing_machine" title="Turing machine">Turing machines</a>. Alternatively, a Turing-equivalent system is one that can simulate, and be simulated by, a universal Turing machine. (All known Turing-complete systems are Turing equivalent, which adds support to the <a href="/wiki/Church%E2%80%93Turing_thesis" title="Church–Turing thesis">Church–Turing thesis</a>.)</dd>/n</dl>/n<dl>/n<dt>(Computational) universality</dt>/n<dd>A system is called universal with respect to a class of systems if it can compute every function computable by systems in that class (or can simulate each of those systems). Typically, the term universality is tacitly used with respect to a Turing-complete class of systems. The term "weakly universal" is sometimes used to distinguish a system (e.g. a <a href="/wiki/Cellular_automaton" title="Cellular automaton">cellular automaton</a>) whose universality is achieved only by modifying the standard definition of <a href="/wiki/Turing_machine" title="Turing machine">Turing machine</a> so as to include input streams with infinitely many 1s.</dd>/n</dl>/n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_completeness&amp;action=edit&amp;section=3" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Turing completeness is significant in that every real-world design for a computing device can be simulated by a <a href="/wiki/Universal_Turing_machine" title="Universal Turing machine">universal Turing machine</a>. The <a href="/wiki/Church%E2%80%93Turing_thesis" title="Church–Turing thesis">Church–Turing thesis</a> states that this is a law of mathematics&#160;–  that a universal Turing machine can, in principle, perform any calculation that any other programmable <a href="/wiki/Computer" title="Computer">computer</a> can. This says nothing about the effort needed to write the <a href="/wiki/Computer_program" title="Computer program">program</a>, or the time it may take for the machine to perform the calculation, or any abilities the machine may possess that have nothing to do with computation.</p>/n<p><a href="/wiki/Charles_Babbage" title="Charles Babbage">Charles Babbage</a>'s <a href="/wiki/Analytical_engine" class="mw-redirect" title="Analytical engine">analytical engine</a> (1830s) would have been the first Turing-complete machine if it had been built at the time it was designed. Babbage appreciated that the machine was capable of great feats of calculation, including primitive logical reasoning, but he did not appreciate that no other machine could do better. From the 1830s until the 1940s, mechanical calculating machines such as adders and multipliers were built and improved, but they could not perform a conditional branch and therefore were not Turing complete.</p>/n<p>In the late 19th century, <a href="/wiki/Leopold_Kronecker" title="Leopold Kronecker">Leopold Kronecker</a> formulated notions of computability, defining <a href="/wiki/Primitive_recursive_functions" class="mw-redirect" title="Primitive recursive functions">primitive recursive functions</a>. These functions can be calculated by rote computation, but they are not enough to make a universal computer, because the instructions which compute them do not allow for an infinite loop. In the early 20th century, <a href="/wiki/David_Hilbert" title="David Hilbert">David Hilbert</a> led a program to axiomatize all of mathematics with precise axioms and precise logical rules of deduction which could be performed by a machine. Soon, it became clear that a small set of deduction rules are enough to produce the consequences of any set of axioms. These rules were proved by <a href="/wiki/Kurt_G%C3%B6del" title="Kurt Gödel">Kurt Gödel</a> in 1930 to be enough to produce every theorem.</p>/n<p>The actual notion of computation was isolated soon after, starting with <a href="/wiki/G%C3%B6del%27s_incompleteness_theorem" class="mw-redirect" title="Gödel's incompleteness theorem">Gödel's incompleteness theorem</a>. This theorem showed that axiom systems were limited when reasoning about the computation which deduces their theorems. Church and Turing independently demonstrated that Hilbert's <i><a href="/wiki/Entscheidungsproblem" title="Entscheidungsproblem">Entscheidungsproblem</a></i> (decision problem) was unsolvable,<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[1]</a></sup> thus identifying the computational core of the incompleteness theorem. This work, along with Gödel's work on <a href="/w/index.php?title=General_recursive_functions&amp;action=edit&amp;redlink=1" class="new" title="General recursive functions (page does not exist)">general recursive functions</a>, established that there are sets of simple instructions, which, when put together, are able to produce any computation. The work of Gödel showed that the notion of computation is essentially unique.</p>/n<h2><span class="mw-headline" id="Computability_theory">Computability theory</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_completeness&amp;action=edit&amp;section=4" title="Edit section: Computability theory">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The first result of computability theory is that it is impossible in general to predict what a Turing-complete program will do over an arbitrarily long time. For example, it is impossible to determine for every program-input pair whether the program, operating on the input, will eventually stop or will continue forever (see <a href="/wiki/Halting_problem" title="Halting problem">halting problem</a>). It is impossible to determine whether the program will return "true" or whether it will return "false". For any characteristic of the program's eventual output, it is impossible to determine whether this characteristic will hold. This can cause problems in practice when analyzing real-world computer programs. One way to avoid this is to cause programs to stop executing after a fixed period of time (<a href="/wiki/Timeout_(computing)" title="Timeout (computing)">timeout</a>), or to limit the power of flow control instructions. Such systems are not Turing complete by design.</p>/n<p>Another theorem shows that there are problems solvable by Turing-complete languages that cannot be solved by any language with only finite looping abilities (i.e., any language that guarantees every program will eventually finish to a halt). Given a guaranteed halting language, the computable function which is produced by <a href="/wiki/Cantor%27s_diagonal_argument" title="Cantor's diagonal argument">Cantor's diagonal argument</a> on all computable functions in that language is not computable in that language.</p>/n<h2><span class="mw-headline" id="Turing_oracles">Turing oracles</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_completeness&amp;action=edit&amp;section=5" title="Edit section: Turing oracles">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Oracle_machine" title="Oracle machine">Oracle machine</a></div>/n<p>A computer with access to an infinite tape of data may be more powerful than a Turing machine: for instance, the tape might contain the solution to the <a href="/wiki/Halting_problem" title="Halting problem">halting problem</a>, or some other Turing-undecidable problem. Such an infinite tape of data is called a <a href="/wiki/Turing_oracle" class="mw-redirect" title="Turing oracle">Turing oracle</a>. Even a Turing oracle with random data is not computable (<a href="/wiki/Almost_surely" title="Almost surely">with probability 1</a>), since there are only countably many computations but uncountably many oracles. So a computer with a random Turing oracle can compute things that a Turing machine cannot.</p>/n<h2><span class="mw-headline" id="Digital_physics">Digital physics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_completeness&amp;action=edit&amp;section=6" title="Edit section: Digital physics">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>All known laws of physics have consequences that are computable by a series of approximations on a digital computer. A hypothesis called <a href="/wiki/Digital_physics" title="Digital physics">digital physics</a> states that this is no accident, because the <a href="/wiki/Universe" title="Universe">universe</a> itself is computable on a universal Turing machine. This would imply that no computer more powerful than a universal Turing machine can be built physically (<i>see</i> <a href="/wiki/Church%E2%80%93Turing_thesis#Philosophical_implications" title="Church–Turing thesis">Church–Turing thesis – Philosophical implications</a>).</p>/n<h2><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_completeness&amp;action=edit&amp;section=7" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The computational systems (algebras, calculi) that are discussed as Turing complete systems are those intended for studying <a href="/wiki/Theoretical_computer_science" title="Theoretical computer science">theoretical computer science</a>. They are intended to be as simple as possible, so that it would be easier to understand the limits of computation. Here are a few:</p>/n<ul>/n<li><a href="/wiki/Automata_theory" title="Automata theory">Automata theory</a></li>/n<li><a href="/wiki/Formal_grammar" title="Formal grammar">Formal grammar</a> (language generators)</li>/n<li><a href="/wiki/Formal_language" title="Formal language">Formal language</a> (language recognizers)</li>/n<li><a href="/wiki/Lambda_calculus" title="Lambda calculus">Lambda calculus</a></li>/n<li><a href="/wiki/Post%E2%80%93Turing_machine" title="Post–Turing machine">Post–Turing machines</a></li>/n<li><a href="/wiki/Process_calculus" title="Process calculus">Process calculus</a></li>/n</ul>/n<p>Most <a href="/wiki/Programming_language" title="Programming language">programming languages</a>, conventional and unconventional, are Turing-complete. This includes:</p>/n<ul>/n<li>All general-purpose languages in wide use./n<ul>/n<li><a href="/wiki/Procedural_programming" title="Procedural programming">Procedural programming</a> languages such as <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>, <a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a>.</li>/n<li><a href="/wiki/Object-oriented_programming_language" class="mw-redirect" title="Object-oriented programming language">Object-oriented</a> languages such as <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a> or <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a>.</li>/n<li><a href="/wiki/Multi-paradigm_programming_language" class="mw-redirect" title="Multi-paradigm programming language">Multi-paradigm</a> languages such as <a href="/wiki/Ada_(programming_language)" title="Ada (programming language)">Ada</a>, <a href="/wiki/C%2B%2B" title="C++">C++</a>, <a href="/wiki/Common_Lisp" title="Common Lisp">Common Lisp</a>, <a href="/wiki/Object_Pascal" title="Object Pascal">Object Pascal</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/R_(programming_language)" title="R (programming language)">R</a>.</li>/n</ul>/n</li>/n<li>Most languages using less common paradigms/n<ul>/n<li><a href="/wiki/Functional_programming" title="Functional programming">Functional</a> languages such as <a href="/wiki/Lisp_(programming_language)" title="Lisp (programming language)">Lisp</a> and <a href="/wiki/Haskell_(programming_language)" title="Haskell (programming language)">Haskell</a>.</li>/n<li><a href="/wiki/Logic_programming" title="Logic programming">Logic programming</a> languages such as <a href="/wiki/Prolog" title="Prolog">Prolog</a>.</li>/n<li><a href="/wiki/Declarative_programming" title="Declarative programming">Declarative</a> languages such as <a href="/wiki/XSLT" title="XSLT">XSLT</a>.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[2]</a></sup></li>/n<li><a href="/wiki/Esoteric_programming_language" title="Esoteric programming language">Esoteric programming languages</a>, a form of <a href="/wiki/Mathematical_recreation" class="mw-redirect" title="Mathematical recreation">mathematical recreation</a> in which programmers work out how to achieve basic programming constructs in an extremely difficult but mathematically Turing-equivalent language.</li>/n</ul>/n</li>/n</ul>/n<p><a href="/wiki/Rewrite_system" class="mw-redirect" title="Rewrite system">Rewrite systems</a> are also Turing-complete.</p>/n<p>Turing completeness is an abstract statement of ability, rather than a prescription of specific language features used to implement that ability. The features used to achieve Turing completeness can be quite different; <a href="/wiki/Fortran" title="Fortran">Fortran</a> systems would use loop constructs or possibly even <a href="/wiki/Goto" title="Goto">goto</a> statements to achieve repetition; Haskell and Prolog, lacking looping almost entirely, would use <a href="/wiki/Recursion" title="Recursion">recursion</a>. Most programming languages are describing computations on von Neumann architectures, which have memory (RAM and register) and a control unit. These two elements make this architecture Turing-complete. Even pure <a href="/wiki/Functional_language" class="mw-redirect" title="Functional language">functional languages</a> are Turing-complete.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[3]</a></sup></p>/n<p>Turing completeness in declarative <a href="/wiki/SQL" title="SQL">SQL</a> is implemented through recursive common table expressions.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[4]</a></sup> Unsurprisingly, procedural extensions to SQL (<a href="/wiki/PLSQL" class="mw-redirect" title="PLSQL">PLSQL</a>, etc.) are also Turing complete. This illustrates one reason why relatively powerful non-Turing-complete languages are rare: the more powerful the language is initially, the more complex are the tasks to which it is applied and the sooner its lack of completeness becomes perceived as a drawback, encouraging its extension until it is Turing complete.</p>/n<p>The untyped <a href="/wiki/Lambda_calculus" title="Lambda calculus">lambda calculus</a> is Turing-complete, but many typed lambda calculi, including <a href="/wiki/System_F" title="System F">System F</a>, are not. The value of typed systems is based in their ability to represent most typical computer programs while detecting more errors.</p>/n<p><a href="/wiki/Rule_110" title="Rule 110">Rule 110</a> and <a href="/wiki/Conway%27s_Game_of_Life" title="Conway's Game of Life">Conway's Game of Life</a>, both <a href="/wiki/Cellular_automaton" title="Cellular automaton">cellular automata</a>, are Turing complete.</p>/n<h3><span class="mw-headline" id="Unintentional_Turing_completeness">Unintentional Turing completeness</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a Gathering</a></i><sup id="cite_ref-Accidentally_Turing-Complete_7-2" class="reference"><a href="#cite_note-Accidentally_Turing-Complete-7">[6]</a></sup></li>/n</ul>/n<p>Zero-person games (simulations):</p>/n<ul>/n<li><i><a href="/wiki/Conway%27s_Game_of_Life" title="Conway's Game of Life">Conway's Game of Life</a></i><sup id="cite_ref-9" class="reference"><a href="#cite_note-9">[8]</a></sup><sup id="cite_ref-10" class="reference"><a href="#cite_note-10">[9]</a></sup></li>/n</ul>/n<p><br /></p>/n<h2><span class="mw-headline" id="Non-Turing-complete_languages">Non-Turing-complete languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_completeness&amp;action=edit&amp;section=9" title="Edit section: Non-Turing-complete languages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Many computational languages exist that are not Turing complete. One such example is the set of <a href="/wiki/Regular_language" title="Regular language">regular languages</a>, which are generated by <a href="/wiki/Regular_expression" title="Regular expression">regular expressions</a> and which are recognized by <a href="/wiki/Finite_state_machine" class="mw-redirect" title="Finite state machine">finite automata</a>. A more powerful but still not Turing-complete extension of finite automata is the category of <a href="/wiki/Pushdown_automaton" title="Pushdown automaton">pushdown automata</a> and <a href="/wiki/Context-free_grammar" title="Context-free grammar">context-free grammars</a>, which are commonly used to generate parse trees in an initial stage of program <a href="/wiki/Compiler" title="Compiler">compiling</a>. Further examples include some of the early versions of the pixel shader languages embedded in <a href="/wiki/Direct3D" title="Direct3D">Direct3D</a> and <a href="/wiki/OpenGL" title="OpenGL">OpenGL</a> extensions.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (December 2010)">citation needed</span></a></i>]</sup></p>/n<p>In <a href="/wiki/Total_functional_programming" title="Total functional programming">total functional programming</a> languages, such as <a href="/wiki/Charity_(programming_language)" title="Charity (programming language)">Charity</a> and <a href="/wiki/Epigram_(programming_language)" title="Epigram (programming language)">Epigram</a>, all functions are total and must terminate. Charity uses a type system and <a href="/wiki/Control_flow" title="Control flow">control constructs</a> based on <a href="/wiki/Category_theory" title="Category theory">category theory</a>, whereas Epigram uses <a href="/wiki/Dependent_type" title="Dependent type">dependent types</a>. The <a href="/wiki/LOOP_(programming_language)" title="LOOP (programming language)">LOOP</a> language is designed so that it computes only the functions that are <a href="/wiki/Primitive_recursive_function" title="Primitive recursive function">primitive recursive</a>. All of these compute proper subsets of the total computable functions, since the full set of total computable functions is not <a href="/wiki/Recursively_enumerable_set" title="Recursively enumerable set">computably enumerable</a>. Also, since all functions in these languages are total, algorithms for <a href="/wiki/Recursively_enumerable_set" title="Recursively enumerable set">recursively enumerable sets</a> cannot be written in these languages, in contrast with Turing machines.</p>/n<p>Although (untyped) <a href="/wiki/Lambda_calculus" title="Lambda calculus">lambda calculus</a> is Turing-complete, <a href="/wiki/Simply_typed_lambda_calculus" title="Simply typed lambda calculus">simply typed lambda calculus</a> is not.</p>/n<h3><span class="mw-headline" id="Data_languages">Data languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Turing_completeness&amp;action=edit&amp;section=10" title="Edit section: Data languages">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>The notion of Turing-completeness does not apply to languages such as <a href="/wiki/XML" title="XML">XML</a>, <a href="/wiki/HTML" title="HTML">HTML</a>, <a href="/wiki/JSON" title="JSON">JSON</a>, <a href="/wiki/YAML" title="YAML">YAML</a> and <a href="/wiki/S-expression" title="S-expression">S-expressions</a>, because they are typically used to represent structured data, not describe computation. These are sometimes referred to as <a href="/wiki/Markup_language" title="Markup language">markup languages</a>, or more properly as "container languages" or "data description languages". However, <a href="/wiki/Rule_110" title="Rule 110">Rule 110</a>, a Turing-complete cellular automaton, has been successfully implemented in <a href="/wiki/CSS_3" class="mw-redirect" title="CSS 3">CSS 3</a>, thus proving, to some extent, its Turing completeness.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">[10]</a></sup></p>/n<h2><span class="mw-headline" id="See_also">See
