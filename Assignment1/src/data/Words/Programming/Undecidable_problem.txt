id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p>In <a href="/wiki/Computability_theory" title="Computability theory">computability theory</a> and <a href="/wiki/Computational_complexity_theory" title="Computational complexity theory">computational complexity theory</a>, an <b>undecidable problem</b> is a <a href="/wiki/Decision_problem" title="Decision problem">decision problem</a> for which it is known to be impossible to construct a single <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> that always leads to a correct yes-or-no answer. The <a href="/wiki/Halting_problem" title="Halting problem">halting problem</a> is an example: there is no algorithm that correctly determines whether arbitrary programs eventually halt when run.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#Background"><span class="tocnumber">1</span> <span class="toctext">Background</span></a></li>/n<li class="toclevel-1 tocsection-2"><a href="#Example:_the_halting_problem_in_computability_theory"><span class="tocnumber">2</span> <span class="toctext">Example: the halting problem in computability theory</span></a></li>/n<li class="toclevel-1 tocsection-3"><a href="#Relationship_with_G.C3.B6del.27s_incompleteness_theorem"><span class="tocnumber">3</span> <span class="toctext">Relationship with GÃ¶del's incompleteness theorem</span></a></li>/n<li class="toclevel-1 tocsection-4"><a href="#Examples_of_undecidable_problems"><span class="tocnumber">4</span> <span class="toctext">Examples of undecidable problems</span></a></li>/n<li class="toclevel-1 tocsection-5"><a href="#Examples_of_undecidable_statements"><span class="tocnumber">5</span> <span class="toctext">Examples of undecidable statements</span></a></li>/n<li class="toclevel-1 tocsection-6"><a href="#See_also"><span class="tocnumber">6</span> <span class="toctext">See also</span></a></li>/n<li class="toclevel-1 tocsection-7"><a href="#References"><span class="tocnumber">7</span> <span class="toctext">References</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="Background">Background</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Undecidable_problem&amp;action=edit&amp;section=1" title="Edit section: Background">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>A decision problem is any arbitrary yes-or-no question on an infinite set of inputs. Because of this, it is traditional to define the decision problem equivalently as the set of inputs for which the problem returns <i>yes</i>. These inputs can be natural numbers, but also other values of some other kind, such as <a href="/wiki/String_(computer_science)" title="String (computer science)">strings</a> of a <a href="/wiki/Formal_language" title="Formal language">formal language</a>. Using some encoding, such as a <a href="/wiki/G%C3%B6del_numbering" title="GÃ¶del numbering">GÃ¶del numbering</a>, the strings can be encoded as natural numbers. Thus, a decision problem informally phrased in terms of a formal language is also equivalent to a set of <a href="/wiki/Natural_numbers" class="mw-redirect" title="Natural numbers">natural numbers</a>. To keep the formal definition simple, it is phrased in terms of subsets of the natural numbers.</p>/n<p>Formally, a decision problem is a subset of the natural numbers. The corresponding informal problem is that of deciding whether a given number is in the set. A decision problem <i>A</i> is called decidable or effectively solvable if <i>A</i> is a <a href="/wiki/Recursive_set" title="Recursive set">recursive set</a>. A problem is called partially decidable, <b>semi-decidable</b>, solvable, or provable if <i>A</i> is a <a href="/wiki/Recursively_enumerable_set" title="Recursively enumerable set">recursively enumerable set</a>. This means that there exists an algorithm that halts eventually when the answer is <i>yes</i> but may run for ever if the answer is <i>no</i>. Partially decidable problems and any other problems that are not decidable are called undecidable.</p>/n<h2><span class="mw-headline" id="Example:_the_halting_problem_in_computability_theory">Example: the halting problem in computability theory</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Undecidable_problem&amp;action=edit&amp;section=2" title="Edit section: Example: the halting problem in computability theory">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>In <a href="/wiki/Computability_theory" title="Computability theory">computability theory</a>, the <a href="/wiki/Halting_problem" title="Halting problem">halting problem</a> is a <a href="/wiki/Decision_problem" title="Decision problem">decision problem</a> which can be stated as follows:</p>/n<dl>/n<dd>Given the description of an arbitrary <a href="/wiki/Computer_program" title="Computer program">program</a> and a finite input, decide whether the program finishes running or will run forever.</dd>/n</dl>/n<p><a href="/wiki/Alan_Turing" title="Alan Turing">Alan Turing</a> proved in 1936 that a general <a href="/wiki/Algorithm" title="Algorithm">algorithm</a> running on a <a href="/wiki/Turing_machine" title="Turing machine">Turing machine</a> that solves the halting problem for <i>all</i> possible program-input pairs necessarily cannot exist. Hence, the halting problem is <i>undecidable</i> for Turing machines.</p>/n<h2><span id="Relationship_with_GÃ¶del's_incompleteness_theorem"></span><span class="mw-headline" id="Relationship_with_G.C3.B6del.27s_incompleteness_theorem">Relationship with GÃ¶del's incompleteness theorem</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Undecidable_problem&amp;action=edit&amp;section=3" title="Edit section: Relationship with GÃ¶del's incompleteness theorem">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The concepts raised by <a href="/wiki/G%C3%B6del%27s_incompleteness_theorem" class="mw-redirect" title="GÃ¶del's incompleteness theorem">GÃ¶del's incompleteness theorems</a> are very similar to those raised by the halting problem, and the proofs are quite similar. In fact, a weaker form of the First Incompleteness Theorem is an easy consequence of the undecidability of the halting problem. This weaker form differs from the standard statement of the incompleteness theorem by asserting that a complete, <a href="/wiki/Consistency_proof" class="mw-redirect" title="Consistency proof">consistent</a> and <a href="/wiki/Soundness" title="Soundness">sound</a> <a href="/wiki/Axiomatization" class="mw-redirect" title="Axiomatization">axiomatization</a> of all statements about natural numbers is unachievable. The "sound" part is the weakening: it means that we require the axiomatic system in question to prove only <i>true</i> statements about natural numbers. It is important to observe that the statement of the standard form of GÃ¶del's First Incompleteness Theorem is completely unconcerned with the question of truth, but only concerns the issue of whether it can be <a href="/wiki/Mathematical_proof" title="Mathematical proof">proven</a>.</p>/n<p>The weaker form of the theorem can be proved from the undecidability of the halting problem as follows. Assume that we have a consistent and complete <a href="/wiki/Axiomatization" class="mw-redirect" title="Axiomatization">axiomatization</a> of all true <a href="/wiki/First-order_logic" title="First-order logic">first-order logic</a> statements about <a href="/wiki/Natural_number" title="Natural number">natural numbers</a>. Then we can build an algorithm that enumerates all these statements. This means that there is an algorithm <i>N</i>(<i>n</i>) that, given a natural number <i>n</i>, computes a true first-order logic statement about natural numbers such that, for all the true statements, there is at least one <i>n</i> such that <i>N</i>(<i>n</i>) yields that statement. Now suppose we want to decide if the algorithm with representation <i>a</i> halts on input <i>i</i>. We know that this statement can be expressed with a first-order logic statement, say <i>H</i>(<i>a</i>, <i>i</i>). Since the axiomatization is complete it follows that either there is an <i>n</i> such that <i>N</i>(<i>n</i>) = <i>H</i>(<i>a</i>, <i>i</i>) or there is an <i>n'</i> such that <i>N</i>(<i>n'</i>) = Â¬ <i>H</i>(<i>a</i>, <i>i</i>). So if we <a href="/wiki/Iterate" class="mw-redirect" title="Iterate">iterate</a> over all <i>n</i> until we either find <i>H</i>(<i>a</i>, <i>i</i>) or its negation, we will always halt. This means that this gives us an algorithm to decide the halting problem. Since we know that there cannot be such an algorithm, it follows that the assumption that there is a consistent and complete axiomatization of all true first-order logic statements about natural numbers must be false.</p>/n<h2><span class="mw-headline" id="Examples_of_undecidable_problems">Examples of undecidable problems</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Undecidable_problem&amp;action=edit&amp;section=4" title="Edit section: Examples of undecidable problems">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/List_of_undecidable_problems" title="List of undecidable problems">List of undecidable problems</a></div>/n<p>Undecidable problems can be related to different topics, such as <a href="/wiki/Logic" title="Logic">logic</a>, <a href="/wiki/Abstract_machine" title="Abstract machine">abstract machines</a> or <a href="/wiki/Topology" title="Topology">topology</a>. Note that since there are <a href="/wiki/Uncountable_set" title="Uncountable set">uncountably</a> many undecidable problems, any list, even one of <a href="/wiki/Countably_infinite" class="mw-redirect" title="Countably infinite">infinite length</a>, is necessarily incomplete.</p>/n<h2><span class="mw-headline" id="Examples_of_undecidable_statements">Examples of undecidable statements</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Undecidable_problem&amp;action=edit&amp;section=5" title="Edit section: Examples of undecidable statements">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div role="note" class="hatnote navigation-not-searchable">See also: <a href="/wiki/List_of_statements_independent_of_ZFC" title="List of statements independent of ZFC">List of statements independent of ZFC</a> and <a href="/wiki/Independence_(mathematical_logic)" title="Independence (mathematical logic)">Independence (mathematical logic)</a></div>/n<p>There are two distinct senses of the word "undecidable" in contemporary use. The first of these is the sense used in relation to GÃ¶del's theorems, that of a statement being neither provable nor refutable in a specified <a href="/wiki/Deductive_system" class="mw-redirect" title="Deductive system">deductive system</a>. The second sense is used in relation to <a href="/wiki/Computability_theory" title="Computability theory">computability theory</a> and applies not to statements but to <a href="/wiki/Decision_problem" title="Decision problem">decision problems</a>, which are countably infinite sets of questions each requiring a yes or no answer. Such a problem is said to be undecidable if there is no <a href="/wiki/Computable_function" title="Computable function">computable function</a> that correctly answers every question in the problem set. The connection between these two is that if a decision problem is undecidable (in the recursion theoretical sense) then there is no consistent, effective <a href="/wiki/Formal_system" title="Formal system">formal system</a> which proves for every question <i>A</i> in the problem either "the answer to <i>A</i> is yes" or "the answer to <i>A</i> is no".</p>/n<p>Because of the two meanings of the word undecidable, the term <a href="/wiki/Independence_(mathematical_logic)" title="Independence (mathematical logic)">independent</a> is sometimes used instead of undecidable for the "neither provable nor refutable" sense. The usage of "independent" is also ambiguous, however. It can mean just "not provable", leaving open whether an independent statement might be refuted.</p>/n<p>Undecidability of a statement in a particular deductive system does not, in and of itself, address the question of whether the <a href="/wiki/Truth_value" title="Truth value">truth value</a> of the statement is well-defined, or whether it can be determined by other means. Undecidability only implies that the particular deductive system being considered does not prove the truth or falsity of the statement. Whether there exist so-called "absolutely undecidable" statements, whose truth value can never be known or is ill-specified, is a controversial point among various <a href="/wiki/Philosophy_of_mathematics" title="Philosophy of mathematics">philosophical schools</a>.</p>/n<p>One of the first problems suspected to be undecidable, in the second sense of the term, was the <a href="/wiki/Word_problem_for_groups" title="Word problem for groups">word problem for groups</a>, first posed by <a href="/wiki/Max_Dehn" title="Max Dehn">Max Dehn</a> in 1911, which asks if there is a finitely presented <a href="/wiki/Group_(mathematics)" title="Group (mathematics)">group</a> for which no algorithm exists to determine whether two words are equivalent. This was shown to be the case in 1952.</p>/n<p>The combined work of GÃ¶del and <a href="/wiki/Paul_Cohen_(mathematician)" class="mw-redirect" title="Paul Cohen (mathematician)">Paul Cohen</a> has given two concrete examples of undecidable statements (in the first sense of the term): The <a href="/wiki/Continuum_hypothesis" title="Continuum hypothesis">continuum hypothesis</a> can neither be proved nor refuted in <a href="/wiki/ZFC" class="mw-redirect" title="ZFC">ZFC</a> (the standard axiomatization of <a href="/wiki/Set_theory" title="Set theory">set theory</a>), and the <a href="/wiki/Axiom_of_choice" title="Axiom of choice">axiom of choice</a> can neither be proved nor refuted in <a href="/wiki/Zermelo-Fraenkel_set_theory" class="mw-redirect" title="Zermelo-Fraenkel set theory">ZF</a> (which is all the ZFC axioms <i>except</i> the axiom of choice). These results do not require the incompleteness theorem. GÃ¶del proved in 1940 that neither of these statements could be disproved in ZF or ZFC set theory. In the 1960s, Cohen proved that neither is provable from ZF, and the continuum hypothesis cannot be proven from ZFC.</p>/n<p>In 1970, Russian mathematician <a href="/wiki/Yuri_Matiyasevich" title="Yuri Matiyasevich">Yuri Matiyasevich</a> showed that <a href="/wiki/Hilbert%27s_tenth_problem" title="Hilbert's tenth problem">Hilbert's Tenth Problem</a>, posed in 1900 as a challenge to the next century of mathematicians, cannot be solved. Hilbert's challenge sought an algorithm which finds all solutions of a <a href="/wiki/Diophantine_equation" title="Diophantine equation">Diophantine equation</a>. A Diophantine equation is a more general case of <a href="/wiki/Fermat%27s_Last_Theorem" title="Fermat's Last Theorem">Fermat's Last Theorem</a>; we seek the <a href="/wiki/Integer_number" class="mw-redirect" title="Integer number">integer roots</a> of a <a href="/wiki/Polynomial" title="Polynomial">polynomial</a> in any number of variables with integer coefficients. Since we have only one equation but <i>n</i> variables, infinitely many solutions exist (and are easy to find) in the <a href="/wiki/Complex_plane" title="Complex plane">complex plane</a>; however, the problem becomes impossible if solutions are constrained to integer values only. Matiyasevich showed this problem to be unsolvable by mapping a Diophantine equation to a <a href="/wiki/Recursively_enumerable_set" title="Recursively enumerable set">recursively enumerable set</a> and invoking GÃ¶del's Incompleteness Theorem.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup></p>/n<p>In 1936, <a href="/wiki/Alan_Turing" title="Alan Turing">Alan Turing</a> proved that the <a href="/wiki/Halting_problem" title="Halting problem">halting problem</a>âthe question of whether or not a <a href="/wiki/Turing_machine" title="Turing machine">Turing machine</a> halts on a given programâis undecidable, in the second sense of the term. This result was later generalized by <a href="/wiki/Rice%27s_theorem" title="Rice's theorem">Rice's theorem</a>.</p>/n<p>In 1973, the <a href="/wiki/Whitehead_problem" title="Whitehead problem">Whitehead problem</a> in <a href="/wiki/Group_theory" title="Group theory">group theory</a> was shown to be undecidable, in the first sense of the term, in standard set theory.</p>/n<p>In 1977, Paris and Harrington proved that the <a href="/wiki/Paris-Harrington_theorem" class="mw-redirect" title="Paris-Harrington theorem">Paris-Harrington principle</a>, a version of the <a href="/wiki/Ramsey_theorem" class="mw-redirect" title="Ramsey theorem">Ramsey theorem</a>, is undecidable in the axiomatization of arithmetic given by the Peano axioms but can be proven to be true in the larger system of <a href="/wiki/Second-order_arithmetic" title="Second-order arithmetic">second-order arithmetic</a>.</p>/n<p><a href="/wiki/Kruskal%27s_tree_theorem" title="Kruskal's tree theorem">Kruskal's tree theorem</a>, which has applications in computer science, is also undecidable from the Peano axioms but provable in set theory. In fact Kruskal's tree theorem (or its finite form) is undecidable in a much stronger system codifying the principles acceptable on basis of a philosophy of mathematics called predicativism.</p>/n<p><a href="/wiki/Goodstein%27s_theorem" title="Goodstein's theorem">Goodstein's theorem</a> is a statement about the <a href="/wiki/Ramsey_theory" title="Ramsey theory">Ramsey theory</a> of the natural numbers that Kirby and Paris showed is undecidable in Peano arithmetic.</p>/n<p><a href="/wiki/Gregory_Chaitin" title="Gregory Chaitin">Gregory Chaitin</a> produced undecidable statements in <a href="/wiki/Algorithmic_information_theory" title="Algorithmic information theory">algorithmic information theory</a> and proved another incompleteness theorem in that setting. Chaitin's theorem states that for any theory that can represent enough arithmetic, there is an upper bound <i>c</i> such that no specific number can be proven in that theory to have <a href="/wiki/Kolmogorov_complexity" title="Kolmogorov complexity">Kolmogorov complexity</a> greater than <i>c</i>. While GÃ¶del's theorem is related to the <a href="/wiki/Liar_paradox" title="Liar paradox">liar paradox</a>, Chaitin's result is related to <a href="/wiki/Berry%27s_paradox" class="mw-redirect" title="Berry's paradox">Berry's paradox</a>.</p>/n<p>In 2007, researchers Kurtz and Simon, building on earlier work by <a href="/wiki/John_Horton_Conway" title="John Horton Conway">J.H.
