Mercury</h1>/n									<div id="bodyContent" class="mw-body-content">/n									<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>/n								<div id="contentSub"></div>/n												<div id="jump-to-nav" class="mw-jump">/n					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>/n				</div>/n				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p>The <b>Mercury</b> was an early commercial <a href="/wiki/Computer" title="Computer">computer</a> from the mid 1950s built by <a href="/wiki/Ferranti" title="Ferranti">Ferranti</a>. It was the successor to the <a href="/wiki/Ferranti_Mark_1" title="Ferranti Mark 1">Ferranti Mark 1</a>, adding a <a href="/wiki/Floating_point_unit" class="mw-redirect" title="Floating point unit">floating point unit</a> for improved performance, and increased reliability by replacing the <a href="/wiki/Williams_tube" title="Williams tube">Williams tube</a> memory with <a href="/wiki/Core_memory" class="mw-redirect" title="Core memory">core memory</a> and using more solid state components. The computer had roughly 2000 <a href="/wiki/Vacuum_tube" title="Vacuum tube">vacuum tubes</a> (mostly type CV2179/A2134 <a href="/wiki/Pentode" title="Pentode">pentodes</a>, EL81 pentodes and CV2493/ECC88 double triodes) and 2000 germanium diodes. Nineteen Mercuries were sold before Ferranti moved on to newer designs.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#Predecessor:_Mark_I"><span class="tocnumber">1</span> <span class="toctext">Predecessor: Mark I</span></a></li>/n<li class="toclevel-1 tocsection-2"><a href="#Prototype:_Meg"><span class="tocnumber">2</span> <span class="toctext">Prototype: Meg</span></a></li>/n<li class="toclevel-1 tocsection-3"><a href="#Commercial_version:_Mercury"><span class="tocnumber">3</span> <span class="toctext">Commercial version: Mercury</span></a></li>/n<li class="toclevel-1 tocsection-4"><a href="#See_also"><span class="tocnumber">4</span> <span class="toctext">See also</span></a></li>/n<li class="toclevel-1 tocsection-5"><a href="#References"><span class="tocnumber">5</span> <span class="toctext">References</span></a></li>/n<li class="toclevel-1 tocsection-6"><a href="#External_links"><span class="tocnumber">6</span> <span class="toctext">External links</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="Predecessor:_Mark_I">Predecessor: Mark I</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ferranti_Mercury&amp;action=edit&amp;section=1" title="Edit section: Predecessor: Mark I">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Manchester_Mark_1" title="Manchester Mark 1">Manchester Mark 1</a></div>/n<p>When the Mark I started running in 1951, reliability was poor. The primary concern was the <a href="/wiki/Drum_memory" title="Drum memory">drum memory</a> system, which broke down all the time. Additionally, the machine used 4,200 <a href="/wiki/Vacuum_tube" title="Vacuum tube">thermionic valves</a>, mostly <a href="/wiki/EF50" title="EF50">EF50</a> pentodes and diodes that had to be replaced constantly. The Williams tubes, used as <a href="/wiki/Random_access_memory" class="mw-redirect" title="Random access memory">random access memory</a> and registers, were reliable but required constant maintenance. As soon as the system went into operation, teams started looking at solutions to these problems.</p>/n<p>One team decided to produce a much smaller and more cost-effective system built entirely with <a href="/wiki/Transistor" title="Transistor">transistors</a>. It first ran in November 1953 and is believed to be the first entirely transistor-based computer. <a href="/wiki/Metropolitan-Vickers" title="Metropolitan-Vickers">Metropolitan-Vickers</a> later built this commercially as the <a href="/wiki/Metrovick_950" title="Metrovick 950">Metrovick 950</a>, delivering seven.</p>/n<h2><span class="mw-headline" id="Prototype:_Meg">Prototype: Meg</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ferranti_Mercury&amp;action=edit&amp;section=2" title="Edit section: Prototype: Meg">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Another team, including the main designers of the Mark I, started with a design very similar to the Mark I but replacing valves used as <a href="/wiki/Diode" title="Diode">diodes</a> with <a href="/wiki/Solid_state_(electronics)" class="mw-redirect" title="Solid state (electronics)">solid state</a> diodes. At that time computers were used almost always in the sciences, and they decided to add a <a href="/wiki/Floating-point" class="mw-redirect" title="Floating-point">floating-point</a> unit to greatly improve performance in this role. Additionally the machine was to run at 1&#160;MHz, eight times faster than the Mark I's 125&#160;kHz, leading to the use of the name <i>megacycle machine</i>, and eventually <b>Meg</b>.</p>/n<p>Meg first ran in May 1954. The use of solid-state diodes reduced valve count by well over half, reducing the power requirement from the Mark I's 25&#160;kW to the Meg's 12&#160;kW. Like the Mark I, Meg was based on a 10-bit "short word", combining two to form a 20-bit address and four to make a 40-bit integer. This was a result of the physical properties of the Williams tubes, which were used to make eight <i>B-lines</i>, or in modern terminology, <a href="/wiki/Accumulator_(computing)" title="Accumulator (computing)">accumulator</a>/<a href="/wiki/Index_register" title="Index register">index registers</a>. Meg could multiply two integers in about 60 microseconds. The floating-point unit used three words for a 30-bit mantissa, and another as a 10-bit exponent. It could add two floating-point numbers in about 180 microseconds, and multiply them in about 360&#160;µs.</p>/n<h2><span class="mw-headline" id="Commercial_version:_Mercury">Commercial version: Mercury</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Ferranti_Mercury&amp;action=edit&amp;section=3" title="Edit section: Commercial version: Mercury">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Ferranti, which had built the Mark I for the university, continued development of the prototype Meg to produce the Mercury. The main change was to replace the Williams tubes with core memory. Although slower to access, at about 10&#160;µs for a 10-bit short word, the system required virtually no maintenance, considerably more important for commercial users. 1024×40-bits of core were provided, backed by four drums each holding 4096×40-bits. Much detailed information both about the Mercury hardware and the <a href="/wiki/Autocode" title="Autocode">Autocode</a> coding system is included in a downloadable Spanish-language Autocode manual.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup></p>/n<p>The first of an eventual 19 Mercury computers was delivered in August 1957. Manchester University received one in February 1958, leasing half the time to commercial users via Ferranti's business unit. Both <a href="/wiki/CERN" title="CERN">CERN</a> at Geneva <sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> and the <a href="/wiki/Atomic_Energy_Research_Establishment" title="Atomic Energy Research Establishment">Atomic Energy Research Establishment</a> at Harwell also installed theirs in 1958. A Mercury bought in 1959 was the UK <a href="/wiki/Met_Office" title="Met Office">Met Office</a>'s first computer.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup> The <a href="/wiki/University_of_Buenos_Aires" title="University of Buenos Aires">University of Buenos Aires</a> in Argentina received another one in 1960.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup></p>/n<p>The machine could run Mercury Autocode, a simplified coding system of the type later described as a <a href="/wiki/High-level_programming_language" title="High-level programming language">high-level programming language</a>.</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span
