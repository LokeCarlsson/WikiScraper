anywhere</h1>/n									<div id="bodyContent" class="mw-body-content">/n									<div id="siteSub" class="noprint">From Wikipedia, the free encyclopedia</div>/n								<div id="contentSub"></div>/n												<div id="jump-to-nav" class="mw-jump">/n					Jump to:					<a href="#mw-head">navigation</a>, 					<a href="#p-search">search</a>/n				</div>/n				<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p>"<b>Write once, run anywhere</b>" (<b>WORA</b>), or sometimes <i>write once, run everywhere</i> (<b>WORE</b>), is a slogan created by <a href="/wiki/Sun_Microsystems" title="Sun Microsystems">Sun Microsystems</a> to illustrate the <a href="/wiki/Cross-platform" title="Cross-platform">cross-platform</a> benefits of the <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java language</a>.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup><sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> Ideally, this means Java can be developed on any device, compiled into a standard <a href="/wiki/Bytecode" title="Bytecode">bytecode</a> and be expected to run on any device equipped with a <a href="/wiki/Java_virtual_machine" title="Java virtual machine">Java virtual machine</a> (JVM). The installation of a JVM or Java interpreter on chips, devices or software packages has become an industry standard practice.</p>/n<p>This means a programmer can develop code on a PC and can expect it to run on Java enabled <a href="/wiki/Cell_phone" class="mw-redirect" title="Cell phone">cell phones</a>, as well as on <a href="/wiki/Router_(computing)" title="Router (computing)">routers</a> and <a href="/wiki/Mainframe_computer" title="Mainframe computer">mainframes</a> equipped with Java, without any adjustments. This is intended to save software developers the effort of writing a different version of their software for each <a href="/wiki/Platform_(computing)" class="mw-redirect" title="Platform (computing)">platform</a> or <a href="/wiki/Operating_system" title="Operating system">operating system</a> they intend to deploy on.</p>/n<p>This idea originated as early as in the late 1970s, when the <a href="/wiki/UCSD_Pascal" title="UCSD Pascal">UCSD Pascal</a> system was developed to produce and interpret <a href="/wiki/P-code_machine" title="P-code machine">p-code</a>. UCSD Pascal (along with the Smalltalk virtual machine) was a key influence on the design of the <a href="/wiki/Java_virtual_machine" title="Java virtual machine">Java virtual machine</a>, as is cited by <a href="/wiki/James_Gosling" title="James Gosling">James Gosling</a>.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2010)">citation needed</span></a></i>]</sup></p>/n<p>The catch is that since there are multiple <a href="/wiki/Java_virtual_machine" title="Java virtual machine">JVM</a> implementations, on top of a wide variety of different operating systems such as <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Windows</a>, <a href="/wiki/Linux" title="Linux">Linux</a>, <a href="/wiki/Solaris_(operating_system)" title="Solaris (operating system)">Solaris</a>, <a href="/wiki/NetWare" title="NetWare">NetWare</a>, <a href="/wiki/HP-UX" title="HP-UX">HP-UX</a>, and <a href="/wiki/MacOS" title="MacOS">macOS</a>, there can be subtle differences in how a program may execute on each JVM/OS combination, which may require an application to be tested on various target platforms. This has given rise to a joke among Java developers, "Write Once, Debug Everywhere".<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup></p>/n<p>In comparison, the <a href="/wiki/Squeak" title="Squeak">Squeak</a> Smalltalk programming language and environment, boasts as being, "truly write once run anywhere",<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup> because it "runs bit-identical images across its wide portability base" <sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup>.</p>/n<p>As Squeak, MicroEJ provides a virtual execution environment (VEE) which guarantees one unique java thread policy across all implementations, insuring a true WORA semantic across millions of devices.</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a
