href="/wiki/Software_engineering" title="Software engineering">software engineering</a> is the process by which a <a href="/wiki/Computer" title="Computer">computer</a> or a <a href="/wiki/Virtual_machine" title="Virtual machine">virtual machine</a> performs the instructions of a <a href="/wiki/Computer_program" title="Computer program">computer program</a>. The instructions in the program trigger sequences of simple actions on the executing machine. Those actions produce effects according to the <a href="/wiki/Formal_semantics_of_programming_languages" class="mw-redirect" title="Formal semantics of programming languages">semantics</a> of the instructions in the program.</p>/n<p>Programs for a computer may execute in a <a href="/wiki/Batch_processing" title="Batch processing">batch process</a> without human interaction, or a <a href="/wiki/User_(computing)" title="User (computing)">user</a> may type <a href="/wiki/Command_(computing)" title="Command (computing)">commands</a> in an <a href="/wiki/Session_(computer_science)" title="Session (computer science)">interactive session</a> of an <a href="/wiki/Interpreter_(computing)" title="Interpreter (computing)">interpreter</a>. In this case the "commands" are simply programs, whose execution is chained together.</p>/n<p>The term <b>run</b> is used almost synonymously. A related meaning of both "to run" and "to execute" refers to the specific action of a user starting (or <i>launching</i> or <i>invoking</i>) a program, as in "Please run the application."</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="toctitle">/n<h2>Contents</h2>/n</div>/n<ul>/n<li class="toclevel-1 tocsection-1"><a href="#Context_of_execution"><span class="tocnumber">1</span> <span class="toctext">Context of execution</span></a></li>/n<li class="toclevel-1 tocsection-2"><a href="#Process"><span class="tocnumber">2</span> <span class="toctext">Process</span></a></li>/n<li class="toclevel-1 tocsection-3"><a href="#Interpreter"><span class="tocnumber">3</span> <span class="toctext">Interpreter</span></a></li>/n<li class="toclevel-1 tocsection-4"><a href="#See_also"><span class="tocnumber">4</span> <span class="toctext">See also</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="Context_of_execution">Context of execution</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Execution_(computing)&amp;action=edit&amp;section=1" title="Edit section: Context of execution">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The context in which execution takes place is crucial. Very few programs execute on a <a href="/wiki/Bare_machine" title="Bare machine">bare machine</a>. Programs usually contain implicit and explicit assumptions about resources available at the time of execution. Most programs execute with the support of an <a href="/wiki/Operating_system" title="Operating system">operating system</a> and <a href="/wiki/Runtime_library" title="Runtime library">run-time libraries</a> specific to the source language that provide crucial services not supplied directly by the computer itself. This supportive environment, for instance, usually decouples a program from direct manipulation of the computer peripherals, providing more general, abstract services instead.</p>/n<h2><span class="mw-headline" id="Process">Process</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Execution_(computing)&amp;action=edit&amp;section=2" title="Edit section: Process">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div role="note" class="hatnote navigation-not-searchable">Further information: <a href="/wiki/Program_lifecycle_phase" title="Program lifecycle phase">Program lifecycle phase</a></div>/n<p>Prior to execution, a program must first be written. This is generally done in <a href="/wiki/Source_code" title="Source code">source code</a>, which is then compiled at <a href="/wiki/Compile_time" title="Compile time">compile time</a> (and statically linked at <a href="/wiki/Link_time" title="Link time">link time</a>) to an executable. This executable is then invoked, most often by an operating system, which loads the program into memory (<a href="/wiki/Load_time" class="mw-redirect" title="Load time">load time</a>), possibly performs <a href="/wiki/Dynamic_linking" class="mw-redirect" title="Dynamic linking">dynamic linking</a>, and then begins execution by moving control to the <a href="/wiki/Entry_point" title="Entry point">entry point</a> of the program; all these steps depend on the <a href="/wiki/Application_Binary_Interface" class="mw-redirect" title="Application Binary Interface">Application Binary Interface</a> of the operating system. At this point execution begins and the program enters <a href="/wiki/Run_time_(program_lifecycle_phase)" title="Run time (program lifecycle phase)">run time</a>. The program then runs until it ends, either normal termination or a <a href="/wiki/Crash_(computing)" title="Crash (computing)">crash</a>.</p>/n<h2><span class="mw-headline" id="Interpreter">Interpreter</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Execution_(computing)&amp;action=edit&amp;section=3" title="Edit section: Interpreter">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>A system that executes a program is called an <a href="/wiki/Interpreter_(computing)" title="Interpreter (computing)">interpreter</a> of the program. Loosely speaking, an interpreter actually does what the program says to do. This contrasts with a language translator that converts a program from one language to another. The most common language translators are <a href="/wiki/Compiler" title="Compiler">compilers</a>. Translators typically convert their source from a high-level, human readable language into a lower-level language (sometimes as low as native <a href="/wiki/Machine_code" title="Machine code">machine code</a>) that is simpler and faster for the <a href="/wiki/CPU" class="mw-redirect" title="CPU">processor</a> to directly execute. The idea is that the ratio of executions to translations of a program will be large; that is, a program need only be compiled once and can be run any number of times. This can provide a large benefit for translation versus direct interpretation of the source language. One trade-off is that development time is increased, because of the compilation. In some cases, only the changed files must be recompiled. Then the executable needs to be relinked. For some changes, the executable must be rebuilt from scratch. As computers and compilers become faster, this fact becomes less of an obstacle. Also, the speed of the end product is typically more important to the user than the development time.</p>/n<p>Translators usually produce an abstract result that is not completely ready to execute. Frequently, the operating system will convert the translator's <a href="/wiki/Object_code" title="Object code">object code</a> into the final executable form just before execution of the program begins.</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a
