Ultimately, in order to execute a program via compilation, it must be translated into a form that can serve as input to an interpreter.</p>/n<p>When a piece of computer hardware can interpret a programming language directly, that language is called <i><a href="/wiki/Machine_code" title="Machine code">machine code</a></i>. A so-called <i>native code compiler</i> is one that compiles a program into machine code. Actual compilation is often separated into multiple passes, like <a href="/wiki/Code_generation_(compiler)" title="Code generation (compiler)">code generation</a> (often for assembler language), <a href="/w/index.php?title=Assembling&amp;action=edit&amp;redlink=1" class="new" title="Assembling (page does not exist)">translator</a> (generating native code), <a href="/wiki/Linker_(computing)" title="Linker (computing)">linking</a>, loading and execution.</p>/n<p>If a compiler of a given <a href="/wiki/High_level_language" class="mw-redirect" title="High level language">high level language</a> produces another high level language, it is called <b><a href="/wiki/Translator_(computing)" class="mw-redirect" title="Translator (computing)">translator</a></b> (source to source translation), which is often useful to add extensions to existing languages or to exploit good and <a href="/wiki/Software_portability" title="Software portability">portable</a> implementation of other language (for example <a href="/wiki/C_(programming_language)" title="C (programming language)">C</a>), simplifying development.</p>/n<p>Many combinations of interpretation and compilation are possible, and many modern programming language implementations include elements of both. For example, the <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a> programming language is conventionally implemented by compilation into <a href="/wiki/Bytecode" title="Bytecode">bytecode</a>, which is then either interpreted or compiled by a <a href="/wiki/Virtual_machine" title="Virtual machine">virtual machine</a> (most popular ways is to use <a href="/wiki/Just-in-time_compilation" title="Just-in-time compilation">JIT</a> or <a href="/wiki/AOT_compiler" class="mw-redirect" title="AOT compiler">AOT compiler</a> compilation<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This statement is dubious. JIT compilation is a relatively new technique, and not one that is straightforward to implement. Personally, I'd guess that naïve interpretation or bytecode compilation would be the most common, since they are the simplest. Moreover, how is &quot;popularity&quot; here being measured? (April 2016)">citation needed</span></a></i>]</sup>). This implementation strategy has been copied by many languages since Smalltalk pioneered it in the 1970s and 1980s.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="I doubt Smalltalk pioneered this. BCPL with its O-code and Pascal with its P-code are both examples of this strategy that predated Smalltalk. (April 2016)">citation needed</span></a></i>]</sup><br /></p>/n<h2><span class="mw-headline" id="Abstract_machines">Abstract machines</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Programming_language_implementation&amp;action=edit&amp;section=1" title="Edit section: Abstract machines">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Every programming language requires abstract machines that bridge the gap between the high level language and the low level of a real machine.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This is a very big claim, and it is doubtful that every programming language has any formal notion of an abstract machine as part of its definition. (April 2016)">citation needed</span></a></i>]</sup></p>/n<ul>/n<li>Abstract machines for object-oriented programming languages. These are basically stack based machines to access data and method of program.</li>/n<li>Abstract machines for string processing</li>/n<li>Abstract machines for functional programming languages</li>/n<li>Abstract machines for logic programming languages. These are based on predicate calculus.</li>/n</ul>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Programming_language_implementation&amp;action=edit&amp;section=2" title="Edit section: See also">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<ul>/n<li><a href="/wiki/Compiler_construction" class="mw-redirect" title="Compiler construction">Compiler construction</a></li>/n<li><a href="/wiki/Functional_compiler" title="Functional compiler">Functional compiler</a></li>/n</ul>/n<table class="metadata plainlinks stub" role="presentation" style="background:transparent">/n<tr>/n<td><a href="/wiki/File:HelloWorld.svg" class="image"><img alt="Stub icon" src="//upload.wikimedia.org/wikipedia/commons/thumb/2/28/HelloWorld.svg/50px-HelloWorld.svg.png" width="50" height="25" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/2/28/HelloWorld.svg/75px-HelloWorld.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/2/28/HelloWorld.svg/100px-HelloWorld.svg.png 2x" data-file-width="512" data-file-height="256" /></a></td>/n<td><i>This <a href="/wiki/Programming_language" title="Programming language">programming-language</a>-related article is a <a href="/wiki/Wikipedia:Stub" title="Wikipedia:Stub">stub</a>.
