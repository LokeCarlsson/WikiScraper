languages</a></td>/n</tr>/n</table>/n<p><b>Simula</b> is the name of two <a href="/wiki/Simulation_language" title="Simulation language">simulation</a> <a href="/wiki/Programming_language" title="Programming language">programming languages</a>, Simula I and Simula 67, developed in the 1960s at the <a href="/wiki/Norwegian_Computing_Center" title="Norwegian Computing Center">Norwegian Computing Center</a> in <a href="/wiki/Oslo" title="Oslo">Oslo</a>, by <a href="/wiki/Ole-Johan_Dahl" title="Ole-Johan Dahl">Ole-Johan Dahl</a> and <a href="/wiki/Kristen_Nygaard" title="Kristen Nygaard">Kristen Nygaard</a>. <a href="/wiki/Syntax_(programming_languages)" title="Syntax (programming languages)">Syntactically</a>, it is a fairly faithful superset of <a href="/wiki/ALGOL_60" title="ALGOL 60">ALGOL 60</a>.<sup id="cite_ref-CommonBase_1-0" class="reference"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:1.3.1</sup></p>/n<p>Simula 67 introduced <a href="/wiki/Object_(computing)" class="mw-redirect" title="Object (computing)">objects</a>,<sup id="cite_ref-CommonBase_1-1" class="reference"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:2, 5.3</sup> <a href="/wiki/Class_(computer_science)" class="mw-redirect" href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:2.2.1</sup> <a href="/wiki/Virtual_function" title="Virtual function">virtual procedures</a>,<sup id="cite_ref-CommonBase_1-4" class="reference"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:2.2.3</sup> <a href="/wiki/Coroutine" reliable sources. (December 2013)">citation needed</span></a></i>]</sup></p>/n<p>Simula is considered the first <a href="/wiki/Object-oriented_programming" title="Object-oriented programming">object-oriented</a> <a href="/wiki/Programming_paradigm" title="Programming paradigm">programming</a> language. As its name suggests, Simula was designed for doing <a href="/wiki/Simulations" class="mw-redirect" title="Simulations">simulations</a>, and the needs of that <a href="/wiki/Problem_domain" title="Problem domain">domain</a> provided the framework for many of the features of object-oriented languages today.</p>/n<p>Simula has been used in a wide range of applications such as simulating <a href="/wiki/Very-large-scale_integration" title="Very-large-scale integration">VLSI</a> designs, <a href="/wiki/Process_modeling" title="Process modeling">process modeling</a>, <a href="/wiki/Protocol_(computing)" class="mw-redirect" title="Protocol (computing)">protocols</a>, <a href="/wiki/Algorithm" title="Algorithm">algorithms</a>, and other applications such as <a href="/wiki/Typesetting" title="Typesetting">typesetting</a>, <a href="/wiki/Computer_graphics" title="Computer graphics">computer graphics</a>, and <a href="/wiki/Education" title="Education">education</a>. The influence of Simula is often understated, and Simula-type objects are reimplemented in <a href="/wiki/C%2B%2B" title="C++">C++</a>, <a href="/wiki/Object_Pascal" title="Object Pascal">Object Pascal</a>, <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a> and several other languages. Computer scientists such as <a href="/wiki/Bjarne_Stroustrup" title="Bjarne Stroustrup">Bjarne Stroustrup</a>, creator of C++, and <a href="/wiki/James_Gosling" title="James Gosling">James Gosling</a>, creator of Java, have acknowledged Simula as a major influence.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup></p>/n<p></p>/n<div id="toc" class="toc">/n<div links</span></a></li>/n</ul>/n</div>/n<p></p>/n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simula&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The following account is based on Jan Rune Holmevik's historical essay.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup><sup id="cite_ref-4" class="reference"><a href="#cite_note-4">[4]</a></sup></p>/n<p><a href="/wiki/Kristen_Nygaard" title="Kristen Nygaard">Kristen Nygaard</a> started writing computer simulation programs in 1957. Nygaard saw a need for a better way to describe the heterogeneity and the <a href="/wiki/Instruction_(computer_science)" class="mw-redirect" title="Instruction (computer science)">operation</a> of a system. To go further with his ideas on a formal <a href="/wiki/Computer_language" title="Computer language">computer language</a> for describing a system, Nygaard realized that he needed someone with more <a href="/wiki/Computer_programming" title="Computer programming">computer programming</a> skills than he had. <a href="/wiki/Ole-Johan_Dahl" title="Ole-Johan Dahl">Ole-Johan Dahl</a> joined him on his work January 1962. The decision of linking the language up to <a href="/wiki/ALGOL_60" title="ALGOL 60">ALGOL 60</a> was made shortly after. By May 1962 the main concepts for a <a href="/wiki/Simulation" title="Simulation">simulation</a> <a href="/wiki/Programming_language" title="Programming language">language</a> were set. "SIMULA I" was born, a special purpose programming language for simulating discrete event systems.</p>/n<p>Kristen Nygaard was invited to <a href="/wiki/UNIVAC" title="UNIVAC">UNIVAC</a> late May 1962 in connection with the marketing of their new <a href="/wiki/UNIVAC_1107" class="mw-redirect" title="UNIVAC 1107">UNIVAC 1107</a> computer. At that visit Nygaard presented the ideas of Simula to <a href="/wiki/Bob_Bemer" title="Bob Bemer">Robert Bemer</a>, the director of systems programming at <a href="/wiki/Univac" class="mw-redirect" title="Univac">Univac</a>. Bemer was a sworn <a href="/wiki/ALGOL" title="ALGOL">ALGOL</a> fan and found the Simula project compelling. Bemer was also chairing a session at the second international conference on information processing hosted by <a href="/wiki/International_Federation_for_Information_Processing" title="International Federation for Information Processing">IFIP</a>. He invited Nygaard, who presented the paper "SIMULA -- An Extension of ALGOL to the Description of Discrete-Event Networks".</p>/n<p><a href="/wiki/Norwegian_Computing_Center" title="Norwegian Computing Center">Norwegian Computing Center</a> got a <a href="/wiki/UNIVAC_1107" class="mw-redirect" title="UNIVAC 1107">UNIVAC 1107</a> August 1963 at a considerable discount, on which Dahl implemented the SIMULA I under contract with UNIVAC. The implementation was based on the UNIVAC <a href="/wiki/ALGOL_60" title="ALGOL 60">ALGOL 60</a> compiler. SIMULA I was fully operational on the UNIVAC 1107 by January 1965. In the following couple of years Dahl and Nygaard spent a lot of time teaching Simula. Simula spread to several countries around the world and SIMULA I was later implemented on <a href="/wiki/Burroughs_large_systems" title="Burroughs large systems">Burroughs B5500 computers</a> and the Russian <a href="/wiki/Ural_(computer)" title="Ural (computer)">URAL-16 computer</a>.</p>/n<p>In 1966 <a href="/wiki/C._A._R._Hoare" class="mw-redirect" title="C. A. R. Hoare">C. A. R. Hoare</a> introduced the concept of record class construct, which Dahl and Nygaard extended with the concept of prefixing and other features to meet their requirements for a generalized process concept. Dahl and Nygaard presented their paper on <a href="/wiki/Class_(computer_science)" class="mw-redirect" title="Class (computer science)">Class</a> and <a href="/wiki/Subclass_(computer_science)" class="mw-redirect" title="Subclass (computer science)">Subclass</a> Declarations at the <a href="/wiki/International_Federation_for_Information_Processing" title="International Federation for Information Processing">IFIP</a> Working Conference on <a href="/wiki/Simulation_language" title="Simulation language">simulation languages</a> in <a href="/wiki/Oslo" title="Oslo">Oslo</a>, May 1967. This paper became the first formal definition of Simula 67. In June 1967 a conference was held to standardize the language and initiate a number of implementations. Dahl proposed to unify the <a href="/wiki/Data_type" title="Data type">type</a> and the class concept. This led to serious discussions, and the proposal was rejected by the board. SIMULA 67 was formally standardized on the first meeting of the SIMULA Standards Group (SSG) in February 1968.</p>/n<p>Simula was influential in the development of <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a> and later <a href="/wiki/Object-oriented_programming_language" class="mw-redirect" title="Object-oriented programming language">object-oriented programming languages</a>. It also helped inspire the <a href="/wiki/Actor_model" title="Actor model">actor model</a> of concurrent computation although Simula only supports <a href="/wiki/Coroutine" title="Coroutine">coroutines</a> and not true <a href="/wiki/Concurrency_(computer_science)" title="Concurrency (computer science)">concurrency</a>.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (December 2015)">citation needed</span></a></i>]</sup></p>/n<p>In the late sixties and the early seventies there were four main implementations of Simula:</p>/n<ul>/n<li><a href="/wiki/UNIVAC" title="UNIVAC">UNIVAC 1100</a> by <a href="/wiki/Norwegian_Computing_Center" title="Norwegian Computing Center">NCC</a></li>/n<li><a href="/wiki/System/360" class="mw-redirect" title="System/360">System/360</a> and <a href="/wiki/System/370" class="mw-redirect" title="System/370">System/370</a> by <a href="/wiki/Norwegian_Computing_Center" title="Norwegian Computing Center">NCC</a></li>/n<li><a href="/wiki/CDC_3000" class="mw-redirect" title="CDC 3000">CDC 3000</a> by <a href="/wiki/University_of_Oslo" title="University of Oslo">University of Oslo</a>'s Joint Computer Installation at Kjeller</li>/n<li><a href="/wiki/TOPS-10" title="TOPS-10">TOPS-10</a> by <a href="/wiki/Swedish_National_Defence_Research_Institute" title="Swedish National Defence Research Institute">Swedish National Defence Research Institute</a> (FOA)</li>/n</ul>/n<p>These implementations were ported to a wide range of platforms. The <a href="/wiki/TOPS-10" title="TOPS-10">TOPS-10</a> implemented the concept of public, protected, and private member variables and procedures, that later was integrated into Simula 87. Simula 87 is the latest standard and is ported to a wide range of platforms. There are mainly three implementations:</p>/n<ul>/n<li>Simula AS</li>/n<li>Lund Simula</li>/n<li>GNU Cim<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">[5]</a></sup></li>/n</ul>/n<p>In November 2001 Dahl and Nygaard were awarded the <a href="/wiki/IEEE_John_von_Neumann_Medal" title="IEEE John von Neumann Medal">IEEE John von Neumann Medal</a> by the <a href="/wiki/Institute_of_Electrical_and_Electronic_Engineers" class="mw-redirect" title="Institute of Electrical and Electronic Engineers">Institute of Electrical and Electronic Engineers</a> "For the introduction of the concepts underlying object-oriented programming through the design and implementation of SIMULA 67". In April 2002 they received the 2001 A. M. <a href="/wiki/Turing_Award" title="Turing Award">Turing Award</a> by the <a href="/wiki/Association_for_Computing_Machinery" title="Association for Computing Machinery">Association for Computing Machinery</a> (ACM), with the citation: "For ideas fundamental to the emergence of object oriented programming, through their design of the programming languages Simula I and Simula 67." Unfortunately neither Dahl nor Nygaard could make it to the ACM Turing Award Lecture,<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">[6]</a></sup> scheduled to be delivered at the November 2002 <a href="/wiki/OOPSLA" title="OOPSLA">OOPSLA</a> conference in Seattle, as they died in June and August of that year, respectively.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">[7]</a></sup></p>/n<p><a href="/wiki/Simula_Research_Laboratory" title="Simula Research Laboratory">Simula Research Laboratory</a> is a research institute named after the Simula language, and Nygaard held a part-time position there from the opening in 2001. The new Computer Science building at the <a href="/wiki/University_of_Oslo" title="University of Oslo">University of Oslo</a> is named Ole Johan Dahl's House, in Dahl's honour, and the main auditorium is named Simula.</p>/n<p>Simula is still used for various types of university courses, for instance, Jarek Sklenar teaches Simula to students at University of Malta.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">[8]</a></sup></p>/n<h2><span class="mw-headline" id="Sample_code">Sample code</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simula&amp;action=edit&amp;section=2" title="Edit section: Sample code">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<h3><span class="mw-headline" id="Minimal_program">Minimal program</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simula&amp;action=edit&amp;section=3" title="Edit section: Minimal program">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>The empty <a href="/wiki/Computer_file" title="Computer file">computer file</a> is the minimal <a href="/wiki/Computer_program" title="Computer program">program</a> in Simula, measured by the size of the <a href="/wiki/Source_code" title="Source code">source code</a>. It consists of one thing only; a dummy <a href="/wiki/Statement_(programming)" class="mw-redirect" title="Statement (programming)">statement</a>.</p>/n<p>However, the minimal program is more conveniently represented as an empty block:</p>/n<pre>/n<b>Begin</b>/n<b>End</b>;/n</pre>/n<p>It begins executing and immediately terminates. The language does not have any <a href="/wiki/Return_value" class="mw-redirect" title="Return value">return value</a> from the program itself.</p>/n<h3><span class="mw-headline" id="Classic_Hello_world">Classic Hello world</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simula&amp;action=edit&amp;section=4" title="Edit section: Classic Hello world">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>An example of a <a href="/wiki/Hello_world_program" class="mw-redirect" title="Hello world program">Hello world program</a> in Simula:</p>/n<pre>/n<b>Begin</b>/n   <b>OutText</b> ("Hello World!");/n   <b>Outimage</b>;/n<b>End</b>;/n</pre>/n<p>Simula is <a href="/wiki/Case-insensitive" class="mw-redirect" title="Case-insensitive">case-insensitive</a>.</p>/n<h3><span id="Classes,_subclasses_and_virtual_procedures"></span><span class="mw-headline" id="Classes.2C_subclasses_and_virtual_procedures">Classes, subclasses and virtual procedures</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simula&amp;action=edit&amp;section=5" title="Edit section: Classes, subclasses and virtual procedures">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>A more realistic example with use of classes,<sup id="cite_ref-CommonBase_1-8" class="reference"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:1.3.3, 2</sup> subclasses<sup id="cite_ref-CommonBase_1-9" class="reference"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:2.2.1</sup> and virtual procedures:<sup id="cite_ref-CommonBase_1-10" class="reference"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:2.2.3</sup></p>/n<pre>/n<b>Begin</b>/n   <b>Class</b> Glyph;/n      <b>Virtual</b>: <b>Procedure</b> print <b>Is</b> <b>Procedure</b> print;/n   <b>Begin</b>/n   <b>End</b>;/n/n   Glyph <b>Class</b> Char (c);/n      <b>Character</b> c;/n   <b>Begin</b>/n      <b>Procedure</b> print;/n        OutChar(c);/n   <b>End</b>;/n/n   Glyph <b>Class</b> Line (elements);/n      <b>Ref</b> (Glyph) <b>Array</b> elements;/n   <b>Begin</b>/n      <b>Procedure</b> print;/n      <b>Begin</b>/n         <b>Integer</b> i;/n         <b>For</b> i:= 1 <b>Step</b> 1 <b>Until</b> UpperBound (elements, 1) <b>Do</b>/n            elements (i).print;/n         OutImage;/n      <b>End</b>;/n   <b>End</b>;/n/n   <b>Ref</b> (Glyph) rg;/n   <b>Ref</b> (Glyph) <b>Array</b> rgs (1 : 4);/n/n   <i>! Main program;</i>/n   rgs (1):- <b>New</b> Char ('A');/n   rgs (2):- <b>New</b> Char ('b');/n   rgs (3):- <b>New</b> Char ('b');/n   rgs (4):- <b>New</b> Char ('a');/n   rg:- <b>New</b> Line (rgs);/n   rg.print;/n<b>End</b>;/n</pre>/n<p>The above example has one <a href="/wiki/Superclass_(computer_science)" class="mw-redirect" title="Superclass (computer science)">super class</a> (Glyph) with two <a href="/wiki/Subclass_(computer_science)" class="mw-redirect" title="Subclass (computer science)">subclasses</a> (Char and Line). There is one <a href="/wiki/Virtual_function" title="Virtual function">virtual procedure</a> with two <a href="/wiki/Implementation" title="Implementation">implementations</a>. The execution starts by executing the main program. Simula does not have the concept of <a href="/wiki/Abstract_class" class="mw-redirect" title="Abstract class">abstract classes</a> since classes with <a href="/wiki/Pure_virtual_function" class="mw-redirect" title="Pure virtual function">pure virtual procedures</a> can be <a href="/wiki/Instantiation_(computer_science)" class="mw-redirect" title="Instantiation (computer science)">instantiated</a>. This means that in the above example all classes can be instantiated. Calling a <a href="/wiki/Pure_virtual_function" class="mw-redirect" title="Pure virtual function">pure virtual procedure</a> will however produce a <a href="/wiki/Run-time_error" class="mw-redirect" title="Run-time error">run-time error</a>.</p>/n<h3><span class="mw-headline" id="Call_by_name">Call by name</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simula&amp;action=edit&amp;section=6" title="Edit section: Call by name">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Simula supports <a href="/wiki/Call_by_name" class="mw-redirect" title="Call by name">call by name</a><sup id="cite_ref-CommonBase_1-11" class="reference"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:8.2.3</sup> so the <a href="/wiki/Jensen%27s_Device" title="Jensen's Device">Jensen's Device</a> can easily be implemented. However, the default transmission mode for simple parameter is <a href="/wiki/Call_by_value" class="mw-redirect" title="Call by value">call by value</a>, contrary to <a href="/wiki/ALGOL" title="ALGOL">ALGOL</a> which used <a href="/wiki/Call_by_name" class="mw-redirect" title="Call by name">call by name</a>. The source code for the <a href="/wiki/Jensen%27s_Device" title="Jensen's Device">Jensen's Device</a> must therefore specify <a href="/wiki/Call_by_name" class="mw-redirect" title="Call by name">call by name</a> for the parameters when compiled by a Simula compiler.</p>/n<p>Another much simpler example is the <a href="/wiki/Arithmetic_function#Summation_functions" title="Arithmetic function">summation function <span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math xmlns="http://www.w3.org/1998/Math/MathML" >/n  <semantics>/n    <mrow class="MJX-TeXAtom-ORD">/n      <mstyle displaystyle="true" scriptlevel="0">/n        <mo>&#x2211;<!-- ∑ --></mo>/n      </mstyle>/n    </mrow>/n    <annotation encoding="application/x-tex">{\displaystyle \sum }</annotation>/n  </semantics>/n</math></span><img src="https://wikimedia.org/api/rest_v1/media/math/render/svg/f1d4e06539576633987e902f402ed46728d573b6" class="mwe-math-fallback-image-inline" aria-hidden="true" style="vertical-align: -1.338ex; width:3.365ex; height:3.843ex;" alt="\sum " /></span></a> which can be implemented as follows:</p>/n<pre>/n<b>Real</b> <b>Procedure</b> Sigma (k, m, n, u);/n   <b>Name</b> k, u;/n   <b>Integer</b> k, m, n; <b>Real</b> u;/n<b>Begin</b>/n   <b>Real</b> s;/n   k:= m;/n   <b>While</b> k &lt;= n <b>Do</b> <b>Begin</b> s:= s + u; k:= k + 1; <b>End</b>;/n   Sigma:= s;/n<b>End</b>;/n</pre>/n<p>The above code uses <a href="/wiki/Call_by_name" class="mw-redirect" title="Call by name">call by name</a> for the controlling variable (k) and the expression (u). This allows the controlling variable to be used in the expression.</p>/n<p>Note that the Simula standard allows for certain restrictions on the controlling variable in a <a href="/wiki/For_loop" title="For loop">for loop</a>. The above code therefore uses a while loop for maximum portability.</p>/n<p>The following:</p>/n<p><span class="mwe-math-element"><span class="mwe-math-mathml-inline mwe-math-mathml-a11y" style="display: none;"><math xmlns="http://www.w3.org/1998/Math/MathML" >/n  <semantics>/n    <mrow class="MJX-TeXAtom-ORD">/n      <mstyle displaystyle="true" scriptlevel="0">/n        <mi>Z</mi>/n        <mo>=</mo>/n        <munderover>/n          <mo>&#x2211;<!-- ∑ --></mo>/n          <mrow class="MJX-TeXAtom-ORD">/n            <mi>i</mi>/n            <mo>=</mo>/n            <mn>1</mn>/n          </mrow>/n          <mrow class="MJX-TeXAtom-ORD">/n            <mn>100</mn>/n          </mrow>/n        </munderover>/n        <mrow class="MJX-TeXAtom-ORD">/n          <mfrac>/n            <mn>1</mn>/n            <mrow>/n              <mo stretchy="false">(</mo>/n              <mi>i</mi>/n              <mo>+</mo>/n              <mi>a</mi>/n              <msup>/n                <mo stretchy="false">)</mo>/n                <mrow class="MJX-TeXAtom-ORD">/n                  <mn>2</mn>/n                </mrow>/n              </msup>/n            </mrow>/n          </mfrac>/n        </mrow>/n      </mstyle>/n    </mrow>/n    <annotation encoding="application/x-tex">{\displaystyle Z=\sum _{i=1}^{100}{1 \over (i+a)^{2}}}</annotation>/n  </semantics>/n</math></span><img src="https://wikimedia.org/api/rest_v1/media/math/render/svg/7e5b4b37fb90cdd6177be7f4c3b1e2f037b248fa" class="mwe-math-fallback-image-inline" aria-hidden="true" style="vertical-align: -3.005ex; width:17.184ex; height:7.343ex;" alt="Z=\sum _{i=1}^{100}{1 \over (i+a)^{2}}" /></span></p>/n<p>can then be implemented as follows:</p>/n<pre>/nZ:= Sigma (i, 1, 100, 1 / (i + a) ** 2);/n</pre>/n<h3><span class="mw-headline" id="Simulation">Simulation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Simula&amp;action=edit&amp;section=7" title="Edit section: Simulation">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>Simula includes a <a href="/wiki/Simulation" title="Simulation">simulation</a><sup id="cite_ref-CommonBase_1-12" class="reference"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:14.2</sup> package for doing <a href="/wiki/Discrete_event_simulation" title="Discrete event simulation">discrete event simulations</a>. This simulation package is based on Simula's <a href="/wiki/Object_oriented" class="mw-redirect" title="Object oriented">object oriented</a> features and its <a href="/wiki/Coroutine" title="Coroutine">coroutine</a><sup id="cite_ref-CommonBase_1-13" class="reference"><a href="#cite_note-CommonBase-1">[1]</a></sup><sup class="reference" style="white-space:nowrap;">:9.2</sup> concept.</p>/n<p>Sam, Sally, and Andy are shopping for clothes. They have to share one fitting room. Each one of them is browsing the store for about 12 minutes and then uses the fitting room exclusively for about three minutes, each following a normal distribution. A simulation of their fitting room experience is as follows:</p>/n<pre>/nSimulation <b>Begin</b>/n   <b>Class</b> FittingRoom; <b>Begin</b>/n      <b>Ref</b> (Head) door;/n      <b>Boolean</b> inUse;/n      <b>Procedure</b> request; <b>Begin</b>/n         <b>If</b> inUse <b>Then</b> <b>Begin</b>/n             Wait (door);/n             door.First.Out;/n         <b>End</b>;/n         inUse:= <b>True</b>;/n      <b>End</b>;/n      <b>Procedure</b> leave; <b>Begin</b>/n         inUse:= <b>False</b>;/n         <b>Activate</b> door.First;/n      <b>End</b>;/n      door:- <b>New</b> Head;/n   <b>End</b>;/n  /n   <b>Procedure</b> report (message); <b>Text</b> message; <b>Begin</b>/n      OutFix (Time, 2, 0); OutText (": " &amp; message); OutImage;/n   <b>End</b>;/n/n   Process <b>Class</b> Person (pname); <b>Text</b> pname; <b>Begin</b>/n      <b>While</b> <b>True</b> <b>Do</b> <b>Begin</b>/n         Hold (Normal (12, 4, u));/n         report  (pname &amp; " is requesting the fitting room");/n         fittingroom1.request;/n         report (pname &amp; " has entered the fitting room");/n         Hold (Normal (3, 1, u));/n         fittingroom1.leave;/n         report (pname &amp; " has left the fitting room");/n      <b>End</b>;/n   <b>End</b>;/n/n   <b>Integer</b> u;/n   <b>Ref</b> (FittingRoom) fittingRoom1;/n/n   fittingRoom1:- <b>New</b> FittingRoom;/n   <b>Activate</b> <b>New</b> Person ("Sam");/n   <b>Activate</b> <b>New</b> Person ("Sally");/n   <b>Activate</b> <b>New</b> Person ("Andy");/n   Hold (100);/n<b>End</b>;/n</pre>/n<p>The main block is prefixed with <code>Simulation</code> for enabling simulation. The simulation package can be used on any block and simulations can even be nested when simulating someone doing simulations.</p>/n<p>The fitting room object uses a queue (<code>door</code>) for getting access to the fitting room. When someone requests the fitting room and it's in use they must wait in this queue (<code>Wait (door)</code>). When someone leaves the fitting room the first one (if any) is released from the queue (<code><b>Activate</b> door.first</code>) and accordingly removed from the door queue (<code>door.First.Out</code>).</p>/n<p>Person is a subclass of Process and its activity is described using hold (time for browsing the store and time spent in the fitting room) and calls procedures in the fitting room object for requesting and leaving the fitting room.</p>/n<p>The main program creates all the objects and activates all the person objects to put them into the event queue. The main program holds for 100 minutes of simulated time before the program terminates.</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span
