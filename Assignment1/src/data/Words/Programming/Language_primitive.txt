href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (May 2016)">citation needed</span></a></i>]</sup> are the simplest elements available in a <a href="/wiki/Programming_language" title="Programming language">programming language</a>. A primitive is the smallest 'unit of processing' available to a programmer of a given machine, or can be an <a href="/wiki/Atomic_operation" class="mw-redirect" title="Atomic operation">atomic</a> element of an <a href="/wiki/Expression_(programming)" class="mw-redirect" title="Expression (programming)">expression</a> in a language.</p>/n<p>Primitives are units with a meaning, i.e., a <a href="/wiki/Semantics_(computer_science)" title="Semantics (computer science)">semantic</a> value in the language. Thus they are different from <a href="/wiki/Token_(parser)" class="mw-redirect" title="Token (parser)">tokens</a> in a <a href="/wiki/Parser" class="mw-redirect" title="Parser">parser</a>, which are the minimal elements of <a href="/wiki/Syntax_(programming_languages)" title="Syntax (programming languages)">syntax</a>.</p>/n<p></p>/n<div id="toc" class="toc">/n<div class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Language_primitive&amp;action=edit&amp;section=1" title="Edit section: Machine level primitives">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>A <a href="/wiki/Machine_instruction" class="mw-redirect" title="Machine instruction">machine instruction</a>, usually generated by an <a href="/wiki/Assembler_(computing)" class="mw-redirect" title="Assembler (computing)">assembler</a> program, is often considered the smallest unit of processing although this is not always the case. It typically performs what is perceived to be one operation such as copying a <a href="/wiki/Byte" title="Byte">byte</a> or <a href="/wiki/String_(computer_science)" title="String (computer science)">string</a> of bytes from one <a href="/wiki/Computer_memory" title="Computer memory">computer memory</a> location to another or adding one <a href="/wiki/Processor_register" title="Processor register">processor register</a> to another.</p>/n<h2><span class="mw-headline" id="Microcode_primitives">Microcode primitives</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Language_primitive&amp;action=edit&amp;section=2" title="Edit section: Microcode primitives">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Many of today's <a href="/wiki/Computer" title="Computer">computers</a>, however, actually embody an even lower unit of processing known as <a href="/wiki/Microcode" title="Microcode">microcode</a> which interprets the <i>machine code</i> and it is then that the microcode instructions would be the <i>genuine</i> primitives. These instructions would typically be available for modification only by the <a href="/wiki/Computer_hardware" title="Computer hardware">hardware</a> vendor's programmers.</p>/n<h2><span class="mw-headline" id="High_level_language_primitives">High level language primitives</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Language_primitive&amp;action=edit&amp;section=3" title="Edit section: High level language primitives">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>A <a href="/wiki/High-level_programming_language" title="High-level programming language">high-level programming language</a> (HLL) program is composed of discrete <a href="/wiki/Statement_(computer_science)" title="Statement (computer science)">statements</a> and <a href="/wiki/Primitive_data_type" title="Primitive data type">primitive data types</a> that may also be <i>perceived</i> to perform a single operation or represent a single data item, but at a higher semantic level than those provided by the machine. Copying a data item from one location to another may actually involve many <a href="/wiki/Machine_instruction" class="mw-redirect" title="Machine instruction">machine instructions</a> that, for instance,</p>/n<ul>/n<li>calculate the address of both <a href="/wiki/Operand" title="Operand">operands</a> in <a href="/wiki/Computer_memory" title="Computer memory">memory</a>, based on their positions within a <a href="/wiki/Data_structure" title="Data structure">data structure</a>,</li>/n<li>convert from one <a href="/wiki/Data_type" title="Data type">data type</a> to another</li>/n</ul>/n<p>before finally</p>/n<ul>/n<li>performing the final store operation to the target destination.</li>/n</ul>/n<p>Some HLL statements, particularly those involving <a href="/wiki/Control_flow#Loops" title="Control flow">loops</a>, can generate thousands or even millions of primitives in a <a href="/wiki/Low-level_programming_language" title="Low-level programming language">low-level programming language</a> (LLL), which comprise the genuine <a href="/wiki/Instruction_path_length" title="Instruction path length">instruction path length</a> the processor has to execute at the lowest level. This perception has been referred to as the <i>abstraction penalty</i>.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup><sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup><sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup></p>/n<h2><span class="mw-headline" id="Interpreted_language_primitives">Interpreted language primitives</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Language_primitive&amp;action=edit&amp;section=4" title="Edit section: Interpreted language primitives">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>An <a href="/wiki/Interpreted_language" title="Interpreted language">interpreted language</a> statement has similarities to the HLL primitives, but with a further added <i>layer</i>. Before the statement can be executed in a manner very similar to an HLL statement: it must first be processed by an <a href="/wiki/Interpreter" class="mw-redirect" title="Interpreter">interpreter</a>, a process that may involve many primitives in the target machine language.</p>/n<h2><span class="mw-headline" id="Fourth_and_fifth-generation_language_primitives">Fourth and fifth-generation language primitives</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Language_primitive&amp;action=edit&amp;section=5" title="Edit section: Fourth and fifth-generation language primitives">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p><a href="/wiki/Fourth-generation_programming_language" title="Fourth-generation programming language">Fourth-generation programming languages</a> (4GL) and <a href="/wiki/Fifth-generation_programming_language" title="Fifth-generation programming language">fifth-generation programming languages</a> (5GL) do not have a simple one-to-many correspondence from high-to-low level primitives. There are some elements of interpreted language primitives embodied in 4GL and 5GL specifications, but the approach to the original problem is less a <a href="/wiki/Procedural_language" class="mw-redirect" title="Procedural language">procedural language</a> construct and are more oriented toward <a href="/wiki/Problem_solving" title="Problem solving">problem solving</a> and <a href="/wiki/Systems_engineering" title="Systems engineering">systems engineering</a>.</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Language_primitive&amp;action=edit&amp;section=6" title="Edit section: See also">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<ul>/n<li><a href="/wiki/Primitive_type" class="mw-redirect" title="Primitive type">Primitive type</a></li>/n<li><a href="/wiki/Hardware-software_codesign" class="mw-redirect" title="Hardware-software codesign">Hardware-software codesign</a></li>/n</ul>/n<h2><span class="mw-headline" id="References">References</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Language_primitive&amp;action=edit&amp;section=7" title="Edit section: References">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div class="mw-references-wrap">/n<ol class="references">/n<li id="cite_note-1"><span class="mw-cite-backlink"><b><a href="#cite_ref-1">^</a></b></span> <span class="reference-text"><cite class="citation journal">Surana P (2006). <a rel="nofollow" class="external text" href="https://web.archive.org/web/20150217154926/http://lispnyc.org/meeting-assets/2007-02-13_pinku/SuranaThesis.pdf">"Meta-Compilation of Language Abstractions."</a> <span style="font-size:85%;">(PDF)</span>.
