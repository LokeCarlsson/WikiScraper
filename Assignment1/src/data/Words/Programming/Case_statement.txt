class="mw-redirect" title="Computer programming language">computer programming languages</a>, a <b>switch statement</b> is a type of selection control mechanism used to allow the value of a <a href="/wiki/Variable_(programming)" class="mw-redirect" title="Variable (programming)">variable</a> or expression to change the <a href="/wiki/Control_flow" title="Control flow">control flow</a> of program execution via a <a href="/wiki/Multiway_branch" title="Multiway branch">multiway branch</a>.</p>/n<p>Switch statements exist in most high-level <a href="/wiki/Imperative_programming" title="Imperative programming">imperative programming</a> languages such as <a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a>, <a title="C++">C++</a>, <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a> and <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a>, and in many other types of language, using such <a href="/wiki/Keyword_(computer_programming)" class="mw-redirect" title="Keyword (computer programming)">keywords</a> as <code>switch</code>, <code>case</code>, <code>select</code> or <code>inspect</code>.</p>/n<p>Switch statements come in two main variants: a structured switch, as in Pascal, which takes exactly one branch, and an unstructured switch, as in C, which functions as a type of <a href="/wiki/Goto" title="Goto">goto</a>. The main reasons for using a switch include improving clarity, by reducing otherwise repetitive coding, and (if the <a href="/wiki/Heuristic" title="Heuristic">heuristics</a> permit) also offering the potential for faster execution through easier <a href="/wiki/Compiler_optimization" class="mw-redirect" title="Compiler optimization">compiler optimization</a> in many cases.</p>/n<table href="/w/index.php?title=Switch_statement&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>In his 1952 text <i>Introduction to Metamathematics</i>, <a href="/wiki/Stephen_Kleene" class="mw-redirect" title="Stephen Kleene">Stephen Kleene</a> formally proved that the CASE function (the IF-THEN-ELSE function being its simplest form) is a <a href="/wiki/Primitive_recursive_function" title="Primitive recursive function">primitive recursive function</a>, where he defines the notion <code>definition by cases</code> in the following manner:</p>/n<dl>/n<dd>"#F. The function φ defined thus/n<dl>/n<dd>φ(x<sub>1</sub> , ... , x<sub>n</sub> ) =/n<dl>/n<dd>/n<ul>/n<li>φ<sub>1</sub>(x<sub>1</sub> , ... , x<sub>n</sub> ) if Q<sub>1</sub>(x<sub>1</sub> , ... , x<sub>n</sub> ),</li>/n<li>. . . . . . . . . . . .</li>/n<li>φ<sub>m</sub>(x<sub>1</sub> , ... , x<sub>n</sub> ) if Q<sub>m</sub>(x<sub>1</sub> , ... , x<sub>n</sub> ),</li>/n<li>φ<sub>m+1</sub>(x<sub>1</sub> , ... , x<sub>n</sub> ) otherwise,</li>/n</ul>/n</dd>/n</dl>/n</dd>/n</dl>/n</dd>/n</dl>/n<dl>/n<dd>where Q<sub>1</sub> , ... , Q<sub>m</sub> are mutually exclusive predicates (or φ(x<sub>1</sub> , ... , x<sub>n</sub>) shall have the value given by the first clause which applies) is primitive recursive in φ<sub>1</sub>, ..., φ<sub>m+1</sub>, Q<sub>1</sub>, ..., Q<sub>m+1</sub>.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">[1]</a></sup></dd>/n</dl>/n<p>Kleene provides a proof of this in terms of the Boolean-like recursive functions "sign-of" sg( ) and "not sign of" ~sg( ) (Kleene 1952:222-223); the first returns 1 if its input is positive and −1 if its input is negative.</p>/n<p>Boolos-Burgess-Jeffrey make the additional observation that "definition by cases" must be both mutually exclusive and collectively exhaustive. They too offer a proof of the primitive recursiveness of this function (Boolos-Burgess-Jeffrey 2002:74-75).</p>/n<p>The IF-THEN-ELSE is the basis of the <a href="/wiki/McCarthy_formalism" class="mw-redirect" title="McCarthy formalism">McCarthy formalism</a>: its usage replaces both primitive recursion and the <a href="/wiki/Mu-operator" class="mw-redirect" title="Mu-operator">mu-operator</a>.</p>/n<h2><span class="mw-headline" id="Typical_syntax">Typical syntax</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Switch_statement&amp;action=edit&amp;section=2" title="Edit section: Typical syntax">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>In most languages, programmers write a switch statement across many individual lines using one or two keywords. A typical syntax involves:</p>/n<ul>/n<li>the first line containing the basic keyword, usually <code>switch</code>, <code>case</code> or <code>select</code>, followed by an expression which is often referred to as the <i>control expression</i> or <i>control variable</i> of the switch statement</li>/n<li>subsequent lines defining the actual cases (the values), with corresponding sequences of statements for execution when a match occurs</li>/n<li>a <code>break</code> statement typically follows a <code>case</code> statement to end said statement. [Wells]</li>/n</ul>/n<p>Each alternative begins with the particular value, or list of values (see below), that the control variable may match and which will cause the control to <a href="/wiki/Go_to" class="mw-redirect" title="Go to">go to</a> the corresponding sequence of statements. The value (or list/range of values) is usually separated from the corresponding statement sequence by a colon or by an implication arrow. In many languages, every case must also be preceded by a keyword such as <code>case</code> or <code>when</code>.</p>/n<p>An optional default case is typically also allowed, specified by a <code>default</code>, <code>otherwise</code>, or <code>else</code> keyword. This executes when none of the other cases match the control expression. In some languages, such as C, if no case matches and the <code>default</code> is omitted the <code>switch</code> statement simply exits. In others, like PL/I, an error is raised.</p>/n<h2><span class="mw-headline" id="Semantics">Semantics</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Switch_statement&amp;action=edit&amp;section=3" title="Edit section: Semantics">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Semantically, there are two main forms of switch statements.</p>/n<p>The first form are structured switches, as in Pascal, where exactly one branch is taken, and the cases are treated as separate, exclusive blocks. This functions as a generalized if–then–else <a href="/wiki/Conditional_(computer_programming)" title="Conditional (computer programming)">conditional</a>, here with any number of branches, not just two.</p>/n<p>The second form are unstructured switches, as in C, where the cases are treated as labels within a single block, and the switch functions as a generalized goto. This distinction is referred to as the treatment of fallthrough, which is elaborated below.</p>/n<h3><span class="mw-headline" id="Fallthrough">Fallthrough</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Switch_statement&amp;action=edit&amp;section=4" title="Edit section: Fallthrough">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>In many languages, only the matching block is executed, and then execution continues at the end of the switch statement. These include the <a href="/wiki/Pascal_(programming_language)" title="Pascal (programming language)">Pascal</a> family (Object Pascal, Modula, Oberon, Ada, etc.) as well as <a href="/wiki/PL/I" title="PL/I">PL/I</a>, modern forms of <a href="/wiki/Fortran" title="Fortran">Fortran</a> and <a href="/wiki/BASIC" title="BASIC">BASIC</a> dialects influenced by Pascal, most functional languages, and many others. To allow multiple values to execute the same code (and avoid needing to <a href="/wiki/Duplicate_code" title="Duplicate code">duplicate code</a>), Pascal-type languages permit any number of values per case, given as a comma-separated list, as a range, or as a combination.</p>/n<p>Languages derived from C, and more generally those influenced by Fortran's <a href="/wiki/Computed_GOTO" class="mw-redirect" title="Computed GOTO">computed GOTO</a>, instead feature fallthrough, where control moves to the matching case, and then execution continues ("falls through") to the statements associated with the <i>next</i> case in the source text. This also allows multiple values to match the same point without any special syntax: they are just listed with empty bodies. Values can be <a href="/w/index.php?title=Special_condition&amp;action=edit&amp;redlink=1" class="new" title="Special condition (page does not exist)">special conditioned</a> with code in the case body. In practice, fallthrough is usually prevented with a <code>break</code> keyword at the end of the matching body, which exits execution of the switch block, but this can cause bugs due to unintentional fallthrough if the programmer forgets to insert the <code>break</code> statement. This is thus seen by many<sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Avoid_weasel_words" class="mw-redirect" title="Wikipedia:Avoid weasel words"><span title="Who says this? (July 2017)">weasel&#160;words</span></a></i>]</sup> as a language wart, and warned against in some lint tools. Syntactically, the cases are interpreted as labels, not blocks, and the switch and break statements explicitly change control flow. Some languages influenced by C, such as <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>, retain default fallthrough, while others remove fallthrough, or only allow it in special circumstances. Notable variations on this in the C-family include <a href="/wiki/C_Sharp_(programming_language)" title="C Sharp (programming language)">C#</a>, in which all blocks must be terminated with a <code>break</code> or <code>return</code> unless the block is empty (i.e. fallthrough is used as a way to specify multiple values).</p>/n<p>In some cases languages provide optional fallthrough. For example, <a href="/wiki/Perl" title="Perl">Perl</a> does not fall through by default, but a case may explicitly do so using a <code>continue</code> keyword. This prevents unintentional fallthrough but allows it when desired. Similarly, bash defaults to not falling through, when terminated with&#160;;; but more recent versions allow fallthrough with&#160;;&amp; instead.</p>/n<p>An example of a switch statement that relies on fallthrough is <a href="/wiki/Duff%27s_device" title="Duff's device">Duff's device</a>.</p>/n<h2><span class="mw-headline" id="Compilation">Compilation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Switch_statement&amp;action=edit&amp;section=5" title="Edit section: Compilation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p><a href="/wiki/Optimization_(computer_science)" class="mw-redirect" title="Optimization (computer science)">Optimizing compilers</a> such as <a href="/wiki/GNU_Compiler_Collection" title="GNU Compiler Collection">GCC</a> or <a href="/wiki/Clang" title="Clang">Clang</a> may compile a switch statement into either a <a href="/wiki/Branch_table" title="Branch table">branch table</a> or a <a href="/wiki/Binary_search_algorithm" title="Binary search algorithm">binary search</a> through the values in the cases.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">[2]</a></sup> A branch table allows the switch statement to determine with a small, constant number of comparisons which branch to execute without having to go through a list of comparisons, while a binary search takes only a logarithmic number of comparisons, measured in the number of cases in the switch statement.</p>/n<p>Normally, the only method of finding out if this optimization has occurred is by actually looking at the resultant <a href="/wiki/Assembly_language" title="Assembly language">assembly</a> or <a href="/wiki/Machine_code" title="Machine code">machine code</a> output that has been generated by the compiler (and is therefore seldom, if ever, done by <a href="/wiki/High-level_programming_language" title="High-level programming language">HLL</a> programmers).</p>/n<h2><span class="mw-headline" id="Advantages_and_disadvantages">Advantages and disadvantages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Switch_statement&amp;action=edit&amp;section=6" title="Edit section: Advantages and disadvantages">edit</a><span Compilers can issue a warning if some enum values are not handled.</li>/n</ul>/n<p>Additionally, an <a href="/wiki/Optimization_(computer_science)" class="mw-redirect" title="Optimization (computer science)">optimized</a> implementation may execute much faster than the alternative, because it is often implemented by using an indexed <a href="/wiki/Branch_table" title="Branch table">branch table</a>. For example, deciding program flow based on a single character's value, if correctly implemented, is vastly more efficient than the alternative, reducing <a href="/wiki/Instruction_path_length" title="Instruction path length">instruction path lengths</a> considerably. When implemented as such, a switch statement essentially becomes a <a href="/wiki/Perfect_hash" class="mw-redirect" title="Perfect hash">perfect hash</a>.</p>/n<p>In terms of the <a href="/wiki/Control_flow_graph" title="Control flow graph">control flow graph</a>, a switch statement consists of two nodes (entrance and exit), plus one edge between them for each option. By contrast, a sequence of "if...else if...else if" statements has an additional node for every case other than the first and last, together with a corresponding edge. The resulting control flow graph for the sequences of "if"s thus has many more nodes and almost twice as many edges, with these not adding any useful information. However, the simple branches in the if statements are individually conceptually easier than the complex branch of a switch statement. In terms of <a href="/wiki/Cyclomatic_complexity" title="Cyclomatic complexity">cyclomatic complexity</a>, both of these options increase it by <i>k</i>−1 if given <i>k</i> cases.</p>/n<p>When implemented with fall-through as the default path, switch/case statements are a frequent source of bugs among even experienced programmers, given that, in practice, the "break" is almost always the desired path, but not the default behavior of the switch/case construct (at least in C and Java).</p>/n<h2><span class="mw-headline" id="Alternative_uses">Alternative uses</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Switch_statement&amp;action=edit&amp;section=7" title="Edit section: Alternative uses">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>Many languages evaluate expressions inside <code>switch</code> blocks at runtime, allowing a number of less obvious uses for the construction. executed.</p>/n<p>In <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a>, due to its handling of <code>===</code> equality, the statement can be used to test for variable’s class:</p>/n<div class="mw-highlight mw-content-ltr" dir="ltr">/n<pre>/n<span class="k">case</span> <span class="n">input</span>/n<span class="k">when</span> <span class="nb">Array</span> href="/w/index.php?title=Switch_statement&amp;action=edit&amp;section=8" title="Edit section: Exception handling">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>A number of languages implement a form of switch statement in <a href="/wiki/Exception_handling" title="Exception handling">exception handling</a>, where if an exception is raised in a block, a separate branch is chosen, depending on the exception. In some cases a default branch, if no exception is raised, is also present. An early example is <a href="/wiki/Modula-3" title="Modula-3">Modula-3</a>, which use the <code>TRY</code>...<code>EXCEPT</code> syntax, where each <code>EXCEPT</code> defines a case. This is also found in <a href="/wiki/Delphi_(programming_language)" title="Delphi (programming language)">Delphi</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/Scala_(programming_language)" title="Scala (programming language)">Scala</a>, and <a href="/wiki/Visual_Basic.NET" class="mw-redirect" title="Visual Basic.NET">Visual Basic.NET</a>.</p>/n<h2><span class="mw-headline" id="Alternatives">Alternatives</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Switch_statement&amp;action=edit&amp;section=9" title="Edit section: Alternatives">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<ul>/n<li>A series of <i>if-else</i> <a href="/wiki/Conditional_(programming)" class="mw-redirect" title="Conditional (programming)">conditionals</a> that examine the target one value at a time. Fallthrough behavior can be achieved with a sequence of <i>if</i> conditionals each without the <i>else</i> clause.</li>/n<li>A <a href="/wiki/Lookup_table" title="Lookup table">lookup table</a>, which contains, as keys, the <code>case</code> values and, as values, the part under the <code>case</code> statement.</li>/n</ul>/n<dl>/n<dd>/n<dl>/n<dd>(In some languages, only actual data types are allowed as values in the lookup table. In other languages, it is also possible to assign <a href="/wiki/Function_(computer_science)" class="mw-redirect" title="Function (computer science)">functions</a> as lookup table values, gaining the same flexibility as a real <code>switch</code> statement. See <a href="/wiki/Control_table" title="Control table">Control table</a> article for more detail on this).</dd>/n<dd><a href="/wiki/Lua_(programming_language)" title="Lua (programming language)">Lua</a> does not support case/switch statements: <a rel="nofollow" class="external free" href="http://lua-users.org/wiki/SwitchStatement">http://lua-users.org/wiki/SwitchStatement</a> . This lookup technique is one way to implement <code>switch</code> statements in the Lua language, which has no built-in <code>switch</code>.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">[3]</a></sup></dd>/n<dd>In some cases, lookup tables are more efficient than non-<a href="/wiki/Program_optimization" title="Program optimization">optimized</a> <code>switch</code> statements since many languages can optimize table lookups, whereas switch statements are not optimized unless the range of values is small with few gaps. A non-optimized, non-<a href="/wiki/Binary_search" class="mw-redirect" title="Binary search">binary search</a> lookup, however, will almost certainly be slower than either a non-optimized switch or the equivalent multiple <i>if-else</i> statements.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">[<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. lookup table) can also be customized to accommodate multiple conditions on multiple inputs if required and usually exhibits greater 'visual compactness' than an equivalent switch (that can occupy many statements).</li>/n<li><a href="/wiki/Pattern_matching" title="Pattern matching">Pattern matching</a>, which is used to implement switch-like functionality in many <a
