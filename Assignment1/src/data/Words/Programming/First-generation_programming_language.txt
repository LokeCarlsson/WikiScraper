Unsourced material may be challenged and removed.</span> <small><i>(September 2013)</i></small> <small class="hide-when-compact"><i>(<a href="/wiki/Help:Maintenance_template_removal" title="Help:Maintenance template removal">Learn how and when to remove this template message</a>)</i></small></div>/n</td>/n</tr>/n</table>/n<p>A <b>first-generation programming language</b> (<b>1GL</b>) is a <a href="/wiki/Machine_code" title="Machine code">machine-level</a> <a href="/wiki/Programming_language" title="Programming language">programming language</a>.<sup id="cite_ref-Computer_Hope_1-0" class="reference"><a href="#cite_note-Computer_Hope-1">[1]</a></sup></p>/n<p>A first <a href="/wiki/Programming_language_generations" title="Programming language generations">generation (programming) language</a> (1GL) is a grouping of programming languages that are machine level languages used to program <a href="/wiki/First-generation_computer" class="mw-redirect" title="First-generation computer">first-generation computers</a>. Originally, no <a href="/wiki/Translator" class="mw-redirect" title="Translator">translator</a> was used to <a href="/wiki/Compiler" title="Compiler">compile</a> or <a href="/wiki/Assembly_language" title="Assembly language">assemble</a> the first-generation language. The first-generation programming instructions were entered through the <a href="/wiki/Front_panel" title="Front panel">front panel</a> switches of the computer system.</p>/n<p>The instructions in 1GL are made of <a href="/wiki/Binary_number" title="Binary number">binary numbers</a>, represented by 1s and 0s. This makes the language suitable for the understanding of the machine but far more difficult to interpret and learn by the human programmer.</p>/n<p>The main advantage of programming in 1GL is that the code can run very fast and very efficiently, precisely because the instructions are executed directly by the <a href="/wiki/Central_processing_unit" title="Central processing unit">central processing unit</a> (CPU). One of the main disadvantages of programming in a <a href="/wiki/Low-level_programming_language" title="Low-level programming language">low level language</a> is that when an error occurs, the code is not as easy to fix.</p>/n<p>First generation languages are very much adapted to a specific computer and CPU, and code portability is therefore significantly reduced in comparison to <a href="/wiki/High-level_programming_language" title="High-level programming language">higher level languages</a>.</p>/n<p>Modern day programmers still occasionally use machine level code, especially when programming lower level functions of the system, such as <a href="/wiki/Device_driver" title="Device driver">drivers</a>, <a href="/wiki/Interface_(computing)#Hardware_interfaces" title="Interface (computing)">interfaces</a> with <a href="/wiki/Firmware" title="Firmware">firmware</a> and hardware devices. Modern tools such as native-code compilers are used to produce machine level from a higher-level language.</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=First-generation_programming_language&amp;action=edit&amp;section=1" title="Edit section: See also">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<ul>/n<li><a href="/wiki/Second-generation_programming_language" title="Second-generation programming language">Second-generation programming language</a></li>/n</ul>/n<h2><span class="mw-headline" id="References">References</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=First-generation_programming_language&amp;action=edit&amp;section=2" title="Edit section: References">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<div class="reflist" style="list-style-type: decimal;">/n<div class="mw-references-wrap">/n<ol class="references">/n<li id="cite_note-Computer_Hope-1"><span class="mw-cite-backlink"><b><a href="#cite_ref-Computer_Hope_1-0">^</a></b></span> <span class="reference-text"><a rel="nofollow" class="external text" href="http://www.computerhope.com/jargon/num/1gl.htm">"Computer Hope, Generation languages"</a></span></li>/n</ol>/n</div>/n</div>/n<h3><span class="mw-headline" id="General">General</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=First-generation_programming_language&amp;action=edit&amp;section=3" title="Edit section: General">edit</a><span class="mw-editsection-bracket">]</span></span></h3>/n<p>1.
