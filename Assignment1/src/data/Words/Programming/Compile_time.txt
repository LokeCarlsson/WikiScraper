Compile time refers to the time duration during which the statements written in any programming language are checked for errors.</p>/n<h2><span class="mw-headline" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Compile_time&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<p>The operations performed at compile time usually include <a href="/wiki/Syntax_analysis" class="mw-redirect" title="Syntax analysis">syntax analysis</a>, various kinds of <a href="/wiki/Semantic_analysis_(computer_science)" class="mw-redirect" title="Semantic analysis (computer science)">semantic analysis</a> (e.g., <a href="/wiki/Datatype" class="mw-redirect" title="Datatype">type checks</a> and <a href="/wiki/Instantiation_of_template" class="mw-redirect" title="Instantiation of template">instantiation of template</a>) and <a href="/wiki/Code_generation_(compiler)" title="Code generation (compiler)">code generation</a>.</p>/n<p>Programming language definitions usually specify compile time requirements that source code must meet to be successfully compiled. For example, languages may stipulate that the amount of storage required by types and variables can be deduced.</p>/n<p>Properties of a program that can be reasoned about at compile time include range-checks (e.g., proving that an array index will not exceed the array bounds), deadlock freedom in concurrent languages, or timings (e.g., proving that a sequence of code takes no more than an allocated amount of time).</p>/n<p>Compile time occurs before <a href="/wiki/Link_time" title="Link time">link time</a> (when the output of one or more compiled files are joined together) and <a href="/wiki/Run_time_(program_lifecycle_phase)" title="Run time (program lifecycle phase)">runtime</a> (when a <a href="/wiki/Computer_program" title="Computer program">program</a> is <a href="/wiki/Execution_(computing)" title="Execution (computing)">executed</a>). In some programming languages it may be necessary for some compilation and linking to occur at runtime. There is a trade-off between compile-time and link-time in that many compile time operations can be deferred to link-time without incurring extra run-time.</p>/n<p>"Compile time" can also refer to the amount of time required for compilation.</p>/n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Compile_time&amp;action=edit&amp;section=2" title="Edit section: See also">edit</a><span class="mw-editsection-bracket">]</span></span></h2>/n<ul>/n<li><a href="/wiki/Link_time" title="Link time">Link time</a></li>/n<li><a href="/wiki/Run_time_(program_lifecycle_phase)" title="Run time (program lifecycle phase)">Run time (program lifecycle phase)</a></li>/n<li><a href="/wiki/Compiling" class="mw-redirect" title="Compiling">Compiling</a></li>/n<li><a href="/wiki/Type_system" title="Type system">Type system</a></li>/n<li><a href="/wiki/Just-in-time_compilation" title="Just-in-time compilation">Just in time compilation</a></li>/n</ul>/n<table class="metadata plainlinks stub" role="presentation" style="background:transparent">/n<tr>/n<td><a href="/wiki/File:LampFlowchart.svg" class="image"><img alt="Stub icon" src="//upload.wikimedia.org/wikipedia/commons/thumb/9/91/LampFlowchart.svg/22px-LampFlowchart.svg.png" width="22" height="30" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/9/91/LampFlowchart.svg/33px-LampFlowchart.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/9/91/LampFlowchart.svg/44px-LampFlowchart.svg.png 2x" data-file-width="324" data-file-height="442" /></a></td>/n<td><i>This <a href="/wiki/Computer_science" title="Computer science">computer science</a> article is a <a href="/wiki/Wikipedia:Stub" title="Wikipedia:Stub">stub</a>.
